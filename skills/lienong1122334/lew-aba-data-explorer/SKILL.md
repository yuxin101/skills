# ABA 数据探索

本skill指导你如何使用 帮助亚马逊卖家从ABA搜索词报告中挖掘有价值的数据。

## 核心概念

ABA（Amazon Brand Analytics）搜索词报告是亚马逊官方提供的搜索行为数据，反映消费者在亚马逊上的真实搜索行为。本工具拥有近3年的**周粒度**数据，覆盖15个亚马逊站点。

**排名逻辑**：`searchFrequencyRank` 值越小，代表搜索热度越高。排名1是最热门的搜索词。这是一个容易混淆的点——当用户说"排名提升"时，意味着数值变小；"排名下降"意味着数值变大。

## 数据字段

| 字段 | 英文名 | 说明 | 示例 |
|------|--------|------|------|
| 搜索词 | searchTerm | 消费者搜索的关键词 | rimel loreal |
| 统计周开始日期 | reportStartDate | 数据统计的周起始日期 | 2025-10-26 |
| 站点 | region | 亚马逊市场站点代码 | US |
| 搜索排名 | searchFrequencyRank | 搜索热度排名（越小越好） | 82135 |
| 点击ASIN | clickedAsin | 被点击的商品ASIN | B0XXXXXXXX |
| 商品名 | clickedItemName | 被点击商品的名称 | xxx |
| 点击占比排名 | clickShareRank | 该ASIN在此搜索词下的点击占比排名 | 1 |
| 点击占比 | clickShare | 该ASIN获得的点击占比（0~1） | 0.28 |
| 转化占比 | conversionShare | 该ASIN获得的转化占比（0~1） | 0.3333 |

## 支持的站点

US（美国）、DE（德国）、BR（巴西）、CA（加拿大）、AU（澳大利亚）、JP（日本）、AE（阿联酋）、ES（西班牙）、FR（法国）、IT（意大利）、SA（沙特）、TR（土耳其）、MX（墨西哥）、SE（瑞典）、NL（荷兰）

默认站点为 **US**。用户未指定站点时使用US。

## API 调用方式

本工具通过 LinkFox 工具网关 API 调用。调用规范、请求参数、响应结构详见 `references/api.md`。可直接执行 `scripts/aba_query.py` 发起查询。

## 如何构建查询

调用时，关键参数是 `analysisDescription`——用自然语言描述你要查询的数据。这段描述会在后台被转化为结构化查询，因此描述需要**精确、具体**。

### 编写 analysisDescription 的原则

1. **明确站点**：始终在描述开头指明站点，如"筛选美国站"
2. **明确筛选条件**：用具体数值范围，而非模糊描述。"排名在5万以内"比"排名较好"有效得多
3. **明确时间范围**：使用具体的时间描述，如"过去12周"、"2024年1-9月"、"近3个月"
4. **明确对比基准**：做趋势分析时，说清比较的时间节点，如"4周前的排名比8周前提升30%"
5. **处理去重逻辑**：当同一搜索词+ASIN有多条数据时，明确保留规则，如"相同搜索词相同ASIN值保留最新的一个"
6. **忠实用户意图**：不要曲解或过度延伸用户的查询需求，精确反映用户想要什么

### 常见分析场景的 analysisDescription 示例

**1. 搜索词热度排名趋势**
```
筛选美国站，关键词"gift"在过去12周的搜索热度排名。
```

**2. 潜力黑马词挖掘**
```
筛选美国站，关键词包含"gift"，2025年Q1和全年的平均搜索排名都大于50万，但最新排名冲进5万-10万的搜索词。
```

**3. 持续增长趋势挖掘**
```
筛选美国站，最新排名在20万以内，且4周前的排名比8周前提升30%，本周的排名比4周前提升30%的搜索词。
```

**4. 市场机会挖掘（高搜索量低垄断）**
```
筛选美国站，筛选当前搜索排名在20000以内，近三个月点击占比Top 1的Asin的转化率占比低于5%的搜索词。相同搜索词相同Asin值保留最新的一个。
```

**5. 季节性/节日关键词定位**
```
筛选美国站，包含"cup"的关键词中，去年（2024年）1-9月份排名未进入50万，10-11月份连续进入20万的词。
```

**6. 高点击低转化ASIN挖掘**
```
筛选美国站关键词包含"hat"的，最新搜索排名在5万-20万之间，且近3个月来点击占比大于20%，转化占比小于10%的ASIN。相同搜索词和ASIN仅保留点击占比和转化占比的比例最小数据。
```

**7. 高ROAS长尾蓝海词库构建**
```
筛选美国站，关键词包含"charger"的，当前排名在20万开外的，近2个月的平均转化占比大于平均转化占比1.5倍的关键词，以及相应的ASIN。
```

**8. 市场新词与新需求侦测**
```
找到美国站"charger"的长尾词中，近一个月才进入排名榜单，且当前排名在50万以内的所有词。
```

**9. 捕捉细分趋势/变体增长**
```
筛选美国站中"table"的长尾词中，排名在10万-30万之间，且近4周的搜索排名增长50%以上的搜索词。
```

## 展示规则

1. **只呈现数据**：将查询结果以清晰的表格形式展示，不做主观商业建议
2. **排名说明**：在涉及排名数据时，提醒用户数值越小表示排名越好
3. **数据量提示**：当结果较多时，展示核心数据并提醒用户可通过下载链接获取完整数据
4. **下载引导**：若返回了 `downloadUrl`，明确告知用户下载地址；若用户需要完整数据但未请求下载，主动提示可以生成下载链接
5. **失败处理**：查询失败时，根据 `msg` 字段向用户说明原因，并建议调整查询条件

## 重要限制

- **数据不入库**：本工具返回的结果不会写入数据库，因此无法使用 `_dataQuery_executeDynamicQuery` 对结果进行二次SQL加工
- **结果上限**：下载链接最多包含10000条数据
- **数据粒度**：数据为周维度，非日维度
- **数据范围**：约近3年的历史数据

## 用户表达方式与场景匹配速查

**适用** — 围绕亚马逊搜索词的数据查询：

| 用户说 | 场景 |
|-------|------|
| "xx词搜索量/热度怎么样" | 排名趋势 |
| "最近突然火的词"、"新冒出来的词" | 黑马词/新词侦测 |
| "蓝海词"、"竞争不大的词" | 市场机会挖掘 |
| "哪些词转化好"、"高转化长尾词" | 高ROAS词库 |
| "季节性的词"、"旺季什么词会涨" | 季节性关键词 |
| "谁在吃流量"、"有没有垄断" | 点击占比/垄断分析 |
| "点击高转化低的asin" | 高点击低转化诊断 |

**不适用** — 超出ABA搜索词数据范畴的需求：
- 广告/PPC相关（出价、投放策略）
- 商品评论、Listing撰写
- ASIN销量预估
- 用户自己已有ABA文件想本地处理

**边界判断**：用户说"选品"、"竞品分析"、"市场有没有机会"时，如果能归结为搜索词维度的数据查询（找蓝海词、看关键词下竞品流量分布），则适用；如果是问利润、价格策略、综合市场报告，则不适用。
