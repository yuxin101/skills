### 4. REFERENCE（正反样例）

**执行标准是什么？**

#### ✅ 正确示例

**用户问题**："如何配置 OpenClaw Gateway？"

**正确执行**：
```
📋 查询结果

## 核心答案
OpenClaw Gateway 配置通过 `config.yaml` 文件完成，主要包含服务器设置、渠道配置和技能管理三大部分。

## 详细说明
Gateway 是 OpenClaw 的核心组件，负责接收消息、处理技能和返回响应。配置文件位于 Gateway 安装目录下的 `config.yaml`。

## 操作步骤

### 1. 定位配置文件
```bash
cd ~/.openclaw/gateway
vi config.yaml
```

### 2. 基础配置
```yaml
server:
  host: 0.0.0.0
  port: 18789

channels:
  - name: wechat
    enabled: true
```

### 3. 重启服务
```bash
openclaw restart
```

## 📖 来源引用
- 文档链接：https://docs.openclaw.ai/configuration/gateway
- 章节位置：Configuration → Gateway Setup
- 最后更新：2024-01-15

## ⚠️ 注意事项
- 修改配置后必须重启服务
- 端口 18789 需要未被占用
- 配置文件使用 YAML 格式，注意缩进
```

### 5. LIMITS（明确限制）

**绝对不能碰的红线是什么？**

#### 🚫 禁止行为

| 禁止项 | 原因 | 正确做法 |
|-------|------|---------|
| **凭记忆回答** | 记忆可能过时或错误 | 必须访问文档验证 |
| **使用模糊表述** | 用户无法执行 | 提供具体路径和参数 |
| **省略来源引用** | 无法验证信息准确性 | 必须提供文档链接 |
| **忽略版本差异** | 不同版本配置可能不同 | 标注适用版本 |
| **跳过注意事项** | 可能导致配置失败 | 必须列出限制条件 |
