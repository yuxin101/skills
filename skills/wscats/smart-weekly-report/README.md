# 周报生成器 · Weekly Report Generator

> 一个基于自然语言的 Prompt-based Skill，帮助各职业角色快速生成结构化周报。无需代码，无需依赖，纯 Markdown + AI 驱动。

---

## ✨ 功能特性

- 🎯 **多职业覆盖**：前端/后端/客户端/测试工程师、运营、销售、HR、会计、市场营销、项目管理等
- 📋 **8 套专业模板**：针对不同职业角色深度定制，开箱即用
- 💡 **洞察优先**：不只是数据罗列，自动提炼趋势、异常和行动建议
- 📊 **对比分析**：引导提供上周数据，自动生成环比分析
- ⚡ **零门槛使用**：纯自然语言输入，2 分钟生成专业周报
- 🔧 **灵活定制**：支持调整风格（正式/简洁）、补充数据、修改措辞

---

## 🚀 快速开始

### 第一步：打开 AI 对话

使用任意支持自定义 Prompt 的 AI 工具（如 ChatGPT、Claude、Kimi、通义千问等）。

### 第二步：加载 Skill

将 [SKILL.md](./SKILL.md) 的全部内容复制粘贴到 AI 对话框中发送，或作为系统提示词（System Prompt）加载。

### 第三步：描述你的工作

用自然语言告诉 AI 你本周的工作情况，AI 将自动生成对应的结构化周报。

## 💬 使用示例

### 最简模式（30 秒）
```
我是前端工程师，本周完成了登录页面重构，修复了 3 个 Bug，下周要做首页性能优化，帮我生成周报。
```

### 标准模式（1 分钟）
```
我是运营，本周完成了双十一活动策划方案，DAU 较上周提升 15%，遇到了供应商资源协调问题，下周要推进活动上线，帮我生成周报。
```

### 详细模式（最完整）
```
请帮我生成周报，信息如下：
- 职业角色：后端工程师
- 汇报对象：技术总监
- 本周完成：用户中心微服务拆分（完成度 80%）、接口性能优化（响应时间从 800ms 降至 200ms）、修复线上 P1 Bug 2 个
- 进行中：订单服务重构，预计下周完成
- 遇到的问题：数据库连接池配置问题导致压测失败，已排查解决
- 下周计划：完成订单服务重构、开始支付模块开发
- 关键数据：接口可用率 99.95%，代码提交 23 次
```

## 🎨 输出风格调整

生成周报后，你可以继续告诉 AI 进行调整：

- `"帮我把语气改得更正式一些"` — 适合向高层汇报
- `"帮我精简一下，只保留最重要的内容"` — 适合快速同步
- `"帮我把数据部分补充得更详细"` — 适合数据驱动型团队
- `"帮我加上英文版本"` — 适合国际化团队
- `"帮我转换成邮件格式"` — 适合通过邮件发送

## 💡 最佳实践

1. **提供上周数据**：有对比才有洞察，尽量提供上周的关键数字
2. **量化你的成果**：用数字说话，例如"提升了 30%"比"有所提升"更有说服力
3. **说明阻碍的影响**：不只是描述问题，说明它影响了什么、需要谁帮助
4. **计划要具体**：给出时间节点和负责人，而不是模糊的"下周继续推进"
5. **定期使用**：每周固定时间生成，形成习惯，积累数据对比

## 🔧 进阶用法

### 作为系统提示词使用
将 `SKILL.md` 内容设置为 AI 的系统提示词（System Prompt），之后每次对话直接输入工作内容即可，无需重复加载。

### 团队统一模板
团队 Leader 可以根据 `SKILL.md` 中的模板，定制团队专属的周报格式，统一汇报标准。

### 历史数据对比
将历史周报保存为 Markdown 文件，下次生成时附上历史数据，AI 可以自动进行多周趋势分析。
