# Physical Safeguards

Last reviewed: March 17, 2026.

Primary authority pointers:

- HIPAA Security Rule physical safeguards
- HHS Security Rule overview

## Why this domain matters

PRS does not create a separate public stage for physical safeguards, but physical safeguards still matter where applicable. They may be direct, inherited, or partially inherited controls that constrain the workload's defensible stage.

## Review questions

- Does the workload rely on facilities, devices, workstations, or media that create physical-safeguard obligations?
- Which physical controls are inherited from cloud or colocation providers?
- Which physical controls remain with the organization?
- Are workstation and device handling expectations documented where relevant?

## Minimum expectations by stage

- PRS-1: applicability and inheritance path are noted
- PRS-2: required physical or inherited controls are documented for the workload
- PRS-3: approval accounts for physical-safeguard responsibilities where applicable
- PRS-4: operational evidence exists where direct physical controls remain in scope

## Strong evidence

- provider attestations or contractual terms
- device handling procedures
- workstation or media controls
- facility responsibility documentation

## Common blockers

- physical safeguards ignored because the workload is cloud-hosted
- inherited controls assumed but not mapped
- local device handling risks left undocumented

## Recommended next actions

- document inherited and retained physical responsibilities
- add device or workstation handling rules where the workload requires them
- retain vendor evidence for inherited facility controls
