# Recognized Security Practices

This document tracks the role of recognized security practices in PRS assessments.

Official source: https://www.hhs.gov/hipaa/for-professionals/security/guidance/index.html

## Key point

Recognized security practices may influence HHS/OCR enforcement context, but they do not replace HIPAA obligations and they are not PRS stages.

## PRS use

- treat recognized security practices as supporting evidence for disciplined security operations
- do not treat them as a shortcut to PRS-2, PRS-3, or PRS-4
- do not use them as a substitute for workload-scoped risk analysis, approval, or live operational evidence

## Good use

- evidence that the organization follows a stable security framework can strengthen confidence in control execution
- evidence that the workload inherits mature practices can support a finding when the inheritance path is explicit

## Bad use

- `we follow framework X, therefore this workload is PRS-4`
- `recognized security practices means we are HIPAA compliant`
