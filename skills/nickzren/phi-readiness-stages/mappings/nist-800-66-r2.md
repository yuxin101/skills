# NIST SP 800-66 Rev. 2 Mapping

NIST SP 800-66 Rev. 2 is an implementation guide for the HIPAA Security Rule. In PRS, it should be used as practical implementation guidance, not as a replacement for the rule itself.

Official source: https://csrc.nist.gov/pubs/sp/800/66/r2/final

## How to use this document

- Use NIST to inform how a control can be implemented or evaluated.
- Use the HIPAA Security Rule and official HHS/OCR guidance to define the baseline requirement.
- If NIST suggests a helpful control pattern that is not explicitly visible in the reviewed workload, treat it as guidance, not proof of a requirement violation by itself.

## Where NIST is most useful in PRS

- translating rule concepts into implementation questions
- structuring risk analysis and risk management discussions
- informing contingency, monitoring, and operational practices
- evaluating inherited and shared-responsibility control models

## Caution

Do not let NIST breadth cause PRS inflation. A workload still needs stage-specific evidence for approval and live operation.
