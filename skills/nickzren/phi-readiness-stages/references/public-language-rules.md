# Public Language Rules

Use these rules for customer-facing, investor-facing, or external status statements.

## Safe

- exact PRS stage codes and frozen public labels
- scoped descriptions tied to a workload and environment
- `HIPAA-informed` only when describing PRS itself

Examples:

- `Claims API is currently PRS-2 PHI-Ready - pending internal approval.`
- `The production data pipeline is PRS-4 PHI-Operational - operating with PHI under ongoing controls.`

## Avoid

- HIPAA compliant
- HIPAA secure
- HIPAA certified
- fully HIPAA ready
- enterprise-grade HIPAA platform

## If HIPAA must be mentioned

- scope the statement to a specific workload and environment
- do not imply regulator certification
- do not use HIPAA as a maturity badge

Preferred construction:

- `We use PHI Readiness Stages (PRS), a HIPAA-informed workload status framework, to describe the lifecycle status of systems that may handle protected health information.`
