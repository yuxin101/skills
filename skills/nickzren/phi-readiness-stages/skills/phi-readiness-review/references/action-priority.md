# Action Priority Rules

Order recommendations using this sequence:

1. actions that remove a hard stage blocker
2. actions that reduce material security risk
3. actions that create durable evidence required for review
4. actions that improve efficiency or polish

## Preferred action shape

Each action should state:

- the concrete task
- the missing required domain it satisfies
- the evidence artifact that should exist after completion
- whether it unlocks the next stage or only strengthens the current stage

## Examples of strong actions

- `Complete and approve a scoped risk analysis for the production claims API.`
- `Run and retain restore-test evidence for the production backup set.`
- `Record formal internal approval for PHI use in production and retain the signed decision artifact.`

## Weak actions to avoid

- `Improve security posture.`
- `Be more compliant.`
- `Tighten HIPAA readiness.`
