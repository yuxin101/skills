# Checklist: PRS-3 to PRS-4

Goal: determine whether the approved workload is now live with PHI and maintaining operational evidence.

## Required checks

- approved production cutover occurred
- PHI is actually live in the approved scope
- access review evidence exists
- monitoring and alert handling evidence exists
- vulnerability and patch evidence exists
- change-control evidence exists
- incident-handling readiness is current
- backup and restore confidence is evidenced
- vendor reassessment process is active
- periodic technical and non-technical evaluation is active
- evidence retention is current

## Common blockers

- PHI is not actually live yet
- logging exists but there is no review or alert response evidence
- backups exist but restore confidence is stale
- reassessment after material changes is missing
- evidence is too stale to defend the current stage

## Minimum evidence

- cutover or launch record
- production evidence that PHI is in scope
- recent access-review artifact
- recent monitoring or alert-response artifact
- recent vulnerability-management artifact
- recent backup or restore evidence
