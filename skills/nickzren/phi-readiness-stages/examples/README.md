# Calibrated Example Assessments

These examples are sanitized calibration references for common PRS archetypes.

Use them to:

- compare scope and role reasoning
- compare evidence-level expectations
- compare the `why not lower` and `why not higher` logic
- avoid stage inflation when a case looks stronger or weaker than it is

Do not treat an example as a shortcut around source verification or scope-specific judgment.

## Available examples

- `examples/prs-0-consumer-health-app.md`
- `examples/prs-1-repo-only-security-aligned.md`
- `examples/prs-2-saas-pending-approval.md`
- `examples/prs-3-customer-approved-defined-scope.md`
- `examples/prs-4-live-operational-phi-workload.md`

## How to use them

- pick the nearest deployment and role archetype
- compare evidence level, freshness, and artifact depth
- compare the crosswalk rows used
- compare the blockers that keep the case from moving up one stage

If the current case is materially different, say so and do not force it into the example pattern.
