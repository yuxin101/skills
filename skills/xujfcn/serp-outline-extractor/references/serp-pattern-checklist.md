# SERP Pattern Checklist

Use this checklist before generating the final outline.

## Intent classification
Identify the dominant intent first:
- informational
- commercial investigation
- comparison
- transactional
- navigational

If mixed, label the primary and secondary intent.

## Patterns to extract
Look for recurring patterns such as:
- definition / overview section
- step-by-step setup section
- pricing / cost section
- feature comparison section
- alternatives / competitors section
- FAQ block
- troubleshooting section
- examples / templates / code section
- pros and cons section

## Signals to preserve in the outline
- beginner vs advanced framing
- year/version freshness (for example 2026)
- direct-answer sections near the top
- table/list opportunities
- comparison angles users are likely evaluating
- objections or friction points implied by the query

## Red flags
- do not generate generic outlines that ignore intent
- do not force a sales page structure onto informational queries
- do not add sections with no obvious search demand signal
- do not bury the likely answer too deep

## Practical rule
The final outline should feel like it came from observing search patterns, not from a random blog template.
