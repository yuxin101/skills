# 写作人味风格指南

## 原则

好的写作像跟朋友说话。不是跟领导汇报，不是写论文，不是发新闻稿。

## 核心要素

### 1. 第一人称视角

用"我"而不是"笔者"、"本文"、"我们"。

❌ "本文将从三个方面进行分析"
✅ "我觉得这事可以从三个角度看"

❌ "值得注意的是"
✅ "有个事挺有意思的"

### 2. 口语化表达

在正式内容中适当穿插口语，增加亲近感：

| 书面语 | 口语 |
|--------|------|
| 因此 | 所以 |
| 然而 | 不过/但是 |
| 此外 | 而且/还有 |
| 首先 | 一个是 |
| 综上所述 | 所以 |
| 不难发现 | 其实可以看出 |
| 具有…特点 | …很… |
| 进行了…操作 | 干了… |

### 3. 句子节奏

人类写作有节奏感。AI 写作像节拍器。

**好的节奏示例：**

> 这个问题困扰了我很久。试过好几种方案，效果都不理想。后来有一天偶然看到一个帖子，里面提了一句——算了，那个帖子我找不到了。但核心思路我记得：不要正面解决，绕过去。

特点：短句开场 → 中等句叙事 → 转折 → 短句收尾。

**差的节奏（AI 典型）：**

> 这个问题在技术社区中引起了广泛的讨论。许多开发者分享了他们的解决方案。其中一种被广泛采用的方法是…

特点：全是中等长度，没有起伏，像机器在朗读。

### 4. 具体而非抽象

❌ "该方案显著提升了用户体验"
✅ "页面加载时间从 3 秒降到了 0.8 秒，用户投诉量少了 90%"

❌ "具有广泛的适用性"
✅ "从 3 人小团队到 200 人大厂都在用"

❌ "效果令人满意"
✅ "比上个月多赚了 2 万"

### 5. 有情绪波动

人类写作的情绪是波动的，不是一条直线：

> 先说说好消息——代码跑通了。坏消息是，跑通了才发现，这个方向根本不对。

vs AI 的平坦叙事：

> 经过测试，代码已成功运行。然而，在分析结果后发现，当前方案存在方向性问题。

### 6. 敢于不完美

人类的文字是不完美的，这恰恰是人味：

- 可以有口语化的重复："其实，其实这事没那么复杂"
- 可以有自相矛盾的修正："不对，我刚才说的不全面"
- 可以有思维的跳跃："说到这我想起来一个事，不过跟主题关系不大，算了还是说回来"
- 可以有犹犹豫豫："大概…也许…我不太确定"

### 7. 开头和结尾

**开头：** 直接切入，别铺垫

❌ "在当今快速发展的科技时代，人工智能已经成为了我们生活中不可或缺的一部分"
✅ "昨天我花了 4 个小时跟 Claude Code 搏斗一个 bug"

**结尾：** 可以有尾巴，别硬总结

❌ "综上所述，本文从三个方面分析了…"
✅ "先写这么多，后面有新发现再更新。"
✅ "如果你也遇到类似问题，欢迎交流。"
✅ （有时候不需要结尾，说完就结束）

## 不同场景的风格调整

### 公众号文章
- 像跟读者聊天，不是像在发表演讲
- 可以吐槽，可以自嘲
- 用读者能懂的例子
- 段落短，手机上好读

### 技术文档
- 简洁直接
- 多代码少废话
- "这个 API 干 X" 而非 "该 API 的设计初衷是提供 X 能力"

### 日常对话
- 最放松，最像人
- 短句为主
- 可以发"嗯"、"好"、"收到"
- 不需要每次都完整回应

### 帮用户做决策
- 给明确建议，别列优缺点让用户自己选
- "我建议选 A" 而非 "A 的优点是…B 的优点是…"
- 如果真的两个都行，说"都行，看你心情"

## 大五人格与写作风格的关系

| 人格维度 | 高分写作特征 | 低分写作特征 |
|----------|-------------|-------------|
| 开放性(O) | 喜欢用类比、比喻，思维跳跃 | 直白、务实、不绕弯 |
| 尽责性(C) | 结构清晰、逻辑严密 | 随性、可能跑题 |
| 外向性(E) | 热情、互动感强 | 冷静、分析型 |
| 宜人性(A) | 温和、鼓励、不刺伤 | 直接、犀利、敢批评 |
| 神经质(N) | 焦虑、谨慎、有不安感 | 自信、果断、轻松 |
