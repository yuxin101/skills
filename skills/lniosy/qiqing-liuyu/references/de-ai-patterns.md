# 去 AI 味完整模式库

## 一、连接词和过渡词

### 🔴 高频 AI 味连接词（每篇各 ≤ 1 次）

| AI 味词 | 替代方案 |
|---------|----------|
| 此外 | 直接说下一点，或用"而且" |
| 然而 | "不过"、"但是"、"可惜" |
| 值得注意的是 | 删掉，直接说重点 |
| 更重要的是 | "关键是"、"核心问题是" |
| 总而言之 | 删掉，直接说结论 |
| 综上所述 | 删掉，直接说结论 |
| 不可否认 | 删掉，几乎每次都是废话 |
| 毋庸置疑 | 同上 |
| 事实上 | "其实"、"老实说" |
| 毋庸讳言 | 删掉 |
| 众所周知 | 删掉或给具体来源 |
| 毋庸赘述 | 删掉 |
| 诚然 | "虽然" |
| 显而易见 | 如果真的显而易见，删掉 |
| 毋庸 | 任何包含"毋庸"的词组，都删掉 |
| 不言而喻 | 删掉 |

### 🟡 需要控制的模式

| 模式 | 问题 | 替代 |
|------|------|------|
| 首先…其次…最后… | 每段都这样 | 用"一是…二是…"或者直接列数字 |
| 随着…的发展 | 万能开头 | 直接切入主题 |
| 在…的背景下 | 同上 | 同上 |
| 让我们… | AI 爱发号施令 | 改成"可以…"或直接做 |
| 在这个…的时代 | 空洞修辞 | 删掉 |
| 深刻地 | 夸大 | 用具体描述替代 |
| 意义深远 | 同上 | 同上 |
| 不可或缺 | 同上 | "很重要"或具体说为什么 |
| 至关重要 | 同上 | 同上 |

## 二、结构模式

### 🔴 否定式排比（每篇 ≤ 1 次）

❌ "不是 X，不是 Y，而是 Z"
❌ "不是…而是…"连续出现
✅ 如果只用 1 次可以保留，更多时拆成独立句子

### 🔴 三段式论证

❌ "首先…其次…最后…"反复出现
✅ 变化结构：有时用数字列表，有时直接说，有时跳过过渡

### 🟡 每段结尾总结

❌ 每段最后一句都是总结句
✅ 有时候让观点自然结束，不需要"所以…"

### 🟡 过度引用

❌ 每个观点都引用来源
✅ 只有争议性观点需要引用，常识直接说

## 三、破折号

### 🔴 破折号零容忍规则

每篇文章"——"不超过 2 处，理想 0 处。

**常见 AI 破折号模式：**
- "XX——这个被广泛讨论的话题——YY" → 删掉或用逗号
- "原因很简单——" → 改为冒号"原因很简单："
- "——一个重要的概念——" → 删掉
- "不仅仅是…而且是…" → 限制 1 次

**替换策略：**
1. 逗号后破折号 → 冒号（引述类）或逗号
2. 补充说明的破折号 → 括号或逗号
3. 兜底：全部替换为逗号

## 四、语气和态度

### 🔴 假客观

❌ "客观来说/客观地讲" — 99% 的时候是废话，删掉
❌ "理性来看" — 同上
❌ "从客观角度来看" — 同上

### 🟡 假谦虚

❌ "作为一个 AI，我…" — 不要提醒用户你是 AI
❌ "我只是一个语言模型" — 同上
❌ "我无法真正理解" — 需要的时候说，不要每次都加

### 🟡 过度礼貌

❌ "请问您需要…" — 大部分时候直接做就行
❌ "如果您需要的话…" — 不确定就直接问
❌ "希望对您有所帮助" — 删掉

## 五、句子层面

### 句子长度变化

AI 倾向于写中等长度的句子（15-25 字），节奏单调。人类写作的特点：
- 短句。制造节奏感。
- 中等长度的句子用于正常叙述。
- 偶尔来一个稍长但好读的句子，用来承载复杂的信息。
- 再来一个短句。收尾。

### 被动语态控制

AI 过度使用被动语态。人类更倾向主动：
- ❌ "这个问题被认为是…" → ✅ "很多人认为…"
- ❌ "被发现" → ✅ "有人发现"

### 具体化

AI 爱用抽象概念，人类爱用具体事物：
- ❌ "提高了用户体验" → ✅ "打开速度快了 2 秒"
- ❌ "具有重大意义" → ✅ "让 100 万人第一次用上了…"
