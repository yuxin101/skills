## ⚠️ Critical Issues

### 1. CloudFront/S3 Routing — BLOCKING MULTI-PAGE SEO 🔴
**Severity:** CRITICAL  
**Impact:** All HTML pages return homepage content, breaking SEO for secondary pages

**Problem:**
```
https://merxex.com/journal.html → returns index.html content
https://merxex.com/terms.html → returns index.html content
https://merxex.com/privacy.html → returns index.html content
```

**Root Cause:**
S3 bucket is configured for Single Page Application (SPA) behavior:
- All 404 errors redirect to index.html
- CloudFront doesn't serve individual HTML files
- This works for React/Vue apps but breaks multi-page static sites

**Solution Options:**

**Option A: Change S3 to Serve Static Files (Recommended)**
```
1. Update S3 bucket settings: Disable "All objects redirect to index.html"
2. Update CloudFront error responses: Remove 404 → index.html redirect
3. Test: Each .html file should serve its own content
4. Deploy: Invalidate CloudFront cache
```

**Option B: Convert to Single Page Application**
```
1. Keep current routing
2. Use JavaScript routing for page navigation
3. Implement server-side rendering or pre-rendering for SEO
4. Add <base> tag and dynamic title/meta updates
```

**Option C: Use AWS Lambda@Edge for Routing**
```
1. Keep SPA behavior for unknown routes
2. Add Lambda@Edge function to serve known .html files correctly
3. Route /terms.html, /privacy.html, etc. to actual files
4. Keep SPA fallback for everything else
```

**Recommendation:** Option A — simplest and most SEO-friendly
