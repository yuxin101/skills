# Execution Reference — CLI, Launcher, Deployment

## Environment Setup (Linux/AIX)

```bash
# Source the ITX setup script (sets DTX_HOME_DIR, PATH, library paths)
. /opt/IBM/itx/setup

# Or set manually:
export DTX_HOME_DIR=/opt/IBM/itx
export PATH=$DTX_HOME_DIR/bin:$PATH
export LD_LIBRARY_PATH=$DTX_HOME_DIR/lib:$LD_LIBRARY_PATH  # Linux
# export LIBPATH=$DTX_HOME_DIR/lib:$LIBPATH               # AIX
export DTX_TMP_DIR=/tmp/itx
```

## Command Server (`dtxcmdsv`) — Direct Execution

Executes a single compiled `.mmc` map:

```bash
# Basic execution
dtxcmdsv -MAP mymap.mmc -IN input.dat -OUT output.xml

# With explicit type override
dtxcmdsv -MAP mymap.mmc -I1 input.dat -O1 output.xml

# Multiple inputs/outputs (card-numbered flags)
dtxcmdsv -MAP mymap.mmc -I1 input1.dat -I2 ref.dat -O1 output.xml

# Enable trace
dtxcmdsv -MAP mymap.mmc -IN input.dat -OUT output.xml -TRACE trace.mtr

# Set return code threshold for non-zero exit
dtxcmdsv -MAP mymap.mmc -IN input.dat -OUT output.xml -EXITON 8
```

### Common CLI Flags
| Flag | Description |
|---|---|
| `-MAP file.mmc` | Map to execute |
| `-IN` / `-I1`, `-I2` | Input card source (numbered for multiple cards) |
| `-OUT` / `-O1`, `-O2` | Output card target |
| `-TRACE file.mtr` | Enable trace output to file |
| `-EXITON n` | Exit with non-zero if return code >= n |
| `-SILENT` | Suppress console output |
| `-LOG file.log` | Redirect log output |

## Launcher (Event-Driven Server)

The Launcher is a server process that activates map execution based on events: file arrival, MQ message, HTTP request, or schedule. It uses an `.msl` system file (generated by IFD).

### Launcher Admin CLI

```bash
# Start/stop mode
launcheradmin.sh -manual         # manual start mode
launcheradmin.sh -auto           # automatic start mode

# Configuration
launcheradmin.sh -lport 4444                    # set listener port
launcheradmin.sh -prange 5000:5100              # map process port range
launcheradmin.sh -ini /opt/IBM/itx/dtx.ini     # specify config file

# Deployment
launcheradmin.sh -addir /deployments/maps       # add deployment directory
launcheradmin.sh -adduser user:admin;login:admin;pwd:secret

# Operations
launcheradmin.sh -status                        # show Launcher status
launcheradmin.sh -list all                      # list all deployed maps
launcheradmin.sh -start MapSystemName           # start a map system
launcheradmin.sh -stop MapSystemName            # stop a map system
launcheradmin.sh -import properties.xml         # import configuration
launcheradmin.sh -export properties.xml         # export configuration
```

### dtx.ini — Key Settings

```ini
[Launcher]
CircularLogSize=10485760          # Max log file size (bytes) — 10MB
CircularLogFileNum=5              # Number of log files before rollover

[Map]
OnSuccess=CreateOnContent         # Prevent 0-byte output files
; OnSuccess=AlwaysCreate          # Default — always create output

[Security]
RequireAuthentication=true
```

**Critical:** Set `OnSuccess=CreateOnContent` for any downstream system that checks for output file existence (e.g., SAP R/3 adapter, FTP pickup).

### Parallelism

- Deploy multiple Launcher processes on different ports for horizontal scale
- For Connect:Direct adapter: set `Max Concurrent Map Instances = 1` to prevent ordering issues
- Use `GETPARTITIONNAME()` in map rules to identify which Launcher partition is executing

## Map Archive (`.mar`) — Deployment Package

A `.mar` file bundles the compiled `.mmc` and all dependent resources for deployment into ACE or containerized environments.

### Creating a `.mar`
1. In Design Studio: `File → Export → Map Archive`
2. Select maps and dependent resources (type trees, lookup tables, functional maps)
3. Specify output `.mar` filename
4. Click Finish

### Deploying to ACE
Reference the `.mar` file in the ACE Toolkit:
1. Add `.mar` to the ACE application project
2. Configure the WTX/ITX Map Node to reference the map by name within the archive
3. Deploy as part of the BAR file

## Docker / Container Deployment

For containerized ITX (e.g., CP4I):

```dockerfile
FROM ibmcom/ace-itx:12.0.x
COPY maps/*.mmc /home/aceuser/maps/
COPY config/dtx.ini /opt/IBM/itx/dtx.ini
ENV DTX_HOME_DIR=/opt/IBM/itx
ENV DTX_TMP_DIR=/tmp/itx
```

Or use ConfigMaps to mount `.mar` files and ITX configuration.

Reference: [ace-itx-ue-cp4i GitHub](https://github.com/efir-tractatus/ace-itx-ue-cp4i)

## Platform Compilation Targets

Compile maps on the same OS and architecture as the deployment target:

| Platform | Notes |
|---|---|
| Windows x64 | Design Studio default; for Windows Launcher/ACE on Windows |
| Linux x64 | Most common for ACE on RHEL/Ubuntu |
| AIX | IBM Power systems |
| z/OS | Mainframe (requires z/OS ITX runtime) |

Compiled `.mmc` files are **not cross-platform**. Always recompile for target OS.
