# Validation

A memory system is not complete when documented. It is complete when tested.

## Minimum validation set

### 1. Continuous execution case
Question: does the system help a complex project keep moving coherently?

### 2. Interruption recovery case
Question: after interruption, can the agent recover stage, reasoning, and next step quickly?

### 3. Cross-project reuse case
Question: can structures, lessons, or templates help the next project start faster?

## Suggested metrics
- recovery accuracy
- follow-up question count
- stage identification accuracy
- reuse success
- structure completeness
- evidence completeness
- subjective usefulness improvement

## Rule
Do not claim broad success from one narrow case.
Always state the tested scope and the non-tested boundary.
