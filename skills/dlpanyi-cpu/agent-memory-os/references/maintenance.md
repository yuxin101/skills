# Maintenance

A memory system survives through maintenance, not just design.

## Minimum maintenance loop

### Daily / milestone-based
- update current project status
- record important decisions
- log meaningful lessons or errors

### Weekly
- review project lessons
- review bridge candidates
- fix missing asset indexes

### Every 2-4 weeks
- review global memory for stale rules
- promote what proved reusable
- deprecate what is outdated

## Core questions during review
- what is stale?
- what is missing from indexes?
- what proved reusable?
- what should not stay global?
- what project should be closed, paused, or archived?

## Rule
If maintenance is optional, decay is inevitable.
