# Publish Checklist

Use this checklist before publishing or widely sharing the skill.

## 1. Privacy check
- [ ] No real person names
- [ ] No private IDs
- [ ] No internal session/message IDs
- [ ] No personal workspace paths
- [ ] No private document URLs
- [ ] No raw private chat excerpts
- [ ] No examples that can be traced back to a specific person or team

## 2. Public usefulness check
- [ ] The opening describes a recognizable pain point
- [ ] The result promise is clear in one sentence
- [ ] The workflow is concrete, not abstract
- [ ] The skill includes a minimal success path
- [ ] The user can tell what they will get by the end

## 3. Reusability check
- [ ] Templates are generalized
- [ ] References are not tied to one project
- [ ] Rules are written as reusable patterns
- [ ] Validation guidance states tested scope and boundaries

## 4. Quality check
- [ ] SKILL.md stays concise enough to trigger well
- [ ] Detailed material lives in references/
- [ ] Assets are useful, not decorative
- [ ] Nothing in the skill depends on hidden private context

## 5. Positioning check
- [ ] This reads like a usable system, not a private project archive
- [ ] The title and description are memorable
- [ ] The skill solves a painful problem that many users recognize

If any privacy item fails, do not publish.
If usability is weak, tighten the opening and minimal success path before publishing.
