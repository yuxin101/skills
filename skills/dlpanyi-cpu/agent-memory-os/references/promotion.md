# Promotion

## Promotion path

Use this evolution path:
- `candidate`
- `promoted`
- `canonical`
- `deprecated` / `superseded` when needed

## Candidate
Use when something looks reusable but is not fully proven yet.

Typical triggers:
- a pattern appears multiple times
- a workflow succeeds once and looks reusable
- a failure creates a durable lesson
- the user explicitly says it should be remembered

## Promoted
Use when the lesson has:
- evidence
- at least one real validation
- some level of generality
- known scope or boundary

## Canonical
Use only when the lesson is:
- proven in multiple contexts
- stable in wording
- strong enough to serve as a default rule or SOP

## Do not promote
- raw chat fragments
- temporary project state
- local workarounds without general value
- unvalidated guesses
