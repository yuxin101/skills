# Failure Modes

Use this reference when the user says their agent has "memory issues" but the real failure mode is unclear.

## Common failure modes

### 1. Single-brain overload
Symptoms:
- everything goes into one place
- long-term memory contains active project clutter
- retrieval gets slower and noisier over time

### 2. Project pollution
Symptoms:
- project-specific state leaks into global rules
- temporary constraints are treated like stable principles
- long-term memory becomes untrustworthy

### 3. Retrieval confusion
Symptoms:
- the agent does not know where to search first
- the same question leads to different memory paths each time
- semantic search is overused without structure

### 4. Knowledge stagnation
Symptoms:
- lessons are written but never reused
- useful patterns never become SOPs or stable rules
- every new project starts almost from scratch

### 5. Maintenance decay
Symptoms:
- the system looked good when created
- status files go stale
- assets are missing from indexes
- outdated rules never get deprecated

## Diagnosis question
Ask: **Is the main problem storage, retrieval, separation, reuse, or maintenance?**

That answer usually determines the next move.
