# Architecture

## Recommended default

Use a three-part memory system:
- **global memory**
- **project memory**
- **bridge/promotions**

## Why this works

### Global memory
Use for:
- durable preferences
- stable operating rules
- long-lived SOPs
- architecture principles
- high-confidence lessons

### Project memory
Use for:
- active project state
- local decisions
- project assets
- project-specific lessons
- temporary complexity

### Bridge/promotions
Use for:
- candidate lessons worth tracking
- promoted patterns with evidence
- canonical rules that proved reusable

## Core rule

The global brain is the parent brain.
Project memory is an attached working brain.
Reusable knowledge can flow upward, but project-local clutter must not overwrite the global layer.
