# YouMind 插画风 PPT 生成技能

## 核心理念

这不是一个"把文字搬到幻灯片上"的工具。它的使命是将用户的大纲**重构为一段视觉故事**——每一页都是故事中的一个场景，由一个贯穿始终的 IP 角色"演"出来。

最终产出是一份 `.pptx` 文件，每页幻灯片包含 AI 生成的 YouMind 风格插画 + 精炼的文案。

---

## 工作流总览

```
用户大纲 → [阶段1] 叙事解构 → [阶段2] 逻辑视觉化 → [阶段3] YouMind 渲染 → 生成 PPTX
```

每个阶段都产出一份中间产物，在进入下一阶段前需要确认。

---

## 第〇步：确定 IP 角色

在一切开始之前，先和用户确定贯穿整个演示的 IP 角色。这个角色是整个 PPT 的灵魂——它会出现在每一页中，用不同的姿态和场景来"演绎"内容。

### 预设角色（参考 `references/ip-presets.md`）

提供 4 个预设角色供用户快速选择，也支持用户自定义。向用户展示预设选项时，简要描述每个角色的性格和适用场景，让用户做出选择。

### 自定义角色

如果用户选择自定义，需要收集：
- 动物/形象类型（如：柴犬、小猫、机器人、小恐龙）
- 标志性服饰或配件（如：穿宇航服、戴围巾、背书包）
- 性格关键词（如：呆萌、热血、优雅）

将自定义角色的描述整理为一段标准化的英文描述短语，格式参照预设角色的描述方式。

## 阶段1：叙事解构 (Narrative Deconstruction)

**目标**：将大纲从"信息罗列"重构为"故事弧线"。

### 思维方式

模仿 TED 演讲和苹果发布会的叙事节奏。不要做内容的搬运工——你是一个故事编剧。

### 操作步骤

1. **扫描大纲**，识别其中隐含的情感曲线：起点在哪？转折在哪？高潮在哪？
2. **重构为"英雄之旅"或"进化路径"**：
   - 起点（现状/痛点）→ 召唤（发现机会）→ 试炼（过程中的挑战）→ 蜕变（成果/突破）→ 新世界（未来愿景）
   - 不是每个大纲都适合完整的英雄之旅，可以截取其中的片段
3. **为每页提炼一个 Golden Sentence**：
   - 每一页只能有**一个**核心观点
   - 这句话要有冲击力，像演讲者会大声说出来的那种
4. **将 Golden Sentence 转化为视觉动作**：
   - 不是说"销量增长" → 而是"小狗背起行囊登上山顶，身后是渐渐变亮的天空"
   - 不是说"团队协作" → 而是"三只小狗围坐在篝火旁，手中各拿着不同颜色的拼图"
   - 不是说"数据驱动" → 而是"小狗站在巨大的仪表盘前，指针正在飞速旋转"

### 阶段1产出

一张表格（在对话中展示给用户确认）：

| 页码 | Golden Sentence | 视觉动作描述 | 叙事角色 |
|------|----------------|-------------|---------|
| 1 | "每个伟大故事都从一个问号开始" | IP角色躺在床上看手机，头顶飘着问号气泡 | 起点 |
| 2 | ... | ... | ... |

**等待用户确认后再进入阶段2。**

---

## 阶段2：逻辑视觉化 (Logic Visualization)

**目标**：为每页内容找到最佳的视觉表达结构。

### 思维方式

模仿手绘笔记（sketchnote）和 Twitter 信息图的视觉逻辑。关键是识别内容背后的**关系类型**，然后匹配对应的视觉结构。

### 关系类型 → 视觉结构映射

详见 `references/visual-logic.md`，核心映射如下：

| 关系类型 | 视觉结构 | 示例 |
|---------|---------|------|
| **递进/成长** | 成长路线图、台阶、登山路径 | Day 1 → Day 16 的进化时间线 |
| **并列/多项** | 悬浮信息卡片、气泡矩阵 | 3 个并列的能力标签气泡 |
| **对比/冲突** | 对比窗口、左右分屏 | Before vs After 对照 |
| **因果/流程** | 箭头流、管道图 | 输入 → 处理 → 输出 |
| **总分/层级** | 中心辐射、树形展开 | 核心概念 + 3 个分支 |
| **数据/成就** | 大数字高亮 + 标签 | "1,157 Messages" 悬浮气泡 |

### 操作步骤

1. 逐页分析阶段1产出的内容，判断每页的逻辑关系类型
2. 为每页选定一个视觉结构
3. 确定每页的**视觉重心（Focus Point）**——用户目光首先落在哪里
4. 规划悬浮 UI 装饰元素的分布（代码框、气泡通知、进度条、点状网络等）

### 阶段2产出

在阶段1的表格基础上补充：

| 页码 | Golden Sentence | 视觉动作 | 逻辑类型 | 视觉结构 | 视觉重心 | UI装饰 |
|------|----------------|---------|---------|---------|---------|--------|

**等待用户确认后再进入阶段3。**

## 阶段3：YouMind 风格渲染 (Visual Rendering)

**目标**：为每页幻灯片生成 YouMind 风格的 AI 图片，并组装为最终 PPTX。

### YouMind 风格规范

| 维度 | 要求 |
|------|------|
| **主体** | 贯穿始终的 IP 角色，每页用不同姿态/场景来表达内容 |
| **背景** | 蓝黄渐变 或 蓝紫渐变，极简留白，不要复杂场景 |
| **装饰** | 悬浮 UI 元素：代码框 `<html><body>`, 点状网络图, 气泡通知, 进度条, 设置齿轮, 文档图标 |
| **风格** | 2D 扁平矢量, 干净线条, 专业信息图感, 温暖友好 |
| **构图** | 16:9 横版, 主体偏左或偏右, 留出充足的文案空间 (copy space) |
| **禁忌** | 不要写实风格, 不要 3D, 不要暗色调, 不要过度复杂的背景 |

### Prompt 生成模板

详见 `references/prompt-templates.md`，基础模板：

```
[IP角色描述] + [视觉动作] + [视觉结构] + [UI装饰元素],
2D flat vector, YouMind illustration style, professional infographic,
clean lines, vibrant gradient background (blue to yellow/blue to purple),
16:9 aspect ratio, ample copy space on the [left/right] side,
floating UI elements, cute and friendly, minimal and clean
```

### 图片生成流程

1. 基于阶段2的规划，为每页组装完整的英文 Prompt
2. **调用可用的图片生成工具**生成图片：
   - 如果有 DALL-E / image generation MCP 可用 → 直接调用
   - 如果没有 → 将所有 Prompt 整理成文档，让用户自行生成
3. 每张图片生成后保存到工作目录

### PPTX 组装

使用 pptx 技能的 PptxGenJS 方式创建最终文件。先阅读 pptx 技能的 `pptxgenjs.md` 了解 API。

#### 每页幻灯片的结构

```
┌─────────────────────────────────────────────┐
│                                             │
│  [AI 生成的 YouMind 插画]                     │
│  （作为全幅背景或大面积主视觉）                   │
│                                             │
│         ┌──────────────────┐                │
│         │  Golden Sentence │                │
│         │  （白色/深色文字）  │                │
│         └──────────────────┘                │
│                                             │
│  [补充说明文字，如果有的话]                      │
│                                             │
└─────────────────────────────────────────────┘
```

#### 排版原则

- **图片为主**：每页的 AI 插画占据主要视觉面积（至少 60%）
- **文字精炼**：Golden Sentence 用大号字体（28-36pt），补充文字用小号（14-16pt）
- **文字位置**：放在 Prompt 中预留的 copy space 区域
- **半透明底板**：如果文字直接放在图片上不够清晰，加一个半透明深色/浅色底板
- **统一字体**：标题用粗体无衬线字体，正文用细体

#### 特殊页面

- **封面页**：IP 角色居中，标题大字号，副标题小字号，背景用最鲜艳的渐变
- **过渡页**：仅有 Golden Sentence + IP 角色的简约动作，大面积留白
- **数据页**：大数字突出显示（48-60pt），IP 角色在旁边做出惊讶/自豪的表情
- **结尾页**：IP 角色以"胜利/展望"姿态出现，配合号召性文字

## 质量检查

生成 PPTX 后，必须进行以下检查：

1. **内容检查**：用 `python -m markitdown output.pptx` 提取文本，确认无遗漏
2. **视觉检查**：转换为图片逐页审查
   ```bash
   libreoffice --headless --convert-to pdf output.pptx
   rm -f slide-*.jpg
   pdftoppm -jpeg -r 150 output.pdf slide
   ```
   如果 LibreOffice 不可用，跳过视觉检查，将文件直接提供给用户人工审查。
3. **审查要点**：
   - 文字是否清晰可读（对比度够不够）
   - 图片是否正确插入且不变形
   - 每页是否只有一个核心观点
   - IP 角色是否贯穿始终
   - 整体色调是否和谐统一

## 依赖

- **pptx 技能**（用于最终 PPTX 生成）— 如果环境中有 pptx 技能，先阅读其 SKILL.md 了解 PptxGenJS API；如果没有，用 `npm install pptxgenjs` 后直接编写 Node.js 脚本生成
- **图片生成工具**（DALL-E / 其他可用的图片生成 MCP）— 如果没有图片生成能力，将 Prompt 整理成文档交给用户自行生成
- `pip install "markitdown[pptx]"` — 内容检查用
- `npm install pptxgenjs` — PPTX 创建
- LibreOffice + Poppler — 视觉检查（可选，不可用时跳过）
