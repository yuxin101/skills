# 性能指标说明

## 核心 Web 指标
- 最大内容绘制时间（LCP）
  - 优秀：<= 2.5 秒
  - 需改进：2.5 秒 - 4.0 秒
  - 较差：> 4.0 秒

- 交互到下次绘制（INP）
  - 优秀：<= 200 毫秒
  - 需改进：200 毫秒 - 500 毫秒
  - 较差：> 500 毫秒

- 累积布局偏移（CLS）
  - 优秀：<= 0.1
  - 需改进：0.1 - 0.25
  - 较差：> 0.25

## 辅助指标
- 首次内容绘制（FCP）
  - 优秀：<= 1.8 秒
  - 需改进：1.8 秒 - 3.0 秒
  - 较差：> 3.0 秒

- 首字节时间（TTFB）
  - 优秀：<= 0.8 秒
  - 需改进：0.8 秒 - 1.8 秒
  - 较差：> 1.8 秒

- DOMContentLoaded（DCL）
  - 优秀：<= 1.5 秒
  - 需改进：1.5 秒 - 3.0 秒
  - 较差：> 3.0 秒

- 总阻塞时间（TBT）
  - 优秀：<= 200 毫秒
  - 需改进：200 毫秒 - 600 毫秒
  - 较差：> 600 毫秒

- 请求总数
  - 优秀：<= 80
  - 需改进：80 - 150
  - 较差：> 150

- JavaScript 总体积
  - 优秀：<= 300 KB
  - 需改进：300 KB - 800 KB
  - 较差：> 800 KB

- CSS 总体积
  - 优秀：<= 100 KB
  - 需改进：100 KB - 250 KB
  - 较差：> 250 KB

- 图片总体积
  - 优秀：<= 500 KB
  - 需改进：500 KB - 2 MB
  - 较差：> 2 MB

- 长任务数量
  - 优秀：0 - 2
  - 需改进：3 - 8
  - 较差：> 8

## 指标作用说明
- LCP 主要反映首屏主要内容是否出现得足够快。
- INP 主要反映页面在点击、输入、交互时是否足够流畅。
- CLS 主要反映页面是否存在明显布局抖动。
- FCP 主要反映用户第一次看到内容的时间。
- TTFB 主要反映服务端响应和首包返回速度。
- DCL 主要反映 DOM 结构完成解析的时间，常用于判断 HTML 体积、阻塞脚本和 DOM 构建压力。
- TBT 主要反映主线程长时间被 JavaScript 占用的程度。
- 请求总数、资源体积和长任务数量用于辅助定位性能瓶颈来源。

## 优先级规则
- 当 LCP 较差时，优先检查首屏图片、服务端响应时间、阻塞渲染资源。
- 当 INP 较差时，优先检查主线程 JavaScript、事件处理函数、长任务和第三方脚本。
- 当 CLS 较差时，优先检查图片尺寸、广告位占位、异步插入内容和字体加载。
- 当 DCL 较差时，优先检查 HTML 体积、同步脚本、head 中的第三方脚本和 DOM 节点数量。
- 当 TTFB 较差时，优先怀疑服务端处理、缓存策略、边缘节点和网络链路。
- 当 TBT 较差时，优先检查大体积 JavaScript、Hydration、长任务拆分和第三方脚本执行。

## 指标关系分析
- 如果 DCL 高但 FCP 不高，通常说明 DOM 构建或同步脚本阻塞较明显。
- 如果 DCL 和 FCP 都高，通常说明初始 HTML、网络或阻塞资源整体偏重。
- 如果 DCL 不高但 LCP 很高，通常优先怀疑首屏大图、关键资源优先级或渲染延迟。
- 如果 TTFB 很高，LCP 往往也会受到拖累。
- 如果 TBT 和 INP 都较差，通常说明主线程 JavaScript 负担过重。
