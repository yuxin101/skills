# 各阶段标准提示词模板

## Phase 1 — 需求理解与论文解析提示

```
你正在对一篇期刊论文进行系统性重新整理。

首先，请完成以下任务：
1. 识别论文的【核心研究问题】（一句话）
2. 识别论文的【研究方法】（简要）
3. 识别论文的【主要结论】（简要）
4. 提炼全文的【核心命题】（一句话，贯穿全文的最高层论点）

输出格式：
---
核心研究问题：
研究方法：
主要结论：
全文核心命题（一句话）：
---
```

---

## Phase 2 — 大纲提取提示

```
基于上述论文内容，提取并重新整理大纲。

要求：
- 大纲层级不超过三级（章 > 节 > 小节）
- 遵循学术论文规范结构：摘要/引言/文献综述/方法/结果/讨论/结论
- 每个标题后附一句话说明该部分的核心功能

输出格式（示例）：
---
一、引言
  功能：提出研究问题，说明研究意义，指出文献空白
  1.1 研究背景
  1.2 研究问题与意义
  1.3 文章结构

二、文献综述
  功能：梳理相关研究，指出现有局限
  ...
---
```

---

## Phase 3 — 大纲评审提示

```
针对上述大纲，从以下维度提出改进意见：

评审维度（参考 references/outline_criteria.md）：
1. 结构合理性：层级是否清晰，比例是否均衡
2. 逻辑一致性：各章是否服务于核心命题，衔接是否自然
3. 学术规范性：是否符合期刊论文写作规范
4. 创新性：研究贡献是否在大纲中有充分体现

输出格式：
---
【总体评价】（2-3句话）

【结构问题】
- 问题1：...  建议：...
- 问题2：...  建议：...

【逻辑问题】
- ...

【规范问题】
- ...

【建议修订大纲】
（给出修改后的完整大纲）
---
```

---

## Phase 4 — 章节分割提示

```
根据确认后的大纲，将论文原文按章节分割，每个章节作为独立处理单元。

分割规则：
- 以一级标题（章）为基本分割单位
- 若某章内容过长（超过 1500 字），可按二级标题（节）进一步细分
- 为每个处理单元标注：章节编号、章节标题、字数、章节功能

输出格式：
---
【分割清单】
单元 1：一、引言（约 XXX 字）
功能：...
原文片段：[摘录前50字]...

单元 2：二、文献综述（约 XXX 字）
...
---
```

## Phase 5 — 单章深化提示（循环使用，多轮迭代）

```
现在对【第 N 章：[章节标题]】进行深化。

深化前必须确认：
- 全文核心命题：[填入 Phase 1 提炼的核心命题]
- 本章在大纲中的功能：[填入章节功能描述]
- 本章原始内容摘要：[简要列出当前要点]

### 多轮深化流程（按顺序执行）：

【第一轮：梳理现有论点】
- 列出本章当前的核心论点
- 识别每个论点的论证缺口（论据不足？缺乏实证？逻辑薄弱？）

【第二轮：文献搜集与理论支撑】
- 针对每个论证缺口，搜集中外文献：
  - 中文文献来源：CNKI、万方、维普
  - 外文文献来源：Web of Science、Google Scholar、PubMed、arXiv
- 优先选取近 5 年高质量期刊论文和经典文献
- 将文献中的理论、方法、数据作为本章论证的支撑

【第三轮：反复论证、层层递进】
- 整合文献与本章论点
- 每补充一个论据，进行 1-2 轮论证迭代
- 确保论点 → 论据 → 结论链条完整

【第四轮：凝练总结】
- 段首呼应全文核心命题
- 段末用过渡句点明本章与整体论证的关系
- 形成本章的完整闭环

### 四大铁律（严格遵守）：
1. 【聚焦本章】：不引入其他章节的具体论点或结论，本章独立完成深化
2. 【联系核心命题】：本章每个论点须与全文核心命题建立显式联系
3. 【不跑题】：所有内容须在本章标题范畴内，无关内容一律删除
4. 【不重复】：不重复论证已在其他章节出现过的内容；若需引用，用"如前所述"带过

### 深化后自查清单：
□ 无超出本章范畴的内容
□ 核心命题在本章中有所呼应
□ 无与其他章节重复的论点或数据
□ 段落间过渡自然
□ 语言符合学术规范
□ 文献引用格式符合目标期刊要求

### 输出格式：
---
【本章核心命题】：[呼应全文核心命题的一句话]

【第一轮 - 现有论点与缺口】：
[列出论点及缺口]

【第二轮 - 文献支撑】：
[列出搜集的文献及用于支撑的论点]

【第三轮 - 论证深化】：
[整合后的深化内容]

【第四轮 - 凝练总结】：
[本章最终文本]

【自查报告】
□ 跑题检查：通过 / 发现问题（说明）
□ 重复检查：通过 / 发现问题（说明）
□ 命题联系：已在第X段明确呼应
□ 文献支撑：已引用X篇中文文献、X篇外文文献
---
```

## Phase 6 — 全文整合与最终检查提示

```
所有章节深化完毕后，进行全文整合与一致性检查。

检查项目：
1. 论点一致性：全文核心命题是否贯穿始终
2. 重复内容：扫描全文，标记重复出现的论点、数据、案例
3. 章节衔接：检查相邻章节过渡是否自然
4. 摘要与结论一致性：摘要的结论与第N章结论是否匹配
5. 引言与结论呼应：引言中的 Gap 是否在结论中得到回应

输出格式：
---
【整合检查报告】
重复内容：[列出位置和内容]
衔接问题：[列出位置和建议]
一致性问题：[列出]

【建议修订项】
[按章节列出需要微调的内容]
---
```

---

## Phase 7 — 章节合并与 Word 输出提示

```
所有章节深化完成并通过一致性检查后，将各章内容合并为完整论文并输出 Word 文档。

### 合并前确认
- [ ] 所有章节已完成深化
- [ ] 各章已通过自查报告
- [ ] Phase 6 一致性检查已完成
- [ ] 用户已提供目标期刊的引用格式要求

### 合并流程
1. 按大纲顺序排列各章内容
2. 添加论文必要元素：
   - 论文标题
   - 摘要（200-300 字，含目的、方法、结论）
   - 关键词（3-5 个）
   - 参考文献（统一格式）
3. 检查章节衔接，确保过渡自然

### Word 格式要求
- 页面：A4，上下 2.54cm，左右 3.17cm
- 字体：中文宋体，英文 Times New Roman
- 标题：一级 16pt 加粗，二级 14pt 加粗，三级 12pt 加粗
- 正文：小四（12pt），1.5 倍行距
- 引用格式：[用户指定 APA 或 GB/T 7714]

### 输出格式：
---
【合并检查清单】
□ 所有章节已按大纲顺序排列
□ 摘要/关键词/参考文献已添加
□ 章节过渡自然
□ 格式符合期刊要求

【Word 文档信息】
文件名：[论文标题].docx
总字数：约 XXX 字
引用格式：[APA/GB/T 7714]

【下一步】
使用 docx skill 生成 .docx 文件
---
```
