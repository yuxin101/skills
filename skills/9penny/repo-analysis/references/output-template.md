# Repo Analysis Output Templates

按用户意图选择一种，不必每次都全写。

## 模式选择速记

- 默认读项目 / 看 repo / 这个项目怎么样 → 速读版
- 默认看主链路 / 模块边界 / 架构分析 / 怎么运行 → 架构版
- 默认值不值得接手 / 能不能二开 / 风险大不大 / 接手评审 → 接手评审版
- 用户显式指定某一版时，以用户指定为准

## 范围识别速记

先判断目标是：
- 整仓 / monorepo
- 子项目 / service / package
- 单模块 / 单目录

整仓 + 速读版：
- 只给第一层模块地图
- 只追一条主链路
- 明确提示可以继续展开成架构版或接手评审版

## GitHub 轻量健康度层

仅在以下条件同时满足时补充：
- 目标是公共 GitHub 仓库
- 用户明显在问“值不值得看 / 跟 / 用 / 参考 / 二开”

只补短块，不单独展开成长报告。
建议只包含：
- stars / forks / license
- 最近 commit / release 节奏
- issue / PR 是否仍在流动
- README / docs / CONTRIBUTING 是否清楚

## 3. 接手评审版

### 问题判断
- 这项目适不适合接手 / 二开 / 参考
- 当前工程成熟度在哪一档

### 原因分析
- 文档质量
- 模块边界
- 测试与部署信号
- 关键复杂度来源
- 历史包袱和维护风险

### 接手评审结论
- 项目定位
- 技术栈与部署形态
- 高风险模块
- 相对安全的切入点
- 接手成本
- 二开建议
- 不建议贸然改动的区域

### 建议方案
- 新人第一阶段阅读顺序
- 第一刀从哪里切
- 上线前要补哪些验证
