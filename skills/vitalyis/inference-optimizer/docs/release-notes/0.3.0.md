# Release Notes: v0.3.0

## Summary

This release refreshes `inference-optimizer` as a runtime-audit-first OpenClaw skill. The command surface stays the same, but the guidance now matches real operational failures seen on production VPS setups.

## Highlights

- Runtime checks now come before tuning suggestions:
  - gateway ownership and duplicate supervisors
  - restart loops and failed services
  - resolved `openclaw` binary path and install type
  - workspace command wiring for the installed skill path
  - updater status and allowlist coverage
  - plugin provenance and unused local extensions
- Diagnosis guidance is stricter:
  - warnings are not root cause by themselves
  - partial or truncated updater output is inconclusive until version, service state, and logs are checked
- Documentation is leaner:
  - `README.md` now follows a lighter landing-page structure
  - `SKILL.md` now uses the social preview image and clearer workflow branches
- Installer behavior is safer:
  - `setup.sh` now replaces managed workspace blocks instead of only appending once
  - `verify.sh` now fails on dead legacy paths such as `skills/public` and repo-local script wiring
  - `/audit` now includes recommended next steps instead of raw token numbers only
