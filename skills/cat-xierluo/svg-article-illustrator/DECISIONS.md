# 决策记录

本文档记录 svg-article-illustrator skill 开发过程中的重要技术决策和设计选择。

## [DEC-001] - 2026-02-07 - 三种输出模式整合设计

### 背景

原始的 illustrate 命令经历了三个版本演进：v1.8 生成 SVG 文件并转 PNG、v2.0.1 直接嵌入静态 SVG、v3.0 添加动态 SVG 效果。如何在一个 skill 中支持这三种不同的输出方式是关键设计决策。

### 选项

1. **分离为三个独立 skill**
   - 优点：职责清晰，每个 skill 专注一种输出方式
   - 缺点：代码重复，维护成本高，用户体验分散

2. **单一模式 + 命令行参数**
   - 优点：统一的入口
   - 缺点：需要在命令中记忆参数，不够直观

3. **多模式 skill + 自动选择（已选择）**
   - 优点：统一入口，AI 根据用户场景自动选择模式，保持向后兼容
   - 缺点：需要明确的模式选择指南

### 决策

选择**多模式 skill + 自动选择**：

- 保留三种输出模式：dynamic-svg（默认）、static-svg、png-export
- 在 SKILL.md 中提供清晰的模式选择指南
- AI 根据用户需求和发布平台自动选择合适的模式
- 使用 Progressive Disclosure 设计，共享核心设计原则

### 理由

1. **向后兼容**：保留原有三种使用方式
2. **用户体验**：AI 自动选择，用户无需记忆参数
3. **代码复用**：共享核心设计原则和尺寸标准
4. **可扩展性**：未来可以添加新的输出模式

### 影响

- 创建 references/ 目录存储模式特定的规范
- core-principles.md 存储共享内容
- SKILL.md 包含模式选择指南和核心工作流程
- 默认使用 dynamic-svg 模式

## [DEC-002] - 2026-02-07 - Progressive Disclosure 设计

### 背景

三个版本的 illustrate 命令文档都非常长（v3.0 有 440+ 行），如果全部合并到 SKILL.md 会导致 context 消耗过大。如何在保持功能完整性的同时优化 token 使用是关键问题。

### 选项

1. **全部内容放入 SKILL.md**
   - 优点：单文件包含所有信息
   - 缺点：context 消耗大，违反 Progressive Disclosure 原则

2. **按输出模式拆分文档（已选择）**
   - 优点：按需加载，优化 token 使用
   - 缺点：需要清晰的文档组织

3. **使用 LLM 摘要技术**
   - 优点：大幅减少 token
   - 缺点：丢失细节信息

### 决策

选择**按输出模式拆分文档**：

- SKILL.md：模式选择指南 + 核心工作流程（<100 行）
- core-principles.md：共享设计原则（所有模式加载）
- dynamic-svg.md：动态 SVG 特定规范（默认模式加载）
- static-svg.md：静态 SVG 特定规范（按需加载）
- png-export.md：PNG 导出特定规范（按需加载）

### 理由

1. **Token 优化**：只加载当前模式需要的文档
2. **可维护性**：共享内容避免重复
3. **扩展性**：添加新模式时只需新增 reference 文件

### 影响

- 创建 references/ 目录结构
- SKILL.md 保持精简（<100 行）
- 各模式文档独立维护
- 需要在 SKILL.md 中提供清晰的引用链接

## [DEC-003] - 2026-02-07 - 模板文件组织方式

### 背景

原始命令中包含大量 SVG 代码示例和动画片段。如何组织这些模板文件以便 AI 在生成配图时快速参考是关键。

### 选项

1. **将模板嵌入 reference 文档**
   - 优点：减少文件数量
   - 缺点：reference 文档变得很长

2. **创建独立的 templates/ 子目录**
   - 优点：清晰的文件组织
   - 缺点：增加目录层级

3. **直接放在 assets/ 根目录（已选择）**
   - 优点：扁平结构，易于访问
   - 缺点：assets 目录可能包含多种类型文件

### 决策

选择**直接放在 assets/ 根目录**：

- animations.smil：SMIL 动画代码片段
- layouts.svg：常用布局模板
- 扁平结构，便于快速访问

### 理由

1. **简洁性**：避免不必要的目录嵌套
2. **可访问性**：AI 可以直接引用模板文件
3. **扩展性**：未来可以添加更多模板类型

### 影响

- assets/ 目录直接包含模板文件
- 无 templates/ 子目录
- 模板文件使用 .smil 和 .svg 扩展名便于识别
