# AI 日报生成 Skill

只做一件事：`从原始新闻表 → 筛选高价值新闻 → 阅读原文 → 生成日报文档和早报导读`

不负责：抓取公开网页、写入原始新闻表、维护新闻源配置 → 交给 `ai_news_fetcher`

## 核心规则

日报 = 汇总过去 24 小时的原始新闻，提炼成 signal 文档。默认时间窗口是过去 24 小时，可由用户指定其他范围。

## 前置依赖

- **飞书多维表格**：需要原始新闻表（`bitable_target.json`）和关注清单表（`watch_target.json`），将表的 `app_token`、`table_id` 等信息填入对应配置文件（参考 `.example.json`）
- **ai_news_fetcher**：日报依赖 fetcher 采集的原始新闻数据
- **飞书 wiki 知识库**：日报文档挂载到飞书知识库（可选）

> **凭证说明**：本 skill 不内置任何飞书凭证。配置文件由用户在本地创建，已被 `.gitignore` 排除。仓库中的 `.example.json` 仅包含占位符，不含真实令牌。

> **第三方服务**：日报生成时，高价值事件需要阅读原文全文。本 skill 使用 [Jina Reader](https://r.jina.ai)（免费、无需 API key）读取公开网页原文。目标文章的 URL 会发送到 `r.jina.ai`，不会发送飞书凭证或用户数据。

> **跨 skill 文件**：本 skill 拥有 `references/watch_chains.md`（产业链观察图谱）和 `watch_target.json`（关注清单表配置），`ai_weekly_reporter` 会读取并更新 `watch_chains.md`。这是设计上的协作关系：日报产出信号 → 周报合成趋势并更新观察链判断基线。

## 执行前必读

- `references/reporting.md` — 执行流程、高价值判断标准、写作规则、门控检查点
- `references/output-template.md` — 日报和早报的输出模板与风格要求
- `references/watch_chains.md` — 产业链关注图谱（筛选高价值事件时参考）
- `references/gotchas.md` — 仅排障时读
- `bitable_target.json` — 原始新闻表配置
- `watch_target.json` — 关注清单表配置

## 管道（不可跳步）

```
读取关注清单 + 产业链图谱
    ↓
读取过去24h原始新闻 → 过滤去重
    ↓
结合关注上下文筛选高价值事件 → 阅读全文
    ↓
按模板生成signal日报 → 保存本地MD副本
    ↓
生成早报导读 → 更新关注清单
```

## 群聊分析反馈记录

当用户在群聊中对分析质量给出反馈时（包括但不限于：指出分析不好、要求换角度重做、追问出更好的分析框架、明确说"这个好/不好"），必须将反馈记录到周报 skill 的分析框架反馈日志中：

**记录位置**：`../ai_weekly_reporter/references/analysis_framework.md` → "反馈日志"表格

**记录格式**：追加一行 `| 日期 | 分类 | 反馈内容（一句话概括） | 来源场景（谁说的、在什么分析上下文） |`

**分类**：填 `日报`、`周报` 或 `深入分析`，根据反馈针对的是哪层产出来判断。

**不要当场修改框架**，只记录。框架的实际修订由周报生成流程（Step 9）统一处理。

**触发条件**：
- 用户明确否定分析结果（"这个不好"、"太抽象了"、"套框架了"）
- 用户要求换视角重做（"从供应链角度看呢"、"技术树要看具体路径"）
- 用户肯定某种分析方式并要求固化（"保持这个质量"、"以后都这么分析"）
- 用户追问出了框架里没有的分析角度

**不触发**：普通的事实性追问（"这个数据来源是？"）、闲聊、与分析框架无关的反馈。

## 硬约束

1. 不做新闻采集（不写入原始新闻表），但高价值事件必须打开原文链接阅读全文
2. 高价值事件必须先阅读全文，再输出判断；证据不足写"待验证问题"
3. 日报必须严格按 `references/output-template.md` 生成，禁止写成"TOP N / 核心事件汇总"式简报
4. 时间范围过滤由查询层控制（优先用 `报道窗口键`），不交给大模型自由判断
5. 原始新闻表只读，不批量回写、删除、重置
6. cron 模式发群，飞书私聊模式只发给发起人
7. 早报是导读不是缩写版，默认地点 `福州`
8. 日报生成后，同时保存一份到 `workspace/daily_reports/YYYY-MM-DD.md`
9. 日报产出的"待验证问题"写入关注清单表；已被本轮回答的问题标记为"已回答"
10. 任务结束需附过程日志：读了多少条、去重多少、最终选了哪些高价值事件
