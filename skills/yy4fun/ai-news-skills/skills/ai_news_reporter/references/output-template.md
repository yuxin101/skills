# 输出模板

默认产出两类内容：

1. 完整日报文档
2. 早报导读消息

## 一、日报文档模板

文档标题：

- `AI日报｜YYYY-MM-DD`

文档开头建议写明：

- 统计范围
- 原始记录数
- 去重后记录数
- 进入正文的高价值事件数

## 二、早报导读消息模板

### 推荐结构

```markdown
早上好。{date}，{weather_summary}。
{greeting_sentence}


**今天最值得先看的 3 个信号**

**1. {signal_1_headline}**
发生了什么：
{signal_1_what_happened}

为什么今天先看它：
{signal_1_why_now}

---

**2. {signal_2_headline}**
发生了什么：
{signal_2_what_happened}

为什么今天先看它：
{signal_2_why_now}

---

**3. {signal_3_headline}**
发生了什么：
{signal_3_what_happened}

{signal_3_why_now}

{open_question_with_reason}

{doc_link}

```

## 三、风格要求

- 日报是 signal 文档，不是热点清单
- 早报是晨间导读，不是机器播报
- 少用“重磅 / 爆了 / 史诗级 / all in / 前夜已至”这类过热词
- 判断可以有，但要克制，优先说明“为什么这条信号重要”
- 开场要像晨间便签，不要像天气播报或客服通知
- `{greeting_sentence}` 是一句动态生成的问候，结合日期（周几、节气、节日）、天气、当天新闻氛围自由发挥，每天不重样。可以是一句感慨、一个轻松的观察、一句和当天内容相关的引子。一句话就好，不要写成段落
- 允许有一点轻松感和人味，但不要鸡汤、不要油腻
- 导读的目标是让人愿意继续点开日报和原文，不是把全文缩写一遍
- 版式上优先使用加粗标题和分隔线，方便在群聊里快速扫读
