# 产品接入建议（中文）

## 推荐触发点

### 方案 A：语音输入结束后自动润色

1. 用户使用输入法麦克风说话。
2. 输入法自身 ASR 输出一段文本。
3. 语音输入结束。
4. 将整段文本传给本技能。
5. 返回书面化后的消息文本。
6. 回填输入框，等待用户发送。

这是最适合当前场景的接法。

### 方案 B：原文先入框，再手动点“润色”

1. 输入法先把识别文本放入输入框。
2. 用户点击“润色”或“整理成书面语”。
3. 调用本技能。
4. 用返回结果覆盖输入框中的文本。

## UI 建议

- 回填到输入框，不要直接发送。
- 保留一次撤销能力，方便用户回到原始识别文本。
- 对非常短的文本（如“好”“到了”“收到”）可直接跳过润色。
- 若接口失败，可保留原文并提示用户手动修改。

## 鉴权建议

- 不要把 `api token` 写进技能包。
- 在运行环境中设置 `IME_MODEL_API_KEY`。
- 若有多环境，使用 `IME_MODEL_BASE_URL`、`IME_MODEL_NAME` 区分配置。

## 建议的产品时序

```text
语音输入结束
→ 获取识别文本
→ 使用 OpenAI SDK 调用 https://models.audiozen.cn/v1
→ 执行 client.chat.completions.create(...)
→ 取 choices[0].message.content
→ 回填输入框
→ 用户确认发送
```

## 适合的文本类型

- 迟到说明
- 请求帮忙
- 确认回复
- 提醒通知
- 简单解释
- 婉拒或改期

## 不建议直接自动发送的原因

即使书面化效果很好，也仍然建议让用户最后看一眼，因为：

- 识别文本可能有误
- 用户可能想手动调整语气
- 某些对话场景对措辞比较敏感
