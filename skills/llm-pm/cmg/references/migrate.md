# CMG 迁移执行引导 (cmg-migrate)

完成资源扫描、选型推荐、TCO 分析后，根据资源类型引导用户使用腾讯云对应迁移工具执行迁移。

---

## 迁移工具速查

| 资源类型 | 迁移工具 | 入口路径 |
|----------|----------|----------|
| **主机**（云服务器 ECS/EC2/ECS 等） | 腾讯云迁移平台 → 主机迁移 | 腾讯云控制台 → 云迁移 → 主机迁移 |
| **对象存储**（OSS/S3/OBS 等） | 腾讯云迁移平台 → 对象存储迁移（MSP） | 腾讯云控制台 → 云迁移 → 对象存储迁移 |
| **数据库**（RDS/MySQL/Redis 等） | 数据传输服务 DTS | 腾讯云控制台 → DTS |
| **文件存储**（NAS/EFS/SFS 等） | 腾讯云迁移平台 → 文件存储迁移（使用迁移集群） | 腾讯云控制台 → 云迁移 → 文件存储迁移 |

---

## 各类型迁移说明

### 主机迁移
- **工具**：腾讯云迁移平台 → 主机迁移
- **适用**：将源端云服务器（阿里云 ECS、AWS EC2、华为云 ECS、IDC 物理机/虚拟机）迁移到腾讯云 CVM
- **方式**：在源端安装迁移 Agent，通过迁移平台控制台创建迁移任务

### 对象存储迁移
- **工具**：腾讯云迁移平台 → 对象存储迁移（MSP）
- **适用**：将源端对象存储（阿里云 OSS、AWS S3、华为云 OBS 等）迁移到腾讯云 COS
- **方式**：在迁移平台控制台配置源端 AK/SK 和目标 COS Bucket，创建迁移任务

### 数据库迁移
- **工具**：数据传输服务 DTS
- **适用**：MySQL、PostgreSQL、Redis、MongoDB、SQL Server 等数据库迁移
- **方式**：在 DTS 控制台创建迁移任务，配置源端和目标端连接信息

### 文件存储迁移
- **工具**：腾讯云迁移平台 → 文件存储迁移（使用迁移集群）
- **适用**：将源端文件存储（阿里云 NAS、AWS EFS、华为云 SFS 等）迁移到腾讯云 CFS
- **方式**：通过迁移集群部署迁移节点，挂载源端和目标端文件系统进行数据同步

---

## 引导话术

当用户完成推荐/TCO 分析后，询问如何迁移时，根据其资源类型给出对应工具入口：

- 有**主机**资源 → 引导使用**腾讯云控制台 → 云迁移 → 主机迁移**
- 有**对象存储**资源 → 引导使用**腾讯云控制台 → 云迁移 → 对象存储迁移**
- 有**数据库**资源 → 引导使用**腾讯云控制台 → DTS**
- 有**文件存储**资源 → 引导使用**腾讯云控制台 → 云迁移 → 迁移集群**
