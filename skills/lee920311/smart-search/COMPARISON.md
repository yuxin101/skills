# Smart Search - 搜索引擎对比与决策逻辑

## 🔍 SearX 1.1.0 vs Tavily

### 核心对比

| 特性 | SearX 1.1.0 | Tavily |
|------|------------|--------|
| **费用** | 🆓 完全免费 | 💰 免费 1000 次/月 |

| **响应速度** | ⚡ ~2-3s | ⚡⚡ ~1-1.5s |
| **AI 摘要** | ❌ 无 | ✅ 有（核心优势） |
| **搜索结果** | 🔍 多引擎聚合 | 🔍 精选高质量 |
| **中文支持** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **稳定性** | ⭐⭐⭐⭐（依赖本地） | ⭐⭐⭐⭐⭐（云服务） |
| **隐私性** | 🔒 完全本地 | 🔒 需传数据到云端 |
| **可定制性** | 🔧 高（可配置引擎） | ⚙️ 低（固定 API） |
| **维护成本** | 📦 需维护容器 | ✅ 零维护 |

## 🧠 智能决策流程

```
用户搜索请求
    ↓
┌─────────────────────────────────┐
│  1. 检查用户是否指定引擎？      │
│     - "用 tavily 搜索" → Tavily  │
│     - "用 searx 搜索" → SearX   │
└─────────────────────────────────┘
    ↓ 未指定
┌─────────────────────────────────┐
│  2. 分析搜索意图                │
│     - AI 内容生成/小红书/文案    │
│       → 优先 Tavily（AI 摘要）   │
│     - 日常查询/调研/简报         │
│       → 优先 SearX（免费）       │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│  3. 执行搜索（带超时）          │
│     - SearX: 10 秒超时          │
│     - Tavily: 15 秒超时         │
└─────────────────────────────────┘
    ↓ 失败
┌─────────────────────────────────┐
│  4. 自动降级                    │
│     - SearX 失败 → Tavily       │
│     - Tavily 失败 → 报错        │
└─────────────────────────────────┘
    ↓
返回结果 + 日志记录
```

## 🔧 配置建议

### ~/.openclaw/.env
```bash
# SearX 本地实例（主力）
SEARXNG_URL=http://localhost:8080

# Tavily API Key（备用 + AI 内容）
TAVILY_API_KEY=tvly-xxx

# 备用 Key（可选，额度用完后轮换）
TAVILY_API_KEY_2=tvly-yyy
TAVILY_API_KEY_3=tvly-zzz
```

### Docker 部署
```bash
# SearX 1.1.0（稳定版，无 bot 检测）
docker run -d --name searx \
  -p 8080:8080 \
  -e SEARX_SECRET='local-secret-2026' \
  --restart unless-stopped \
  searx/searx:1.1.0-69-75b859d2
```

## 💡 最佳实践

### ✅ 推荐做法
1. **默认用 SearX** - 90% 场景免费搞定
2. **AI 内容用 Tavily** - 利用 AI 摘要优势
3. **配置备用 Key** - 防止单点故障

5. **记录降级日志** - 便于问题排查

### ❌ 避免做法
1. ~~全部用 Tavily~~ - 额度不够用

3. ~~不部署 SearX~~ - 失去免费搜索能力
