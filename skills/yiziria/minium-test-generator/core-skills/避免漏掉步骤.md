# 避免漏掉步骤的技能

**版本**: 1.0.0  
**创建时间**: 2026-03-23  

---

## 🎯 技能目标

确保在将录制脚本转换为测试用例时：
1. ✅ **不漏掉任何步骤**
2. ✅ **不简化特殊逻辑**（如三层判断）
3. ✅ **不混淆相邻步骤的元素**

---

## 📋 核心方法

### 方法 1: 步骤编号对照法

**操作**: 为录制脚本的每个步骤编号，生成检查清单

**工具**: `scripts/parse_steps.py`

**步骤**:
```bash
# 1. 运行解析脚本
python scripts/parse_steps.py \
    --input recorded_script.py \
    --output steps_checklist.md

# 2. 查看生成的步骤清单
cat steps_checklist.md
```

**输出示例**:
```markdown
# 步骤清单

## 普通步骤 (25 个)
- step_1: 点击首页按钮
- step_2: 滚动页面
...

## ⚠️ 特殊逻辑步骤 (5 个)
- step_14: 点击设置按钮【三层判断】
- step_15: 选择选项 A【else 判断】
- step_16: 点击确认图标【else 判断】
...

## 🔄 循环步骤 (2 组)
- step_9-13: 滚动页面 (5 次)
- step_20-23: 滚动页面 (4 次)
```

**关键点**:
- ✅ 特殊步骤会被标记 ⚠️
- ✅ 循环步骤会被分组
- ✅ 每个步骤都有唯一编号

---

### 方法 2: 逐行复制法

**操作**: 先把录制脚本的代码完整复制到测试用例，再重构

**步骤**:

#### 第 1 步：完整复制（不改任何逻辑）

```python
def set_option_with_mode(self, value):
    # step_14: 完整复制录制脚本
    element_is_exists = self.page.element_is_exists("view.setting-btn...")
    if element_is_exists:
        element = self.page.get_element("view.setting-btn...")
    else:
        element_is_exists = self.page.element_is_exists(xpath="/page-wrapper/...")
        if element_is_exists:
            element = self.page.get_element_by_xpath("/page-wrapper/...")
        else:
            element = self.page.get_element_by_xpath("//*", inner_text="未设置")
    element.tap()
    
    # step_15: 完整复制录制脚本
    element_is_exists = self.page.element_is_exists(xpath="/page-wrapper/.../view[2]/view")
    if element_is_exists:
        element = self.page.get_element_by_xpath("/page-wrapper/.../view[2]/view")
    else:
        element = self.page.get_element_by_xpath("//*", inner_text="选项 A")
    element.tap()
    
    # step_16: 完整复制录制脚本
    element = self.page.get_element_by_xpath("/page-wrapper/.../icon[2]")
    element.tap()
    
    # ... 更多步骤
```

#### 第 2 步：验证每个步骤的 XPath 都不同

```bash
# 使用工具检查
python scripts/validate_steps.py \
    --input recorded_script.py \
    --test testcase/xxx/test_xxx.py
```

#### 第 3 步：再考虑是否需要提取公共方法

**只有在验证通过后才考虑重构**

---

### 方法 3: 相邻步骤对比检查

**操作**: 对连续步骤，必须对比元素 XPath 是否相同

**工具**: `steps_checklist.md` 中的"相邻步骤检查"部分

**步骤**:
```markdown
## 相邻步骤检查

- [ ] step_14 和 step_15 的元素 XPath 是否不同？
  - step_14 XPath: `/page-wrapper/.../setting-btn`
  - step_15 XPath: `/page-wrapper/.../view[2]/view`
  - 结论：✅ 不同

- [ ] step_15 和 step_16 的元素 XPath 是否不同？
  - step_15 XPath: `/page-wrapper/.../view[2]/view`
  - step_16 XPath: `/page-wrapper/.../icon[2]`
  - 结论：✅ 不同
```

**规则**:
- ✅ 如果不同，打勾 ✅
- ❌ 如果相同，必须标注原因
- ⚠️ 如果不确定，运行 `validate_steps.py` 检查

---

### 方法 4: 元素定义分离

**操作**: 不同步骤的元素，即使名称相似，也要分开定义

**错误案例**:
```python
# ❌ 错误：同一个变量用于两个步骤
option_icon = "..."  # step_15 和 16 都用这个
```

**正确做法**:
```python
# ✅ 正确：分开定义
class SettingPage(BaseDef):
    # step_15: 选项入口按钮
    option_entrance = "/page-wrapper/.../view[2]/view"
    
    # step_16: 弹窗中的选项图标
    option_icon = "/page-wrapper/.../icon[2]"
    
    # step_17: 确认按钮
    confirm_btn = "/page-wrapper/.../footer-button/..."
```

**命名规则**:
- 入口按钮：`xxx_entrance`
- 图标：`xxx_icon`
- 确认按钮：`confirm_xxx`

---

### 方法 5: 特殊逻辑标记法

**操作**: 识别并标记有特殊逻辑的步骤

**工具**: `parse_steps.py` 自动识别

**标记类型**:
- `⚠️ 三层判断` - 有 2 个以上 if 判断
- `⚠️ else 判断` - 有 else 分支
- `🔄 循环` - 滚动等重复操作
- `📝 输入` - 文本输入操作
- `🔗 页面跳转` - 有页面跳转验证

**处理方式**:
- ⚠️ 特殊逻辑步骤：**必须逐行对照录制脚本**
- 🔄 循环步骤：**必须确认循环次数**
- 🔗 页面跳转：**必须添加等待**

## 🛠️ 工具使用

### parse_steps.py

**功能**: 解析录制脚本，生成步骤清单

**用法**:
```bash
python scripts/parse_steps.py \
    --input recorded_script.py \
    --output steps_checklist.md
```

**输出**:
- 普通步骤列表
- 特殊逻辑步骤列表（带⚠️标记）
- 循环步骤列表（带🔄标记）
- 步骤对照表模板

---

### validate_steps.py

**功能**: 验证测试用例是否漏掉步骤或简化逻辑

**用法**:
```bash
python scripts/validate_steps.py \
    --input recorded_script.py \
    --test testcase/xxx/test_xxx.py \
    --report validation_report.md
```

**检查项**:
1. ✅ 步骤数量是否一致
2. ⚠️ **新增**: 相邻步骤是否使用了相同的元素
3. ⚠️ **新增**: 特殊逻辑是否保留
4. ⚠️ **新增**: 页面跳转是否都有等待

## ⚠️ 常见错误及避免方法

### 错误 1: 漏掉特殊逻辑

**案例**: step_14 的三层判断被简化成一层

**避免方法**:
1. ✅ 使用 `parse_steps.py` 标记特殊逻辑步骤
2. ✅ 使用逐行复制法，不改任何逻辑
3. ✅ 运行 `validate_steps.py` 检查特殊逻辑

---

### 错误 2: 混淆相邻步骤的元素

**案例**: step_15 和 step_16 使用同一个元素

**避免方法**:
1. ✅ 填写步骤对照表的"元素是否相同"列
2. ✅ 运行 `validate_steps.py` 的相邻步骤检查
3. ✅ 分开定义不同步骤的元素变量

---

### 错误 3: 漏掉页面跳转等待

**案例**: 页面跳转后没有等待，元素找不到

**避免方法**:
1. ✅ 查看步骤清单的"页面跳转检查"部分
2. ✅ 运行 `validate_steps.py` 的页面跳转等待检查
3. ✅ 按照代码审查清单逐项打勾
