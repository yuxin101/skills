# 代码审查清单

**版本**: 1.0.0  
**用途**: 在提交测试用例代码前，逐项检查确保质量

---

## 📋 审查流程

### 阶段 1: 步骤完整性审查

- [ ] 步骤数量是否与录制脚本一致？
- [ ] 是否使用 `parse_steps.py` 生成了步骤清单？
- [ ] 是否填写了 `steps_checklist.md` 对照表？
- [ ] 是否运行了 `validate_steps.py` 验证？

**验证命令**:
```bash
# 生成步骤清单
python scripts/parse_steps.py \
    --input recorded_script.py \
    --output steps_checklist.md

# 验证步骤
python scripts/validate_steps.py \
    --input recorded_script.py \
    --test testcase/xxx/test_xxx.py \
    --report validation_report.md
```

---

### 阶段 2: 相邻步骤检查 ⚠️

**规则**: 如果连续步骤在同一个方法中，必须验证元素 XPath 是否不同

- [ ] step_1 和 step_2 的元素 XPath 是否不同？ → 已验证 ✅
- [ ] step_2 和 step_3 的元素 XPath 是否不同？ → 已验证 ✅
- [ ] step_3 和 step_4 的元素 XPath 是否不同？ → 已验证 ✅
- [ ] step_14 和 step_15 的元素 XPath 是否不同？ → 已验证 ✅
- [ ] step_15 和 step_16 的元素 XPath 是否不同？ → 已验证 ✅
- [ ] step_16 和 step_17 的元素 XPath 是否不同？ → 已验证 ✅

**检查方法**:
1. 打开 `steps_checklist.md`
2. 查看"相邻步骤检查"部分
3. 逐项对比 XPath 是否相同

**常见错误案例**:
```python
# ❌ 错误：step_15 和 step_16 使用同一个元素
self.tap(self.free_commission_mode_icon)  # step_15
self.tap(self.free_commission_mode_icon)  # step_16 - 完全一样！

# ✅ 正确：使用不同的元素
self.tap(self.free_commission_mode_entrance)  # step_15: 入口按钮
self.tap(self.free_commission_mode_icon)      # step_16: 弹窗图标
```

---

### 阶段 3: 特殊逻辑检查 ⚠️

**规则**: 录制脚本中的特殊判断逻辑必须完整保留

- [ ] 三层判断是否保留？（CSS→XPath→inner_text）
- [ ] else 判断是否保留？
- [ ] 循环次数是否正确？
- [ ] 条件判断是否保留？

**检查方法**:
1. 查看 `steps_checklist.md` 中的"特殊逻辑步骤"
2. 对照录制脚本，逐行检查特殊逻辑
3. 运行 `validate_steps.py` 的"特殊逻辑检查"

**常见错误案例**:
```python
# ❌ 错误：简化了三层判断
self.tap(self.element_leaderHelpSetting)  # 只有一层

# ✅ 正确：保留三层判断
element_is_exists = self.page.element_is_exists("view.publish-set-detail-text...")
if element_is_exists:
    element = self.page.get_element("view.publish-set-detail-text...")
else:
    element_is_exists = self.page.element_is_exists(xpath="/page-wrapper/...")
    if element_is_exists:
        element = self.page.get_element_by_xpath("/page-wrapper/...")
    else:
        element = self.page.get_element_by_xpath("//*", inner_text="未设置")
element.tap()
```

### 阶段 4: 页面跳转等待检查 ⚠️

**规则**: 每个页面跳转后都必须添加 `wait_for_page_success()`

- [ ] step_1 跳转后是否添加了等待？ → 已验证 ✅
- [ ] step_4 跳转后是否添加了等待？ → 已验证 ✅
- [ ] step_14 跳转后是否添加了等待？ → 已验证 ✅
- [ ] step_19 跳转后是否添加了等待？ → 已验证 ✅
- [ ] step_25 跳转后是否添加了等待？ → 已验证 ✅
- [ ] step_27 跳转后是否添加了等待？ → 已验证 ✅
- [ ] step_30 跳转后是否添加了等待？ → 已验证 ✅

**检查方法**:
1. 查看 `steps_checklist.md` 中的"页面跳转检查"
2. 搜索测试用例中的 `wait_for_page_success` 调用
3. 对照录制脚本的 `wait_for_page` 调用

**常见错误案例**:
```python
# ❌ 错误：页面跳转后没有等待
def click_and_navigate(self):
    self.tap(self.element)
    # 缺少等待，下一步可能找不到元素

# ✅ 正确：页面跳转后添加等待
def click_and_navigate(self):
    self.tap(self.element)
    self.wait_for_page_success("/pro/pages/xxx/xxx")
```

---

### 阶段 5: 元素定义检查

**规则**: 不同步骤的元素必须分开定义

- [ ] 不同步骤的元素是否分开定义？
- [ ] 变量命名是否能区分不同元素？
- [ ] 是否优先使用 XPath？
- [ ] 是否有降级策略（CSS→XPath→inner_text）？

**检查方法**:
1. 查看页面类的元素定义部分
2. 检查是否有重复使用的元素变量
3. 检查命名是否清晰

**常见错误案例**:
```python
# ❌ 错误：同一个变量用于不同步骤
free_commission_mode_icon = "..."  # step_15 和 16 都用这个

# ✅ 正确：分开定义
free_commission_mode_entrance = "/page-wrapper/.../view[2]/view"  # step_15
free_commission_mode_icon = "/page-wrapper/.../icon[2]"           # step_16
```

### 阶段 6: 代码规范检查

- [ ] 所有元素声明在类顶部？
- [ ] 所有方法在元素声明下面？
- [ ] 方法注释是否标注了对应步骤？
- [ ] 是否有中文注释说明元素功能？
- [ ] 是否避免了 `tap()` 后的多余等待？

**检查方法**:
1. 查看 `docs/代码规范.md`
2. 逐项对照检查

- [ ] 是否有元素找不到的错误？
- [ ] 是否有页面跳转超时的错误？
