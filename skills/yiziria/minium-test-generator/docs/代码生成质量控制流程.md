# 代码生成质量控制流程

**版本**: 1.0.0  
**创建时间**: 2026-03-23  
**目的**: 确保生成的测试用例与录制脚本 100% 一致，避免漏掉步骤或简化逻辑

---

## 🎯 质量目标

1. ✅ **步骤完整性** - 录制脚本的每一步都在测试用例中有对应
2. ✅ **逻辑准确性** - 特殊判断逻辑（如三层判断）完全保留
3. ✅ **页面跳转验证** - 每个页面跳转后都有等待校验
4. ✅ **元素准确性** - 不同步骤的元素不混淆

---

### 阶段 2: 代码生成

**规则**:

#### 2.1 逐行复制法

```python
# 第 1 步：完整复制录制脚本（不改任何逻辑）
def method_name(self):
    # step_X: 完整复制录制脚本的代码
    element_is_exists = self.page.element_is_exists(xpath="...")
    if element_is_exists:
        element = self.page.get_element_by_xpath("""...""")
    else:
        element = self.page.get_element_by_xpath("//*", inner_text="""...""")
    element.tap()
    
    # step_X+1: 继续复制
    ...

# 第 2 步：验证每个步骤的 XPath 都不同
# 第 3 步：再考虑是否需要提取公共方法
```

#### 2.2 元素分离原则

```python
# ❌ 错误：同一个变量用于不同步骤
free_commission_mode_icon = "..."  # step_15 和 16 都用这个

# ✅ 正确：分开定义
free_commission_mode_entrance = "/page-wrapper/.../view[2]/view"  # step_15
free_commission_mode_icon = "/page-wrapper/.../icon[2]"           # step_16
confirm_choose = "/page-wrapper/.../footer-button/..."            # step_17
```

#### 2.3 页面跳转等待

```python
# 规则：只要录制脚本中有 wait_for_page，就必须添加等待
def click_and_wait(self):
    self.tap(self.element)
    self.wait_for_page_success("/pro/pages/xxx/xxx")
```

## ⚠️ 常见错误及避免方法

### 错误 1: 漏掉步骤

**案例**: 漏掉 step_14 的三层判断

**避免方法**:
- ✅ 使用 `parse_steps.py` 生成步骤清单
- ✅ 标记特殊逻辑步骤
- ✅ 逐行复制录制脚本代码

---

### 错误 2: 混淆相邻步骤的元素

**案例**: step_15 和 step_16 使用同一个元素

**避免方法**:
- ✅ 填写步骤对照表，检查"元素是否相同"
- ✅ 运行 `validate_steps.py` 检查相邻步骤
- ✅ 分开定义不同步骤的元素变量

---

### 错误 3: 漏掉页面跳转等待

**案例**: 页面跳转后没有等待，元素找不到

**避免方法**:
- ✅ 在代码审查清单中强制检查
- ✅ `validate_steps.py` 自动检查
- ✅ 录制脚本有 `wait_for_page` 就必须添加
