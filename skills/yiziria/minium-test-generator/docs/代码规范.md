## 📝 页面类规范

### 1. 文件结构

```python
# -*- coding: utf-8 -*-
"""
页面名称
页面路径：/pages/xxx/xxx
"""

from time import sleep
from base.basedef import BaseDef


class PageName(BaseDef):
    """页面对象类"""
    
    # ========== 元素声明（所有元素在顶部） ==========
    # 按钮名称：元素描述
    element_name = '选择器'
    
    # ========== 页面方法（所有方法在元素下面） ==========
    def method_name(self, param):
        """方法说明"""
        # 操作代码
```

### 2. 元素命名规范

| 元素类型 | 命名前缀 | 示例 |
|---------|---------|------|
| 普通元素 | `element_` | `element_submit` |
| 输入框 | `input_` | `input_username` |
| 按钮 | `btn_` 或 `element_` | `btn_confirm` |
| 避免冲突 | 添加后缀 | `input_priceAmount`（避免与方法名冲突） |

### 3. 元素选择器规范 ⭐

**优先级**: **CSS Selector > XPath**

```python
# ✅ 首选（CSS 选择器，简洁高效）
element = 'view.submit-btn'
element = 'image.choose-img'
element = 'textarea.publish-input'

# ⚠️ 备选（XPath，当 CSS 无法定位时使用）
element = '/page-wrapper/view/view[1]/button'
element = '//view[@text="提交"]'

# ✅ 组合使用（先 CSS，失败后用 XPath）
def click_element(self):
    # 先尝试 CSS 选择器
    element_is_exists = self.page.element_is_exists("view.submit-btn")
    if element_is_exists:
        element = self.page.get_element("view.submit-btn")
    else:
        # CSS 失败，使用 XPath
        element = self.page.get_element_by_xpath("/page-wrapper/.../button")
    element.tap()
```

**选择器优先级说明**:

| 优先级 | 选择器类型 | 优点 | 缺点 | 使用场景 |
|--------|-----------|------|------|---------|
| 1️⃣ | CSS 选择器 | 简洁、执行快、易读 | 复杂结构可能无法定位 | 首选 |
| 2️⃣ | XPath | 精确、支持复杂定位 | 路径长、执行慢 | CSS 无法定位时 |
| 3️⃣ | inner_text | 语义化、易维护 | 文本变化会失效 | 按钮文字稳定时 |

**最佳实践**:
```python
# ✅ 推荐：三层降级策略
def click_setting_btn(self):
    # 第 1 层：CSS 选择器（首选）
    element_is_exists = self.page.element_is_exists("view.setting-btn")
    if element_is_exists:
        element = self.page.get_element("view.setting-btn")
    else:
        # 第 2 层：XPath（备选）
        element_is_exists = self.page.element_is_exists(xpath="/page-wrapper/.../view[2]/view")
        if element_is_exists:
            element = self.page.get_element_by_xpath("/page-wrapper/.../view[2]/view")
        else:
            # 第 3 层：inner_text（最后手段）
            element = self.page.get_element_by_xpath("//*", inner_text="未设置")
    element.tap()
```

### 4. 方法封装规范

**原则**: 同一个页面的连续操作封装到一个方法

```python
# ✅ 推荐（连续操作封装）
def set_option_and_submit(self, value):
    """
    设置选项并提交
    
    录制脚本对应步骤:
        step_5-10: 输入值，滚动，点击设置
        step_11-15: 选择选项，确认
    """
    # step_5-6: 输入值
    self.send_key(self.input_value, value)
    
    # step_7-9: 滚动页面
    self.page.scroll_to(300)
    self.page.scroll_to(600)
    self.page.scroll_to(800)
    
    # step_10: 点击设置
    self.tap(self.element_setting)
    # ... 更多步骤

# ❌ 不推荐（一个操作一个方法）
def tap_element_1(self): ...
def tap_element_2(self): ...
def tap_element_3(self): ...
```

### 5. 等待时间规范

```python
# ✅ tap() 不需要额外等待（内部已有重试机制）
self.tap(self.element_name)

# ✅ send_key() 后如有必要可加等待
self.send_key(self.input_name, text)
self.page.wait_for(2)

# ✅ 滚动操作后建议加等待
self.page.scroll_to(1000)
self.page.wait_for(1)

# ❌ 不要每个操作都加等待
self.tap(self.element_1)
self.page.wait_for(2)  # 多余
self.tap(self.element_2)
self.page.wait_for(2)  # 多余
```

## ⚠️ 常见错误

### 错误 1: 元素命名冲突

```python
# ❌ 错误（元素名与方法名冲突）
input_price = '...'  # 元素
def input_price(self, price): ...  # 方法

# ✅ 正确（添加后缀区分）
input_priceAmount = '...'  # 元素
def input_price(self, price): ...  # 方法
```

### 错误 2: 选择器优先级错误

```python
# ❌ 错误：优先使用 XPath（路径长，执行慢）
element = '/page-wrapper/view/view[1]/view[2]/button'

# ✅ 正确：优先使用 CSS（简洁高效）
element = 'view.submit-btn'
```

### 错误 3: 步骤遗漏

```python
# ❌ 错误（漏掉了滚动步骤）
def publish(self):
    self.send_key(self.input_title, "标题")
    self.tap(self.element_publish)  # 缺少滚动步骤

# ✅ 正确（完整步骤）
def publish(self):
    self.send_key(self.input_title, "标题")
    self.page.scroll_to(300)  # step_7
    self.page.scroll_to(600)  # step_8
    self.tap(self.element_publish)  # step_9
```

### 错误 4: 混淆相邻步骤的元素

```python
# ❌ 错误：step_15 和 step_16 使用同一个元素
option_icon = "..."  # step_15 和 16 都用这个

# ✅ 正确：分开定义
option_entrance = "/page-wrapper/.../view[2]/view"  # step_15
option_icon = "/page-wrapper/.../icon[2]"           # step_16
```
