# Archives

Completed work. Projects that shipped, areas you're no longer managing, resources you're retiring.

## Structure

Each archive folder mirrors the active structure:
- `project.md` / `area.md` / `resource.md` — Completed item
- `items.json` — Facts from this item (for historical reference and pattern-finding)

## Purpose

- **Audit trail:** Track what you've accomplished
- **Pattern recognition:** Look back on old projects to find lessons learned
- **Retirement:** Declutter active areas by moving old projects here

## Examples

- `completed-projects/` — Projects that shipped
- `retired-areas/` — Areas you're no longer managing
- `obsolete-resources/` — Resources you no longer use
