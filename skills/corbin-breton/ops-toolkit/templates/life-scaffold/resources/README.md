# Resources

Reference materials. Things you want to keep but don't actively work on.

## Structure

Each resource folder contains:
- `index.md` — Overview of the resource
- `items.json` — Atomic facts about this resource (sources, key points, applications)

## Examples

- `research/papers/` — Academic papers you've read
- `tools/` — Software tools and their configurations
- `frameworks/` — Methodologies and decision frameworks
- `templates/` — Reusable templates and boilerplate

Delete or modify based on your actual reference materials.
