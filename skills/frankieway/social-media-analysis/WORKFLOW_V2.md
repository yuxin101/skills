# Social Media Analysis v2 - 优化工作流


```
┌─────────────────────────────────────────────────────────────────┐
│  步骤 1: 创建字段（一次性）                                       │
│  - 链接是否有效（单选：是/否）                                    │
│  - 包含多媒体类型（单选：纯文本/图片/视频）                        │
│  - 内容解析（文本）                                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  步骤 2: 快速有效性检查（批量）                                   │
│                                                                 │
│  2.1 HEAD 请求探测（不下载内容）                                  │
│      curl -I -L --max-time 5 URL                                │
│      - HTTP 200/302 → 有效                                      │
│      - HTTP 404/403/超时 → 无效                                 │
│                                                                 │
│  2.2 更新「链接是否有效」字段（批量 10 条/次）                       │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  步骤 3: 完整媒体处理（仅有效链接）                               │
│                                                                 │
│  3.1 平台识别 + 类型判断                                         │
│      抖音：先图集脚本探测 → 0 张=视频，N 张=图集                   │
│      微博：yt-dlp --dump-json → duration=视频，无=图集            │
│      B 站：yt-dlp --dump-json → 必有视频                          │
│      今日头条：Playwright → 文章图片                             │
│      微信：Camoufox → 文章图片                                   │
│                                                                 │
│  3.2 全部下载（保证完整性）                                      │
│      - 图集：下载全部图片 + 音频                                  │
│             node scripts/extract-douyin-images.js URL output    │
│      - 视频：下载完整视频                                        │
│             node scripts/parse-douyin-video.js URL --download   │
│             yt-dlp -o video.mp4 URL                             │
│      - 今日头条/微信：下载全部文章图片                            │
│             node scripts/parse-toutiao-playwright.js URL output │
│                                                                 │
│  3.3 视频抽帧（统一标准）                                        │
│      所有视频：每 5 秒抽 1 帧                                        │
│      命令：ffmpeg -i video.mp4 -vf "fps=1/5" frames/frame_%03d.jpg │
│                                                                 │
│  3.4 更新「包含多媒体类型」字段                                   │
│      - 图片：图集/文章图片                                       │
│      - 视频：含视频内容                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  步骤 4: 大模型分析（全面分析）                                   │
│                                                                 │
│  4.1 输入内容                                                    │
│      - 必选：标题 + 正文（已有字段）                              │
│      - 必选：所有抽帧图片（视频）                                 │
│      - 必选：所有图集图片（图集）                                 │
│      - 必选：所有文章图片（今日头条/微信）                        │
│                                                                 │
│  4.2 提示词模板                                                  │
│      基于以下信息生成 100 字以内内容解析：                         │
│      - 标题：{title}                                            │
│      - 正文：{text}                                             │
│      - 平台：{platform}                                         │
│      - 图片/视频内容：{visual_content}                          │
│      要求：                                                      │
│      1. 突出与"小爱/小米/小爱同学"相关的内容（如有）               │
│      2. 无相关信息则如实描述画面                                  │
│      3. 严格控制在 100 字以内                                      │
│                                                                 │
│  4.3 分析策略                                                    │
│      - 视频：分析所有抽帧图片（每 5 秒 1 帧）                        │
│      - 图集：分析所有图片                                       │
│      - 文章：分析所有配图                                       │
│                                                                 │
│  4.4 统一解析器兜底（必须执行）                                    │
│      node scripts/build-content-analysis.js \                   │
│        --title "{title}" --text "{text}" \                      │
│        --platform "{platform}" --media-type "{media_type}" \    │
│        --visual "{visual_content}"                               │
│      - 强制保障“内容解析”非空                                      │
│      - 输出限制 100 字以内                                         │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  步骤 5: 批量更新表格                                            │
│                                                                 │
│  5.1 每 10 条记录批量更新一次（减少 API 调用）                       │
│  5.2 更新字段：链接是否有效、包含多媒体类型、内容解析             │
└─────────────────────────────────────────────────────────────────┘
```
