# 极客数据服务 工作流指南

## 核心概念

### 平台定位
极客数据服务是抖音内容营销数据平台，为荣耀手机内容团队提供竞品分析、内容对标、热点追踪等数据能力。核心用户是**内容架构师**，通过数据找到可复制的优质内容。

### 5 个场景

| # | 场景 | 优先级 | 核心能力 |
|---|------|--------|---------|
| ① | 对标账号搜索 | 中 | 按品牌/粉丝/标签多维筛选 KOS/KOL 账号 |
| ② | 热点关键词搜索 | 中 | 搜索词发现、竞争分析、关键词-视频关联 |
| ③ | 平台热点搜索 | 中 | 38 分类热榜视频、热门话题趋势 |
| ④ | 选题内容建议 | 高 | 基于卖点找对标内容，供内容架构师分析 |
| ⑤ | 对标内容寻找 | 高 | 汇总账号+关键词+热点，输出对标视频列表 |

### 工作流（数据流向）

```
① 对标账号搜索 ──┐
                  │
② 热点关键词搜索 ──┼──→ ④ 选题内容建议 ──→ ⑤ 对标内容寻找
                  │
③ 平台热点搜索 ──┘
```

- **主动路径**：卖点 → 找账号 → 找内容 → 选题建议
- **被动路径**：看热点 → 找可复制的内容 → 对标分析

### 数据覆盖

| 数据 | 更新频率 | 说明 |
|------|---------|------|
| 用户账号 | 每2分钟 | Scrapy 爬虫持续采集 |
| 品牌/内容标签 | 每分钟 | 自动打标脚本 |
| 关键词视频 | 每5分钟 | 关键词搜索排名爬虫 |
| 热榜视频 | 每5分钟 | 热点宝榜单爬虫（每次1个分类，全量约3小时） |
| 关键词指标 | 每24小时 | Go 服务内置同步 |

## 典型工作流

### 流程一：主动路径 — 卖点驱动找内容（最常用）

适用场景：荣耀新品上市，内容架构师拿到产品卖点（如"人像拍摄"），需要找对标内容。

```
Step 1: 找竞品账号
  → accounts search --brand-tags "华为,小米,OPPO,vivo" --content-tags "拍摄" --follower-min 1000 --exclude-star

Step 2: 看这些账号的拍摄内容
  → videos search --sec-uids "uid1,uid2,..." --keyword "拍照" --interaction-min 1000 --sort-by interaction_count --page-size 50

Step 3: 同时看关键词排名内容
  → keyword-videos search --keyword "手机拍照"

Step 4: 汇总输出 top 100 视频，供内容架构师人肉分析
```

**关键筛选标准：**
- KOS 账号内容：互动量 > 1000
- KOL 账号内容：互动量 > 10000
- 优先最近 30 天的内容

### 流程二：被动路径 — 热点驱动找内容

适用场景：内容架构师日常监控，看什么内容火可以复制。

```
Step 1: 查看科技分类热榜
  → hot-videos search --category "科技" --page-size 50

Step 2: 查看近期热门话题
  → hashtags trending --days 7 --size 50

Step 3: 搜索手机相关热榜内容
  → hot-videos search --keyword "手机" --page-size 50

Step 4: 筛选中等互动量（可复制）的内容
  → 关注互动量在 1000-10000 之间的视频，这类内容更容易被 AIGC 复刻
```

### 流程三：关键词竞争分析

适用场景：了解荣耀和竞品在抖音搜索中的表现。

```
Step 1: 搜索荣耀相关关键词
  → keywords search --keyword "荣耀" --page-size 50

Step 2: 对比竞品关键词
  → keywords search --keyword "华为" --page-size 50
  → keywords search --keyword "小米" --page-size 50

Step 3: 查看高搜索量关键词的排名视频
  → keyword-videos search --keyword "荣耀Magic7"

Step 4: 添加新关键词到监控
  → keywords add --word "荣耀GT7 Pro拍照"
```

### 流程四：账号对标分析

适用场景：深入分析特定竞品账号的内容策略。

```
Step 1: 找到目标账号
  → accounts search --brand-tags "华为" --follower-min 5000 --user-type "1,2" --page-size 20

Step 2: 查看账号详情
  → accounts get --sec-uid "MS4wLjABAAAA..."

Step 3: 搜索该账号的所有视频
  → videos search --sec-uids "MS4wLjABAAAA..." --sort-by interaction_count --page-size 50

Step 4: 查看其爆款视频的趋势
  → videos trend --aweme-id "7321234567890" --days 7
```

## 内容筛选标准

### KOS 内容（品牌销售账号）
- 粉丝 > 1000
- 互动量 > 1000
- 发布时间：最近 30 天
- 排除明星、排除纯带货

### KOL 内容（头部达人）
- 粉丝 > 10000
- 互动量 > 10000
- 发布时间：最近 30 天

### 搜索排名内容
- 只看最近 14 天内生产的
- 只看之前没出现过的新内容

### 热榜可复制内容
- 互动量在 1000-10000 之间（中等热度，可复制）
- 和手机品类相关

## 错误处理

### 原则
- 同一接口最多重试 1 次，连续失败 2 次后停止
- 不要反复尝试
- 明确告知用户失败原因和替代方案

### 常见错误

| 错误场景 | 处理方式 |
|---------|---------|
| 连接失败 | 运行 `check` 确认 API 地址，检查网络 |
| 搜索无结果 | 放宽条件：降低粉丝门槛、减少标签、扩大时间范围 |
| 500 服务错误 | 重试 1 次，仍失败则建议稍后再试 |
| 数据量过大 | 分页查询，每次 50 条，循环多页 |
