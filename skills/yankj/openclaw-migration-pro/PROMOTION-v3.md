🦞 OpenClaw Migrate v3 发布！

## 核心理念

搬家时，你会：
1. 打包你的家具、衣服、个人物品
2. 消耗品（食物、日用品）到新家重新买
3. 个人配置（钥匙、密码）随身带

OpenClaw Migrate v3 就是这样设计的。

## v2 vs v3

### v2（错误的设计）
❌ 排除 credentials/（API Keys 丢失）
❌ 排除 node_modules/（Skills 无法运行）
❌ 归位后手动配置一切

### v3（正确的设计）
✅ 打包 credentials/（你的身份）
✅ 打包所有配置（你的个性化设置）
✅ 不打包 node_modules/（到新家重新 npm install）
✅ 归位自动安装依赖

## 使用示例

```bash
# 打包
openclaw-migrate pack --output ~/my-openclaw/

# 传输
rsync -av ~/my-openclaw/ user@new-pc:~/

# 归位
openclaw-migrate unpack --input ~/my-openclaw/
# ✅ 自动安装依赖
# ✅ 恢复所有配置
```

## 打包内容

### 你的资产（打包）
- credentials/ - API Keys（你的身份）
- openclaw.json - 配置
- SOUL.md, USER.md - 个人配置
- memory/ - 所有数据
- skills/ - 工具
- cron/ - 任务

### 消耗品（不打包）
- node_modules/ - 重新 npm install
- completions/ - 自动重新生成
- *.log - 不需要

## 安装

```bash
npx clawhub install openclaw-migrate-v3
```

## 源码

欢迎试用和提建议！🙏

## 💬 常见问题回复

### Q: 为什么打包 credentials？不安全吧？

**A**: 
```
credentials 是你的个人配置，就像搬家时的钥匙和密码。

我们建议：
1. 使用加密传输（rsync over SSH）
2. 传输后删除打包文件
3. 或使用密码管理器

安全很重要，但方便也很重要。你可以根据需要选择。
```

**English**:
```
credentials are your personal configs, like keys and passwords when moving house.

We recommend:
1. Use encrypted transfer (rsync over SSH)
2. Delete pack files after transfer
3. Or use password manager

Security is important, but convenience matters too. Choose what works for you.
```

---

### Q: node_modules 不打包，会不会很慢？

**A**:
```
不会。node_modules 通常只有 10-50MB，npm install 只需几秒到几分钟。

而且：
1. 避免了打包大文件
2. 确保依赖版本一致
3. 减少传输时间

这是值得的权衡。
```

**English**:
```
No. node_modules is usually 10-50MB, npm install takes seconds to minutes.

Plus:
1. Avoids packing large files
2. Ensures consistent dependency versions
3. Reduces transfer time

It's a worthy trade-off.
```
