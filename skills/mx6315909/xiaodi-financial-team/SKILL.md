# 金融分析师团队 (Financial Analyst Team)

**版本**: v2.1.0  
**定位**: 帮投资者做专业的金融分析和投资决策

---

## ⚠️ 安装前须知

在安装本技能前，请了解以下内容：

### 环境变量要求
本技能会读取以下环境变量（需用户自行配置）：
- `ZHIPU_API_KEY` - 智谱 AI 密钥（可选，用于 AI 分析）
- `BROWSERLESS_URL` - 浏览器服务地址（可选，用于 PE/PB 数据）
- `BROWSERLESS_API_KEY` - 浏览器服务密钥（可选）

### 配置文件访问
本技能会读取 `~/.openclaw/openclaw.json` 以自动适配 OpenClaw 配置的模型。

### 外部 HTTP 请求
本技能会向以下外部 API 发起请求：
- `https://open.bigmodel.cn/api/paas/v4` - 智谱 AI API
- `https://push2.eastmoney.com` - 东方财富行情 API
- `https://quote.eastmoney.com` - 东方财富行情页面

### 安装脚本
运行 `./install.sh` 前，建议先查看脚本内容：
```bash
cat install.sh | less
```

## 🎯 核心价值

**痛点**：
- 金融产品多 → 分析不过来
- 持仓复杂 → 复盘耗时
- 决策难 → 不敢下手

**解决方案**：
- 6 个专业分析师协作
- 股票 + 基金双覆盖
- 三档报告（快速/标准/专业）
- 分析师自动迭代优化

## 🔄 工作流程

### 流程 1：标准分析（Sequential + 迭代）

```
用户请求 → 产业分析师 → 估值分析师 → 技术分析师 → 风险分析师 → 资金分析师
                ↓              ↓              ↓              ↓              ↓
            [信息交换] ← ← ← ← [信息交换] ← ← ← ← [信息交换] ← ← ← ← [信息交换]
                ↓
           策略师综合 → 报告输出
```

**迭代机制**：
- 每个分析师完成分析后，自动分享关键发现
- 后续分析师可以参考前序分析结果
- 策略师汇总时，如有矛盾点会触发二次分析

---

### 流程 2：快速分析（Parallel）

```
          ├→ 产业分析师（精简版）
用户请求 → ├→ 估值分析师（精简版） → 策略师快速汇总
          └→ 风险分析师（精简版）
```

**适用场景**：需要快速决策、初步筛选

---

### 流程 3：专业分析（多轮迭代）

```
Round 1: 6个分析师独立分析
    ↓
Round 2: 分析师交换意见，修正判断
    ↓
Round 3: 策略师组织讨论，解决分歧
    ↓
Round 4: 最终报告 + 置信度评估
```

**适用场景**：大额投资决策、重要持仓分析

## 📊 三档报告

| 档位 | 分析时间 | 分析深度 | 适用场景 |
|------|----------|----------|----------|
| **快速版** | ~1分钟 | 3个分析师精简版 | 快速决策、初步筛选 |
| **标准版** | ~5分钟 | 6个分析师完整版 | 日常分析、持仓复盘 |
| **专业版** | ~15分钟 | 多轮迭代+置信度 | 大额投资、重要决策 |

### 快速版输出
- 综合评分
- 核心逻辑（3条）
- 简要建议

### 标准版输出
- 完整分析报告
- 各维度详细分析
- 操作建议 + 止盈止损

### 专业版输出
- 标准版全部内容
- 分析师讨论纪要
- 置信度评估
- 风险情景分析
- 备选方案

## 📦 安装与配置

### 快速安装

```bash
# 方法 1: 使用安装脚本（推荐）
cd skills/financial-analyst-team
./install.sh

# 方法 2: 手动安装
pip install -r requirements.txt
```

### 安装脚本功能

运行 `./install.sh` 会：

1. **自动检测环境**
   - Python 版本
   - pip 可用性
   - akshare 是否已安装
   - browserless 服务状态

2. **给客户选择权**
   ```
   是否安装 akshare? (y/n) [默认: y]
   是否配置 browserless? (A/B/C) [默认: C]
   ```

3. **三种配置方案**
   - **完整模式**: akshare + browserless（所有功能）
   - **标准模式**: akshare（实时行情可用）
   - **纯 AI 模式**: 无额外依赖（仅 AI 分析）

### 环境变量配置

```bash
# browserless 服务地址（可选，请替换为您自己的地址）
export BROWSERLESS_URL="http://your-browserless-host:3000"

# 或使用远程服务
export BROWSERLESS_URL="https://chrome.browserless.io"
export BROWSERLESS_API_KEY="your_api_key"
```
