# For site owners

You do **not** need to know commands. You tell the assistant (e.g. in OpenClaw) **what you want to achieve**—in plain language.

## Examples you can simply write

- “Create a **draft** blog post titled …”
- “Change the second paragraph on the **About** page to …”
- “Show me which **plugins** are active on the site.”
- “Is **WooCommerce** installed and what version?”

The assistant uses tools on the computer or server in the background (as your setup allows).

## What you set up once (often with help)

For the assistant to work on **your** WordPress site you typically need:

1. Your site **URL** (e.g. `https://your-domain.com`).
2. Access WordPress supports—often an **application password** for a suitable user. Create it in WordPress under Users → Profile (do not share your normal login password).

Technical details for setup people: [AUTH.md](AUTH.md).

## What the assistant cannot do without setup

- Without this setup it **cannot** reliably reach your site.
- **Deleted data** or large rebuilds: prefer a backup first—you can ask the assistant to use **drafts** only or to **read** first and suggest changes.

## If something goes wrong

Say what you saw (“error message …”, “page is blank”). The assistant should answer **calmly and clearly** and suggest the next step.
