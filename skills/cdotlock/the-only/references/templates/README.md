# HTML Reference Templates

Place your reference HTML files in this directory. Ruby will read them before coding each interactive webpage to match your preferred design language.

## How It Works

1. Add `.html` files to this directory — these are your design benchmarks.
2. When Ruby generates an interactive webpage, it will first read all templates here.
3. Ruby treats these as **visual and structural inspiration**, not copy-paste templates. Each article is still coded fresh, but the design quality, color palette, typography, and animation style will be guided by your references.

## Tips

- Name files descriptively: `dark_magazine.html`, `minimal_reader.html`, `tech_article.html`.
- Include CSS and JS inline (single-file templates work best).
- The more polished your reference, the better Ruby's output will be.
- You can add multiple templates — Ruby will analyze all of them and synthesize the best patterns.
