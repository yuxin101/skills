# /gen-design - 系统设计技能

**版本**: 1.0.0  
**阶段**: Phase 1

---

## 概述

`/gen-design` 技能用于通过交互式技术选型，生成完整的系统设计文档。

**核心特性**：
- 交互式技术栈确认（3-5 个关键问题）
- 基于领域知识库的数据模型设计
- 基于领域知识库的接口设计
- 基于领域知识库的集成设计
- 数据流主线与集成点必填机制
- 与 PRD 数据流对齐的可追溯设计

---

## 使用方式

### 基本用法

```bash
# 基于 PRD 生成设计
/gen-design

# 基于需求描述生成设计
/gen-design 我要做一个积分系统

# 指定 PRD 文件
/gen-design --prd=docs/prd/xxx.md

# 存量项目（参考设计主线）
/gen-design --design-line=docs/design/design-line.md
```

### 参数说明

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `prd` | string | 否 | 自动查找 | PRD 文档路径 |
| `requirement` | string | 否 | - | 需求描述 |
| `design-line` | string | 否 | - | 设计主线路径（存量项目） |
| `output` | string | 否 | `docs/design/YYYY-MM-DD-{project-name}-design.md` | 输出文件路径 |

---

## 输入

- PRD 文档（推荐）或需求描述
- 技术栈偏好（通过交互问答确认）
- 设计主线（存量项目时参考）

---

## 输出

- `docs/design/YYYY-MM-DD-{project-name}-design.md` - 系统设计文档

## 设计文档结构

| 章节 | 说明 | 必填 | 知识库支持 |
|------|------|------|------------|
| 系统架构 | 架构图、架构说明 | 是 | - |
| 技术栈 | 前后端技术、数据库 | 是 | 架构方案库 |
| 模块划分 | 模块列表、职责描述 | 是 | - |
| 数据流主线与集成点 | 数据流表、集成点清单 | **是（必填）** | 领域知识库 |
| 接口设计 | API 规范、接口列表 | 是 | 领域知识库 |
| 数据模型 | ER 图、表结构 | 是 | 领域知识库 |
| 部署方案 | 部署架构、资源配置 | 是 | - |

### 领域知识库支持

本技能基于领域知识库进行增强设计：

| 设计阶段 | 知识库内容 | 用途 |
|---------|-----------|------|
| 数据模型设计 | 核心实体关系、数据库表设计参考 | 设计表结构 |
| 接口设计 | 核心接口设计 | 设计 API 接口 |
| 集成设计 | 常见集成点、典型数据流骨架 | 设计系统集成 |

**可用领域知识库**：
- `skills/libs/domain-knowledge/points.md` - 积分系统
- `skills/libs/domain-knowledge/user-center.md` - 用户中心
- `skills/libs/domain-knowledge/order.md` - 订单系统
- `skills/libs/domain-knowledge/payment.md` - 支付系统
- `skills/libs/domain-knowledge/ecommerce.md` - 电商通用

## 技术栈选择

### 默认推荐

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Ant Design Vue | 企业级应用首选 |
| 后端 | Spring Boot 3 + MyBatis-Plus | 成熟稳定 |
| 数据库 | MySQL 8.0 + Redis | 标准配置 |
| AI 能力 | LangChain4j + LangGraph4j | 大模型应用 |
| 微服务 | Spring Cloud Alibaba | 大规模场景 |

### 规模决策

| 用户规模 | 推荐架构 | 说明 |
|----------|----------|------|
| 日活 <1 万 | 单体架构 | 开发效率高 |
| 日活 1-10 万 | 单体 + 读写分离 | 性能提升 |
| 日活 >10 万 | 微服务架构 | 高可扩展 |

## 交互问答流程

本技能通过交互问答收集关键信息：

### 阶段 1：技术栈确认（3-5 个问题）

1. **技术栈偏好**
   - 前端框架：Vue/React/其他
   - 后端框架：Spring Boot/Node.js/Python

2. **预期用户规模**（决定架构复杂度）
   - A) 日活 < 1 万（单体架构）
   - B) 日活 1-10 万（单体 + 读写分离）
   - C) 日活 > 10 万（微服务架构）

3. **是否需要 AI 能力**
   - 是（集成 LangChain4j）/ 否

4. **部署环境**
   - 云服务器 / K8s / Serverless

5. **特殊需求**（可多选）
   - 高并发 / 实时通信 / 大数据量

### 阶段 2：架构方案推荐

基于用户选择，从 `architecture-options.md` 匹配最佳方案：
- 展示推荐方案详情（架构图、技术栈、成本估算）
- 提供 2-3 个可选方案对比
- 用户确认后进入下一步

### 阶段 3：领域知识库驱动设计

1. **领域识别**：分析 PRD 识别涉及领域
2. **知识库加载**：自动加载对应领域知识库
3. **数据模型设计**：基于知识库设计实体和表结构
4. **接口设计**：基于知识库设计 API 接口
5. **集成设计**：基于知识库设计系统集成

## 常见问题

### Q1: 设计文档与 PRD 的关系？

设计文档中的「数据流主线与集成点」应与 PRD 中的「核心数据流与闭环」对齐，保证可追溯。

### Q2: 存量项目如何使用本技能？

指定 `--design-line` 参数，参考现有设计主线生成新功能的设计文档。

### Q3: 架构方案如何选择？

本技能会根据用户规模、技术栈偏好自动推荐合适的架构方案，详见 libs/architecture-options.md。
