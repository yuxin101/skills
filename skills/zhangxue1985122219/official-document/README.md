# 机关公文技能（official-document）

> 基于 GB/T 9704-2012 党政机关公文格式标准，程序化生成规范红头文件

---

## 一句话介绍

只需填入发文字号、标题、正文内容，运行一行命令，即可生成符合国家标准的正式公文（通知、请示、报告、函等）。

---

## 核心特性

### ✅ 严格对标 GB/T 9704-2012

| 项目 | 标准值 |
|------|--------|
| 纸张 | A4 |
| 上/下/左/右边距 | 3.7 / 3.5 / 2.8 / 2.6 cm |
| 公文标题 | 方正小标宋简体 22pt，居中 |
| 正文 | 仿宋_GB2312 16pt，首行缩进 2 字符 |
| 一级标题 | 黑体 16pt，首行缩进 |
| 二级标题 | 楷体_GB2312 16pt，首行缩进 |
| 行距 | 固定 28.5 磅 |
| 发文字号年份 | **六角括号** `〔〕`（非方括号） |

### ✅ 中文引号自动还原

AI 生成脚本时容易将中文弯引号（`""` `''`）错写成英文直引号。脚本内置 `normalize_chinese_quotes()` 函数，在文本写入 Word 前自动将 ASCII 直引号交替还原为正确的中文弯引号，无需手动处理。

### ✅ 结构完整

- 发文字号 → 标题（空 2 行）→ 主送机关（空 1 行）→ 正文 → 附件 → 落款（空 3 行）→ 联系方式
- 落款、日期右对齐，联系方式自动套中文括号

### ✅ 零门槛使用

复制脚本 → 修改 CONFIG 和 CONTENTS → 运行，无需了解 Word XML 底层细节。

## 快速开始

### 前置依赖

```bash
pip install python-docx
```

### 使用步骤

1. **加载技能**
   ```
   use_skill("official-document")
   ```

2. **复制生成脚本**到工作目录：
   ```
   official-document/scripts/generate_docx.py
   ```

3. **修改配置区**（CONFIG）和**正文区**（CONTENTS）：

   ```python
   CONFIG = {
       "docNumber": "陕XX〔2026〕28号",         # 六角括号！
       "title": "关于做好全国模范单位奖牌颁发工作的通知",
       "mainRecipients": "各设区市、各高校：",
       "author": "陕西省XXXX",
       "date": "2026年1月15日",
       "attachments": ["全国模范单位奖牌颁发情况统计表"],
       "contact": "联系人：XXX  电话：189XXXXXXXX",
       "outputFile": "公文.docx"
   }

   CONTENTS = [
       ("h1", "一、充分认识奖牌颁发工作的重要意义"),
       ("normal", "全国模范单位奖牌是表彰先进、树立典型的重要载体……"),
       ("h2", "（一）领取方式"),
       ("normal", "获奖单位请于2026年X月X日前联系所在设区市领取奖牌。"),
       # 继续添加正文……
   ]
   ```

4. **运行生成**：
   ```bash
   python generate_docx.py
   ```

---

## CONTENTS 格式说明

| 类型 | 代码 | 说明 |
|------|------|------|
| 一级标题 | `("h1", "一、xxx")` | 黑体，16pt |
| 二级标题 | `("h2", "（一）xxx")` | 楷体，16pt |
| 正文段落 | `("normal", "正文内容")` | 仿宋，16pt，自动首行缩进 |
| 空行 | `("blank", None)` | 手动空一行 |

---

## 发文字号格式规范

**必须使用六角括号** `〔〕`，严禁方括号 `【】`。

```
✅ 正确：陕XX〔2026〕28号
❌ 错误：陕XX【2026】28号
```
