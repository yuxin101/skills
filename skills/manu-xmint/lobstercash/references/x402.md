# x402 API Payments

Fetch a paid API endpoint, paying automatically from the agent wallet. x402 is a payment protocol: the server returns HTTP 402, the CLI pays, and the server returns the content.

## Before fetching — check balance

Run: lobstercash balance --agent-id <id>

If the balance is 0 or below the expected payment amount on the expected chain, and token, stop and tell the user funds are needed first.

## Command

lobstercash x402 fetch <url> \
 --network <mainnet-beta|devnet> \
 --agent-id <id>

For POST requests add: `--json '{"key": "value"}'`
For custom headers add: `--header "Authorization: Bearer <token>"`
The `--network` flag selects the Solana cluster for payment. Accepted values are `mainnet-beta` (default — real funds) and `devnet` (test funds). Use `devnet` only when the target server is explicitly running on devnet; most production APIs expect `mainnet-beta`.

## Reading the output

Key fields:

- `status`: HTTP response status code (200 = success)
- `body`: the response body as a string — parse as JSON if needed
- `paymentSucceeded`: boolean — true means payment succeeded and content received
- `contentType`: the response content type

## If the fetch fails

Add `--debug` to the same command and run it again. Debug output goes to
stderr and shows exactly where the payment flow broke.

Common failures:

- `"x402 payment failed"`: the server's facilitator does not support
  settlement accounts. This is a server-side configuration issue.
- Network error before 402: the URL is wrong or the server is down.
- Balance insufficient: top up via lobster.cash/dashboard.

## Reporting to the user

Report what the API returned (`body`), not the payment mechanics.
Only mention payment if the user asks.
