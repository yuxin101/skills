# 量化管理系统 (Quantified Self) - 使用指南

## 简介

**量化管理系统** 是 OpenClaw 的一个全方位数据化呈现技能，将 AI 的成长、任务完成、学习成果、效率指标、成就记录等所有方面都用数据化的形式呈现。

**核心理念：一切皆可量化，让进步看得见。**

---

## 功能特性

### 📊 七大核心模块

| 模块 | 功能 | 输出示例 |
|------|------|----------|
| **成长追踪** | 知识、能力、质量、效率的量化评估 | 从 48.5% → 52.3% (+3.8%) |
| **任务统计** | 任务完成情况、质量评分、类型分布 | 完成 18/23 任务 (78.3%) |
| **学习成果** | 新掌握概念、学习进度、知识图谱 | 新学 12 个概念，总计 94 个 |
| **效率分析** | 响应速度、一次解决率、迭代次数 | 一次解决率 76.5% |
| **成就记录** | 里程碑、高光时刻、突破性进展 | 解锁 3 个成就 |
| **目标管理** | 目标设定、进度追踪、达成率 | 目标进度 67%，预计 3 天完成 |
| **综合日报** | 所有模块的汇总报告 | 今日量化报告 |

## 安装步骤

### 1. 复制 Skill 文件夹

```bash
# 将整个 growth-tracker 文件夹复制到 OpenClaw 的 skills 目录
cp -r growth-tracker ~/.openclaw/workspace/skills/quantified-self

# Windows 用户
xcopy /E /I growth-tracker %USERPROFILE%\.openclaw\workspace\skills\quantified-self
```

### 2. 重启 OpenClaw

```bash
# 方法 1：重启 Gateway
openclaw restart

# 方法 2：刷新 Skills
在 OpenClaw 中输入："刷新 skills"
```

### 3. 验证安装

```
输入："汇报成长"
如果看到量化报告，说明安装成功！✅
```

## 配置选项

编辑 `~/.openclaw/config.json` 来自定义行为：

```json
{
  "skills": {
    "quantified_self": {
      "weights": {
        "knowledge": 0.25,
        "capability": 0.30,
        "quality": 0.25,
        "efficiency": 0.20
      },
      "report_detail": "full",
      "auto_track": true,
      "show_trends": true,
      "show_progress_bars": true,
      "retention_days": 90
    }
  }
}
```

### 配置说明

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `weights` | 各维度权重 | 0.0 - 1.0，总和应为 1.0 |
| `report_detail` | 报告详细程度 | `full`, `summary`, `minimal` |
| `auto_track` | 自动追踪 | `true`, `false` |
| `show_trends` | 显示趋势图 | `true`, `false` |
| `show_progress_bars` | 显示进度条 | `true`, `false` |
| `retention_days` | 数据保留天数 | 数字，默认 90 |

## 常见问题

### Q1: 首次使用显示什么？

**A:** 首次使用会初始化数据文件，建立能力基准：
```
📊 正在初始化量化管理系统...
✅ 基准已建立：当前能力水平为 40.0%
📝 下次汇报时将显示具体的量化数据
```

### Q2: 百分比是如何计算的？

**A:**
- **总体能力** = Σ(各维度 × 权重) / 100
- **成长%** = (当前值 - 基准值) / (100 - 基准值) × 100
- 百分比是相对于基准的增量，不是绝对能力值

### Q3: 如何提高某个维度的成长速度？

**A:** 查看报告中的"下一步建议"，会针对薄弱维度提供具体建议。例如：
- 知识广度低 → 多学习新概念、工具
- 能力深度低 → 练习更复杂的任务
- 响应质量低 → 提高准确性，减少修正
- 效率指标低 → 减少迭代次数

### Q4: 数据可以导出吗？

**A:** 可以，输入"导出所有数据"即可获得 JSON/CSV 格式的完整数据。

### Q5: 如何重新开始？

**A:** 输入"重置量化数据"并确认，会删除所有历史数据并重新初始化。

### Q6: 数据存储在哪里？

**A:** 所有数据存储在本地 `~/.openclaw/data/quantified_self.json`，不会上传到云端。
