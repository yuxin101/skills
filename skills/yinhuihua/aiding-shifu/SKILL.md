## 🔧 API 工具定义

### 基础配置

| 配置项 | 值 |
|--------|-----|
| **API 基础地址** | `https://mes.dderp.cn/mob` |
| **认证方式** | Header: `Authorization: Bearer {token}` |

### 认证工具

#### 微信扫码登录

**触发场景**：
- 用户说"登录"、"扫码登录"、"绑定账号"
- 用户首次使用需要认证身份

**工具路径**：`tools/login/qr-login.html` 或 `tools/login/login.js`

**流程**：
1. 调用 `POST /wechat/createLoginQrCode` 生成二维码
2. 用户用微信扫码并在手机端确认
3. 调用 `POST /wechat/checkLoginStatus?sceneId={id}` 轮询状态
4. 状态变为 `confirmed` 时获取 Token

**返回 Token 后**：
- 保存到 `~/.workbuddy/skills/aiding-shifu/config.json`
- 后续 API 调用使用此 Token

### 师傅管理工具

#### 搜索师傅

**触发场景**：
- 用户说"找师傅"、"搜索师傅"、"附近有哪些师傅"
- 用户需要查找特定区域或技能的师傅

**API**：`POST /worker/pageWorkerForMobile`
**参数**：
- `provinceCode` - 省份编码（可选）
- `cityCode` - 城市编码（可选）
- `districtCode` - 区县编码（可选）
- `skillType` - 技能类型（可选）
- `pageNum` - 页码
- `pageSize` - 每页条数

**返回**：师傅列表（姓名、评分、擅长技能、距离等）

---

#### 获取师傅详情

**触发场景**：
- 用户说"师傅详情"、"看看这个师傅的信息"

**API**：`POST /worker/getWorkerDetailById`
**参数**：
- `workerId` - 师傅ID

**返回**：师傅详细信息（头像、评分、从业年限、服务区域、已完成订单数等）

---

### 工单管理工具

#### 创建工单（派单）

**触发场景**：
- 用户说"派单"、"创建工单"、"下单"

**API**：`POST /workorder/addWorkOrder`
**参数**：
- `dealerId` - 经销商ID
- `customerName` - 客户姓名
- `customerPhone` - 客户电话
- `customerAddress` - 客户地址
- `skillType` - 技能类型（如：吊顶、木门、地板）
- `workContent` - 工作内容描述
- `expectTime` - 期望上门时间
- `workerId` - 指定的师傅ID（可选，不指定则系统自动派单）

**返回**：工单ID、状态

---

#### 查询工单列表

**触发场景**：
- 用户说"查工单"、"我的订单"、"订单列表"

**API**：`POST /workorder/pageWorkOrder`
**参数**：
- `status` - 工单状态（待接单/进行中/已完成等）
- `pageNum` - 页码
- `pageSize` - 每页条数

**返回**：工单列表

---

#### 查询工单详情

**触发场景**：
- 用户说"工单详情"、"看看这个订单"

**API**：`POST /workorder/getWorkOrderDetail`
**参数**：
- `workOrderId` - 工单ID

**返回**：工单详细信息（客户信息、师傅信息、进度等）

---

#### 更新工单状态

**触发场景**：
- 用户说"确认完工"、"取消订单"、"开始处理"

**API**：`POST /workorder/updateWorkOrderStatus`
**参数**：
- `workOrderId` - 工单ID
- `status` - 新状态
