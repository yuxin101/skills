# memory/ conventions

This directory stores durable session notes.

## Daily task-intake rule

For any non-trivial task, append a short intake note to `memory/YYYY-MM-DD.md` immediately.

Suggested shape:
- root task
- scope / boundaries
- unusual constraints
- current task slug or related plan/status/handoff files when useful

## What belongs here

Write down:
- task intake
- major decisions
- durable conclusions
- blockers worth remembering
- pointers to important handoff/status files

## What does not belong here

Avoid using daily memory for:
- large scratch dumps better kept in `temp/`
- secrets unless explicitly asked
- redundant copies of full tool output when a concise summary is enough
