# 中国财政数据收集需求文档

## 1. 文档目的

定义中国财政数据收集项目的范围、规则、存储与交付要求。

## 2. 项目目标

基于财政部官网“信息公开 -> 财政数据发布”栏目公告，构建按月组织的财政指标采集与加工流程。首期目标：

1. 采集财政部官网财政数据发布栏目中的相关公告。
2. 从公告正文中抽取财政收支相关指标。
3. 将公告中披露的累计值还原为单月值。
4. 基于当月财政收支数据计算当月财政赤字。
5. 支持对原始提取结果和推导结果进行统一输出。

## 3. 数据来源

### 3.1 官方来源

财政部官网财政数据发布栏目：

`https://www.mof.gov.cn/gkml/caizhengshuju/`

### 3.2 来源要求

1. 仅采集官方发布内容。
2. 结构化数据必须可追溯到具体公告。
3. 原始公告需保留标题、发布日期、来源部门、链接、正文。

## 4. 业务范围

### 4.1 首期范围

首期纳入以下业务范围：

1. 财政收支情况公告的抓取与解析。
2. 财政收入、财政支出及其分项指标的结构化存储。
3. 累计值向单月值的推导。
4. 当月财政赤字的计算。
5. 原始提取层和推导层结果的 Excel 输出。

### 4.2 暂不纳入首期的范围

以下内容不作为首期强制交付范围：

1. 政府债务数据的正式采集接入。
2. 非财政部官网来源的数据接入。
3. 图表生成、可视化展示和分析报表。

## 5. 核心指标范围

### 5.1 财政收支类指标

从“财政收支情况”类公告中按指标拆分采集。指标划分以公告正文实际结构为准，分为财政收入、财政支出两大类，并按预算口径和原文子项展开。

### 5.2 财政收入类指标

#### 5.2.1 一般公共预算收入

1. 全国一般公共预算收入
2. 全国税收收入
3. 非税收入
4. 中央一般公共预算收入
5. 地方一般公共预算本级收入

#### 5.2.2 一般公共预算收入下的主要税收收入项目

1. 国内增值税
2. 国内消费税
3. 企业所得税
4. 个人所得税
5. 进口货物增值税、消费税
6. 关税
7. 出口货物退增值税、消费税
8. 城市维护建设税
9. 车辆购置税
10. 印花税
11. 证券交易印花税
12. 资源税
13. 契税
14. 房产税
15. 城镇土地使用税
16. 土地增值税
17. 耕地占用税
18. 环境保护税
19. 车船税、船舶吨税、烟叶税等其他各项税收收入合计

#### 5.2.3 政府性基金预算收入

1. 全国政府性基金预算收入
2. 中央政府性基金预算收入
3. 地方政府性基金预算本级收入
4. 国有土地使用权出让收入

### 5.3 财政支出类指标

#### 5.3.1 一般公共预算支出

1. 全国一般公共预算支出
2. 中央一般公共预算本级支出
3. 地方一般公共预算支出

#### 5.3.2 一般公共预算支出下的主要支出科目

1. 教育支出
2. 科学技术支出
3. 文化旅游体育与传媒支出
4. 社会保障和就业支出
5. 卫生健康支出
6. 节能环保支出
7. 城乡社区支出
8. 农林水支出
9. 交通运输支出
10. 债务付息支出

#### 5.3.3 政府性基金预算支出

1. 全国政府性基金预算支出
2. 中央政府性基金预算本级支出
3. 地方政府性基金预算支出
4. 国有土地使用权出让收入相关支出

### 5.4 指标划分原则

1. 以公告正文实际章节为一级划分依据，如“一般公共预算收入情况”“一般公共预算支出情况”“政府性基金预算收入情况”“政府性基金预算支出情况”。
2. 以公告原文列举的“主要税收收入项目”和“主要支出科目”作为二级指标集合。
3. 对于原文中明确出现的“其中”子项，应允许单独建项并保留父子关系，例如“印花税”与“证券交易印花税”。
4. 不得将政府性基金预算指标混入一般公共预算指标。
5. 对于名称较长或包含多个税种的指标，应保留原文名称作为标准化前的基础字段。
6. 首期覆盖范围以本节列出的指标为准。

### 5.5 财政赤字

财政赤字为派生指标，首期同时支持两种口径。

1. 窄口径赤字

`窄口径赤字 = 当月一般公共预算支出 - 当月一般公共预算收入`

仅涵盖一般公共预算，对应官方赤字率口径。

2. 宽口径赤字

`宽口径赤字 = （当月一般公共预算支出 + 当月政府性基金预算支出） - （当月一般公共预算收入 + 当月政府性基金预算收入）`

用于一般公共预算与政府性基金预算的合并观察。首期至少完成两类预算的合并口径组织，并为后续扩展债务工具预留能力。

### 5.6 政府债务

政府债务相关内容当前不纳入本期需求范围，暂不考虑。

## 6. 数据粒度与标准化要求

### 6.1 数据粒度

财政收支类指标按“月份 + 指标”粒度落地。每条记录至少包含：

1. 月份
2. 指标名称
3. 指标值
4. 单位
5. 数据来源公告
6. 数据生成方式

### 6.2 单位标准

1. 金额类指标统一使用“亿元”作为存储单位。
2. 增速类指标统一使用百分比数值存储。
3. 若原文单位发生变化，入库前需统一换算。

### 6.3 时间标准

1. 公告发布日期应按标准日期格式存储。
2. 指标月份应统一存储为标准月份格式，如 `2025-10`。
3. 公告中的自然语言期间表达需转换为标准截止月份。

示例：

1. `1-10月` 对应截止月份 `2025-10`
2. `前三季度` 对应截止月份 `2025-09`
3. `上半年` 对应截止月份 `2025-06`

## 7. 数据处理规则

### 7.1 公告抓取规则

1. 公告标题
2. 公告链接
3. 发布日期
4. 来源部门
5. 公告正文

### 7.2 公告识别规则

1. 统计年份
2. 统计期间表达
3. 截止月份
4. 公告类别

### 7.3 指标抽取规则

1. 以指标为单位进行结构化拆分。
2. 同一公告中的多个指标应分别存储。
3. 保留指标原文名称，且映射到标准指标名称。
4. 对于同比增长、同比下降等表述，应统一解析为数值字段。

### 7.4 累计值转单月值规则

1. 若统计期间为 1 月，则累计值即当月值。
2. 若统计期间为 2 月及以后，则当月值 = 当前累计值 - 上一累计期间值。
3. 上一累计期间应为同一指标、同一年份下的相邻截止月份。
4. 单月值的生成过程必须保留计算依据，便于追溯。

### 7.5 财政赤字计算规则

1. 同时支持窄口径赤字和宽口径赤字两种计算结果。
2. 窄口径赤字使用一般公共预算口径下的当月收入和当月支出。
3. 宽口径赤字使用一般公共预算与政府性基金预算合并观察口径下的当月收入和当月支出。
4. 将财政赤字作为派生指标单独存储。
5. 保留公式、口径类型和依赖的原始指标记录。

## 8. 数据存储要求

至少拆分为“原始公告层”“原始公告提取的结构化指标层”“推导的结构化指标层”三层。

### 8.1 原始公告层

1. 公告标题
2. 公告链接
3. 发布日期
4. 来源部门
5. 公告正文
6. 统计期间原文
7. 截止月份

### 8.2 原始公告提取的结构化指标层

存储从公告正文直接提取出的结构化指标，通常为累计值或公告原文直接披露值。字段包括：

1. 指标月份
2. 标准指标名称
3. 指标原文名称
4. 指标分类
5. 指标值
6. 单位
7. 同比增速
8. 来源公告标识
9. 来源公告标题
10. 来源公告发布日期
11. 统计期间原文
12. 截止月份

约束：

1. 该层只保存公告原文可直接提取的数据，不保存通过差值计算得到的月度值。
2. 必须保留同比增速字段。
3. 该层中的指标通常为累计值，但也应允许存储公告中直接给出的其他原始披露值。

### 8.3 推导的结构化指标层

存储根据原始公告提取指标加工得到的结果，包括单月值和派生指标值。字段包括：

1. 指标月份
2. 标准指标名称
3. 指标分类
4. 指标值
5. 单位
6. 同比增速
7. 推导类型
8. 计算公式
9. 依赖的原始指标记录标识
10. 依赖的来源公告标识
11. 是否派生指标

约束：

1. 推导类型至少应区分“累计值推导的单月值”和“计算得到的派生指标值”。
2. 财政赤字应存储在该层，作为派生指标。
3. 该层需支持多次重跑和幂等更新，避免重复入库。
4. 该层需支持同比增速字段。

### 8.4 对外输出层

1. 原始提取层输出一套 Excel 结果。
2. 推导层输出一套 Excel 结果。
3. Excel 输出应保留字段名、月份、指标名称、指标值、单位、来源信息和必要的计算说明。
4. 若存在多 Sheet 方案，应保证原始提取层与推导层清晰区分。

## 9. 处理流程要求

1. 抓取公告列表。
2. 抓取公告详情。
3. 保存原始公告内容。
4. 解析公告所属期间。
5. 抽取公告中的财政指标。
6. 存储累计值指标。
7. 根据相邻累计期间推导单月值。
8. 根据单月收入和单月支出计算窄口径赤字与宽口径赤字。
9. 执行数据校验并输出异常记录。

## 10. 校验与异常处理要求

1. 同一月份、同一指标不得重复生成多条有效月度记录。
2. 单月值推导必须能追溯到两个有效的累计值来源，1 月除外。
3. 若缺少上一个累计期间数据，则该月指标不生成单月值，并记录异常。
4. 若同一指标在不同公告中的名称不一致，应通过映射规则归一化。
5. 若正文未能提取出有效数值，应记录解析失败信息，便于后续修复。
6. 若单位不一致，应在入库前完成统一换算。

## 11. 验收标准

1. 能稳定抓取财政部财政数据发布栏目中的目标公告。
2. 能对“财政收支情况”类公告完成正文存储。
3. 能抽取主要财政收入、支出及税收指标。
4. 能将累计值正确推导为单月值。
5. 能基于单月收支数据生成财政赤字。
6. 每条月度数据均可追溯到来源公告和计算过程。
7. 对缺失上期数据、解析失败、重复数据等情况有明确异常记录。

## 12. 非功能要求

### 12.1 可追溯性

结构化数据必须能追溯到原始公告与计算过程。

### 12.2 可扩展性

指标体系、时间解析规则和来源接入方式应支持新增财政指标和新增统计口径。

### 12.3 可维护性

指标映射、时间解析规则和异常规则应尽量配置化，减少硬编码。
