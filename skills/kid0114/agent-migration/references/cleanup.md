# Cleanup Notes

## 清理原则
- 不要在迁移刚完成时立刻删除旧 agent
- 先观察新 agent 是否稳定运行
- 删除旧 agent 前，必须再次询问用户
- 会话内容迁移属于默认步骤，不能用“活跃会话判断”替代或跳过

## 推荐清理对象
- 旧 agent 目录：`~/.openclaw/agents/<old-id>`
- 旧 session 历史
- 明确不再使用的旧索引或残留文件

## 不建议直接做的事
- 未确认 UI / runtime 已切稳就删旧历史
- 未经确认直接删除旧 session 文件
- 默认假设不需要备份

## 建议清理顺序
1. 再次询问用户是否删除旧 agent
2. 确认是否需要备份
3. 删除旧 agent 目录 / 历史
4. 展示删除证据
5. 观察新 agent 是否仍正常工作
