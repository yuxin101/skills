#### 流程图重构建议
建议将当前流程图重构为以下交付物：
1. **L1 主干图**：进入列表 → 触发授权 → 获取信息 → 创建乘机人 → 完成（5 个节点，线性）
2. **L2 泳道图**：用户 | App前端 | OTA后端 | 支付宝（4 条泳道，明确异步回调）
3. **L3 授权状态 FSM**：`IDLE → REDIRECTING → AUTH_PENDING → AUTHORIZED / CANCELLED / TIMEOUT → CREATING → CREATED / FAILED`
4. **L4 去重规则表**：去重字段 + 匹配策略 + 冲突处理
5. **L5 交互流**：已有 HTML 稿，补充错误态即可

## 改进建议（建议优化）

1. 需求背景增加量化数据，例如"当前手动添加乘机人平均耗时 X 秒，信息填错率 Y%"
2. 上线方案增加灰度扩量条件和回滚触发标准
3. 接口设计增加错误码定义表（目前只有 code=0 的成功响应）
4. 安全章节补充个人信息保护法合规说明
5. 兼容性说明补充各端支付宝 SDK 版本矩阵
6. 将去重规则（姓名+证件类型+证件号匹配）独立为规则表，明确匹配精度
7. 流程图按 L1-L5 分层重构，当前一张图混合了所有层级
