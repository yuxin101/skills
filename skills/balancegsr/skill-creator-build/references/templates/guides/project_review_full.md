# 项目回顾指南（完整版）

> 项目结项时加载此文件，引导生成结构化的项目回顾。

---

## 回顾流程

### 第一步：决策回顾

1. 读取 decisions.md 全文
2. 按阶段梳理关键技术决策（每阶段提取 2-3 个最重要的）
3. 对每个关键决策评估：
   - 当时的选择现在看是否正确？
   - 如果重做，会做不同的选择吗？为什么？
   - 这个决策带来了哪些预期之外的影响（正面或负面）？

### 第二步：架构演变

1. 对比项目初始架构和最终架构
2. 识别关键转折点：哪些事件导致了架构变化？
3. 总结架构演变的模式：是渐进式优化，还是有大的重构节点？

### 第三步：目标达成评估

1. 对照启动时定义的项目目标
2. 逐条评估达成度：完全达成 / 部分达成 / 未达成 / 超额达成
3. 分析偏差原因（如有）

### 第四步：知识沉淀

提炼可复用的经验，格式：

```
## 经验：[一句话标题]
- 场景：[什么情况下适用]
- 做法：[具体怎么做]
- 原因：[为什么有效]
- 适用边界：[什么情况下不适用]
```

重点关注：
- 技术选型中的经验（"选 X 而不选 Y 的理由，以及实际验证结果"）
- 架构决策中的经验（"提前验证核心假设避免了后期返工"或"没有提前验证导致了重构"）
- 流程中的经验（"阶段拆分粒度是否合适"、"回退机制是否有效"）

### 第五步：生成 project_summary.md

按以下结构生成最终文档：

1. 项目回顾（目标 vs 结果）
2. 关键技术决策回顾（3-5 个最重要的）
3. 架构演变概要
4. 经验教训（按上述格式）
5. 数据/指标（如适用）

---

## 回顾质量检查

- [ ] 决策回顾覆盖了每个阶段的关键决策
- [ ] 每个关键决策都有"回头看"的评价
- [ ] 经验教训有具体场景，不是泛泛的"要做好规划"
- [ ] 适用边界明确——知道什么时候该用、什么时候不该用
- [ ] project_summary.md 独立可读，不依赖 decisions.md 的上下文
