# skill-creator-design

> **MethodCraft** series — 把经过验证的方法论编译为可执行的 AI Skill

根据你的设计需求，生成一个定制化的设计项目 Skill。

生成的 Skill 会在你的整个设计过程中持续引导——从需求定义、调研、方案设计、评审到交付。不是一次性的头脑风暴，而是一个贯穿项目全程的设计伴侣。

主要面向产品设计（product design）场景优化，同时对其他设计领域（服务设计、空间设计、品牌设计等）提供通用引导——核心框架不变，术语和侧重点由 agent 根据具体领域适配。

## 核心特性：自带 Memory 系统

生成的 Skill 内置文件级 memory 机制，让 agent 跨对话记住你的设计进度和决策：

- **决策追溯**：每个关键设计决策自动记录到 decisions.md——选了什么、为什么、放弃了什么——项目再长也不会忘记"当初为什么这么决定"
- **按步骤组织**：产出物按设计步骤分文件夹（requirements/、research/、design/、review/、output/），结构清晰
- **跨对话恢复**：每次开新对话，Skill 自动读取 plan.md 和最近的决策记录，声明"当前进度：步骤 3 方案设计"——不需要你重复交代上下文
- **智能读取优先级**（完整模式）：四层读取策略，优先加载当前步骤相关文件，上游产出物按需参考
- **严禁删除**：产出物更新是覆盖写入最新内容，变更历史在 decisions.md 中保留

## 快速开始

安装后，对 agent 说：

- "我想设计一个用户 onboarding 流程"
- "帮我生成一个设计 skill"
- "创建设计项目"
- "I want to design a notification system"

Creator 会用 2-3 轮对话了解你的情况（设计什么、目标、范围、约束），然后生成一个定制的 SKILL.md。

## 它会问你什么

| 信息 | 为什么需要 | 示例 |
|---|---|---|
| 设计主题 | 确定领域和方向 | "用户 onboarding 流程"、"通知系统" |
| 设计目标 | 决定成功标准 | "新用户 7 天留存率从 30% 提到 50%" |
| 设计范围 | 确定复杂度和结构 | "0→1 新产品" / "现有 App 的功能迭代" |
| 约束条件 | 划定可行边界 | "技术栈限制 React Native"、"2 周内交付" |

不会问太多。目标受众、交付物偏好之类的，如果你在对话中自然提到了会采用，没提到就用合理默认值。

## 两种模式

| | 轻量模式 | 完整模式 |
|---|---|---|
| 适合 | 单功能设计 / 小型迭代 | 新产品 0→1 / 大型功能 |
| 调研深度 | 竞品分析为主 | 竞品 + 可行性 + 用户调研 |
| 评审 | 三视角快速检查 + 聚焦 1-2 个关键风险 | 三视角深度评审 + 多角度系统质疑 |
| 结项 | 精简设计总结 | 完整决策回顾 + 原则提炼 + 迭代方向 |
| 文件管理 | 全量加载（文件少） | 四层优先级动态调整 |

Creator 会根据设计范围推荐模式，你可以覆盖。两种模式在覆盖范围内的引导质量一致——轻量不是"缩水版"，是为小型项目裁剪了深度调研和系统评审的环节。

## 生成物长什么样

生成的项目 Skill 安装在项目文件夹中。随着设计推进，项目目录会自动生长为这样：

```
design_user_onboarding/
├── {skill_prefix}/design_user_onboarding/
│   ├── SKILL.md                          # 主控逻辑：启动协议 + 五步流程 + 回退/验证
│   └── references/
│       ├── review_checklist.md           # 评审执行清单
│       └── summary_guide.md              # 设计总结生成指南
├── plan.md                               # 设计计划 + 步骤进度
├── decisions.md                          # 决策记录（追加式，永不覆盖）
├── requirements/
│   └── requirements.md                   # 需求边界文档
├── research/
│   └── research.md                       # 调研摘要
├── design/
│   └── design.md                         # 方案文档
├── review/
│   └── review.md                         # 评审记录
├── output/
│   └── [交付物]                          # PRD / 技术方案 / 其他
└── summary_user_onboarding.md            # 设计总结（结项时生成）
```

安装后，Skill 引导你的设计项目走完五个步骤：

1. **需求定义** — 聚焦核心场景、划定边界、定义成功指标
2. **调研** — 竞品分析、技术可行性、用户调研（按需）
3. **方案设计** — 核心流程 + 异常路径 + 决策记录
4. **评审** — 三视角（用户/技术/业务）检查 + 多角度质疑
5. **输出** — 产出交付物（PRD 等）

贯穿全程的两个机制：

- **推进与回退** — 每步完成后主动推进下一步；当新信息与上游结论矛盾时，建议回退重新梳理
- **即时验证** — 识别到关键假设或决策时，当场从多视角质疑，不等到步骤 4 才发现问题

## 交付方式

生成完成后你可以选择：

- **直接创建** — 在当前目录的上级创建项目文件夹，Skill 直接可用
- **ZIP 打包** — 生成 zip 文件，可自行解压到任意位置或分享

## 兼容性

主要为 OpenClaw 设计，同时兼容能操作本地文件系统的 AI Agent 产品（Claude Code、WorkBuddy、CodeBuddy 等）。

- 所有指令使用描述性语言，不依赖特定平台 API
- 安装路径自动适配平台（探测已有 skill 目录，兜底 `.agents/skills/`）
- 文件路径使用相对路径
- 文件格式统一为 Markdown
- frontmatter 为标准 YAML，非 OpenClaw 环境下 agent 忽略 frontmatter 仍可读取 body

## 设计说明

这个 Creator 背后是经过实际设计项目验证的方法论——有向五步流程（需求定义→调研→方案设计→评审→输出）配合智能回退机制，确保设计过程既有推进方向又不会一条路走到黑。

几个关键设计决策：

- **文件即 memory**：通过 plan.md + decisions.md 实现跨对话状态保持，不依赖任何平台的记忆机制
- **设计伴侣，不是项目管理器**：专注帮你完成一次设计任务，不替你管日程和看板
- **即时验证 + 集中评审**：关键假设当场验证，不积压到最后；步骤 4 做系统性三视角评审
- **引导而非强制**：步骤可以跳过、评审深度可以调整——Skill 给建议，你做决定
