# 设计总结生成指南（轻量模式）

> 本文件在结项模式触发时加载。指导如何从项目决策记录和产出物中生成简明设计总结。

## 什么是设计总结

设计总结不是各步骤产出物的拼接，而是对整个设计过程的**反思性回顾**——提炼出关键决策和取舍逻辑。

好的设计总结应该能让未来的自己（或接手的人）快速理解：为什么方案长这样、关键取舍是什么、哪些仍有风险。

## 生成步骤

### 第一步：加载素材

读取以下文件：
- decisions.md 全文——这是最核心的素材
- requirements.md（需求边界）
- design.md（方案文档）
- review.md（评审记录）

### 第二步：提炼决策链

从 decisions.md 中梳理关键决策，识别：
- 哪些决策经历了反复或触发了回退——这些是最有价值的取舍
- 每个关键决策的核心权衡是什么

### 第三步：标注风险和开放问题

从 review.md 的风险清单和 decisions.md 中汇总：
- 已识别并已应对的风险（记录应对方案）
- 仍然开放的问题（记录后续需要关注的点）

### 第四步：组装并保存

按以下结构组装设计总结：

```
# [设计主题] 设计总结

## 设计概要
[一段话概括：设计了什么、解决什么问题、核心方案是什么]

## 关键设计决策
### [决策1标题]
- 决策：[选了什么]
- 权衡：[考虑过什么、为什么放弃]
（列出最重要的决策，通常 2-3 个）

## 风险与开放问题
| 类型 | 内容 | 状态 | 备注 |
|---|---|---|---|
| 已应对 | [风险] | [应对方案] | |
| 开放 | [问题] | 待观察 | [后续关注点] |

## 后续方向
- [可能的优化或迭代方向]
```

保存为 `summary_[topic_slug].md`，通知用户项目完成。
