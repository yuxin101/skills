# Flash Company 快速参考

## 🚀 一键启动

### 自动匹配团队

```
用户说："临时组队，帮我写一个登录功能"
→ 自动匹配：开发组
→ 创建：产品经理 + 程序员 + 测试员
```

### 指定团队

```
用户说："flash 开发组，帮我设计API"
用户说："临时组队 内容组，写一篇技术文章"
用户说："flash biz，分析这个商业模式"
```

## ⚡ vs Virtual Company

| 场景 | 推荐 |
|------|------|
| 快速一次性任务 | ✅ Flash Company |
| 不需要长期记忆 | ✅ Flash Company |
| 简单协作需求 | ✅ Flash Company |
| 长期运营项目 | ✅ Virtual Company |
| 需要独立记忆 | ✅ Virtual Company |
| 复杂团队协作 | ✅ Virtual Company |

**Flash = 临时工，Virtual = 正式员工**

---

## 🔧 实现方式

当用户说"临时组队"时：

1. **识别任务类型** → 匹配合适的团队
2. **生成子代理命令** → 根据模板生成
3. **执行创建** → 调用 sessions_spawn
4. **分发任务** → 给每个成员分配任务
5. **协作完成** → 收集结果
6. **清理** → 任务完成后销毁

## 💡 使用示例

### 示例 1：开发功能

```
用户："临时组队，帮我写一个用户注册功能"

执行：
1. 匹配 → 开发组
2. 创建子代理：
   - 产品经理 (qwen3-max) → 分析需求
   - 程序员 (glm-5) → 写代码
   - 测试员 (qwen3.5-plus) → 写测试
3. 协作完成
4. 输出：代码 + 测试用例
```

### 示例 2：内容创作

```
用户："flash 内容组，帮我写一篇关于AI的文章"

执行：
1. 匹配 → 内容组
2. 创建子代理：
   - 策划 (qwen3-max) → 选题大纲
   - 写手 (qwen3.5-plus) → 撰写正文
   - 编辑 (glm-5) → 润色优化
3. 协作完成
4. 输出：完整文章
```

### 示例 3：商业分析

```
用户："临时组队 分析组，评估这个创业项目"

执行：
1. 匹配 → 分析组
2. 创建子代理：
   - 分析师 (qwen3-max) → 数据分析

   - 研究员 (qwen3.5-plus) → 市场调研

```

## ⚠️ 注意事项

1. **临时性质** - 任务完成即销毁，不保留记忆
2. **规模建议** - 每次 3-5 人，避免协作复杂度过高
3. **任务边界** - 适合明确边界的任务
4. **协作模式** - 默认串行，可指定并行
