# Phase 2 — Execution Chain

**Purpose:** From "Business Model Canvas" to "knowing what to do this week."
**Prerequisite:** Phase 1 complete, BMC nine-cell grid filled.
**Preceding file:** Read `~/opc-guide/bmc-[project-name]-[date].md`
**Output:** 90-day plan + this week's action + validation metrics

---

## Read Preceding File

```
~/opc-guide/bmc-[project-name]-[date].md
```

Start from Q8's three failure assumptions to derive obstacles.

## Link 1: What are your obstacles?

Based on Phase 1 Q8 (failure assumptions), derive the top three execution obstacles.

> "In Phase 1, you named three likely failure reasons. Now it's time to act, not think. From these three, which one will you hit this week?"

**Format:**

```
Obstacle 1: [specific — not "no users", but "don't know where to find users"]
Obstacle 2: [specific]
Obstacle 3: [specific]
```

**Follow-up:**

> "Which of these three obstacles could you start pushing on today?"

## Link 3: Validation Metrics

> "Over these 90 days, what single metric proves you're moving in the right direction?"

**Not "how many users" — ask:**

- What is the one number you check every week?
- Does growing or shrinking that number tell you whether to change direction next week?

> "If this number doesn't change for two weeks in a row, what does that tell you?"

**One-person companies are especially prone to "busyness" — feeling productive without going anywhere. This number is your compass.**

Format:
```
Validation metric: [the one number you check every week]
衡量标准: [what the number needs to be to confirm the direction is right]
```
