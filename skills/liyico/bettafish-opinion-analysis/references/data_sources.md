# BettaFish 数据来源指南

本指南说明 BettaFish 舆情分析 Skill 的数据获取方式。

## 重要声明

**本 Skill 不使用任何数据库！**

所有数据通过 WebSearch/WebFetch 实时获取，确保分析的时效性和独立性。

## 数据获取方式

### 1. WebSearch（主要方式）

使用网络搜索获取实时数据：

```python
# 搜索新闻和网页
WebSearch("某咖啡连锁品牌 评测")

# 搜索特定平台内容
WebSearch("site:xiaohongshu.com 某咖啡品牌")

# 搜索近期内容
WebSearch("某咖啡品牌 口碑 2024")
```

### 2. WebFetch（补充方式）

获取具体页面的详细内容：

```python
# 获取文章详情
WebFetch("https://zhuanlan.zhihu.com/p/xxx")
```

### 3. Curl 命令行（备用方式）

当 WebFetch 无法获取特定页面时，使用 curl 命令行获取内容：

```bash
# 基础请求
curl -s "https://api.example.com/data"

# 带 User-Agent 的请求（模拟浏览器）
curl -s -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36" \
  "https://www.example.com/page"

# 跟随重定向
curl -s -L "https://www.example.com/redirect"

# 获取 JSON 数据并格式化
curl -s "https://api.example.com/data" | python -m json.tool

# 保存到文件
curl -s -o output.html "https://www.example.com/page"
```

**使用场景**：
- API 接口数据获取
- 需要特殊 Headers 的页面
- WebFetch 返回不完整内容时
- 批量抓取静态页面

## 支持的数据源

### 新闻资讯
- 新浪财经、腾讯科技、网易科技
- 36氪、虎嗅、钛媒体
- 主流门户科技频道

**获取方式**: `WebSearch("品牌名+新闻/评测/报道")`

### 社交媒体

#### 微博
- **特点**: 实时性强、热点发酵快
- **获取**: `WebSearch("site:weibo.com 品牌名")`

#### 小红书
- **特点**: 种草属性、详细体验
- **获取**: `WebSearch("site:xiaohongshu.com 品牌名")`

#### 抖音
- **特点**: 短视频、传播力强
- **获取**: `WebSearch("抖音 品牌名 口碑/评测")`

#### B站
- **特点**: 深度分析、弹幕文化
- **获取**: `WebSearch("site:bilibili.com 品牌名 评测")`

#### 知乎
- **特点**: 理性讨论、专业性强
- **获取**: `WebSearch("site:zhihu.com 品牌名")`

### 论坛社区
- 贴吧、豆瓣小组、V2EX、虎扑
- **获取**: `WebSearch("site:tieba.baidu.com 品牌名")`

## 搜索策略

### 关键词组合

1. **基础搜索**
   ```
   "品牌名 + 口碑"
   "品牌名 + 评价"
   "品牌名 + 怎么样"
   ```

2. **深度搜索**
   ```
   "品牌名 + 优点/缺点"
   "品牌名 + 问题/故障"
   "品牌名 + 对比/比较"
   ```

3. **事件搜索**
   ```
   "品牌名 + 事件名"
   "品牌名 + 危机/负面"
   "品牌名 + 回应/声明"
   ```

### 时间筛选

通过关键词控制时间范围：
- 近24小时：添加"今天/最新"
- 近7天：添加"本周/最近一周"
- 近30天：添加"本月/最近一个月"

## 数据可信度评估

| 可信度 | 数据源 | 标识 |
|--------|--------|------|
| ⭐⭐⭐ 高 | 官方声明、权威媒体 | 官方/权威 |
| ⭐⭐ 中 | KOL观点、用户评论 | 参考/分析 |
| ⭐ 低 | 匿名爆料、单一信源 | 待验证 |

## 数据保存

由于不使用数据库，数据以以下形式保存：

1. **分析过程**: 保存在内存中
2. **最终结果**: 导出为 HTML/Markdown 文件
3. **图谱数据**: 嵌入 HTML 的 JSON 中

## 隐私与合规

1. **数据脱敏**: 用户ID使用化名（@用户A）
2. **引用规范**: 注明内容来源
3. **时效标注**: 明确数据采集时间
4. **免责声明**: 说明数据局限性

## 与数据库方案的对比

| 特性 | 数据库方案 | 本Skill方案 |
|------|-----------|------------|
| 数据时效性 | 依赖更新频率 | 实时获取 |
| 存储成本 | 需要存储空间 | 无存储需求 |
| 部署复杂度 | 需要配置数据库 | 零配置 |
| 数据量 | 受存储限制 | 按需获取 |
| 隐私风险 | 需保护用户数据 | 不存储敏感数据 |
| 离线能力 | 可离线查询 | 需要网络 |

## 最佳实践

1. **多源验证**: 同一信息至少2个来源确认
2. **交叉分析**: 不同平台数据相互印证
3. **时效关注**: 优先使用近期数据
4. **样本标注**: 明确数据样本量和时间范围
5. **误差说明**: 说明搜索数据的局限性
