# evolve-rules.md — Workspace 持续生长规则

> 这套规则是让 Agent 从"能用"走向"好用"的核心机制。
> 必须写入每个 Agent 的 AGENTS.md，不是可选功能，是基础工作规范。
>
> 核心理念：人在行动中暴露自己，而不是在问答中。
> Workspace 的质量，不由创建时决定，而由使用过程中的持续积累决定。
>
> 人伴型 Agent 的进化路径：了解他 → 理解他的工作 → 预判他 → 替代他 → 超越他。
> 触发式写入是这条路径的引擎——不只积累偏好，更要积累业务判断模式。

---

## 一、触发式写入规则（每次对话中执行）

### 何时触发

以下情况发生时，**立刻写入对应文件**，不等积累，不等 heartbeat：

| 触发条件 | 写入位置 | 写入内容 |
|---|---|---|
| 用户明确说"记住这个" | MEMORY.md 或 memory/ | 用户说的内容（判断是否长期，见下方规则） |
| 用户对输出说"不用那么细" / "详细点" | USER.md | 输出偏好条目 |
| 用户反复用同一个词/说法（3次以上） | USER.md | 术语习惯条目 |
| 用户提到新项目/客户名 | memory/当天文件 | 业务背景 |
| 用户明确表达不喜欢某种方式 | USER.md | 偏好限制条目 |
| 用户做出一个决策，并说明理由 | MEMORY.md | 业务判断模式（这类情况他倾向于X）|
| 用户否定了某个方向，说"不是这样的" | MEMORY.md | 业务判断边界（他不会这样决策）|
| 用户对某个方案说"对，就是这个感觉" | MEMORY.md | 业务判断偏好（他认可的方向）|
| 发现 USER.md 里有错误信息 | USER.md | 更正 |

### 写入后的动作

每次写入后，简短告知用户：
```
"已经记下来了，以后会[具体说明]。"
```

不需要详细解释，一句话即可。

### MEMORY.md vs memory/ 的写入判断

```
用户说"记住这个"时：
  问自己：这个信息一年后还有效吗？
  → 是 → MEMORY.md（长期知识）
  → 否 → memory/当天文件（临时上下文）

观察到稳定行为（3次以上）：
  → USER.md + MEMORY.md（双写，偏好是稳定的）

当次对话的重要事件、决定：
  → memory/当天文件
```

## 二、对话中的触发式追问规则

当发现重要信息缺失时，不在对话开头集中提问，而是在对话流程里自然问一句。

### 原则

- **一次只问一个**
- **在自然节点问**（任务完成后，不是任务进行中）
- **问完立刻写入**，不积累
- **一周内同类问题不重复问**（避免烦人）

### 示例

```
帮用户写完文案后：
"我注意到你比较喜欢短句，以后默认用这个风格吗？"
→ 用户确认 → 立刻写入 USER.md
```

```
用户第三次把"客户"叫"甲方"：
"你习惯叫'甲方'，我以后也用这个词。"
→ 写入 USER.md，不需要用户回答
```

## 三、Heartbeat 精炼规则（每 3 天执行一次）

### 执行步骤

```
1. 读最近 3 天的 memory/YYYY-MM-DD.md 文件
   （只看最近 3 天，不要试图读所有历史）

2. 识别以下内容：
   □ 新的稳定偏好（出现 2 次以上）
   □ 新的业务背景（项目/客户）
   □ 用户明确表达过的喜好或厌恶
   □ 需要更新的旧信息

3. 提炼写入：
   □ 稳定偏好 → USER.md
   □ 长期有效的业务背景 → MEMORY.md
   □ SOUL.md 有没有需要微调的（风格已经变化了？）

4. 清理过时内容：
   □ USER.md 里有没有已经不准确的条目
   □ MEMORY.md 里有没有已经过期的信息
```

### 执行完毕后

在当天 memory/ 文件里简短记录：
```
# Heartbeat 精炼 YYYY-MM-DD
- 提炼进 USER.md：[内容摘要]
- 提炼进 MEMORY.md：[内容摘要]
- 清理：[删除了什么]
```

## 四、写入 AGENTS.md 的具体段落

以下内容**逐字复制**进每个 Agent 的 AGENTS.md 的"记忆规则"部分：

```markdown
## 记忆规则（核心，每次对话执行）

### 触发式写入
以下情况发生时，立刻写入对应文件：

- 用户明确说"记住这个" → 判断长期性：
  - 长期有效 → MEMORY.md
  - 临时上下文 → memory/当天文件
- 用户对输出表达偏好（简洁/详细/风格调整）→ USER.md
- 用户反复使用同一个词/说法（3次以上）→ USER.md 术语习惯
- 用户提到新项目/客户 → memory/当天文件
- 用户明确说不喜欢某种方式 → USER.md

每次写入后告知用户一句话：
"已经记下来了，以后会[具体说明]。"

### 追问规则
发现重要信息缺失时：
- 在任务完成后自然问一句
- 一次只问一个
- 问完立刻写入
- 同类问题一周内不重复

### Heartbeat 精炼
每 3 天，读最近 3 天 memory/ 文件，提炼进 USER.md 和 MEMORY.md，
清理过时内容。只看最近 3 天，不要试图一次读完所有历史。
```
