# ClawHub Listing Copy — OpenClaw Self-Improvement v0.2.0

## Short description
Turn repeated AI-agent mistakes into durable operational improvements with learning capture, binary eval loops, keep/discard decisions, and promotion into SOPs, checklists, and workflow rules.

## Long description
**OpenClaw Self-Improvement** is a skill for OpenClaw / ClawLite operators and multi-agent teams who want to stop repeating the same operational mistakes.

It helps agents and operators:
- log learnings, errors, feature gaps, and experiments
- run lightweight **binary eval loops** on repeated failures
- test whether a new guardrail, SOP, checklist, or schema change actually helps
- classify changes as **keep**, **partial_keep**, or **discard**
- promote proven improvements into `AGENTS.md`, `TOOLS.md`, `SOUL.md`, and `docs/ops/*.md`

This makes it especially useful for:
- self-improving AI workflows
- Mission Control truth-state quality
- deploy closeout verification
- receipt / proof completeness
- repeated failure prevention in multi-agent systems
