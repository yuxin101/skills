## 工作原理（安全模式）

```
用户提问 → web_search(搜索引擎) → 读取公开搜索结果 → 整理成报告
```

### 为什么安全？

| 操作 | 传统爬虫 | 本技能 |
|------|----------|--------|
| 请求目标 | 直接访问电商平台 | 只访问搜索引擎 |
| 数据获取 | 解析 HTML/DOM | 读取搜索摘要 |
| Cookie 处理 | 尝试登录态 | 不处理任何 cookie |
| 反爬绕过 | 可能尝试规避 | 完全遵守 robots.txt |
| 法律风险 | 中到高 | **极低** |

✅ **关键点**: 搜索引擎索引的数据本来就是公开的，就像你在 Google/百度上搜索一样自然。

## 局限性（重要！）

### ❌ 搜索能显示的 vs 不能显示的

| 数据项 | 搜索能看到吗？ | 说明 |
|--------|----------------|------|
| 标价 | ✅ 通常能看到 | 搜索引擎索引的公开价格 |
| 会员价 | ❌ 看不到 | 需要登录后才能看到 |
| 优惠券 | ❌ 不确定 | 部分会显示在摘要里 |
| 运费 | ⚠️ 有时能看到 | 取决于平台是否公开 |
| 实时库存 | ❌ 看不到 | 搜索快照可能过期 |
| 促销活动 | ⚠️ 部分看到 | 复杂活动可能被简化 |

### 💡 怎么办？

**方案 A: 接受近似值**
- 用于初步筛选和研究
- 最后决定前再去平台核实
- 通常误差 ±10-20%

**方案 B: 半手动模式**
```
1. OpenClaw 提供各平台链接给你
2. 你手动打开这些页面
3. 告诉我看到的实际价格
4. 我帮你整理最终对比
```

**最准确的做法**: 
> "帮我找这几个平台的链接，我自己去查实际价格"

## 常见问答

### Q: 为什么有些价格显示"未找到"？

A: 
- 该商品在该网站可能没有上架
- 或者没有被搜索引擎收录
- 建议换更简单的关键词重试

### Q: 价格和实际买的时候不一样怎么办？

A:
- 这是正常的，搜索结果是某个时间点的快照
- 电商价格经常变动
- 最终以下单页面显示为准

### Q: 这个真的没有法律风险吗？

A:
- ✅ 我们只是用搜索引擎查公开信息
- ✅ 就像你自己在百度搜一样
- ✅ 不会直接访问或爬取各平台
- ⚠️ 但请遵守当地法律，不要商用

### Q: 有会员价我怎么知道？

A:
- 当前方法看不到登录后才有的价格
- 建议：用这个做初步研究，真买的时候再登录查看会员价

## 技术原理（给好奇的你）

### Data Flow

```mermaid
graph LR
    A[用户提问] --> B[web_search]
    B --> C[搜索引擎结果]
    C --> D[提取公开数据]
    D --> E[整理分析]
    E --> F[生成 Markdown 报告]
```

### What We Use

- `web_search` - 调用搜索引擎 API
- `web_fetch` - 获取完整搜索结果页
- Built-in NLP - 理解问题意图

### What We Don't Do

- ❌ No direct HTTP requests to shopping sites
- ❌ No JavaScript execution
- ❌ No cookie manipulation
- ❌ No login attempts
- ❌ No DOM parsing
