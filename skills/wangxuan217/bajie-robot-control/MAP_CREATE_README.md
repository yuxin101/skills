# 建图任务优化说明

## 问题背景

建图任务 (`map_create`) 在执行时报错：
- **错误码**: `0x000028D0` (10448)
- **错误信息**: 桩充状态检测未在充电

## 根本原因

服务端建图任务的先决条件检查要求：
1. 机器人必须在充电桩上
2. 充电状态必须为 `isCharge=1`

但实际场景中建图需要机器人**离开充电桩探索环境**，这个要求存在设计矛盾。

## 重要更新 (v2.1)

**任务失败后不自动重试**

- ❌ 旧逻辑：失败后自动重试 3 次
- ✅ 新逻辑：失败后立即返回，等待用户指令

**原因**：避免在已知问题下重复尝试，节省时间并提供清晰的错误信息。

## 优化方案

### 1. 优化版脚本：`map_create_optimized.py`

**功能**:
- 自动回充并等待充电状态确认
- 可配置是否强制要求充电状态
- 详细的进度反馈和错误诊断
- 建图后自动验证地图数据

**使用**:
```bash
# 标准模式（强制充电状态）
python3 map_create_optimized.py

# 跳过充电状态检查
python3 map_create_optimized.py --no-force-charge

# 自定义等待时间
python3 map_create_optimized.py --max-wait 180
```

### 2. 完整测试脚本：`test_map_create_final.py`

**功能**:
- 5 步完整流程（状态→回充→建图→结果→验证）
- 详细的日志输出
- 友好的错误提示和建议

**使用**:
```bash
python3 test_map_create_final.py
```

### 3. 核心库修改：`bajie_robot.py`

**修改内容**:
```python
async def map_create(self, timeout: float = 900.0, auto_retry: bool = True) -> TaskResult:
    """
    建图任务
    
    Args:
        timeout: 超时时间（秒），默认 900 秒（15 分钟）
        auto_retry: 是否自动重试（默认 True）
    """
    return await self._execute_mission("map_create", {}, timeout=timeout, auto_retry=auto_retry)
```

**改进**:
- 增加超时参数（默认 15 分钟）
- 支持关闭自动重试
- 更清晰的文档说明

## 使用建议

### 场景 1: 正常建图

```bash
python3 test_map_create_final.py
```

**注意**：任务失败后不会自动重试，会直接返回错误信息。

### 场景 2: 充电状态检测失败

如果报错 `0x000028D0`：

1. **检查硬件**:
   - 充电桩是否通电
   - 充电触点是否清洁
   - 机器人屏幕是否显示充电图标

2. **手动回充后重试**:
   - 手动将机器人放回充电桩
   - 等待 30 秒确认充电状态
   - 重新执行建图命令

3. **如硬件正常但仍失败**:
   - 记录错误码和日志
   - 联系技术支持检查服务端逻辑

### 场景 3: 建图超时

如果建图超时：

1. 检查网络连接
2. 确认机器人移动空间充足
3. 清理 LDS 激光雷达
4. 增加超时时间：
   ```python
   result = await robot.map_create(timeout=1200.0)  # 20 分钟
   ```

## 错误码速查

| 错误码 | 十六进制 | 含义 | 处理 |
|--------|----------|------|------|
| 10448 | 0x000028D0 | 桩充状态检测未在充电 | 检查充电硬件 |
| 10449 | 0x000028D1 | 桩充状态检测未在充电桩上 | 手动回充 |
| 28674 | 0x00007002 | 全局规划失败 | 清理障碍物 |
| 28676 | 0x00007004 | 路径被阻挡 | 清除路径障碍 |
| -1 | N/A | 超时 | 检查网络/增加超时 |
