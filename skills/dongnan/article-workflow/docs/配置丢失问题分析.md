# 配置丢失问题 - 根因分析与解决方案

## 🔍 问题分析

### 症状
每次重构或优化 Skill 时，都会提示"没有配置"，但用户之前已经配置过。

### 根本原因

1. **配置文件在.gitignore 中**（正确做法）
   ```
   # .gitignore
   config.json  # 包含敏感信息，不应提交
   .env         # 环境变量，不应提交
   ```

2. **修改工具覆盖配置**
   - 使用 `write` 工具修改文件时会覆盖整个文件
   - 没有检查是否已有本地配置
   - 没有备份和恢复机制

3. **缺乏配置保护意识**
   - 开发时没有先备份配置
   - 升级后没有恢复配置
   - 没有配置合并机制

## ✅ 解决方案

### 1. 配置保护脚本（已创建）

|------|------|---------|

| `scripts/config_manager.py` | 配置管理工具 | 检查/备份/恢复/向导/合并 |

### 2. 配置管理工具（已创建）

```bash

# 备份配置

# 恢复配置

# 配置向导（首次使用）

# 合并配置（升级时）
python3 scripts/config_manager.py merge
```

### 3. 配置合并逻辑

**智能合并规则：**
- ✅ **保留** 用户敏感信息（Bitable Token、Table ID）
- 🔄 **更新** 新增配置项
- 🔄 **更新** 默认配置（workflow、paths 等）

**合并代码示例：**
```python
def merge_config(new_config: dict, existing_config: dict) -> dict:
    """合并配置，保留用户自定义设置"""
    merged = existing_config.copy()
    
    for key, value in new_config.items():
        if key == 'bitable' and 'bitable' in merged:
            # Bitable 配置保留现有的（敏感信息）
            if existing_config['bitable'].get('app_token'):
                merged['bitable']['app_token'] = existing_config['bitable']['app_token']
            if existing_config['bitable'].get('table_id'):
                merged['bitable']['table_id'] = existing_config['bitable']['table_id']
        elif isinstance(value, dict) and key in merged:
            merged[key] = merge_config(value, merged[key])
        else:
            merged[key] = value
    
    return merged
```
