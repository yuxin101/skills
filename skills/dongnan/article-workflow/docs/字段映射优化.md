# Bitable 字段映射优化指南

## 📊 字段映射配置

### 完整字段映射表

| 内部字段 | Bitable 字段名 | 字段类型 | 必填 | 说明 |
|---------|--------------|---------|------|------|
| `primary` | 多行文本 | 文本 | ✅ | 主字段（自动生成） |
| `title` | 标题 | 文本 | ✅ | 文章标题 |
| `url` | URL 链接 | 文本 | ✅ | 原文链接 |
| `summary` | 简短摘要 | 文本 | ✅ | 3-5 句摘要 |
| `read_date` | 阅读日期 | 日期 | ✅ | 分析日期 |
| `assignee` | 负责人 | 人员 | ✅ | 自动填充当前用户 |
| `review_date` | 复盘日期 | 日期 | ❌ | 后续复盘用 |
| `source` | 来源 | 单选 | ✅ | 平台映射（微信/GitHub） |
| `tags` | 关键词标签 | 多选 | ✅ | 自动提取标签 |
| `importance` | 重要程度 | 单选 | ✅ | 高/中/低 |
| `status` | 状态 | 单选 | ✅ | 已完成/处理中/待处理 |
| `creation_method` | 创建方式 | 单选 | ✅ | 自动分析/手动触发 |
| `doc_url` | 详细报告链接 | 文本 | ✅ | 飞书文档链接 |
| `cover_image` | 封面图片 | 附件 | ❌ | 文章封面图 |
| `attachments` | 附件 | 附件 | ❌ | 相关附件 |

## 🔧 配置说明

### 1. 基础配置

在 `config.json` 中配置：

```json
{
  "bitable": {
    "app_token": "YOUR_TOKEN",
    "table_id": "YOUR_TABLE_ID"
  },
  "bitable_fields": {
    "primary": "多行文本",
    "title": "标题",
    "assignee": "负责人",
    "source": "来源",
    "tags": "关键词标签"
    // ... 其他字段
  }
}
```

### 2. 字段映射规则

**来源平台映射：**
```json
{
  "bitable_field_mapping": {
    "source_platform": {
      "微信": "微信",
      "今日头条": "微信",
      "知乎": "微信",
      "GitHub": "GitHub",
      "其他": "微信"
    }
  }
}
```

**说明：** 由于 Bitable 单选字段只能选择已有选项，需要将不同平台映射到已有选项。

### 3. 自动填充配置

```json
{
  "auto_assign": {
    "enabled": true,
    "default_assignee": "ou_e84c86a385551ed902a3d8d2c57c63c3"
  }
}
```

**说明：**
- `enabled`: 是否自动填充负责人
- `default_assignee`: 默认负责人 open_id

## 📝 字段填充规则

### 1. 负责人字段（人员类型）

**格式：**
```json
{
  "负责人": [
    {
      "id": "ou_e84c86a385551ed902a3d8d2c57c63c3",
      "name": "董楠",
      "type": "user"
    }
  ]
}
```

**自动生成：**
```python
from field_mapper import format_person_field

assignee_field = format_person_field(
    user_id="ou_e84c86a385551ed902a3d8d2c57c63c3",
    name="董楠"
)
```

### 2. 日期字段（日期类型）

**格式：** 毫秒时间戳
```json
{
  "阅读日期": 1773541200000
}
```

**自动生成：**
```python
from field_mapper import format_date_field

# 使用当前日期
read_date = format_date_field()

# 使用指定日期
read_date = format_date_field("2026-03-15")
```

### 3. 标签字段（多选类型）

**格式：** 字符串数组
```json
{
  "关键词标签": ["OpenClaw", "百度搜索", "AI 智能体"]
}
```

**说明：** 如果标签不存在，Bitable 会自动创建新选项。

### 4. 来源字段（单选类型）

**格式：** 字符串（必须是已有选项）
```json
{
  "来源": "微信"
}
```

**映射规则：**
- 今日头条 → 微信
- 知乎 → 微信
- GitHub → GitHub
- 其他 → 微信

### 5. 封面图片（附件类型）

**格式：**
```json
{
  "封面图片": [
    {
      "file_token": "baxxxxxxx",
      "name": "cover.jpg"
    }
  ]
}
```

**自动提取：**
```python
from field_mapper import extract_cover_image_url

# 从文章内容提取第一张图片
cover_url = extract_cover_image_url(content)

# 下载并上传到 Bitable（需要飞书 API）
```

## ⚠️ 常见问题

### 问题 1：负责人字段为空

**原因：** 人员字段格式不正确

**解决：**
```python
# ❌ 错误格式
{"负责人": "ou_e84c86a385551ed902a3d8d2c57c63c3"}

# ✅ 正确格式
{"负责人": [{"id": "ou_xxx", "name": "董楠", "type": "user"}]}
```

### 问题 2：来源字段报错

**原因：** 单选字段值不在选项中

**解决：** 使用映射函数
```python
from field_mapper import map_source_platform

mapped_source = map_source_platform("今日头条", config)
# 返回："微信"
```

### 问题 3：日期字段格式错误

**原因：** 日期格式不是毫秒时间戳

**解决：**
```python
from field_mapper import format_date_field

# ✅ 正确格式
{"阅读日期": format_date_field("2026-03-15")}
# 返回：1773541200000
```

### 问题 4：标签字段不显示

**原因：** 标签字段是多选类型，需要数组格式

**解决：**
```python
# ❌ 错误格式
{"关键词标签": "OpenClaw"}

# ✅ 正确格式
{"关键词标签": ["OpenClaw", "AI"]}
```

### 3. 自动化

**在 archiver.py 中自动填充：**
```python
def archive_to_bitable(self, analysis_result: dict, doc_url: str) -> dict:
    """归档到 Bitable，自动填充所有字段"""
    
    # 获取当前用户 ID
    assignee_id = get_current_user_id()
    
    # 创建字段映射
    fields = create_bitable_record_fields(
        title=analysis_result['title'],
        url=analysis_result['url'],
        summary=analysis_result['summary'],
        source=analysis_result.get('source', '其他'),
        tags=analysis_result.get('tags', []),
        importance=analysis_result.get('importance', '中'),
        doc_url=doc_url,
        assignee_id=assignee_id
    )
    
    # 创建记录
    return self.bitable_api.create_record(fields)
```
