# Step 5: 生成配图（HTML生成方式）

**工具：** 生成HTML配图并使用 `temp-convert.js` + Puppeteer 直接截图，支持多种图片类型

## 尺寸设置规范

| 类型    | 尺寸规格           | 比例     | 用途            |
| ----- | -------------- | ------ | ------------- |
| 封面图   | **1280 × 544** | 2.35:1 | 公众号首图（官方推荐尺寸） |
| 竖版封面  | 1080 × 1280    | 4:5    | 公众号分享卡片封面     |
| 文章配图  | 1080 × 任意高度    | -      | 章节插图、信息图、流程图  |
| 小红书配图 | 1080 × 1440    | 3:4    | 小红书笔记配图       |
| 横幅图   | 1280 × 720     | 16:9   | 横幅信息图         |

## HTML格式要求

所有CSS**内联在`<style>`标签中**，不依赖外部样式表或资源文件。每个需要截图的区块必须添加 `.page` 类。

## 使用方法

1. **准备HTML文件**：按照上述规范编写HTML，所有CSS内联，每个图片区块用`.page`包裹
2. **运行转换**：使用命令行参数指定HTML文件路径和输出目录

```bash
node ${SKILL_DIR}/scripts/temp-convert.js --html=your-html-file.html --output=output-images-directory --width=1280 --height=544
```

**参数说明：**

- `--html`：HTML文件路径（必填）
- `--output`：输出目录路径（必填）
- `--width`：页面宽度（可选，默认1400）
- `--height`：页面高度（可选，默认1200）

脚本会自动：

- 启动Puppeteer浏览器
- 加载HTML文件
- 等待字体和样式完全加载
- 对每个 `.page` 元素逐个截图
- 保存PNG格式图片到输出目录

## 截图准确性保证

为了确保截图范围准确无空白：

1. 必须给每个需要截图的元素添加 `.page` 类
2. 固定尺寸图片（封面、海报）必须在CSS中设置精确的width和height
3. 可变高度配图只需要设置width，高度会自适应内容
4. 脚本会等待 `document.fonts.ready` 再截图，确保文字渲染完成
5. 截图使用元素边界，不会产生额外空白

## 输出路径规范

所有图片输出到：

```
工作区/work/output/{YYYYMMDD}-{文章标题}/images/
```

## 图片命名与存放规范

必须遵循以下命名和存放规则：

### 封面图片

- **输出位置**：`images/cover.png`
- **尺寸要求**：建议使用 1280 × 544 像素（公众号官方推荐尺寸）

### 其他图片

- **存放位置**：`images/` 目录下
- **命名建议**：使用有意义的英文或数字命名，避免中文和特殊字符

## 输出要求

- 封面图输出为PNG格式保持最佳质量
- 文章配图根据大小选择PNG/JPEG，确保文件大小 < 1000KB
- 公众号对图片大小限制为 1MB 以内，建议压缩到 500KB - 1000KB 保持加载速度
- 所有图片必须存放在 `images/` 目录或根目录
