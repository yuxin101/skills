# ProcessOn Diagrams - AI Agent 绘图技能

这是一个通用的 AI Agent Skill（技能），兼容 Trae 以及其他支持标准 Skill 规范的智能体。它能够将自然语言描述或代码上下文自动转化为专业美观的 ProcessOn 图形（支持流程图、架构图、时序图、网络拓扑图等）。

## 🚀 核心特性
- **开箱即用 (零依赖)**：底层脚本使用纯 Python 标准库 (`urllib`) 编写。无需 `pip install` 任何第三方库，在 macOS/Linux/Windows 上即插即用。
- **智能意图识别**：根据您的 Prompt 或选中的代码，自动匹配并推荐最合适的图形类型。
- **多轮澄清机制**：拒绝 AI “幻觉”盲目画图。当您的需求描述存在逻辑断层时，它会主动提问并给出专业建议供您确认。
- **双通路稳定执行**：优先调用已安装并可用的 `processon-diagrams` MCP 服务；若未安装或不可用，则降级使用内置 API 脚本生成。
- **配置指引兜底**：如果 MCP 和 API 回退都不可用，技能会明确说明失败原因，并提示如何配置 `PROCESSON_API_KEY`，附带可直接复制的示例命令，避免再自动输出与图片版可能不一致的 Mermaid 结果。
- **本地文件落盘**：脚本模式下会将生成图片保存到工作区 `outputs/processon/`。文件名标题应由大模型根据用户输入先总结成一个简短名称，再传给脚本；默认格式为 `标题-时间戳.png`，若同一次生成了多张图，后续图片才追加序号。脚本本身不负责语义提取，只负责落盘和返回保存路径，方便直接打开或二次引用。

## 📦 安装指南

1. 下载或克隆当前文件夹。
2. 将整个 `processon-diagrams` 文件夹放入您的 AI Agent/IDE 的 Skills 目录中。
   - **Trae 用户**：放入 `~/.trae/skills/processon-diagrams` 或项目级 `.trae/skills/` 目录下。
   - **其他 Agent 用户**：请参考对应客户端的 Skill 安装文档。
3. 重启您的客户端或重新加载窗口即可激活该技能。

## ⚙️ MCP 与 API 配置

本技能有两种接入方式：

### 方式一：已安装 MCP

如果您的宿主已经安装并启用了 `processon-diagrams` MCP 服务，请直接在宿主的 MCP 配置中填写服务地址和密钥。MCP 模式下的鉴权完全由宿主或用户自行配置。若宿主支持凭证交互，技能返回的结构化字段可用于触发“配置凭证”类控件；若宿主不支持，则会回退为明确的手动配置提示。

示例配置：

```json
{
  "mcpServers": {
    "processon-diagrams": {
      "type": "http",
      "url": "https://smart.processon.com/mcp",
      "headers": {
        "Authorization": "Bearer <your-processon-api-key>"
      }
    }
  }
}
```

### 方式二：未安装 MCP，走 API 回退

如果当前宿主没有安装 MCP，技能会自动回退到本地 API 脚本。此时只需要您自行设置 `PROCESSON_API_KEY`：

```bash
export PROCESSON_API_KEY="<your-processon-api-key>"
```

说明：
- `PROCESSON_API_KEY`：API 回退模式必填
- API 回退请求地址由技能内部固定，不对外开放配置
- 当缺少凭证时，技能会返回可直接复制的配置命令、下一步重试提示，以及供宿主识别的凭证交互元数据；是否真正弹出输入控件，取决于当前宿主是否支持该类 UI。
- 当成功生成图片时，技能会尽量返回可内联展示的图片内容与本地保存路径；若宿主支持图片渲染，建议直接展示首图并同步告知用户文件路径。

## 🎯 如何使用

无需输入复杂的命令，直接在聊天窗口中用自然语言提出需求即可。当它检测到“生成图”、“画图”、“架构分析”、“流程图”、“ProcessOn”等关键词时会自动触发。

**经典 Prompt 示例：**
- *"帮我画一个标准电商下单的**时序图**"*
- *"分析当前项目的代码结构，并生成一张**架构图**"*
- *"基于这段代码逻辑，生成一张**流程图**"*
- *"帮我画一张包含 Web 层和 DB 层的**网络拓扑图**"*
