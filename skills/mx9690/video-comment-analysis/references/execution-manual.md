# 视频评论分析 Skill 执行手册

## 目的

这份手册用于统一后续所有 **视频评论分析 / 带货评论分析 / 评论区商业分析** 任务的执行方式。

它解决 3 个核心问题：

1. **评论应该读取多少**
2. **报告应该输出哪些模块**
3. **网页报告应该采用什么设计风格与规范**

这份手册是执行层规则；可视化指标定义与图表边界，请配合《视频评论分析可视化口径规范》一起使用。

关联文档：
- `docs/analysis-standards/video-comment-analysis-visualization-spec.md`

---

## 一、评论读取规则

### 1. 默认读取规则

以后默认按以下规则执行：

- **评论总量 ≤ 100 条：全量读取**
- **评论总量 > 100 条：至少读取 100 条有效评论**

这里的关键词是：

> **有效评论**

不是只凑够数量，而是要读到足以支撑六维分析的评论样本。

---

### 2. 有效评论定义

**有效评论 = 能支撑六维分析中至少一个维度判断的评论**。

通常包含：
- 问价格 / 运费
- 问链接 / 购买入口
- 问材质 / 味道 / 容量 / 颜色 / 细节
- 表达购买意向
- 表达已购买
- 表达质疑 / 犹豫 / 劝退点
- 明确反馈使用体验

通常不算有效评论或仅低权重处理：
- 纯表情
- 单纯 “cute” / “omg” / “love it” 且无更多信息
- 明显重复刷屏
- 没有实际判断价值的无信息评论

---

### 3. 回复处理规则

回复内容单独看待：

- **回复用于辅助解释**
- **默认不混入主评论基础统计**
- 仅在需要验证高价值问题时展开，例如：
  - 价格争议
  - 运费争议
  - 购买反馈
  - 品质质疑
  - 链接问题

默认建议：
- 小评论区：按需展开 3–5 组回复
- 大评论区：按需展开 8–15 组高价值回复

---

### 4. 样本口径写法（固定模板）

每次报告都默认写清楚：

> 本次分析基于 X 条有效主评论；额外展开 Y 组高价值回复；回复内容用于辅助解释，不纳入主评论主题占比统计。

如果评论总量少于 100 条且已全读，可写：

> 本次分析基于评论区全部主评论，共 X 条；额外展开 Y 组高价值回复；回复内容用于辅助解释，不纳入主评论主题占比统计。

---

## 二、默认报告模块

以后默认采用 **10 模块标准结构**。

除非用户明确要求删改，否则默认按这个顺序输出。

---

### 模块 1：封面 / 项目概览

内容建议：
- 视频标题 / 链接
- 分析对象说明
- 评论总量（若可见）
- 本次读取的有效评论数
- 是否展开回复
- 一句话结论

作用：让读者一上来就知道分析了什么、看了多少、结论是什么。

---

### 模块 2：核心结论摘要

建议用结论卡呈现，例如：
- 购买意向强弱
- 主要卖点
- 最大转化阻力
- 整体卖货判断

作用：给先看结论的人一个快速判断入口。

---

### 模块 3：评论主题分布

对应六维之一。

回答问题：
> 评论区主要在讨论什么？

适合数据化呈现。

---

### 模块 4：用户关注点分析

对应六维之一。

回答问题：
> 用户在做购买判断时最在意什么？

适合数据化呈现。

---

### 模块 5：购买意向分析

对应六维之一。

回答问题：
> 这条内容的评论区，是“有人觉得不错”，还是“已经具备成交能力”？

适合结构化呈现。

---

### 模块 6：成交驱动因素

对应六维之一。

回答问题：
> 这条内容为什么能卖？

建议用：
- 核心驱动
- 强驱动
- 辅助驱动

不建议伪精确评分。

---

### 模块 7：影响转化因素

对应六维之一。

回答问题：
> 到底是什么在拦截成交？

适合做阻力排行或优先级呈现。

---

### 模块 8：优化建议

对应六维之一。

回答问题：
> 下一步怎么改？

建议固定使用：
- P1 立即优化
- P2 下一轮内容补充
- P3 后续测试项

---

### 模块 9：代表性评论证据

建议保留 4–8 条代表性评论。

要求：
- 能支撑主要结论
- 尽量覆盖购买、犹豫、价格、链接、质量、细节等关键面向

作用：增强结论可信度，避免页面只有图表没有证据。

---

### 模块 10：统计口径 / 方法说明

必须保留。

说明内容建议包括：
- 评论读取规则
- 有效评论定义
- 回复是否纳入统计
- 哪些图是计数 / 哪些是分析判断

作用：降低“数字从哪来”的质疑，提高报告可信度。

---

## 三、数据表达优先级规则

以后默认按以下优先级表达数据：

### 1. 先用硬指标的维度
以下四类维度，默认必须优先使用：
- **条数**
- **占比**
- **提及率**

适用维度：
- 评论主题分布
- 用户关注点分析
- 购买意向分析
- 影响转化因素

也就是说，这四类维度默认应优先写成：
- X 条
- X / N
- X%
- 提及率 X%

而不是先写“高 / 中 / 强”。

### 2. 可以优先用判断型表达的维度
以下两类维度允许优先使用判断型表达：
- 成交驱动因素
- 优化建议

推荐表达：
- 核心驱动 / 强驱动 / 辅助驱动
- P1 / P2 / P3
- 第一优先优化项 / 第二优先优化项

### 3. 禁止的混用方式
以后默认避免：
- 在评论主题分布里只写“高 / 中 / 低”而不给条数或占比
- 在用户关注点分析里只写“关注度强”而不给提及率或样本数量
- 在购买意向分析里只写“意向较强”而不给分层条数 / 占比
- 在影响转化因素里只写“阻力很大”而不给阻力提及情况

## 四、网页报告设计风格与规范

### 1. 总体风格方向

以后默认网页报告采用：

> **Warm Editorial 商业提案风**

关键词：
- 商业提案感
- 编辑感 / 杂志感
- 高级但克制
- 温暖中性色
- 不是科技 dashboard 感

目标不是做成后台数据面板，而是做成：

> **能给人看、也愿意继续往下看的商业分析页**

---

### 2. 基础配色方向

#### 背景色
优先：
- 暖白
- 米白
- 浅奶咖
- 浅沙色

避免：
- 纯黑背景默认化
- 大面积深蓝科技风
- 强烈霓虹 / 仪表盘风格

#### 主文字色
优先：
- 深棕灰
- 暖黑
- 柔和深灰

避免死黑、硬对比过强的阅读体验。

#### 强调色
建议控制为：
- **1 个主强调色**
- **1 个辅助强调色**

推荐方向：
- 棕金 / 焦糖 / 赭石
- 橄榄绿 / 灰粉 / 暖金

避免：
- 紫蓝渐变默认化
- 彩虹配色
- 高饱和色同时大量出现

---

### 3. 图表配色规则

图表颜色必须有业务逻辑：

- **正向 / 驱动 / 利好** → 绿色系或暖金系
- **阻力 / 风险 / 劝退** → 柔和红棕系
- **中性信息** → 米棕 / 沙色 / 灰褐

单个图表内颜色建议：
- 不超过 5 个主色

避免图表颜色过杂，导致页面像信息噪音墙。

---

### 4. 版式规范

#### 页面定位
页面不是 dashboard，而是 proposal。

所以应该：
- 一屏一重点
- 有阅读节奏
- 有留白
- 结论先行

#### 信息层级建议
固定层级：
- H1：项目标题 / 总判断
- H2：模块标题
- H3：子结论 / 图标题
- 正文：解释
- 注释：统计口径 / 方法说明

#### 图表与文字关系
图表应该：
> 为结论服务

不是：
> 结论只是给图表配文

---

### 5. 风格禁忌清单

以后默认避免：
- 黑底科技风作为默认样式
- 高饱和紫蓝渐变
- AI 仪表盘式密集卡片
- 太多没有业务意义的图
- 伪精确数字堆叠
- 纯文字大段无节奏
- 图很多但证据评论很少
- 页面像后台而不是商业提案

---

## 五、默认执行顺序（落地版）

以后涉及视频评论分析时，默认按这个顺序执行：

1. 打开视频并锁定评论区
2. 根据评论总量决定全读或至少读 100 条有效评论
3. 按需展开高价值回复串
4. 严格按六维框架分析
5. 对主题 / 关注点 / 意向 / 阻力四类维度，先做条数 / 占比 / 提及率表达
6. 对成交驱动与优化建议，再做分析判断表达
7. 按 10 模块结构输出报告
8. 默认采用 Warm Editorial 商业提案风网页
9. 报告完成后自动打开最终分析报告页面
10. 在报告末尾写清统计口径 / 方法说明

## 六、执行检查清单

### 评论读取
- [ ] 是否判断了评论总量是否小于等于 100

- [ ] 是否明确定义有效评论

### 数据表达
- [ ] 评论主题分布是否优先使用条数 / 占比
- [ ] 用户关注点分析是否优先使用提及率 / 条数 / 占比
- [ ] 购买意向分析是否给出分层条数 / 占比
- [ ] 影响转化因素是否给出阻力提及情况
- [ ] 是否只在成交驱动和优化建议中优先使用判断型表达

- [ ] 是否包含 10 个默认模块
- [ ] 是否缺少“代表性评论证据”
- [ ] 是否缺少“统计口径 / 方法说明”
- [ ] 是否仍严格围绕六维结构

### 设计风格

- [ ] 是否避免黑底科技 dashboard 感

- [ ] 是否保证页面有节奏和留白

## 七、固定可复用话术

### 评论读取规则说明
> 评论总量小于等于 100 条时，默认全量读取；评论总量大于 100 条时，默认至少读取 100 条有效评论。

### 有效评论说明
> 有效评论指能够支撑至少一个分析维度判断的评论，例如价格、链接、购买意向、质量疑虑、细节问题与使用反馈等。

### 网页风格说明
> 页面采用 Warm Editorial 商业提案风，以暖白、米色、沙色和棕灰为基础配色，强调可读性、判断效率与展示质感，而非仪表盘式堆叠。

### 未达正式版门槛说明
> 由于平台网页端动态加载 / 可见区域限制，本轮仅稳定获得 X 条有效主评论，未达到正式版所要求的样本下限，因此本版为阶段性样本分析。

### 数据表达说明
> 评论主题分布、用户关注点分析、购买意向分析、影响转化因素，默认先使用条数、占比、提及率；成交驱动因素与优化建议再使用分析判断表达。

### 报告完成后动作说明
> 完成网页报告后，默认自动打开最终分析报告页面，方便用户直接查看结果。
