# Soyoung Clinic Tools — 使用说明

新氧诊所工具集包含 1 个配置工具和 2 个功能技能：

| 类型 | 名称 | 功能 |
|---|---|---|
| ⚙️ 配置工具 | `setup/apikey` | 绑定主人、配置 API Key、保存主人位置 |
| 🛠️ 功能技能 | `appointment` | 门店查询、预约切片、预约审批与执行 |
| 🛠️ 功能技能 | `project` | 医美项目与商品查询 |

## 核心安全规则

1. API Key 只能由主人在私聊中发送和配置，绝不能发到多人群聊中。
2. Key、主人绑定、位置、审批单按 workspace 隔离，目录为 `~/.openclaw/state/soyoung-clinic-tools/workspaces/<workspace_key>/`。
3. 高风险预约动作只有 4 个：
   - 查询我的预约
   - 提交预约
   - 修改预约
   - 取消预约
4. 非主人在群聊中发起高风险动作时，必须先 `@主人`，再由主人回复 `确认 #审批单号` 或 `拒绝 #审批单号`。

## 快速开始

### 第一步：主人私聊配置 API Key

```text
配置新氧 API Key 为 your-api-key
```

1. 打开浏览器访问 `https://www.soyoung.com/loginOpenClaw`
2. 登录账号并复制 API Key
3. 只能在与主人私聊中发送上述配置语句
4. 首次配置会自动把当前私聊发送者绑定为该 workspace 的主人

### 第二步：普通查询

```text
附近有新氧门店吗
查询北京的门店
什么是童颜水光
热玛吉价格
```

### 第三步：高风险预约审批

群成员发起：

```text
@主人 帮我取消预约 12345
```

机器人返回审批单后，主人在同一群里回复：

```text
确认 #A1B2C3
```

如果要拒绝：

```text
拒绝 #A1B2C3
```

## API Key 管理

以下操作都只能由主人在私聊中执行：

- 配置：`配置新氧 API Key 为 xxx`
- 替换：`重置新氧 API Key 为 xxx`
- 查看状态：`我的新氧 API Key 配置了吗`
- 查看主人：`当前新氧主人是谁`
- 删除：`删除新氧 API Key`

也支持这些自然说法：

- `设置新氧 API Key`
- `查看新氧 API Key`
- `谁是新氧主人`
- `我保存的位置是什么`

## 位置管理

位置也视为主人私密信息，只允许主人在私聊中查看和更新：

```text
我在北京市朝阳区
我保存的位置是什么
```

群聊或非主人场景下，优先让用户直接提供城市/区域，不要读取主人已保存位置。

## 预约管理

### 低风险查询

- 查门店
- 查预约切片

这些动作可以直接执行，不需要审批。

常见说法示例：

```text
附近有新氧门店吗
查下北京有哪些新氧
新氧在哪
新氧还有号吗
明天下午还有新氧面诊号吗
```

### 高风险动作

- 查询我的预约
- 提交预约
- 修改预约
- 取消预约

规则如下：

- 主人本人：可直接执行
- 非主人私聊：直接拒绝
- 非主人群聊：必须先 `@主人`
- 主人确认：`确认 #审批单号`
- 主人拒绝：`拒绝 #审批单号`

高风险动作的常见说法：

```text
查询我的预约
我的预约
预约记录
帮我预约新氧
改预约到明天下午三点
取消这个预约
@主人 帮我取消预约 12345
```

## 常见问题

**Q: 提示“未找到 API Key”**  
A: 请让主人在私聊中说「配置新氧 API Key 为 xxx」。

**Q: 为什么群聊里不能发 Key？**  
A: 因为群聊里任何成员都可能看到消息，Key 一旦泄露就能直接操作预约账户。

**Q: 为什么我在群里不能直接取消预约？**  
A: 取消预约属于高风险动作，非主人只能先 `@主人` 发起审批。

**Q: README/使用说明写短了，会不会影响 OpenClaw 识别？**  
A: 影响远小于 `skill.yaml` 和 bootstrap hook。真正决定识别精度的是触发词、description、执行规则和 hook 注入内容，这些我已经一起补强了。

## 卸载

```bash
# 1. 由主人在私聊中删除当前 workspace 的 Key
bash ~/.openclaw/skills/soyoung-clinic-tools/setup/apikey/scripts/main.sh --delete-key --confirm --workspace-key <workspace_key> --chat-type direct --chat-id <chat_id> --sender-open-id <owner_open_id>

# 2. 禁用并移除 bootstrap hook
openclaw hooks disable soyoung-clinic-tools
rm -rf ~/.openclaw/hooks/soyoung-clinic-tools/

# 3. 删除 skill 目录
rm -rf ~/.openclaw/skills/soyoung-clinic-tools/
```
