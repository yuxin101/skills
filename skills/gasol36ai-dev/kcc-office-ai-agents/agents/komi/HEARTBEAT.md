# HEARTBEAT.md - Proactive Checks (Komi - CTO Agent)

When you receive a heartbeat poll, follow this checklist:

- [ ] Review any urgent technical alerts or system notifications
- [ ] Check calendar for upcoming technical deadlines or milestones
- [ ] Look for interesting technical findings or improvement opportunities
- [ ] Ensure no critical technical issues have been missed since last check
- [ ] Update memory files if significant technical events occurred
- [ ] Review and update MEMORY.md with distilled technical learnings (every few days)
- [ ] Check on technical projects (git status, build status, etc.)
- [ ] Review agent collaboration effectiveness and communication
- [ ] Check system performance metrics if available
- [ ] Verify security posture and update if needed
- [ ] Commit and push your own technical changes if needed

If nothing needs attention after checking, reply HEARTBEAT_OK.

## CTO-Specific Heartbeat Items

### Technical Health
- [ ] Review system logs for errors or warnings
- [ ] Check resource usage (CPU, memory, disk) if applicable
- [ ] Verify backup systems are functioning
- [ ] Monitor for any degradation in response times

### Architecture Review
- [ ] Assess if current technical decisions still align with goals
- [ ] Identify any emerging technical debt that needs attention
- [ ] Evaluate if new technologies should be considered for evaluation
- [ ] Review inter-agent communication effectiveness

### Code Quality
- [ ] Scan for any obvious code quality issues in recent changes
- [ ] Verify documentation is kept up-to-date with changes
- [ ] Check if coding standards are being followed
- [ ] Review test coverage if automated tests exist

### Agent Coordination
- [ ] Check if all agent roles are functioning as expected
- [ ] Verify communication channels between agents are working
- [ ] Look for opportunities to improve collaboration efficiency
- [ ] Identify any blockers preventing agent productivity

### Proactive Technical Improvement
- [ ] What technical improvement could I implement today that would delight the team?
- [ ] Are there any recurring technical issues that indicate a deeper problem?
- [ ] What have I learned recently that should be shared with the team?
