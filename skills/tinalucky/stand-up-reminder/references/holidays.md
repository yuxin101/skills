# 节假日跳过指南 / Holiday Skip Guide

## 背景

cron 表达式本身不支持节假日感知。要实现「节假日跳过」，需要在 `payload.message` 中
让 agent 在执行时自行判断当天是否为节假日，若是则静默跳过。

---

## 实现方式：在 agentTurn message 中加入节假日检查指令

将 `payload.message` 改为包含判断逻辑的指令，而不是直接的提醒文案：

```
请先判断今天（{{今天日期}}）是否为中国法定节假日或调休休息日。
- 如果是节假日/休息日：直接回复 NO_REPLY，不发送任何提醒。
- 如果是正常工作日：发送以下提醒消息给用户：
  "⏰ 久坐提醒：已经坐了一个小时啦！站起来活动一下吧 🚶 伸个懒腰、走几步、喝杯水，对身体好！"
```

> **注意**：`agentTurn` 的 message 在执行时会由 isolated agent 处理，agent 可以调用
> `session_status` 获取当前日期，再结合自身知识库判断节假日。

---

## 节假日判断逻辑（供 agent 参考）

### 中国法定节假日（固定规律）

| 节假日 | 通常日期 |
|--------|----------|
| 元旦 | 1月1日前后 |
| 春节 | 农历正月初一前后（7天） |
| 清明节 | 4月4日或5日 |
| 劳动节 | 5月1日前后（5天） |
| 端午节 | 农历五月初五 |
| 中秋节 | 农历八月十五 |
| 国庆节 | 10月1日–7日 |

### 调休说明

中国节假日存在「调休」制度：部分周六/周日需要上班（补班），部分工作日放假。
Agent 应结合当年官方调休安排进行判断（知识库截止日期内的年份可直接判断，
超出截止日期的年份建议提示用户手动确认或查询官方日历）。

---

## 多国节假日扩展

若用户在其他国家/地区工作，可在创建任务时询问：

- 🇺🇸 美国：Federal Holidays（感恩节、独立日等）
- 🇯🇵 日本：国民の祝日（元日、成人の日等）
- 🇬🇧 英国：Bank Holidays

并将对应的节假日判断指令写入 `payload.message`。

---

## 完整 payload 示例（含节假日跳过 + 中文活泼风格）

```json
{
  "kind": "agentTurn",
  "message": "请先用 session_status 获取今天的日期，判断今天是否为中国法定节假日或调休休息日。如果是节假日/休息日，直接回复 NO_REPLY。如果是正常工作日，请通过 message 工具发送以下提醒：「⏰ 久坐提醒：已经坐了一个小时啦！站起来活动一下吧 🚶 伸个懒腰、走几步、喝杯水，对身体好！」"
}
```
