# Conventions - 代码规范和约定

本文件记录团队在工作中达成的规范和约定。

## 用户验证

- **【2026-03-16】芒格：** 用户验证应该在服务端完成，不要只在前端
- **【2026-03-16】费曼：** 使用bcrypt进行密码哈希，不要使用MD5

## JWT认证

- **【2026-03-16】芒格：** JWT token过期时间设置为7天（登录功能标准）
- **【2026-03-16】费曼：** JWT secret应该从环境变量读取，不要硬编码

## 错误处理

- **【2026-03-16】德明：** 所有API返回应该有统一的错误格式：`{error: {code, message}}`
- **【2026-03-16】费曼：** 数据库错误不要直接暴露给用户，应该记录到日志
