# API Review Checklist

## 设计审查清单

使用此清单审查 API 设计的各个方面。

---

## 1. URL 结构

### 资源命名
- [ ] 使用复数名词（`/users` 而非 `/user`）
- [ ] 使用小写字母和连字符（`user-profiles` 而非 `userProfiles`）
- [ ] 避免缩写（使用 `products` 而非 `prods`）
- [ ] URL 层级不超过 3 层

### 端点设计
- [ ] 使用正确的 HTTP 方法（GET/POST/PATCH/DELETE）
- [ ] 操作使用资源而非动词（`POST /users/123/activate`）
- [ ] 查询参数用于过滤、排序、分页
- [ ] 路径参数用于资源标识

---

## 2. HTTP 状态码

### 正确性
- [ ] 2xx 用于成功（200/201/204）
- [ ] 4xx 用于客户端错误（400/401/403/404）
- [ ] 5xx 用于服务器错误（500/502/503）
- [ ] 不使用 200 表示业务错误

### 场景对应
- [ ] `201 Created` 用于创建资源成功
- [ ] `204 No Content` 用于删除成功（无返回体）
- [ ] `400 Bad Request` 用于参数验证失败
- [ ] `401 Unauthorized` 用于未认证
- [ ] `403 Forbidden` 用于无权限
- [ ] `404 Not Found` 用于资源不存在
- [ ] `429 Too Many Requests` 用于频率超限

---

## 3. 错误响应格式

### 结构
- [ ] 错误响应格式统一
- [ ] 包含错误码（`code`）
- [ ] 包含用户可读消息（`message`）
- [ ] 包含请求追踪 ID（`requestId`）
- [ ] 包含时间戳（`timestamp`）

### 错误码规范
- [ ] 错误码格式为 `{DOMAIN}_{NUMBER}`
- [ ] 错误域划分清晰（`AUTH`, `VAL`, `RES` 等）
- [ ] 错误码文档完整

### 安全性
- [ ] 不暴露内部错误细节（堆栈、SQL 等）
- [ ] 错误消息不包含敏感信息

---

## 4. 请求/响应格式

### 请求
- [ ] 使用 JSON 格式
- [ ] 设置 `Content-Type: application/json`
- [ ] 参数验证规则明确
- [ ] 必填参数有明确标识

### 响应
- [ ] 使用 JSON 格式
- [ ] 字段命名一致（camelCase）
- [ ] 日期格式为 ISO 8601
- [ ] 不返回不必要的敏感字段

### 分页
- [ ] 分页响应包含元数据
- [ ] 支持分页参数（`page`, `limit` 或 `cursor`）
- [ ] 返回 `total` 或 `hasMore`

---

## 5. 认证与授权

### 认证
- [ ] 使用标准认证方式（JWT/OAuth）
- [ ] Token 通过 Header 传递
- [ ] Token 有过期时间
- [ ] 支持 Token 刷新

### 授权
- [ ] 实现资源级权限控制
- [ ] 用户只能访问自己的资源
- [ ] 敏感操作有额外验证

---

## 6. 安全性

### 输入验证
- [ ] 所有输入参数验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护（转义输出）

### 安全 Header
- [ ] `X-Content-Type-Options: nosniff`
- [ ] `X-Frame-Options: DENY`
- [ ] `Strict-Transport-Security`（HTTPS）
- [ ] `Content-Security-Policy`

### 速率限制
- [ ] 实现速率限制
- [ ] 返回限制 Header（`X-RateLimit-*`）
- [ ] 超限时返回 `429`

---

## 7. 版本管理

### 版本策略
- [ ] 使用 URL 版本（`/v1/`）
- [ ] 版本号语义清晰（只增不减）
- [ ] 有版本迁移策略

### 兼容性
- [ ] 新版本兼容旧版本
- [ ] 破坏性变更升级主版本
- [ ] 有废弃警告机制

---

## 8. 文档完整性

### OpenAPI 文档
- [ ] 提供 OpenAPI 3.0 规范
- [ ] 所有端点有描述
- [ ] 所有参数有说明
- [ ] 所有响应码有文档
- [ ] 示例数据完整

### 使用指南
- [ ] 认证方式说明
- [ ] 错误码参考
- [ ] 速率限制说明
- [ ] 迁移指南（如适用）

---

## 9. 性能考虑

### 效率
- [ ] 避免 N+1 查询问题
- [ ] 支持字段过滤（减少数据传输）
- [ ] 使用缓存（合适的场景）

### 限制
- [ ] 请求体大小有限制
- [ ] 查询结果有限制（`limit`）
- [ ] 有超时机制

---

## 10. 一致性

### 命名一致性
- [ ] 资源命名统一
- [ ] 字段命名统一
- [ ] 错误消息风格统一

### 行为一致性
- [ ] 类似操作有类似行为
- [ ] 错误处理方式统一
- [ ] 排序规则统一
