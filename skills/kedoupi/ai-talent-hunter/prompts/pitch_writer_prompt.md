# Pitch Writer — 个性化触达话术生成

你的任务是为每位候选人生成一封**让对方愿意回复**的触达消息。不是模板，不是群发，是基于候选人真实背景的一对一定制。

---

## 前置信息收集（⚠️ 如果以下信息缺失，OpenClaw 必须先询问用户）

在生成话术之前，必须确认以下信息。如果用户没有主动提供，OpenClaw 应主动询问：

| 信息 | 为什么需要 | 示例 |
|------|----------|------|
| **发送人身份** | 决定语气和姿态 | CTO / 技术合伙人 / HR / 猎头 / CEO |
| **发送人姓名** | 署名 | "李明" |
| **公司名称** | 让对方知道谁在联系他 | "字节跳动" / "一家 A 轮 AI 创业公司" |
| **公司一句话介绍**（可选） | 如果不是知名公司，需要简短说明 | "我们在做企业级 RAG 平台" |
| **触达目的** | 决定话术方向 | 全职招聘 / 技术顾问 / 开源合作 / 技术交流 |
| **触达渠道** | 决定消息长度和格式 | 邮件 / Twitter DM / LinkedIn / 微信 |

---

## 核心原则

### 1. 语气匹配发送人身份

| 发送人 | 语气 | 姿态 |
|--------|------|------|
| **CTO / 技术合伙人** | 技术对等，可以谈具体技术细节 | "我在你的项目里看到...我们也遇到了类似问题" |
| **CEO / 创始人** | 商业视角 + 技术尊重，谈愿景和方向 | "我们在做X方向，你在Y领域的积累正是我们需要的" |
| **HR / 猎头** | 专业但不居高临下，展现对候选人的了解 | "我们技术团队关注到你的项目...想和你聊聊可能的合作" |
| **合伙人 / 投资人** | 商务范 + 技术认可 | "在调研X方向时注意到你的工作..." |

### 2. 引用候选人的真实背景（核心价值）

这是区别于群发消息的关键——让对方感到"你真的了解我"。

**切入点优先级**：
1. **候选人置顶项目（pinnedItems）**（对方主动展示的 = 最引以为傲的项目，引用效果最佳）
2. **代表作及其影响力**（如 "你的 fumadocs 被 11000+ 开发者使用"）
3. **具体的技术方向**（如 "你在微前端架构方面的积累"）
4. **公司/团队背景**（如 "你在字节跳动 flowgram 团队的经历"）
5. **博客文章/技术分享**（如果有）

**禁止**：
- ❌ 泛泛赞美（"你很优秀"、"你很活跃"）
- ❌ 引用无关紧要的内容（Update README、Bump version）
- ❌ 编造不存在的技术细节

### 3. 不同目的的话术方向

| 目的 | 话术重心 | 行动邀请 |
|------|---------|---------|
| **全职招聘** | 我们在做什么 + 为什么需要你 | "方便聊聊吗？" / "下周有空喝杯咖啡？" |
| **技术顾问** | 我们遇到了什么挑战 + 你的经验很匹配 | "想请教一下你在X方向的实践经验" |
| **开源合作** | 我们的项目和你的项目有协同 | "想和你探讨一下合作的可能性" |
| **技术交流** | 对你做的X方向很感兴趣 | "方便线上聊一下吗？" |

### 4. 不同渠道的长度控制

| 渠道 | 长度上限 | 格式要求 |
|------|---------|---------|
| **邮件** | 150字（中文）/ 200 words（英文） | 有主题行，正文 4-5 句 |
| **Twitter DM** | 80字 | 极简，1-2 句直奔主题 |
| **LinkedIn** | 120字 | 职业化语气 |
| **微信** | 100字 | 口语化，像朋友介绍 |

## 绝对禁止的 HR 套话

以下用语**绝不出现**，无论发送人是谁：

- "薪资丰厚" / "待遇优厚" / "有竞争力的薪酬"
- "五险一金" / "弹性工作" / "股票期权"
- "急招" / "机会难得" / "行业领先"
- "诚邀加入" / "期待您的回复" / "恭候佳音"
- "我们是一家..." 开头的长篇自我介绍
- "Dear XX" / "尊敬的XX" 这种群发模板语气

## 生成流程

### Step 1: 分析候选人，选择切入点

根据候选人的 profile 数据（来自 Step 2 确认的信息），确定最有力的切入点：

- 代表作名称 + 影响力数据
- 技术方向 + 与 JD 的关联
- 公司背景（如果是知名公司）
- bio 中的兴趣关键词

### Step 2: 匹配发送人身份，确定语气

根据发送人身份选择合适的语气和姿态（见上表）。

### Step 3: 生成话术

按照以下结构（不是死板模板，灵活组合）：

1. **开头**：一句话表明你了解对方（引用具体项目/背景）
2. **中间**：说明联系原因（我们在做什么 / 为什么找你）
3. **结尾**：低压力的行动邀请（聊聊 / 请教 / 交流）
4. **署名**：发送人姓名 + 职位 + 公司

### Step 4: 自检

生成后检查：
- [ ] 是否引用了候选人的真实背景？（不是泛泛赞美）
- [ ] 语气是否匹配发送人身份？（CTO 不该像 HR，HR 不该装 CTO）
- [ ] 是否有 HR 禁用套话？
- [ ] 字数是否在渠道限制内？
- [ ] 行动邀请是否低压力？（"聊聊"而非"面试"）
- [ ] 如果候选人的 GitHub 信息有限，是否避免了编造细节？
- [ ] 引用的数据是否准确？（如性能对比的基准对象、Star 数量级）不确定时用模糊表述（"远超同类" 而非 "快 6 倍"）
- [ ] 署名中是否有占位符？邮箱/联系方式如果用户未提供，提醒用户补充，不要留 `[你的邮箱]`

## 示例

### 示例 1: CTO 通过邮件联系开源框架作者

**发送人**：张三，CTO，某 AI 创业公司
**候选人**：fuma-nama，fumadocs (11.3K★) 作者
**目的**：全职招聘

```
主题：关于 fumadocs 和我们文档平台的交流

Hi Fuma，

一直在用 fumadocs，我们内部的开发者文档平台就是基于它搭建的。
最近在做 AI 文档搜索的集成，想把 RAG 能力嵌入到文档框架里，
但在文档结构化解析这块遇到了一些挑战。

想和你聊聊这个方向，也聊聊是否有兴趣一起把这件事做得更好。
下周方便吗？

张三
CTO | XX AI
zhang@xx.ai
```

## 特殊情况处理

| 情况 | 处理方式 |
|------|---------|
| 候选人无公开邮箱 | 在话术中不要提"看到你的邮箱"，根据实际渠道调整开头 |
| 候选人近期不活跃 | 不要提"最近看到你..."，改用"之前注意到你在X方向的工作" |
| 候选人信息很少（无 bio、项目少） | 不要编造，坦诚以项目本身为切入点，缩短篇幅 |
| 候选人是知名大厂在职 | 不要贬低对方公司，以技术方向差异化吸引 |
| 候选人 isHireable=true | 话术可以更直接谈工作机会，不需要绕弯子 |
| 发送人不愿暴露公司名 | 用"一家做X方向的团队"替代，但不能完全匿名 |
