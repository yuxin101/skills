## 质量评分系统

### 评分维度

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| **字数** | 20% | ≥1000 字=100 分，≥500 字=80 分，≥300 字=60 分 |
| **数据点** | 30% | ≥10 个=100 分，≥5 个=80 分，≥3 个=60 分 |
| **结构** | 20% | 标题/列表/表格数量 × 25 |
| **引用** | 15% | ≥5 个=100 分，≥3 个=80 分，≥1 个=60 分 |
| **具体性** | 15% | 百分比 + 日期 + 预测 + 洞察综合评分 |

### 等级划分

| 总分 | 等级 | 说明 |
|------|------|------|
| ≥85 | 优秀 | 数据丰富、结构清晰、引用充分 |
| ≥70 | 良好 | 数据较充分、结构完整 |
| ≥55 | 一般 | 基本满足要求 |
| <55 | 较差 | 数据不足或结构混乱 |

### 评分示例

**某平台回复评分**：
```
总分：78.5 (良好)

详细评分:
  word_count: 80 (权重：20%) - 650 字
  data_points: 100 (权重：30%) - 12 个数据点
  structure: 75 (权重：20%) - 3 个结构标记
  sources: 60 (权重：15%) - 1 个引用
  specificity: 70 (权重：15%) - 3%+2dates+2predictions
```

```python
    {'platform': 'DeepSeek', 'content': '...', 'extracted_data': data, 'quality_score': quality},
    # ... 其他平台

table = extractor.generate_comparison_table(results)
```

```python
patterns = {

```

### 评分计算逻辑

```python
total_score = (
    score_word * 0.20 +      # 字数 20%
    score_data * 0.30 +      # 数据点 30%
    score_structure * 0.20 + # 结构 20%
    score_sources * 0.15 +   # 引用 15%
    specificity_score * 0.15 # 具体性 15%
)
```

## 局限性与改进方向

### 当前局限

1. **语言限制**：主要针对中文优化，英文提取效果一般
2. **上下文理解**：基于正则，无法理解复杂语义
3. **数据关联**：无法自动关联相关数据（如"油价"和"95 美元"）
4. **表格提取**：Markdown 表格内数据提取不完整

### 未来改进（v2.2+）

- [ ] 引入 NLP 模型（如 spaCy、NLTK）提升语义理解
- [ ] 支持数据关联和实体识别
- [ ] 增加图表生成（matplotlib）
- [ ] 支持多语言提取
- [ ] 数据趋势分析（时间序列）

## 常见问题

### Q1: 为什么某平台的数据点很少？

**A**: 可能原因：
- 回复内容较短
- 缺少具体数字和日期
- 语言表述过于笼统

**解决**：在问题中明确要求"提供具体数据和日期"

### Q2: 质量评分是否准确？

**A**: 评分系统基于启发式规则，能反映大致质量，但不是绝对标准。建议：
- 作为参考，不完全依赖
- 结合人工判断
- 关注各维度得分，了解优劣势

### Q3: 可以自定义评分权重吗？

**A**: 可以，修改 `extractor.py` 中的 `scoring_weights`：

```python
self.scoring_weights = {
    'word_count': 0.2,    # 调整这里
    'data_points': 0.3,   # 调整这里
    'structure': 0.2,
    'sources': 0.15,
    'specificity': 0.15,
}
```
