# 统一错误码

## 网关返回码

网关层面的错误（请求未到达业务处理层）：

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 00000000 | 交易成功 | 正常处理 |
| 00000100 | 参数校验失败 | 检查请求参数格式和必填项 |
| 00000103 | 验签失败 | 检查 RSA 公私钥配置是否正确 |
| 00000104 | 产品号不存在 | 检查 product_id 是否正确 |
| 00000105 | 商户不存在 | 检查 huifu_id 和 sys_id 是否正确 |
| 00000200 | 系统异常 | 稍后重试，若持续出现联系汇付技术支持 |
| 00000300 | 请求超时 | 稍后重试 |

## 线上交易返回码（支付托管相关）

| 返回码 | 返回描述 | 处理建议 |
|-------|---------|---------|
| 00000000 | 处理成功 | 正常处理 |
| 99010002 | 预下单请求流水重复 | 使用新的 req_seq_id 重新发起 |
| 99010003 | 交易金额不合法 | 检查金额格式（保留两位小数，最小 0.01） |
| 99010004 | 商户状态异常 | 联系汇付确认商户状态 |
| 99010005 | 产品未开通 | 联系汇付开通对应产品 |
| 99020001 | 原交易不存在 | 检查 org_req_date 和 org_req_seq_id 是否正确 |
| 99020002 | 原交易状态不允许退款 | 确认原交易状态为成功后再发起退款 |
| 99020003 | 退款金额超过可退金额 | 检查退款金额不超过原交易金额 |
| 99020004 | 退款请求重复 | 使用新的 req_seq_id 发起退款 |
| 99030001 | 交易已关闭 | 交易已被关单，无需重复操作 |

## 通用错误处理建议

1. **resp_code = 00000000**：交易成功，继续后续业务逻辑
2. **resp_code 以 990 开头**：业务错误，根据错误描述排查
3. **resp_code 以 000 开头（非 00000000）**：网关/系统级错误，检查配置或稍后重试
4. **未收到响应或超时**：调用**查询接口**确认交易最终状态，切勿直接判断为失败
5. **trans_stat = P**：交易处理中，等待异步通知或轮询查询接口
