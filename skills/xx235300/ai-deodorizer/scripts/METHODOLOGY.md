## 一、核心设计原则

### 1.1 两轮改写策略（核心框架）

**第一轮：模式修复**
- 识别并修复 25 种 AI 写作模式
- 将"明显是AI写的"改为"看起来正常"

**第二轮：Anti-AI Audit（关键创新）**
- 问自己："这段文字哪里看起来明显是AI写的？"
- 回答并列出残留的AI特征
- 根据答案再次改写，消除这些特征
- **这一轮是去味的精髓**，解决"修干净了但还是像AI"的问题

### 1.2 Soul 注入原则（去味的另一半）

> "避免AI模式只是基础，sterile，voiceless的写作同样可疑。好的写作背后有一个真实的人。"

**Soul 检测维度**：
- 每句话是否长度结构完全相同？
- 有没有真实观点，还是只有中性陈述？
- 有没有承认不确定性或复杂情感？
- 有没有在合适的地方使用第一人称？
- 有没有幽默、棱角、个性？
- 是否读起来像维基百科或新闻稿？

**Soul 注入方法**：
1. **有观点** — 不要只报告事实，要对事实做出反应
2. **变化节奏** — 短句+长句混合
3. **承认复杂性** — 真实的人有复杂感受
4. **使用"我"** — 第一人称是诚实的表现
5. **允许混乱** — 完美的结构反而显得机械
6. **感受要具体** — 用具体细节替代抽象概括

### 1.3 统计指标（辅助维度，可选）

| 指标 | 说明 | 异常阈值 |
|------|------|---------|
| Burstiness | 句子长度变化程度 | AI通常变化小 |
| Type-Token Ratio (TTR) | 词汇重复率 | AI通常词汇重复高 |
| 可读性指数 | 文本易读程度 | 过高=AI写作 |

## 二、AI写作特征清单（25种模式）

### 类别一：内容模式（Content Patterns）

#### P1: 意义膨胀（Significance Inflation）
**识别特征**：
- 中文："具有重要意义"、"关键作用"、"标志性时刻"、"不可或缺的核心"
- 过度使用"里程碑"、"转折点"、"分水岭"

**修复策略**：删除夸大陈述，用具体事实替代

#### P2: 知名度堆砌（Notability Name-Dropping）
**识别特征**：
- "据《纽约时报》《BBC》《金融时报》报道"
- "多位业内专家表示"、"权威人士指出"

**修复策略**：给出具体引用上下文，不要堆砌媒体名称

#### P3: -ing 结尾的肤浅分析（Superficial -ing Analyses）
**识别特征**：
- 大量使用 "...象征着..." "...体现了..." "...反映出..."

**修复策略**：删除或补充真实来源

#### P4: 宣传性语言（Promotional Language）
**识别特征**：
- "风景如画"、"令人叹为观止"、"美不胜收"
- "不容错过的"、"绝佳的选择"、"天花板级别"

**修复策略**：降级为中性描述

#### P5: 模糊归因（Vague Attributions）
**识别特征**：
- "研究表明"、"数据显示"、"专家普遍认为"
- "有研究指出"、"业内人士分析"

**修复策略**：写出具体研究名称、年份、机构，或删除

#### P6: 公式化挑战描述（Formulaic Challenges）
**识别特征**：
- "尽管面临诸多挑战，该行业仍然蓬勃发展"
- "挑战与机遇并存"、"危中有机"

**修复策略**：写出具体挑战是什么，放弃套话

### 类别二：语言模式（Language Patterns）

#### P7: AI 高频词汇（AI Vocabulary）
| AI常用词 | 人类替换词 |
|---------|-----------|
| 此外 | 还有、另外 |
| 由此可见 | 总之 |
| 值得注意的是 | （直接说内容） |
| 从某种意义上说 | 说白了 |
| 本质上 | 说起来 |
| 综上所述 | （删掉结论废话） |

#### P8: 系动词回避（Copula Avoidance）
**识别特征**：
- "起到了...作用"、"具备...功能"、"拥有...特性"

**修复策略**：还原为"是"和"有"

#### P9: 否定式排比（Negative Parallelisms）
**识别特征**：
- "这不是...而是..."（滥用）
- "既不是...也不是..."

**修复策略**：直接陈述观点，删除"不是...而是"

#### P10: 三段式法则滥用（Rule of Three Abuse）
**识别特征**：
- "创新、创造、突破"
- "智能化、数字化、信息化"

**修复策略**：用自然数量列出，不刻意凑三

#### P11: 同义词刻意循环（Synonym Cycling）
**识别特征**：同一段话对同一事物用不同称呼来回换

**修复策略**：统一称呼，不要刻意换词

#### P12: 虚假范围（False Ranges）
**识别特征**：
- "从宇宙大爆炸到黑洞"
- "从古至今"、"从源头到终端"

**修复策略**：直接列出实际涉及的主题

---

### 类别三：风格模式（Style Patterns）

#### P13: 破折号滥用（Em Dash Overuse）
**识别特征**：一段话出现三个以上破折号

**修复策略**：改为逗号、分号或拆成两句

#### P14: 粗体/斜体滥用（Markup Overuse）
**识别特征**：
- 连续使用加粗强调大量术语
- "**XXX**、**YYY**、**ZZZ**"

**修复策略**：只在真正需要强调时用加粗

#### P15: 内联标题列表（Inline Header Lists）
**识别特征**：
- "**性能**：性能提升了30%"
- 把正文写成标题+内容的垂直列表

**修复策略**：改为段落叙述

#### P16: 标题大写（Title Case Headings）
**识别特征**：标题中每个实词首字母大写

**修复策略**：改为 Sentence case

#### P17: 表情符号滥用（Emoji Abuse）
**识别特征**：
- "🚀 性能突破：💡 关键洞察"
- 在正式书面文本中使用emoji作为标题装饰

**修复策略**：删除所有emoji

#### P18: 弯引号（Curly/typographic Quotes）
**识别特征**：中文文本中使用英文弯引号

**修复策略**：统一使用直角引号

---

### 类别四：交流模式（Communication Patterns）

#### P19: Chatbot 交流痕迹（Chatbot Artifacts）
**识别特征**：
- "希望这对你有帮助！"
- "如果你有任何问题，请随时告诉我"

**修复策略**：完全删除

#### P20: 截止日期免责声明（Cutoff Disclaimers）
**识别特征**：
- "我的知识截止到XXX"
- "由于信息有限，以下分析可能不够全面"

**修复策略**：删掉或改为"我查到的最新信息是..."

#### P21: 谄媚语气（Sycophantic Tone）
**识别特征**：
- "您说得太对了！"
- "完全赞同您的看法"

**修复策略**：直接回应，不要恭维

---

### 类别五：填充词与过度限定（Hedging & Filler）

#### P22: 填充短语（Filler Phrases）
| 填充短语 | 简化 |
|---------|-----|
| 从根本上来说 | 说白了/其实 |
| 从某种意义上说 | 说起来 |
| 综上所述 | （删掉） |
| 值得注意的是 | （直接说） |

#### P23: 过度限定（Over-hedging）
**识别特征**：
- "可能"、"也许"、"或许"出现过于频繁
- "在一定程度上..."连续出现

**修复策略**：适度确定性，在能确定的地方直接说确定

#### P24: 通用积极结论（Generic Positive Conclusions）
**识别特征**：
- "综上所述，该研究具有重要的理论和实践意义"
- "相信在不久的将来，该技术将得到广泛应用"

**修复策略**：写具体结论或直接不写结尾

#### P25: 僵尸模板（Zombie Templates）
**识别特征**：
- 任何段落都以固定模板开头
- "随着...的发展"、"近年来..."

**修复策略**：多样化开头句式，自然引入

## 三、中文 AI 词汇对照表

### 3.1 高频 AI 词汇替换表

| 序号 | AI词汇（禁用） | 人类替代表达 |
|------|-------------|-----------|
| 1 | 此外 | 还有、另外、并且 |
| 2 | 由此可见 | 总之、所以、看来 |
| 3 | 值得注意的是 | （直接说内容） |
| 4 | 从某种意义上说 | 说白了、说实在的 |
| 5 | 本质上 | 说起来、说到底 |
| 6 | 综上所述 | （直接删掉） |
| 7 | 不言而喻 | 很明显、这很明显 |
| 8 | 令人瞩目的是 | （直接说事实） |
| 9 | 可以说 | （直接说内容） |
| 10 | 与此同时 | 同时、这期间 |

### 3.2 中文三段式词汇禁用表

| 禁用表达 | 问题 | 替换建议 |
|---------|------|---------|
| "创新、突破、发展" | 强行三段 | 只用最相关的一个 |
| "智能化、数字化、网络化" | 强行三段 | 写具体技术名称 |
| "想清楚、看明白、做出来" | 鸡汤三段 | 写具体行动步骤 |

## 四、两轮改写Prompt设计（核心）

### 第一轮：模式修复Prompt

```
你是一个文字编辑，负责将AI生成的文本改写得更自然、更像人类书写。

你的任务：
1. 识别并修复文本中的AI写作模式（共25种，包括：意义膨胀、知名度堆砌、-ing肤浅分析、宣传性语言、模糊归因、公式化挑战、AI高频词汇、系动词回避、否定式排比、三段式滥用、同义词循环、虚假范围、破折号滥用、粗体滥用、内联标题列表、标题大写、emoji滥用、弯引号、Chatbot痕迹、截止日期免责声明、谄媚语气、填充短语、过度限定、通用积极结论、僵尸模板）
2. 用自然表达替换AI模式
3. 保留原文的核心信息和基本结构
4. 保持原有语气（正式/口语/技术性）

禁止：
- 不要添加任何你自己的观点
- 不要改变原文的事实内容
- 不要简化内容让文章变短
- 不要在结尾添加总结性废话

请直接输出去味后的文本，不需要解释你做了什么修改。
```

### 第二轮：Anti-AI Audit Prompt

```
请完成以下两步：

第一步：快速浏览以下文本，列出它最明显的一到两个"AI味"特征（是什么让它看起来像AI写的？）。回答要简短。

第二步：根据第一步的答案，将文本改写，让它明显不再是AI写的。

改写原则：
- 加入真实的人类声音（有观点、有态度）
- 变化句子长度（短句+长句混合）
- 在合适的地方使用"我"
- 承认复杂性和不确定性
- 允许一些"混乱"——完美的结构反而可疑
- 不要刻意追求"正确"，要有具体细节

直接输出去味后的文本。
```

## 五、中文场景特殊处理

### 5.1 中文特有AI模式

| 模式 | 修复策略 |
|------|---------|
| 过度使用"赋能" | 写具体怎么帮 |
| "抓手"滥用 | 写具体行动 |
| "拉齐" | 直接写"统一认识" |
| "落地" | 写具体做什么 |
| "打法" | 写具体策略 |
| "赛道" | 写具体行业 |

## 六、效果评估

| 维度 | 评估方法 | 达标标准 |
|------|---------|---------|
| 模式清除率 | 25种模式残留数量 | ≤3种残留 |
| 自然度 | 人类评审（自己读） | 无明显AI味 |
| 信息保留 | 与原文对比 | 核心信息不丢失 |
| 长度变化 | 字数增减 | ±20%以内 |
| Soul注入 | 有无观点/节奏/第一人称 | 至少有1项 |
