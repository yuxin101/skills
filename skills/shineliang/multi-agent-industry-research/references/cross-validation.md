# CLI 交叉验证协议

## 核心目的

不同 AI 模型有不同的知识覆盖、推理偏好和盲区。通过 CLI 调用其他模型对你的分析结果做独立验证，能发现盲区和偏见，显著提升调研质量。这不是"补充视角"，而是**质量控制的关键步骤**。

## 执行条件

当 `RESEARCH_CROSS_MODEL` 为 `cli` 或 `both` 时，研究 agent **必须**执行此协议。

## 详细步骤

### 步骤 1：准备验证材料

完成 web_search 和初步分析后，用 exec 将核心发现写入临时文件：

```bash
cat > /tmp/research-cross-validate.txt << 'FINDINGS'
## 我的核心发现（请审核）

1. [发现1 — 包含具体数据和来源]
2. [发现2]
3. [发现3]
...

## 关键数据点
- [数据A]: [数值] (来源: [xxx])
- [数据B]: [数值] (来源: [xxx])

## 我的主要结论
[你的分析结论，500-1000字]
FINDINGS
```

### 步骤 2：调用 CLI 做交叉验证

用 exec 调用推荐的 CLI 工具（timeout 设为 config.cliTimeout 秒）：

**Codex 调用：**
```bash
cat /tmp/research-cross-validate.txt | codex exec "你是一个行业分析师。请审核以下调研发现：1）指出可能有误或过时的数据，说明理由；2）补充遗漏的重要观点或数据；3）对有争议的结论给出你的独立判断；4）总结哪些发现是可靠的，哪些需要进一步验证。以中文输出。" 2>&1
```

**Gemini 调用：**
```bash
cat /tmp/research-cross-validate.txt | gemini -p "你是一个行业分析师。请审核以下调研发现：1）指出可能有误或过时的数据，说明理由；2）补充遗漏的重要观点或数据；3）对有争议的结论给出你的独立判断；4）总结哪些发现是可靠的，哪些需要进一步验证。以中文输出。" 2>&1
```

**Claude 调用：**
```bash
cat /tmp/research-cross-validate.txt | claude -p "你是一个行业分析师。请审核以下调研发现：1）指出可能有误或过时的数据，说明理由；2）补充遗漏的重要观点或数据；3）对有争议的结论给出你的独立判断；4）总结哪些发现是可靠的，哪些需要进一步验证。以中文输出。" 2>&1
```

### 步骤 3：降级处理

如果推荐 CLI 调用失败（超时、命令不存在、返回错误）：

```
降级链：{推荐CLI} → {次选CLI} → {第三选CLI}

示例：
- 角色推荐 Gemini → 失败 → 尝试 Codex → 失败 → 尝试 Claude
- 角色推荐 Codex → 失败 → 尝试 Gemini → 失败 → 尝试 Claude
```

全部失败才跳过，并在报告中标注：`[交叉验证：CLI 不可用，已跳过]`

### 步骤 4：整合到报告

将 CLI 返回结果整合到报告的"交叉验证"章节，格式如下：

```markdown
## 交叉验证
> 使用 {CLI工具名} 对核心发现进行独立审核

### 验证结果
{CLI 返回的分析内容}

### 交叉验证小结
| 核心发现 | 本 agent 结论 | CLI 验证结论 | 是否一致 |
|----------|--------------|-------------|---------|
| 发现1 | 你的结论 | CLI 结论 | 一致/矛盾/补充 |
| 发现2 | 你的结论 | CLI 结论 | 一致/矛盾/补充 |
```

### 步骤 5：清理

```bash
rm -f /tmp/research-cross-validate.txt
```

## 各 CLI 工具特点（供角色分配参考）

| CLI | 擅长 | 适合分配给 |
|-----|------|-----------|
| Codex (OpenAI) | 代码分析、结构化推理、技术细节 | 技术架构、平台能力分析师 |
| Gemini (Google) | 搜索增强、多源信息综合、时效性 | 市场规模、竞品格局分析师 |
| Claude (Anthropic) | 深度推理、逻辑严密性、长文分析 | 战略分析、商业化、实施路径分析师 |
