# 神策分析手册转飞书文档生成器（升级版）

将神策（Sensors Analytics）官方分析手册转换为飞书使用说明文档。

## 功能说明

根据用户提供的神策分析手册链接，自动抓取官方文档内容，并按照标准化的文档框架生成飞书文档。

### 支持的功能模块

#### 元数据管理（一次性6个手册）
| 模块 | 说明 |
|------|------|
| 元事件 | 事件配置管理 |
| 事件属性 | 事件级属性配置 |
| 虚拟属性 | 虚拟属性配置 |
| 虚拟事件 | 虚拟事件配置 |
| 维度表 | 维度表配置 |
| 物品属性 | 物品属性配置 |

#### 人群与标签
| 模块 | 说明 |
|------|------|
| 用户分群 | 用户群体划分 |
| 用户标签 | 用户标签管理 |

---

## 交互流程

### Step 1: 用户输入手册链接

用户需要提供神策分析手册的链接。

**元数据管理（一次性提供6个链接）：**
- 元事件：`https://manual.sensorsdata.cn/sa/docs/guide_metadata_meta`
- 事件属性：`https://manual.sensorsdata.cn/sa/docs/guide_metadata_event_attr`
- 虚拟属性：`https://manual.sensorsdata.cn/sa/docs/guide_metadata_virtual_attr`
- 虚拟事件：`https://manual.sensorsdata.cn/sa/docs/guide_metadata_virtual_event`
- 维度表：`https://manual.sensorsdata.cn/sa/docs/guide_metadata_dimension`
- 物品属性：`https://manual.sensorsdata.cn/sa/docs/guide_metadata_item_attr`

**人群与标签：**
- 用户分群：`https://manual.sensorsdata.cn/sa/docs/User_Group_Create`
- 用户标签：`https://manual.sensorsdata.cn/sa/docs/guide_tags`

### Step 2: 判断文档类型

根据用户提供的链接，自动判断文档类型：
- 如果是元数据管理（6个功能），使用**元数据管理规范**
- 如果是人群/标签，使用**原规范**

### Step 3: 显示标准框架规范

在生成文档之前，必须先向用户展示标准框架规范，确认是否需要调整。

（二）其他功能模块：全部表格化，使用`<lark-table>`组件，列宽120/350/180

##### 三、常见问题解答（FAQ）

##### 四、文档维护

**关键注意事项：**

1. **标题与序号规范（必须遵守）**
   - **文档标题**：使用飞书文档的标题功能，格式为【XX】xxx使用说明
   - **正文H1标题**：使用中文数字：一、二、三（如：一、快速入门）
   - **正文H2标题**：使用数字有序列表：1.1、1.2、1.3（与H1标题联动）
   - **正文H3标题**：使用数字有序列表：1.1.1、1.1.2、1.1.3

2. **高亮框标准**：信息说明light-orange，截图标记light-yellow
3. **截图标记**：使用📸 emoji标识
4. **截图提示**：高亮块内开头固定语句"截图提示：截图可按需插在表格最后一列"，然后另起一行有序分点罗列截图建议

### Step 4: 确认框架规范

使用 `question()` 向用户确认：

```
question({
  questions: [{
    header: "文档框架确认",
    question: "以下是生成文档的标准框架规范，请确认是否需要调整：\n\n【元数据管理规范】（整合6个手册）\n一、快速入门：定义说明（高亮框）→ 实际应用场景\n二、核心功能详解：基础全局功能 + 其他功能模块（全部表格化，lark-table，列宽120/350/180）\n三、常见问题解答\n四、文档维护\n\n【人群/标签规范】（原规范）\n一、快速入门：定义说明 → 实际应用场景 → 2分钟完成第一个分析\n二、核心功能：基础全局功能 + 其他功能模块\n三、实践应用模板（2-3个）\n四、常见问题解答\n五、文档维护\n\n请确认：",
    options: [
      { label: "直接按此框架生成 (Recommended)", description: "不调整框架，直接生成文档" },
      { label: "调整框架规范", description: "修改部分框架内容后再生成" }
    ]
  }]
})
```

如果用户选择"调整框架规范"，则收集用户的具体调整需求，然后重新确认。

### Step 5: 抓取文档内容

#### 元数据管理（6个链接）
同时抓取6个手册内容，然后整合：

```bash
# 批量抓取
web_fetch url="<元事件链接>" maxChars=50000
web_fetch url="<事件属性链接>" maxChars=50000
web_fetch url="<虚拟属性链接>" maxChars=50000
web_fetch url="<虚拟事件链接>" maxChars=50000
web_fetch url="<维度表链接>" maxChars=50000
web_fetch url="<物品属性链接>" maxChars=50000
```

#### 人群/标签（单/多链接）
按需抓取1-2个手册内容：

```bash
web_fetch url="<用户提供的链接>" maxChars=50000
```

### Step 6: 生成飞书文档

使用 `feishu_create_doc` 工具创建文档：

**元数据管理文档标题：**
- 【飞书】元数据管理使用说明

**人群/标签文档标题：**
- 【飞书】用户分群使用说明
- 【飞书】用户标签使用说明

## 常见问题

### Q: 元数据管理需要一次性提供6个链接吗？
A: 是的，元数据管理模块需要一次性提供6个手册链接，系统会整合后生成一份统一的文档。

### Q: 用户提供的链接不是神策官网怎么办？
A: 提示用户需要提供神策官方手册链接，格式为 `https://manual.sensorsdata.cn/sa/docs/...`

### Q: 抓取内容失败怎么办？
A: 可能页面使用了JavaScript渲染，尝试使用其他方式获取内容，或告知用户手动复制文档内容

### Q: 如何区分使用哪种规范？
A: 根据用户提供的链接判断：
- 链接包含 metadata（6个）：使用元数据管理规范（去掉2分钟/去掉实践应用模板）
- 链接包含 User_Group 或 tag：使用原规范
