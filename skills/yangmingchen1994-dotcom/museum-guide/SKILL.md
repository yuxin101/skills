# 博物馆参观路线规划师

## ⚠️ 使用前提

本技能需要以下配置方可正常运行：

### 1. 大模型 API 配置

需要在环境变量中设置（优先级最高）：

| 环境变量 | 说明 | 示例 |
|---------|------|------|
| `API_KEY` | 模型 API Key | `sk-xxx` |
| `API_BASE` | 模型 Base URL | `https://api.example.com/v1` |
| `MODEL_NAME` | 模型名称 | `xxx` |

> 可选：若你在 `scripts/config.json` 放置了与脚本兼容的配置，脚本会在环境变量未配置时回退读取该文件（若文件不存在则不生效）。

或在 `scripts/config.json` 中配置：

```json
{
  "api_key": "sk-xxx",
  "api_base": "https://api.example.com/v1",
  "model_name": "xxx"
}
```

### 2. ProSearch 代理

需要本地运行 ProSearch 代理服务：

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| `AUTH_GATEWAY_PORT` | ProSearch 代理端口 | `19000` |

### 3. 依赖二进制

- `curl`：用于发起 HTTP 请求

### 4. 安全注意事项

- **API Key 安全**：只配置你信任的模型 API 端点，不要复用其他系统的高权限凭证
- **本地代理信任**：脚本通过 curl 调用本地代理，确保该代理是你信任的服务
- **数据访问**：`museum_assistant.py` 负责调度整体流程；`extract_profile.py`、`plan_route.py` 和 `search_artifacts.py` 都会优先读取环境变量 `API_KEY/API_BASE/MODEL_NAME`，若未配置则回退读取 `scripts/config.json`

## 工作流程总览

```
入口脚本：`python3 scripts/museum_assistant.py` 串联执行
阶段一：画像提取 + 智能推断 + 确认（内部调用 `extract_profile.py`）
阶段二：文物检索（内部调用 `search_artifacts.py`）
阶段三：路线规划（内部调用 `plan_route.py`）
```

## 完整工作流程

### 阶段一：用户画像提取 + 确认

#### 1. 提取用户输入

从用户输入中解析以下维度：
- 博物馆名称（**必填**，不可跳过）
- 参观时长
- 是否携带儿童
- 关注领域（如：绘画、书法、玉器、瓷器等）
- 文物种类偏好
- 朝代偏好（如：明清、唐宋等）

运行脚本提取用户画像：

```bash
python3 scripts/extract_profile.py "<用户输入>"
```

#### 2. 缺失维度推断填充

若用户未提供某维度，按以下默认规则：

| 字段 | 默认值 |
|------|--------|
| 参观时长 | 2-3小时 |
| 是否带儿童 | 否 |
| 关注领域 | 绘画, 书法, 瓷器, 玉器 |
| 朝代偏好 | 隋唐, 辽宋夏金元, 明清 |

#### 3. 确认话术

当存在推断填充的字段时，输出以下确认话术：

```
📋 我已根据您提供的信息，为您规划参观路线：
• 博物馆：{博物馆名}
• 参观时长：{时长}
• 携带儿童：{是/否}
• 关注领域：{领域列表}
• 文物种类：{种类列表}
• 关注朝代：{朝代列表}

（以上未提供的信息，我已根据您的画像进行了推断）

请确认是否合适，或告诉我需要调整的地方。
```

#### 4. 用户响应处理

| 用户响应 | 处理方式 |
|----------|----------|
| **确认**（"好的"、"可以"、"ok"） | 直接进入阶段二 |
| **调整**（"我喜欢瓷器"、"改成4-5小时"） | 合并原始输入 + 调整内容，更新对应字段，直接进入阶段二（不再二次确认） |
| **提供完整信息**（所有维度都已明确） | 直接进入阶段二 |

---

### 阶段二：文物检索

#### 检索流程
调用脚本检索文物：

```bash
python3 scripts/search_artifacts.py "<博物馆名称>"
```

脚本会使用 ProSearch 搜索博物馆相关关键词（镇馆之宝、精品文物、文物列表等），自动解析和结构化文物信息。

#### 收集字段
每件文物收集：名称、展馆、时期、文物种类、是否镇馆之宝、简介

#### 检索目标
根据搜索结果尽可能多地检索文物（通常 20-50 件），供后续筛选使用。

---

### 阶段三：路线规划与输出

#### 1. 确定文物数量

| 参观时长 | 输出数量 |
|----------|----------|
| 参观时长 | 输出数量 |
| 2-3小时 | 30 件 |
| 4-5小时 | 50 件 |

运行脚本生成路线：

```bash
python3 scripts/plan_route.py --profile profile.json --artifacts artifacts.json --output route_result.json
```

#### 2. 筛选原则

- **朝代均衡**：避免超过 40% 集中在同一朝代
- **种类多样**：至少覆盖 3 种以上文物类型
- **携带儿童**：优先 child_friendly 文物，减少碎片化文物数量

#### 3. 输出格式

按参观顺序输出 Markdown 表格：

| 序号 | 文物名称 | 展馆 | 时期 | 推荐理由 |
|---|---------|------|------|---------|
| 1 | 文物名 ⭐ | 展馆名 | 朝代 | 推荐原因 |

⭐ = 镇馆之宝

## 注意

- 博物馆名称为**必填**，不可跳过
- 文物数量不足目标时，适当降低筛选标准
- 当文物展馆信息不足时，最终 Markdown 表格可能会省略“展馆”列
- 同一展馆内的文物尽量安排在一起，减少折返
