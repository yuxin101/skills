# Output Contract

本文件定义 `seo-geo-page-factory` 的标准输出结构。
所有页面生成任务必须遵守此结构，除非用户明确要求删减。

---

## 1. 总规则

输出必须按固定顺序生成，不能随意打乱。
其目的是：
- 保证不同 agent 产出一致
- 保证不会遗漏关键 SEO / GEO 模块
- 方便团队成员、设计师、前端、内容编辑直接复用

---

## 2. 标准输出顺序

### 1）页面定位
必须包括：
- 页面类型判断
- 本页的主要搜索意图
- 本页的主要转化目标
- 本页适合承接的用户类型
- 本页在整站结构中的作用
- 若有输入缺失，需明确写出关键假设

### 2）SEO / GEO 基础信息
必须包括：
- 推荐 URL slug
- 推荐 Title（2 个版本：稳健版 / 更强点击版）
- 推荐 Meta Description
- 推荐 H1
- 推荐页面副标题
- 推荐核心关键词（1 个主关键词 + 5-10 个相关词）
- 推荐可覆盖的问题型长尾词（至少 8 个）

### 3）页面信息架构
必须包括：
- 完整页面模块结构
- 每个模块的目标说明
- 若为分类页 / 聚合页 / 市场页，需预留动态内容区

### 4）完整页面文案模板
必须包括：
- 每个模块的标题
- 每个模块的正文文案
- 可直接上站
- 可给设计 / 内容 / 前端使用

### 5）FAQ 模块
必须包括：
- 5-8 个 FAQ
- 问题自然真实
- 回答简洁且可引用
- 与正文不重复

### 6）结构化数据建议
必须包括：
- 推荐 schema 类型
- 为什么适合
- 不适合什么 schema
- 一个 JSON-LD 示例

### 7）技术实现与抓取建议
必须包括：
- canonical 建议
- robots / noindex 建议
- sitemap 建议
- breadcrumb 建议
- 首屏文本可见性建议
- 图片 alt 与文件名建议
- 至少 5 条内链建议
- 若相关，AI 搜索可见性建议
- 若相关，llms.txt 作为可选项说明

### 8）上线质检清单
必须包括：
- 标题唯一性
- description 可用性
- H1 唯一
- canonical 正确
- 索引策略正确
- 内容文本可见
- FAQ 与正文不冲突
- schema 与正文一致
- CTA 清晰
- 内链已加
- 图片 alt 已加
- 页面是否真正有独特价值
- 页面是否避免 AI 批量空内容风险

### 9）可直接交付版
必须包括：
- 仅保留页面模块标题 + 成稿文案
- 不带解释
- 可直接给设计 / 前端 / 内容编辑使用

---

## 3. 标题规范

输出中各大段标题必须使用以下标准名称：

- 页面定位
- SEO / GEO 基础信息
- 页面信息架构
- 完整页面文案模板
- FAQ 模块
- 结构化数据建议
- 技术实现与抓取建议
- 上线质检清单
- 可直接交付版

若缺失任一标题，视为输出不合格。

---

## 4. 文案规范

### 4.1 首屏文案
必须强，必须直接。
至少回答：
- 是什么
- 给谁
- 为什么重要

### 4.2 中段文案
必须承担以下至少 3 项：
- 解释概念
- 说明价值
- 指出适用场景
- 给出分类 / 内容组织方式
- 建立信任
- 支撑转化

### 4.3 末段文案
必须明确承接 CTA，不得草草结束。

---

## 5. 动态内容区规范

当页面属于以下类型时，必须明确预留动态区域：
- 分类页
- 聚合页
- 市场页
- 工具目录页
- 资源集合页

动态区可包括：
- 卡片流
- 标签筛选
- 排序
- 热门内容
- 最新内容
- 编辑精选
- 相关推荐

---

## 6. 代码块规范

若输出 JSON-LD、head 标签、robots 示例等，必须使用代码块包裹。
不得把结构化数据散写成普通文本。

---

## 7. 质量底线

输出若出现以下情况，视为不合格：

- 缺少任意一个固定大模块
- 没有 FAQ
- 没有 schema 建议
- 没有技术建议
- 没有上线 checklist
- 没有成稿版
- 页面像“概念草稿”而不是“生产可用模板”
- 使用大量空话填充

---

## 8. 简化输出的允许条件

只有当用户明确要求“只给我简版”时，才允许压缩内容。
即便压缩，也必须保留：
- 页面定位
- SEO / GEO 基础信息
- 页面信息架构
- 核心文案
- 技术建议
- 成稿版

---

## 9. 最终交付目标

输出必须能够被以下角色直接理解并接手：

- 产品经理
- 内容编辑
- SEO 负责人
- 设计师
- 前端开发
- 项目负责人

也就是说，输出不是“灵感”，而是“可执行交付物”。
