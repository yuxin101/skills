# Input Contract

本文件定义 `seo-geo-page-factory` skill 在生成页面前应收集、推断和约束的输入信息。

---

## 1. 目标

确保 agent 在生成网页前，先形成最小可用输入集，避免出现：
- 页面主题模糊
- 页面类型判断错误
- 搜索意图与转化目标混乱
- 输出文案空泛
- SEO / GEO 结构缺失
- 技术建议与页面类型不匹配

---

## 2. 最小可用输入集（Minimum Viable Input）

当且仅当以下 5 项可确定时，允许直接产出完整页面：

1. 页面主题
2. 页面类型（可由 agent 推断）
3. 品牌名
4. 目标用户
5. 主要转化目标

如果缺失上述任意 2 项以上，仍可输出，但必须：
- 进行最小合理假设
- 在“页面定位”部分明确写出假设内容
- 禁止因此输出空泛模板

---

## 3. 建议收集的完整输入项

### 3.1 基础信息
- 页面主题
- 页面类型（首页 / 落地页 / 功能页 / 分类页 / 聚合页 / 详情页 / 博客页 / 文档页 / FAQ 页 / 定价页 / 对比页 / 其他）
- 品牌名
- 品牌一句话介绍

### 3.2 用户与业务信息
- 目标用户
- 页面主要转化目标
- 产品 / 服务核心能力
- 页面希望覆盖的主要使用场景
- 核心竞争差异点

### 3.3 搜索与内容信息
- 页面主要关键词
- 希望覆盖的问题型长尾词
- 是否特别考虑 Google
- 是否特别考虑百度
- 是否特别考虑 AI 搜索 / 大模型引用
- 是否需要 FAQ
- 是否需要结构化数据建议

### 3.4 风格与交付信息
- 语气偏好（专业 / 平台感 / 极简 / 友好 / 锋利 / 科技感）
- 是否需要动态区预留（卡片流 / 筛选区 / 排序区 / 推荐区）
- 是否需要可直接给设计师 / 前端 / 内容团队使用的成稿版

---

## 4. 输入解释规则

### 4.1 页面主题
页面主题必须是“一个页面要讲什么”，而不是整个网站要做什么。

正确示例：
- AI Skills 页面
- AI 工具分类页
- MCP 介绍页
- 企业 AI 工作流引擎功能页

错误示例：
- 做一个很厉害的网站
- 写一点关于 AI 的内容
- 我们平台很强

### 4.2 页面类型
如果用户未明确说明页面类型，agent 必须根据目标推断。

推断优先级：
1. 如果是网站总入口，优先判断为首页
2. 如果围绕单一能力介绍，优先判断为功能页 / 详情页
3. 如果围绕一组内容分发，优先判断为分类页 / 聚合页
4. 如果围绕教育、说明、步骤，优先判断为文档页 / FAQ 页
5. 如果围绕广告投放或单一转化，优先判断为落地页

### 4.3 目标用户
目标用户必须具备可操作性，不能只写“所有人”。

正确示例：
- 想快速找到 AI Tools 的普通用户
- 需要发现 MCP 工具的开发者
- 想搭建自动化流程的业务团队
- 想提交并分发 Skills 的构建者

### 4.4 主要转化目标
转化目标必须具体可执行，至少包含一个动作。

正确示例：
- 点击分类
- 浏览全部工具
- 进入详情页
- 提交工具
- 联系销售
- 注册 / 下载 / 试用

错误示例：
- 让用户喜欢我们
- 提高品牌感知
- 让页面很高级

---

## 5. 缺失信息时的处理规则

当输入不完整时，按以下顺序处理：

### 5.1 优先推断，而不是停滞
如果大部分信息已足够支持页面生成，优先做合理推断，不要因为缺少次要信息而停下。

### 5.2 只做最小假设
假设应尽量少，且必须贴近已知上下文。

示例：
- 若页面主题为“AI Skills 页面”，可合理假设是分类页 / 聚合页
- 若用户提到“联系我们试用”，可合理假设主要转化目标为“申请试用 / 联系销售”

### 5.3 必须显式写出假设
所有关键假设必须在“页面定位”中写出，不得偷偷补全。

### 5.4 禁止用无意义话术逃避
禁止出现：
- “可根据实际情况调整”
- “其他信息可自行补充”
- “按需修改即可”
- “视品牌情况而定”

除非确实属于非核心可选项。

---

## 6. 输入优先级规则

当用户输入存在冲突时，按以下优先级取舍：

1. 页面主题
2. 页面转化目标
3. 目标用户
4. 页面类型
5. 品牌定位
6. 关键词偏好
7. 文风偏好

例如：
- 若用户说“这是首页”，但目标与内容完全围绕单一功能，必须指出：首页定位与内容结构存在冲突，并给出更合理判断
- 若用户要求“极简”，但又要求“覆盖大量 FAQ / 分类 / 模块”，优先保证信息完整，再在文风上尽量克制

---

## 7. 禁止接受的危险输入意图

若用户意图明显属于以下类型，必须拒绝相应部分或进行改写：

- 隐藏关键词
- 白底白字
- 诱导搜索引擎抓取与用户不同内容
- 伪造用户评价、评分、价格、库存、权威认证
- 伪造结构化数据
- 承诺“保证排名”“保证收录”
- 纯为操纵搜索结果而批量制造空内容
