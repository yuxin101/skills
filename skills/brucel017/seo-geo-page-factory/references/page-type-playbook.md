# Page Type Playbook

本文件定义不同页面类型的判断逻辑、结构模板、主要意图、CTA 类型和 SEO / GEO 重点。

---

## 1. 总原则

所有页面都必须遵守：

- 一页一个核心意图
- 首屏清楚回答“这是什么、给谁、为什么值得看”
- 中段回答“为什么、怎么用、适合谁”
- 末段承接 CTA
- 页面结构应服务内容意图，而不是机械套模板

---

## 2. 首页（Homepage）

### 何时使用
- 网站总入口
- 品牌总定位页
- 用户第一次认识产品的主要入口页

### 核心意图
- 品牌理解
- 价值建立
- 站内分发
- 承接高层级品牌词

### 结构建议
1. Hero
2. 品牌/平台定位说明
3. 核心能力总览
4. 适用人群 / 场景
5. 为什么选择我们
6. 重点内容 / 分类分发
7. FAQ
8. 最终 CTA

### CTA
- 立即探索
- 浏览产品
- 查看分类
- 开始使用
- 联系我们

### SEO / GEO 重点
- 品牌词 + 核心业务定义
- 首页不要塞太多细枝末节
- 强化实体信息与站点定位
- 适合作为站内导航与内链分发中心

---

## 3. 落地页（Landing Page）

### 何时使用
- 单一广告/投放承接页
- 单一活动、单一目标、单一受众页面
- 需要强转化闭环的场景

### 核心意图
- 说服 + 转化

### 结构建议
1. Hero
2. 核心问题 / 痛点
3. 解决方案
4. 功能/价值点
5. 场景 / 客户对象
6. 信任信息
7. FAQ
8. 强 CTA

### CTA
- 立即试用
- 预约演示
- 联系销售
- 开始注册

### SEO / GEO 重点
- 不要写成品牌总介绍页
- 页面围绕一个转化动作服务
- 问题与答案段要尽量清晰可引用

---

## 4. 功能页（Feature Page）

### 何时使用
- 介绍某一个功能或能力
- 承接“某功能是什么 / 有什么用 / 怎么用”类搜索

### 核心意图
- 功能理解
- 功能比较
- 功能转化

### 结构建议
1. Hero
2. 功能定义
3. 功能解决的问题
4. 核心能力拆解
5. 使用场景
6. 为什么选择这个功能
7. FAQ
8. CTA

### CTA
- 了解更多
- 立即体验
- 查看文档
- 联系团队

### SEO / GEO 重点
- 一页只讲一个功能主题
- 定义句必须清楚
- 场景与边界必须写清楚
- 适合承接对比型和定义型搜索

---

## 5. 产品页 / 详情页（Product / Detail Page）

### 何时使用
- 单个工具、单个 Skill、单个产品详情页
- 希望承接长尾 SEO 与具体转化

### 核心意图
- 具体能力理解
- 进入使用、下载、安装、跳转或提交

### 结构建议
1. Hero
2. 产品 / Skill 定义
3. 适合谁
4. 核心卖点
5. 使用方式 / 接入方式
6. 相关能力 / 关联内容
7. FAQ
8. CTA

### CTA
- 立即使用
- 查看详情
- 获取接入方式
- 提交 / 收藏 / 安装

### SEO / GEO 重点
- 适合长尾词
- 要有具体实体信息
- 不能写成泛泛的介绍文
- FAQ 很重要

---

## 6. 分类页（Category Page）

### 何时使用
- 聚合同一类主题或工具
- 例如：AI Tools、AI Skills、MCP、写作工具、编程工具

### 核心意图
- 分类理解
- 浏览分发
- 承接主题级搜索流量

### 结构建议
1. Hero
2. 该分类是什么
3. 为什么重要 / 为什么要看这一类
4. 子分类导航
5. 精选推荐
6. 适用场景
7. FAQ
8. CTA

### CTA
- 浏览全部
- 查看热门分类
- 提交内容
- 进入详情页

### SEO / GEO 重点
- 应兼顾“解释页 + 分发页”
- 需要预留动态卡片区 / 筛选区 / 排序区
- 内容不能只有列表，没有解释

---

## 7. 聚合页 / Collection Page

### 何时使用
- 主题集合、资源总入口、市场页
- 例如：AI Skills 聚合页、全部工具页、精选资源页

### 核心意图
- 浏览
- 发现
- 分发

### 结构建议
1. Hero
2. 聚合主题定义
3. 分类 / 标签 / 筛选入口
4. 精选内容
5. 推荐逻辑说明
6. FAQ
7. CTA

### CTA
- 浏览全部
- 筛选查看
- 提交你的内容
- 进入分类

### SEO / GEO 重点
- 必须有文字解释，不只是数据表
- 需要告诉用户“这页收集的到底是什么”
- 适合作为站内流量枢纽页

---

## 8. 博客页 / 文章页（Blog / Article Page）

### 何时使用
- 教育内容
- 新闻分析
- 教程说明
- 观点型、解释型文章

### 核心意图
- 回答问题
- 建立理解
- 内链导流

### 结构建议
1. 标题与导语
2. 核心结论 / 定义
3. 正文展开
4. 小结
5. FAQ 或延伸阅读
6. CTA / 内链引导

### CTA
- 查看相关页面
- 阅读更多
- 了解产品
- 订阅 / 联系

### SEO / GEO 重点
- 首段先给答案
- 段落尽量短且可引用
- 不要为了长度灌水

---

## 9. 文档页 / Help Page（Documentation / Help Page）

### 何时使用
- 操作步骤
- 使用说明
- 安装教程
- 技术接入指导

### 核心意图
- 帮助完成任务
- 降低理解成本

### 结构建议
1. 页面目的说明
2. 前置条件
3. 步骤说明
4. 注意事项
5. 常见问题
6. 相关文档入口

### CTA
- 开始配置
- 查看下一步
- 返回目录
- 联系支持

### SEO / GEO 重点
- 步骤型写法
- 适合承接 how-to 搜索
- 结构必须清晰，避免废话

---

## 10. FAQ 页（FAQ Page）

### 何时使用
- 用户问题密集
- 售前疑问多
- 概念误解多
- 有大量长尾问句可承接

### 核心意图
- 快速回答
- 降低咨询成本
- 承接搜索

### 结构建议
1. 页面目的说明
2. 问题分组
3. 简洁回答
4. 相关链接
5. CTA

### CTA
- 查看详情页
- 联系支持
- 开始使用
- 返回分类

### SEO / GEO 重点
- 问题必须真实自然
- 回答简洁且可引用
- 不要堆出几十个低质量问题

---

## 11. 对比页（Comparison Page）

### 何时使用
- 两个产品 / 方案 / 路径对比
- 承接“X vs Y”类意图

### 核心意图
- 帮用户做决策

### 结构建议
1. Hero
2. 对比对象定义
3. 关键维度对比
4. 适合谁
5. 如何选择
6. FAQ
7. CTA

### CTA
- 查看方案 A
- 查看方案 B
- 联系咨询
- 开始体验

### SEO / GEO 重点
- 对比维度必须明确
- 不得伪造竞争对手信息
- 不得恶意贬低

---

## 12. 定价页（Pricing Page）

### 何时使用
- 有套餐、价格、方案结构
- 承接商业化阶段用户

### 核心意图
- 解释价格
- 建立信任
- 促成购买/咨询

### 结构建议
1. Hero
2. 套餐说明
3. 价格表
4. 功能差异
5. 适合谁
6. FAQ
7. CTA

### CTA
- 开始试用
- 立即购买
- 联系销售

### SEO / GEO 重点
- 不得虚构价格
- 若价格可能变化，应谨慎写法
- schema 仅在信息完整时使用

---

## 13. 页面类型选择优先级

当页面类型不确定时，按以下问题判断：

1. 这是网站总入口吗？如果是，优先首页
2. 这是单一转化承接吗？如果是，优先落地页
3. 这是讲一个能力吗？如果是，优先功能页 / 详情页
4. 这是讲一类内容吗？如果是，优先分类页 / 聚合页
5. 这是讲操作步骤吗？如果是，优先文档页
6. 这是回答问题吗？如果是，优先 FAQ 页
7. 这是比较选择吗？如果是，对比页
8. 这是商业方案说明吗？如果是，定价页

---

## 14. 自检规则

在确定页面类型后，agent 必须自问：

- 这个页面的主意图是否单一？
- 这个页面的 CTA 是否与页面类型匹配？
- 这个页面的结构是否服务于该类型最核心的问题？
- 这个页面是否被错误写成了另一种页面？

若答案是否定，必须先修正页面类型再继续生成。
