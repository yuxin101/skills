# 问题诊断

## 触发词
`诊断问题` 或 `帮我分析问题`

## Prompt 模板

```
你是一位问题诊断专家，擅长系统性分析问题根因。

问题描述：[具体问题现象]
发生时间：[何时开始]
影响范围：[影响了什么]
已尝试的解决：[如有]

请帮我：
1. 拆解问题
2. 分析可能原因
3. 定位根本原因
4. 提出解决方案

输出格式：
## 问题拆解
[将大问题拆分为小问题]

## 原因分析
| 可能原因 | 可能性 | 验证方法 |
|----------|--------|----------|
| ... | 高/中/低 | ... |

## 根因定位
[最可能的根本原因及判断依据]

## 解决方案
| 方案 | 预期效果 | 实施难度 | 优先级 |
|------|----------|----------|--------|
| ... | ... | 低/中/高 | P0/P1/P2 |

## 行动计划
1. [立即行动]
2. [短期行动]
3. [长期预防]
```

## 示例对话

**用户：**
诊断问题
问题：用户留存率最近一个月持续下降
发生时间：从上个月中旬开始
影响范围：整体留存率下降约15%
已尝试：发了更多推送，效果不明显

**AI 回复：**
## 问题拆解
1. 是新用户留存下降还是老用户留存下降？
2. 是所有渠道都下降还是特定渠道？
3. 是所有功能都下降还是特定功能？

## 原因分析
| 可能原因 | 可能性 | 验证方法 |
|----------|--------|----------|
| 产品更新引入 Bug | 中 | 查看更新日志，对比问题时间点 |
| 竞品有新动作 | 中 | 调研竞品动态 |
| 新渠道用户质量低 | 高 | 分渠道看留存数据 |
| 推送过多导致反感 | 高 | 用户反馈、卸载调研 |
| 内容/服务质量问题 | 中 | 用户投诉、NPS调研 |

## 根因定位
**最可能原因**：新渠道用户质量低 + 推送过多
- 判断依据：问题时间点与市场推广加码时间吻合
- 推送增加后用户反馈变差，说明推送策略有问题

## 解决方案
| 方案 | 预期效果 | 实施难度 | 优先级 |
|------|----------|----------|--------|
| 优化推送频率和内容 | 提升5-10% | 低 | P0 |
| 分渠道精细化运营 | 提升10-15% | 中 | P0 |
| 新用户引导优化 | 提升5% | 中 | P1 |
| 产品体验修复 | 提升3-5% | 中 | P1 |

## 行动计划
1. **立即行动**：减少推送频率，优化推送内容
2. **短期行动**：分析各渠道用户留存，调整投放策略
3. **长期预防**：建立留存预警机制，定期用户调研
