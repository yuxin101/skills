# 知识整理器

## 触发词
`整理笔记` 或 `知识卡片`

## Prompt 模板

### 模式 1：读书笔记整理

```
你是一位知识管理专家，擅长从信息中提炼精华。

我的阅读笔记/摘录：
"""
[粘贴内容]
"""

请帮我整理为知识卡片格式：

## 核心概念
[用一句话概括核心观点]

## 关键要点（3-5 个）
- **[要点1]**：[解释]
- **[要点2]**：[解释]
...

## 金句摘录
- "[原句]" —— [页码/章节]

## 我的思考
[联系自己的经验或问题]

## 行动清单
- [ ] [可以立即执行的动作]

## 关联知识
[可以和什么已有知识建立联系]
```

### 模式 2：知识体系构建

```
我正在学习：[主题]
已掌握的知识点：
- [知识点1]
- [知识点2]
...

请帮我：
1. 构建知识框架（思维导图结构）
2. 识别知识缺口
3. 推荐学习路径
4. 设计知识卡片模板
```

## 示例对话

**用户：**
整理笔记

原文：
"""
费曼学习法的核心是用自己的话解释概念，如果你不能用简单的话解释清楚，说明你没有真正理解。
步骤：
1. 选择一个概念
2. 用自己的话解释给 8 岁小孩听
3. 识别卡壳的地方，回去重学
4. 简化语言，用类比

费曼说：如果你不能简单地解释它，你就没有理解它。
"""

**AI 回复：**
## 核心概念
费曼学习法 = 通过"教会别人"来检验和深化自己的理解。

## 关键要点

- **输出倒逼输入**：不是学完再输出，而是通过输出发现不懂的地方
- **降维解释**：用 8 岁小孩能懂的话解释，才能证明真正理解
- **精准定位盲点**：卡壳的地方就是知识漏洞
- **类比思维**：用熟悉的事物类比新概念，加深记忆

## 金句摘录
- "如果你不能简单地解释它，你就没有理解它" —— 费曼

## 我的思考
这和学习编程很像：觉得自己懂了，但写代码时卡住，才发现没真懂。以后学习新概念时，试着讲给老婆/同事听。

## 行动清单
- [ ] 下次学新概念时，尝试用大白话写在笔记里
- [ ] 准备"费曼测试"：能不能用一句话说清楚今天学的东西？

## 关联知识
- **第一性原理**：回到最基础的层面理解
- **帕累托法则**：用 20% 的核心概念解释 80% 的现象
- **间隔重复**：配合记忆曲线，定期"教学"
