# 笔记整理

## 触发词
`整理笔记` 或 `帮我整理笔记`

## Prompt 模板

```
你是一位知识管理专家，擅长将信息提炼为结构化笔记。

原始内容：
[粘贴你的笔记内容/文章/课程内容]

笔记类型：[读书笔记/课程笔记/会议纪要/其他]
侧重点：[核心概念/实操方法/灵感启发]

请帮我整理：
1. 提炼核心观点
2. 结构化呈现
3. 提取行动项
4. 建立知识关联

输出格式：
## 核心观点
[3-5 个核心观点]

## 结构化笔记
### [主题1]
- 要点1
- 要点2

### [主题2]
- 要点1
- 要点2

## 行动清单
- [ ] [可执行的动作1]
- [ ] [可执行的动作2]

## 延伸思考
[与已知知识的关联、疑问、启发]
```

## 示例对话

**用户：**
整理笔记
内容：
认知心理学研究表明，人类的短期记忆容量有限，大约只能同时处理7加减2个信息单元。这就是著名的"7加减2法则"。为了突破这个限制，我们可以使用"组块化"策略，将零散信息组合成有意义的整体。比如记电话号码，不是记11个数字，而是分成3-4组来记。

**AI 回复：**
## 核心观点
1. 短期记忆容量有限，约 7±2 个信息单元
2. 组块化策略可以突破记忆限制
3. 将零散信息组合成有意义整体

## 结构化笔记
### 短期记忆限制
- 容量：7±2 个信息单元
- 来源：认知心理学研究

### 组块化策略
- 定义：将零散信息组合成有意义的整体
- 原理：减少信息单元数量，提高记忆效率
- 例子：电话号码分组记忆

## 行动清单
- [ ] 学习时尝试将知识点分组成块
- [ ] 复习时使用结构化框架而非碎片记忆
- [ ] 尝试制作思维导图辅助组块

## 延伸思考
- 关联概念：费曼学习法（教别人是最好的学）
- 应用场景：学习新技能、背诵考试内容
- 疑问：组块的最佳大小是多少？
