# 痛点-解决模板

## 结构

```
[Hook] 2-3s
  └─ 抛出痛点问题，引发共鸣
  
[Pain] 3-8s  
  └─ 放大痛点，展示糟糕现状
  └─ 可用多个短镜头快切

[Solution] 占总时长 50-60%
  └─ 引入产品/方案
  └─ 展示核心功能（demo_insert: true）
  └─ 强调差异化优势

[Result] 3-5s
  └─ 展示使用后的效果
  └─ 数据/对比/用户反馈

[CTA] 2-3s
  └─ 明确行动指引
```

## Hook 公式

- "你是不是也..." + 痛点描述
- "每次...都要..." + 繁琐过程
- "如果你还在..." + 旧方法
- 数字开头："90%的人不知道..."

## Pain 表现手法

- 特写皱眉/叹气表情
- 快切多个失败场景
- 时间流逝蒙太奇
- 对话框堆积/通知轰炸

## Solution 展示要点

- 一句话说清楚产品是什么
- 聚焦 1-2 个核心功能，不要贪多
- 操作演示要流畅，标记 demo_insert
- 用对比突出效果差异

## CTA 话术

- "链接在评论区/主页"
- "点击头像私信获取"
- "关注我，下期教你..."
- "点赞收藏，随时回看"
