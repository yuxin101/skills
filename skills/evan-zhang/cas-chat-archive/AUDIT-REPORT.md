## 一、严重问题 (Critical - 必须修复)

### C1. 文件锁超时未实现
**位置**: `cas_archive.py:append_locked()`
**问题**: 代码注释说 `timeout_seconds: int = 5`，但实际上 `fcntl.flock` 没有等待超时，会立即阻塞。
**影响**: 高并发时可能死锁。
**修复**: 使用 `fcntl.flock(lockf.fileno(), fcntl.LOCK_EX | fcntl.LOCK_NB)` 并捕获 `BlockingIOError`。

### C2. Gateway 名称未验证
**位置**: `cas_archive.py:cmd_init(), cmd_record_message(), cmd_record_asset()`
**问题**: `gateway` 参数直接用于路径拼接，没有验证是否包含路径分隔符（如 `../`）。
**影响**: 可能导致目录遍历攻击，写入任意目录。
**修复**: 添加验证 `if "/" in gateway or "\\" in gateway: raise ValueError("Invalid gateway name")`

### C3. 附件复制失败后仍记录日志
**位置**: `cas_archive.py:cmd_record_asset()`
**问题**: `shutil.copy2(source, target)` 在 `append_locked` 之前执行，如果复制失败但异常被捕获，可能记录不存在的文件。
**影响**: 日志与实际文件不一致。
**修复**: 先复制文件，确认成功后再写入日志。

### C4. Hook 脚本性能问题
**位置**: `cas_hook.py:main()`
**问题**: 每次调用启动 3-5 个子进程（init + 消息 + 附件），开销巨大。
**影响**: 严重影响 Gateway 响应速度。
**修复**: 直接调用 `cas_archive` 的函数，而不是启动子进程。

## 二、高风险问题 (High - 应尽快修复)

### H1. 磁盘满未处理
**位置**: 所有写入操作
**问题**: 没有检查磁盘剩余空间，也没有捕获 `OSError: No space left on device`。
**影响**: 磁盘满时日志丢失，且可能导致 Gateway 崩溃。
**修复**: 添加磁盘空间检查（阈值 100MB），捕获 OSError 并告警。

### H2. 文件权限问题未处理
**位置**: 所有文件操作
**问题**: 没有检查文件权限，`PermissionError` 可能导致整个记录失败。
**影响**: 权限错误时日志丢失。
**修复**: 捕获 `PermissionError`，记录错误但不影响主流程。

### H3. 附件路径未验证
**位置**: `cas_hook.py:main()`
**问题**: `attachment` 路径直接传给 `record-asset`，没有验证是否在允许的目录内。
**影响**: 可能被利用读取任意文件。
**修复**: 验证附件路径必须在 Gateway upload 目录内。

### H4. 大文件未限制
**位置**: `cas_archive.py:cmd_record_asset()`
**问题**: 没有限制附件大小，可能复制超大文件导致磁盘满。
**影响**: 磁盘空间耗尽。
**修复**: 添加 `--max-size` 参数，默认 100MB。

### H5. 跨时区问题
**位置**: `cas_archive.py:parse_ts()`
**问题**: 如果传入的 timestamp 没有时区信息，会添加本地时区，但可能导致跨时区使用时混乱。
**影响**: 时间记录不准确。
**修复**: 强制要求 timestamp 带时区，或在日志中明确记录时区。

## 三、中等风险问题 (Medium - 建议修复)

### M1. 性能优化：目录创建
**位置**: `cas_archive.py:ensure_dirs()`
**问题**: 每次调用都尝试创建所有目录，即使已存在。
**影响**: 轻微性能损失。
**修复**: `mkdir(parents=True, exist_ok=True)` 已经处理，无需修复。

### M2. 性能优化：文件锁粒度
**位置**: `cas_archive.py:append_locked()`
**问题**: 每次写入都获取文件锁，高并发时可能成为瓶颈。
**影响**: 高并发时性能下降。
**修复**: 考虑使用队列缓冲，批量写入。

### M3. 错误信息不够详细
**位置**: 所有 `except` 块
**问题**: 捕获异常后只打印简单错误信息，缺少上下文。
**影响**: 难以调试。
**修复**: 使用 logging 模块，记录完整异常栈。

### M4. 缺少配置文件
**位置**: 全局
**问题**: 所有参数都是硬编码或命令行参数，没有配置文件。
**影响**: 不便于批量部署和配置管理。
**修复**: 添加 `config.yaml` 支持。

### M5. 缺少监控指标
**位置**: 全局
**问题**: 没有暴露任何监控指标（如记录成功率、磁盘使用量）。
**影响**: 无法监控运行状态。
**修复**: 添加 Prometheus metrics 或简单的统计接口。

## 四、低风险问题 (Low - 可选修复)

### L1. 文件名长度限制
**位置**: `cas_archive.py:sanitize_filename()`
**问题**: 限制 200 字符，但某些文件系统可能更短。
**影响**: 极少数情况下文件名过长。
**修复**: 降低到 255 字节（不是字符）。

### L2. 隐藏文件未处理
**位置**: `cas_archive.py:sanitize_filename()`
**问题**: 没有检查文件名是否以点开头（Unix 隐藏文件）。
**影响**: 可能创建隐藏文件。
**修复**: 添加检查 `if name.startswith('.'): name = '_' + name`

### L3. MIME 类型检测不准确
**位置**: `cas_archive.py:cmd_record_asset()`
**问题**: `mimetypes.guess_type` 基于扩展名，可能不准确。
**影响**: 日志中 MIME 类型可能错误。
**修复**: 使用 `python-magic` 库检测实际类型。

### L4. 帮助文档不完整
**位置**: `argparse` 定义
**问题**: 很多参数没有详细说明。
**影响**: 用户体验差。
**修复**: 为所有参数添加详细帮助文本。

### L5. 缺少版本信息
**位置**: 全局
**问题**: 没有 `--version` 参数。
**影响**: 无法确认运行版本。
**修复**: 添加版本参数。

## 五、架构级建议

### A1. 日志轮转机制
**问题**: 没有自动压缩旧日志的机制。
**建议**: 添加定期任务，压缩 90 天前的日志。

### A2. 备份机制
**问题**: 没有自动备份。
**建议**: 添加定期备份到外部存储的功能。

### A3. 搜索优化
**问题**: 纯文本搜索效率低。
**建议**: 考虑添加可选的索引服务（如 SQLite FTS）。

### A4. Hook 集成方式
**问题**: 当前 Hook 是外挂式，需要手动配置。
**建议**: 考虑做成 Gateway 插件，自动集成。
