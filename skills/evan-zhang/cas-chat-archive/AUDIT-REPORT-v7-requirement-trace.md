## 1) 关键需求对照（结论）

### R1. 用户无感知（不影响正常聊天）
- 现状：**基本满足**
- 依据：
  - hook 采用 fail-soft（默认 `CAS_STRICT_MODE=false`），归档异常不阻断主流程。
  - 长消息已避免 argv 崩溃（payload/file 传输）。
  - 附件越界会阻断归档但不会让 hook 硬失败。
- 风险：低（主要是归档质量风险，不是聊天可用性风险）。

- 现状：**未完全满足（关键缺口）**

  - 即：代码已就绪，但“自动触发链路”尚未挂载到网关运行时。
- 结论：这是当前最关键的上线缺口。

### R3. 全量记录 + 可搜索
- 现状：**部分满足**
- 文本消息：满足（inbound/outbound append-only）
- 附件：满足（在白名单范围内）
- 可搜索：满足（markdown + grep/rg）
- 说明：白名单外附件不会归档（安全优先）。

### R4. 安全与稳定
- 现状：**满足灰度标准**
- 已修：路径逃逸、长消息崩溃、并发 state 冲突、包污染、发布假阳性。
- 仍可优化：进程内直调、告警通道、事件流模型。
