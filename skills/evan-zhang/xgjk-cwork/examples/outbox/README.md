# outbox 示例

## 模块说明
获取当前用户的发件箱汇报列表，支持分页。

## 依赖脚本
`../../scripts/outbox/get-list.py`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在拉取发件箱列表之前，先给出建议方案。**

- 建议合适的分页参数（默认50条/页）
- 建议是否需要按接收人类型或时间范围过滤
- 提示发件箱数据量与用户发送频率相关

```
建议：默认拉取最近 50 条发件箱记录，按时间倒序。
如需查看特定接收对象的发送记录，可补充 receiver 过滤条件。
```

### Step 2 — Decide（确认/决策）
**涉及数据查询前，必须向用户确认参数。**

- 确认分页大小（默认50）
- 确认时间范围是否需要限定
- 确认是否需要按接收人类型过滤

```
请确认查询参数：
□ 分页大小：50（默认）/ 其他：____
□ 时间范围：全部 / 本周 / 本月 / 自定义：____
□ 接收人类型过滤（可选）：____
```

### Step 3 — Execute（执行）
执行发件箱查询脚本。

### Step 4 — Log（留痕）
**查询结果必须完整记录。**

- 记录查询时间、分页参数、过滤条件、返回条数
- 格式：`[LOG] outbox | page:1 size:50 | result:N | ts:ISO8601`

```
[LOG] outbox | page:1 size:50 | result:50 | ts:2026-03-25T13:52:00+08:00
```

---

## 输出格式

```json
{
  "page": 1,
  "size": 50,
  "total": N,
  "items": [
    {
      "reportId": "...",
      "receiver": "...",
      "title": "...",
      "createTime": "...",
      "type": "..."
    }
  ]
}
```

---

## 注意事项
- 发件箱为只读查询
- 分页参数需严格校验，避免异常偏移
- 日志保留完整查询上下文，供审计使用
