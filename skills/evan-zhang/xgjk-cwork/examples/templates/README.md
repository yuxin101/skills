# templates 示例

## 模块说明
获取汇报模板清单，帮助用户快速创建标准格式汇报。

## 依赖脚本
`../../scripts/templates/get-list.py`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在拉取模板列表之前，先给出建议方案。**

- 说明可用模板的类型（日报、周报、月报、项目汇报等）
- 建议按用途场景选择合适模板
- 提示模板内容为只读参考，可直接引用结构

```
建议：默认拉取全部汇报模板清单。
如需特定类型模板（日报/周报/月报），可补充 type 过滤参数。
模板内容仅供参照，不涉及任何数据变动。
```

### Step 2 — Decide（确认/决策）
**涉及数据查询前，必须向用户确认参数。**

- 确认是否需要按模板类型过滤
- 确认是否需要特定场景模板

```
请确认：
□ 模板类型：全部 / 日报 / 周报 / 月报 / 项目汇报
□ 是否有其他过滤需求：____
```

### Step 3 — Execute（执行）
执行模板列表查询。

### Step 4 — Log（留痕）
**查询结果必须完整记录。**

- 记录查询条件、返回模板数量、时间戳
- 格式：`[LOG] templates | filter:weekly | result:N | ts:ISO8601`

```
[LOG] templates | filter:all | result:12 | ts:2026-03-25T13:56:00+08:00
```

---

## 输出格式

```json
{
  "items": [
    {
      "templateId": "...",
      "name": "...",
      "type": "daily|weekly|monthly|project",
      "description": "...",
      "fields": ["title", "content", "attachments"]
    }
  ]
}
```

---

## 注意事项
- 模板查询为只读操作
- 日志需记录查询参数与返回数量
- 如模板数量较少，可考虑缓存结果避免重复查询
