# plan-create 示例

## 模块说明
快速创建一个简单的工作计划/汇报，支持快捷提交。

## 依赖脚本
`../../scripts/plan-create/create-simple.py`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在创建计划之前，先给出建议方案。**

- 说明该接口用于快速创建工作计划，字段精简（仅 title + content）
- 建议用户确认计划内容的完整性和准确性
- 提示提交后将直接进入待审核/已发布状态

```
建议：使用 create-simple 接口快速创建工作计划。
该操作会生成新的汇报记录，请确保 title 和 content 填写完整。
提交后状态为"已发布"，无法直接撤回，请仔细确认。
```

### Step 2 — Decide（确认/决策）
**涉及数据变动前（创建操作），必须获取用户明确决策。**

```
⚠️ 确认执行数据变动操作：
□ 计划标题：____
□ 计划内容：____（请确保内容完整准确）
□ 提交后不可直接撤回，了解此风险：是
□ 输入 "确认提交" 以继续：____
```

### Step 3 — Execute（执行）
执行计划创建脚本。

### Step 4 — Log（留痕）
**数据变动结果必须完整记录。**

- 记录创建的计划标题、计划ID（reportId）、提交时间、操作人
- 格式：`[LOG] plan-create | reportId:xxx | title:xxx | ts:ISO8601 | operator:user`

```
[LOG] plan-create | reportId:REP-20260325001 | title:三月第三周工作计划 | ts:2026-03-25T13:59:00+08:00 | operator:evan
```

---

## 输入格式

**Body：**
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 计划标题 |
| `content` | string | 是 | 计划内容 |

---

## 输出格式

```json
{
  "success": true,
  "reportId": "REP-xxx",
  "title": "三月第三周工作计划",
  "createTime": "2026-03-25T13:59:00+08:00"
}
```

---

## 注意事项
- ⚠️ 该接口为写操作（POST），执行后会在服务器创建新记录
- 提交前必须经过完整的 Suggest → Decide → Log 闭环
- Log 必须包含 reportId，供后续查询或撤回使用
- 该接口仅支持简化字段，如需附件/多媒体请使用完整创建接口
