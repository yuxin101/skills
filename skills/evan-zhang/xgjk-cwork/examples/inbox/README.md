# inbox 示例

## 模块说明
获取当前用户的收件箱汇报列表，支持分页。

## 依赖脚本
`../../scripts/inbox/get-list.py`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在拉取收件箱列表之前，先给出建议方案。**

- 根据用户当前状态，建议合适的分页参数（默认50条/页）
- 建议是否需要限定时间范围（如：本周/本月/自定义）
- 提示收件箱数据量通常较大，建议按需索取

```
建议：默认拉取最近 50 条收件箱记录，按时间倒序。
如需查看历史记录，可指定 startDate / endDate 过滤。
```

### Step 2 — Decide（确认/决策）
**涉及数据查询前，必须向用户确认参数。**

- 确认分页大小（默认50，是否接受）
- 确认时间范围是否需要限定
- 确认是否需要按发送人/类型过滤

```
请确认查询参数：
□ 分页大小：50（默认）/ 其他：____
□ 时间范围：全部 / 本周 / 本月 / 自定义：____
□ 按发送人过滤（可选）：____
```

### Step 3 — Execute（执行）
执行收件箱查询脚本。

### Step 4 — Log（留痕）
**查询结果必须完整记录。**

- 记录查询时间、分页参数、过滤条件、返回条数
- 记录本次拉取的关键元信息
- 格式：`[LOG] inbox | page:1 size:50 | result:N | ts:ISO8601`

```
[LOG] inbox | page:1 size:50 | result:50 | ts:2026-03-25T13:51:00+08:00
```

## 输出格式

```json
{
  "page": 1,
  "size": 50,
  "total": N,
  "items": [
    {
      "reportId": "...",
      "sender": "...",
      "title": "...",
      "createTime": "...",
      "type": "..."
    }
  ]
}
```

---

## 注意事项
- 收件箱为只读查询，无需写操作确认
- 大量数据时需分页拉取，避免单次超时
- 日志需保留每次查询的完整参数，供后续比对
