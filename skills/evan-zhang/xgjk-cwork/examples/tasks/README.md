# tasks 示例

## 模块说明
获取工作任务与计划的分页索引列表。

## 依赖脚本
`../../scripts/tasks/get-page.py`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在拉取任务列表之前，先给出建议方案。**

- 建议合适的分页参数（默认20条/页）
- 建议是否需要按任务状态（待完成/进行中/已完成）过滤
- 提示时间范围过滤的适用场景

```
建议：默认拉取最近 20 条任务记录，按创建时间倒序。
如需按状态筛选，可补充 status 参数（pending/running/completed）。
如需查看特定时间段的任务，可指定 dateRange。
```

### Step 2 — Decide（确认/决策）
**涉及数据查询前，必须向用户确认参数。**

- 确认分页大小和页码
- 确认状态过滤条件（如适用）
- 确认时间范围（如适用）

```
请确认查询参数：
□ 分页：第____页，每页____条（默认第1页，20条）
□ 状态过滤：全部 / 待完成 / 进行中 / 已完成
□ 时间范围：全部 / 本周 / 本月 / 自定义：____
```

### Step 3 — Execute（执行）
执行任务列表查询脚本。

### Step 4 — Log（留痕）
**查询结果必须完整记录。**

- 记录查询参数、返回条数、时间戳
- 格式：`[LOG] tasks | page:1 size:20 status:all | result:N | ts:ISO8601`

```
[LOG] tasks | page:1 size:20 status:pending | result:8 | ts:2026-03-25T13:54:00+08:00
```

---

## 输出格式

```json
{
  "page": 1,
  "size": 20,
  "total": N,
  "items": [
    {
      "taskId": "...",
      "title": "...",
      "status": "...",
      "assignee": "...",
      "dueDate": "...",
      "createTime": "..."
    }
  ]
}
```

---

## 注意事项
- 任务状态变更需通过 todos 模块的 complete 接口，不在本模块范围内
- 查询为只读操作
- 日志需保留查询参数与结果摘要，供后续分析
