# user-search 示例

## 模块说明
按姓名模糊搜索内部员工，返回匹配列表。

## 依赖脚本
`../../scripts/user-search/search-emp.py`

---

## 标准流程（含 3S1R 管理闭环）

### Step 1 — Suggest（建议）
**在执行任何搜索动作之前，先给出建议方案。**

- 根据用户提供的姓名片段，预估可能的匹配范围（模糊匹配特性）
- 建议最可能的搜索关键词写法
- 提示搜索可能返回多条结果，需进一步确认

```
建议：使用 "姓名关键词" 进行模糊搜索，返回所有包含该关键词的员工记录。
如需缩小范围，可补充部门或职位条件。
```

### Step 2 — Decide（确认/决策）
**涉及重要查询前，必须向用户确认意图。**

- 确认搜索关键词是否准确
- 确认是否需要限定部门范围
- 确认是否接受模糊匹配带来的多条结果

```
请确认搜索关键词：[用户提供]
□ 接受模糊匹配结果（默认勾选）
□ 指定部门过滤（可选）
```

### Step 3 — Execute（执行）
执行搜索脚本，返回员工列表。

### Step 4 — Log（留痕）
**所有查询结果必须记录到会话或外部日志。**

- 记录搜索关键词、查询时间戳、命中数量
- 记录选取的具体员工信息（用于后续引用）
- 格式：`[LOG] user-search | keyword:xxx | hits:N | ts:ISO8601`

```
[LOG] user-search | keyword:张 | hits:12 | ts:2026-03-25T13:50:00+08:00
```

---

## 输出格式

```json
{
  "keyword": "张",
  "hits": [
    { "empId": "...", "name": "...", "dept": "...", "title": "..." }
  ],
  "total": N
}
```

---

## 注意事项
- 模糊匹配可能返回大量无关结果，执行前务必通过 Decide 步骤确认范围
- 日志需包含每次搜索的完整上下文，供审计追溯
