# 通用填写规范

> 本文件是四套模板的通用填写规则。基础模板结构见同目录下的基础模板文件。

---

## 四类模板的核心区别

| 维度 | 月报 | 季报 | 半年报 | 年报 |
|------|------|------|--------|------|
| 时间跨度 | 1个月 | 3个月 | 6个月 | 12个月 |
| 核心功能 | 过程监控 | 趋势复盘 | 中程校准 | 全年评价+传承 |
| 偏差判断 | 当月vs月度目标 | 季度累计vs年度节点 | H1 vs 全年目标 | 全年最终数字 |
| 趋势分析 | 不要求 | 必须（三个月环比） | 必须 | 必须 |
| 中程校准 | 不要求 | 第7章 | 必须正式校准方案 | 不要求 |
| 独有章节 | 无 | 下季度安排 | H2校准方案 | 价值贡献+下年建议 |
| 林刚介入 | 事项升级 | 季度复盘 | 半年校准会议 | 年度决策 |

## 颜色预警规则

### 通用预警规则

| 指标类型 | 🔴 红 | 🟡 黄 | 🟢 绿 |
|---------|------|------|------|
| 财务类（收入/利润/费用率） | 偏离>5% | 3-5% | <3% |
| 得分类（上市得分/引进得分/管线得分） | 偏离>10% | 5-10% | <5% |
| 节点类（审批/交付/系统上线） | delay>2周 | delay 1-2周 | 正常 |
| 人员类（离职率/引进达成率） | 偏离>5% | 3-5% | <3% |

### 零容忍项（任何偏离即红）

- 重大合规事故 = 0
- BP签约率 < 100%
- 奖金发放 delay
- 重大法律败诉
- 因内部协同导致核心产品重大延期

## BP原文引用规则

**核心原则：BP原文怎么写，模板就怎么填**

### 引用格式

```
**对标BP：** PXXXX-X.X（个人）/ AXX-X.X（组织）

**衡量标准：**
[引用BP原文的衡量标准内容]

**数字锚点：**
[引用BP原文的具体数字，如"≥90%"、"≤5%"、"≥100%"等]

**举措：**
[引用BP原文的举措编号和内容]
```

### 引用示例

正确：
```
**对标BP：** P1001-6.1（个人）/ A8-2（组织）

**衡量标准：**
关键岗位（TOP 30%）年度主动离职率≤5%。

**数字锚点：**
离职率≤5%，引进达成率≥90%，试用期转正率≥80%。
```

## 章节填写要点

### 第1章 汇报综述

**月报：** 总判断 + 关键进展 + 关注问题 + 下月判断
**季报：** 整体评价 + 阶段成果 + 主要偏差 + 下季策略
**半年报：** 总体判断 + 上下半程 + 可达区间 + 最重要结论
**年报：** 总体结论 + 最重要成果 + 最主要失分点 + 下年判断

### 第2章 BP目标承接

每个BP维度对应一节，按以下结构填充：
1. 对标编码（个人 + 组织）
2. 承接重点（引用BP原文举措）
3. 阶段/当前成果（引用BP原文衡量标准）
4. 数字锚点（引用BP原文具体数字）
5. 偏离判断（颜色预警）

### 第7章 下期安排（季报独有）

**季报：** 下季度目标 + 关键成果 + 关键举措 + 需校准事项

**半年报：** 下半程目标 + 关键成果 + 中程校准方案 + 成功标准

**年报：** 无第7章（改为第7章"下一年度安排"）

---

## 审查要点

1. **数字必须来自BP原文**——不得自己编造数字
2. **编码必须与BP原文一致**——检查P系列和A系列编码是否匹配
3. **颜色预警必须正确**——对照零容忍项检查
4. **示例必须标注"示例:"前缀**——区分示例和正式内容
