# CMB 招行工资批量导入 — 列映射参考

## 工资表 → 银行模板 的字段对应关系

| 银行模板列 | 字段名 | 说明 | 工资表列 |
|-----------|--------|------|---------|
| A | 账号 | 银行卡号 | D列（第4列） |
| B | 户名 | 员工姓名 | B列（第2列） |
| C | 金额 | 税后实发工资 | W列（第23列） |
| D | 开户行 | 银行名称 | E列（第5列） |
| E | 开户地 | 留空或从开户行推断 | — |
| F | 汇款备注 | 留空 | — |

## 工资表（工资表 sheet）结构

```
第2行：表头（序号、姓名、身份证号、银行卡号、开户行...）
第3行：子表头（基本工资、绩效工资、通勤奖...）
第4-12行：员工数据（通常9人）
第13行：合计行
```

### 关键列说明

| 列号 | 列字母 | 字段 | 备注 |
|------|--------|------|------|
| B | 2 | 姓名 | 用于匹配 |
| D | 4 | 银行卡号 | 必须脱敏/格式化 |
| E | 5 | 开户行 | 含支行名称 |
| W | 23 | 税后实发 | `税后实发` 列 = `应税工资 - 个税` |
| X | 24 | 人员状态 | 空=在职，其他=离职/停发 |

### 税后实发计算逻辑
```
税前合计(S列) → 应税工资(U列) → 个税(V列) → 税后实发(W列)
税后实发 = 应税工资 - 个税
```

## 银行模板结构（AgencyPayment）

```
第1行：标题行（高度47.25，含logo图）
第2行：表头（*账号、*户名、*金额、开户行、开户地、汇款备注）
第3行起：数据行（最多13行空白格）
```

### 保留内容（不得修改）
- 第1行：标题 logo/图片
- 第2行：表头格式和文字
- D列：下拉数据验证（23家银行列表）
- 单元格样式（边框、字体、颜色、对齐）
- 绘图对象（drawing）

## 脚本用法

```bash
python3 import_salary.py <工资文件.xlsx> <银行模板.xlsx> <输出文件.xlsx>
```
