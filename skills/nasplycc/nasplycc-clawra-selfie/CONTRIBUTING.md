# Contributing

感谢你对 `clawra-selfie` 感兴趣。

这个项目当前主要围绕几个方向演进：

- OpenClaw 下的自拍 / 状态图工作流
- 更稳定的人设与视觉一致性
- 更清晰的安装、配置、分发体验
- 为后续更强后端（reference-image editing / ComfyUI / LoRA）预留升级空间

## 欢迎贡献的内容

- README / 文档改进
- 安装流程优化
- 新的示例场景与演示素材
- Prompt 工程改进
- 更稳的路径配置与 workspace 适配
- 错误处理与调试信息优化
- 与 OpenClaw / ClawHub 更标准的集成方式

## 提交建议

### 1. 先开 issue（如果改动较大）
如果你要做的是下面这些内容，建议先开 issue 讨论：

- 更换默认后端
- 大幅调整人物设定
- 改动 skill 目录结构
- 引入新的依赖或发布方式

### 2. 保持改动小而清晰
尽量避免一个 PR 同时做太多无关改动。

更推荐：
- 一个 PR 修一个问题
- 一个 PR 做一类文档改进
- 一个 PR 引入一个明确的小功能

### 3. 说明你改了什么
提交 PR 时，建议写清楚：

- 改动目的
- 改了哪些文件
- 是否影响安装 / 使用方式
- 是否需要更新 README / examples / SKILL.md

## 开发注意事项

- 尽量不要把敏感 token、私人参考图、工作区隐私数据提交进仓库
- `generated/`、个人数据集、临时调试文件不应进入公开仓库
- 如果涉及真实人物相似度，请保持克制，避免把项目导向对真人高度模仿的方向
- 新增文档时，优先保持中文 README 的一致风格

## 提交前检查

建议至少确认：

- README 是否需要同步更新
- SKILL.md 是否仍与实际能力一致
- 安装命令是否还能跑通
- ClawHub 安装说明是否仍然正确
- shell 脚本是否有基本语法检查

## 许可证

提交代码即默认同意项目使用当前仓库的开源许可证进行分发。
