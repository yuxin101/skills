# Security Policy

感谢你帮助改进 `clawra-selfie` 的安全性。

## 安全边界

这个项目主要是一个 OpenClaw skill，涉及：

- 本地 shell 脚本执行
- 第三方图像生成后端调用
- 参考图与工作区路径读取
- 与消息发送链路的集成

因此，最值得关注的安全点包括：

- token / API key 泄露
- 路径处理不当
- 调试日志误暴露敏感信息
- 将私人参考图、聊天数据、工作区文件误提交到公开仓库
- 对外部命令或下载链路缺少边界控制

## 报告方式

如果你发现的是安全问题，而不是普通 bug，请不要直接公开贴出可利用细节。

建议先通过以下方式私下联系维护者：

- GitHub issue 中仅做最小描述，并说明希望私下沟通
- 或通过仓库 owner 资料中的可用联系渠道联系

## 不建议公开披露的内容

请不要在公开 issue / PR 中直接贴出：

- 有效 token / API key
- 完整私人参考图数据
- 含有隐私路径、用户标识、聊天内容的日志
- 可直接复现利用的敏感 payload

## 使用者自我保护建议

如果你部署这个 skill：

- 使用独立的 Hugging Face token
- 不要把私人参考图和训练候选集提交到公开仓库
- 检查 `.gitignore` 是否覆盖生成物、数据集、临时文件
- 在公开演示图前，先确认图中不包含隐私信息
- 对 `curl | bash` 安装方式保持基本审查习惯

## 项目当前安全定位

这个项目当前更偏实验性 / 工作流型 skill，而不是经过完整安全审计的生产级系统。

如果你打算把它部署到更正式的环境里，建议额外做好：

- token 隔离
- 运行环境隔离
- 日志脱敏
- 外部 API 配额与错误监控
