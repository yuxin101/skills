# 中国个人信息保护法（PIPL）合规Skill

## ⚠️ 安全声明

### 功能范围和安全限制
1. **检查范围限定**：本Skill仅提供合规性检查、风险评估和文档生成指导
2. **数据安全**：所有脚本不收集、存储或传输用户敏感数据
3. **文件操作**：仅限于读取参考文档和写入合规报告/文档
4. **权限最小化**：运行脚本不需要特殊系统权限

### 使用注意事项
1. **测试验证**：建议先在测试环境验证功能，再应用于生产环境
2. **授权确认**：所有文件写入操作需要用户明确授权
3. **合规责任**：本Skill提供指导和工具，不替代专业法律意见
4. **数据保护**：生成的报告可能包含业务信息，请妥善保管

## 快速开始

### 1. 使用场景识别
**典型场景**：
- 新产品/新功能上线前的合规检查

## 核心功能模块

### 1. PIPL法规库
**文件**：`references/pipl-law.md`
- 核心法规条款摘要
- 监管要求重点提示
- 合规实践指导

### 2. 合规检查工具
**脚本**：`scripts/pipl-check.py`

#### 主要检查项：
- **告知义务**：是否充分告知收集目的、方式等
- **同意有效性**：是否获得有效同意（特别是单独同意）
- **数据最小化**：是否仅收集必要信息
- **跨境传输**：是否符合传输要求
- **安全保障**：是否采取必要安全措施
- **用户权利**：是否保障用户各项权利

#### 使用方式：
```bash
# 基础检查
python3 scripts/pipl-check.py --scenario "用户注册"

# 详细报告
python3 scripts/pipl-check.py --scenario "营销活动" --detailed

# 输出JSON格式
python3 scripts/pipl-check.py --scenario "数据分析" --format json
```

### 3. 风险评估框架
**脚本**：`scripts/risk-assessment.py`

#### 风险评估维度：
1. **数据敏感度**：信息类型和敏感程度
2. **处理规模**：数据量和覆盖范围
3. **处理目的**：必要性和正当性
4. **第三方参与**：共享和委托处理
5. **安全保障**：技术和管理措施

#### 风险等级：
- 🔴 **高风险**：需要立即整改
- 🟡 **中风险**：需要限期改进
- 🟢 **低风险**：建议持续优化

### 4. 文档生成工具
**脚本**：`scripts/document-generator.py`

#### 主要模板类型：
1. **隐私政策模板**：符合中国PIPL要求的完整模板
2. **用户同意书模板**：告知同意和单独同意专项模板
3. **数据处理协议模板**：委托处理、共同处理协议模板

#### 使用方式：
```bash
# 生成隐私政策
python3 scripts/document-generator.py --type privacy-policy

# 生成用户同意书
python3 scripts/document-generator.py --type consent-form

# 指定公司信息
python3 scripts/document-generator.py --type privacy-policy --company "公司名称"
```
