Agent认知：OS与Application

垂类不应该做Agent，应该做Agent上面的应用。从Prompt到Script到Application，是逐步把复杂度搬到Agent外部的过程。

A-01
三层光谱
01
Prompt 指令
给Agent点拨方向，但活还是Agent干，带宽不减。文本可复制，壁垒为零。
02
Script 工具
外部代劳返回结果，带宽降低。但没有外部状态，逻辑可复现，壁垒低。
03
Application
外部代劳＋持久状态＋基础设施，带宽和容量都大幅降低。不可复制。
B-05
Comparison
维度 Prompt Script Application
壁垒 零 低 高
带宽释放 无 部分 大幅
可复制性 一键 可复现 不可
价值天花板 卖copy 卖copy 卖服务
C-12
Analysis

认知共生：Application让OS更聪明

好的Application把领域推理从Agent注意力里搬走，干扰消失，其他任务更准，于是更精准地调用更多Application，飞轮转起来。

2
物理约束：容量 + 带宽
3
不可复制：状态·设施·规模

飞轮效应

垂类做OS的三个致命原因

  • 战场错位——Agent战场是推理能力和编排效率，垂类的领域深度在OS战场上一点用都没有
  • 市场收敛——OS层天然赢家通吃，PC两个、移动两个，Agent不会有几十个赢家
  • 机会成本——花三年正面硬刚，本可以在自己领域建一座别人进不来的根据地
