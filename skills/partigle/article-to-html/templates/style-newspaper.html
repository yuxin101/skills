Agent认知：OS与Application
"垂类不应做Agent，应做Agent上面的应用"
A-01
头条 · LEAD STORY

垂类做OS是找死，战场完全不同

Agent是一种交互范式，就像手机是一种交互范式。你不会为了购物专门买一部手机。垂类不应该尝试做Agent，应该做Agent上面的应用。
Agent的战场是推理能力、编排效率、交互体验。垂类的优势是领域深度、业务理解、行业数据——这些优势在OS战场上一点用都没有。你拿领域知识去跟OpenAI、Anthropic比推理能力，是拿刀去打坦克。而且OS层的市场结构天然收敛：PC时代活下来Windows和Mac，移动时代活下来iOS和Android。Agent OS也不会有几十个赢家。
B-01
物理约束 · CONSTRAINTS
01
上下文容量
Context是有限的容器，塞得越多性能越差。那些塞不进context的领域知识和用户历史，需要Application在外面管着。
02
注意力带宽
左手画圆、右手画方，合在一起两个都变形。Agent同时做法律推理、追踪意图、规划行动——每件事的质量都下降。注意力是零和的。
C-01
三层光谱 · SPECTRUM
壁垒：零

指令 Prompt

给Agent一个点拨方向，但推理工作仍然是Agent自己干的，带宽不减。而且prompt是文本，文本可以被复制。

壁垒：低

工具 Script

封装好的外部逻辑，从"指路"变成了"代劳"，带宽降低。但没有外部状态，别人看懂思路就能重写一个。

壁垒：高

服务 Application

外部代劳 + 持久状态 + 基础设施。Skill背后有领域状态、基础设施成本、规模经济，就不再是Skill，是Agent-native Application。

D-01
壁垒对比 · BARRIER MATRIX
维度 Prompt Script Application
执行方式 Agent自己推理 外部代劳 外部代劳+持久状态
带宽消耗 不减 降低 大幅降低
可复制性 文本即复制 逻辑可复现 不可复制
商业天花板 卖copy 卖copy 认知共生飞轮
E-01 核心洞察
好的Application让Agent OS更聪明，更聪明的OS更精准地调用Application——飞轮转起来了。这是认知共生：一个通用智能和一个专精智能通过接口耦合，比各自单独运行都更强。The best context is no context. Agent越轻量，表现越好。
