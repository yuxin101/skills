# 工作流程图

## 单章生产流程

```
用户输入
    │
    ▼
┌─────────────┐
│  任务分析   │ ◄── skill 判断
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────┐
│  是否需要规划？                      │
│  - 模糊需求 → planner               │
│  - 明确需求 → 跳过                  │
└─────────────────────────────────────┘
       │
       ▼
┌─────────────┐
│   writer    │ ◄── 写初稿
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   checker   │ ◄── 审稿
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────┐
│  是否通过？                          │
│  - 通过 → 结束                      │
│  - 不通过 → 返修（最多2次）         │
└─────────────────────────────────────┘
       │
       ▼
┌─────────────┐
│   writer    │ ◄── 返修
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   checker   │ ◄── 复审
└─────────────┘
```

## 多章协作流程

```
用户：写第10-15章
    │
    ▼
┌─────────────┐
│   manager   │ ◄── 统筹规划
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   planner   │ ◄── 设计6章结构
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────┐
│  循环：第10章 → 第11章 → ... → 第15章 │
│  每章：writer → checker → 通过/返修   │
└─────────────────────────────────────┘
       │
       ▼
┌─────────────┐
│   manager   │ ◄── 汇总结果
└──────┬──────┘
       │
       ▼
    交付用户
```

## 返修流程

```
checker 输出审查报告
        │
        ▼
┌─────────────────┐
│  返修次数 < 2？  │
└────────┬────────┘
         │
    是 ──┼──► writer 返修 ──► checker 复审
         │
    否 ──┴──► 输出最终报告 ──► 提示用户介入
```

## 角色协作关系

```
        ┌─────────┐
        │   用户   │
        └────┬────┘
             │
    ┌────────┼────────┐
    │        │        │
    ▼        ▼        ▼
┌───────┐ ┌───────┐ ┌───────┐
│planner│ │ writer│ │checker│
└───┬───┘ └───┬───┘ └───┬───┘
    │         │         │
    └─────────┼─────────┘
              │
              ▼
        ┌─────────┐
        │ manager │ ◄── 统筹（按需启用）
        └─────────┘
```
