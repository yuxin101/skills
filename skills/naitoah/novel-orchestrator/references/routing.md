# 默认路由规则

## 直接路由

### writer

当用户明确要求以下任务时，默认路由给 `writer`：

- 写一章
- 续写一章
- 改写一章
- 重写一章
- 扩写正文
- 把已有章节写顺、写完整、写得更好看

### planner

当用户明确要求以下任务时，默认路由给 `planner`：

- 分卷规划
- 大纲设计
- 主线梳理
- 章节拆解
- 下一章写什么
- 某段剧情怎么推进
- 某个桥段怎么落到章节层面

### checker

当用户明确要求以下任务时，默认路由给 `checker`：

- 审这章行不行
- 看这章能不能发
- 判断问题在哪
- 给修改意见
- 判断是否需要返修
- 判断当前版本是否通过

### manager

当用户明确要求以下任务时，优先启用 `manager`：

- 帮我统筹一下
- 你来安排流程
- 汇总这几方意见
- 现在到底该怎么推进
- 这轮任务怎么分工
- 现在该进入下一步了吗

## 升级为协作流程

如果用户请求同时包含以下任意两个或以上阶段，不直接只路由给单个角色，而是由 skill 判断是否升级为协作流程：

- 规划
- 写作
- 审查
- 返修
- 验收

## 强制升级条件

遇到以下情况时，直接升级为协作流程：

- 用户明确要求最终可用、可发、可交付的结果。
- 用户明确要求重写、大修、救文或整体拉升已有内容。
- 用户明确要求最终审定、最终结论或是否通过的判断。
- 用户要求把模糊想法直接落成章节成品。
- 用户明确要求由 skill 统筹整个过程。
- 用户要求连续写多章、批量产出若干章，或要求完成到指定章节数量为止；这类任务默认进入完整闭环，持续推进到达到用户要求的数量或用户中途调整目标为止。
