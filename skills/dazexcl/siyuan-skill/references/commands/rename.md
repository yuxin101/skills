# 重命名文档命令

重命名思源笔记文档。
## 命令格式
```bash
siyuan rename <docId> <title> [--force]
```

## 参数说明
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `<docId>` | string | ✅ | 文档 ID（位置参数） |
| `<title>` | string | ✅ | 新标题（位置参数） |
| `--force` | flag | ❌ | 强制重命名（忽略重名检测） |

## 使用示例
### 基本用法
```bash
# 重命名文档
siyuan rename <docId> "新标题"

# 完整示例
siyuan rename 20260304051123-doaxgi4 "更新后的标题"

# 强制重命名（忽略重名检测)
siyuan rename <docId> "已存在的标题" --force
```

## 返回格式
```json
{
  "success": true,
  "data": {
    "id": "20260227231831-yq1lxq2",
    "title": "新标题",
    "renamed": true,
    "notebookId": "20260227231831-yq1lxq2"
  },
  "message": "文档重命名成功",
  "timestamp": 1646389200000
}
```
## 注意事项
1. **文档ID格式**：文档 ID 格式为 14 位数字 + 短横线 + 7 位字母数字
2. **标题长度**：标题长度建议不超过 512 字符
3. **特殊字符**：标题中避免使用特殊字符，如 `/`、`\`、`:` 等
4. **权限限制**：需要相应的权限才能重命名文档
5. **重名检测**：如果同级已存在同名文档，会返回错误

## 重名检测
重命名文档前会自动检测同一目录下是否存在同名文档：
- **检测范围**：当前文档所在目录下的所有同级文档
- **排除自身**：重命名时排除当前文档自身（重命名为相同名称不会触发冲突）
- **冲突处理**：返回错误信息
- **强制重命名**：使用 `--force` 参数跳过重名检测

```bash
# 检测到重名时的返回
# {
#   "success": false,
#   "error": "文档名称冲突",
#   "message": "在目标位置已存在标题为"xxx"的文档（ID: xxx），请使用 --force 参数强制重命名"
# }

# 重命名为相同名称（不触发冲突）
siyuan rename <docId> "当前标题"

# 重命名为已存在的其他名称（触发冲突）
siyuan rename <docId> "已存在的标题"
# 使用 --force 强制重命名
siyuan rename <docId> "已存在的标题" --force
```
