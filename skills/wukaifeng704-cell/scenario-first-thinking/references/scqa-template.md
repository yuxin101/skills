# SCQA专项写作模板

> 针对场景3（表达/说服）的结构化输出模板

---

## SCQA黄金公式

```
S（情境）：[大家都在做的事]
C（冲突）：[但是有一个问题]
Q（问题）：[如何解决这个矛盾？]
A（答案）：[我们的方案/方法]
```

**核心原则**：S和C要让对方说"对！"，A要让对方说"原来如此"

---

## 场景3完整执行模板

### Step 1：先确定核心观点

在写SCQA之前，先问自己：**"读者读完最应该记住的一句话是什么？"**

这句话就是A的核心。

### Step 2：构建S（情境）

**公式**：`[描述一个普遍现象] + [具体数字或案例]`

**要点**：
- 要有共鸣，大多数人都经历过
- 要有数据或细节，不能太泛
- 长度：1-2句话

**示例**：
```
S（情境）：2025年开始，AI工具爆发式增长。ChatGPT、Midjourney、Claude…
职场人每天被"AI焦虑"包围，打开公众号全是"AI取代你"的文章。
```

### Step 3：构建C（冲突）

**公式**：`[但是] + [一个让人皱眉的问题]`

**要点**：
- 不能和S同一方向，要有转折
- 要戳到痛点，不是泛泛的痛
- 长度：1-2句话

**示例**：
```
C（冲突）：但真正上手时，教程要么太技术（代码+配置），
要么太浅（"只要会说话就能用"）。
在"会用"和"用好"之间，隔着十万八千里。
```

### Step 4：构建Q（问题）

**公式**：`[如何] + [一个具体的挑战]`

**要点**：
- 问句要具体，不是开放性问题
- 读者要能对号入座
- 长度：1句话

**示例**：
```
Q（问题）：有没有一条路，能让普通人绕过"AI科普"直接上手，
从"听说过AI"变成"真正用AI解决工作问题"？
```

### Step 5：构建A（答案）

**公式**：`[核心观点] + [为什么有效] + [一个具体行动]`

**要点**：
- 核心观点一句话说清楚
- 为什么有效要短，给读者信心
- 具体行动要可操作，读者能立刻做
- 长度：2-4句话

**示例**：
```
A（答案）：这条路叫"任务驱动学习法"——
不学概念，直接用AI做一个你真实工作中的任务。
做完了，AI就学会了。不用背命令，不用懂原理。
比如：明天你要写一份周报，直接让AI帮你写，
然后对比"AI写的"和"你写的"，哪里不一样，为什么？
这就是最真实的AI学习。
```

## SCQA变体：长文版

用于公众号文章、深度提案

```
【开篇S】
描述一个让目标读者产生共鸣的场景

【问题C】
描述这个场景下读者面临的核心困境

【转折Q】
抛出核心问题，这个问题必须是读者的真实疑问

【答案A1】
给出第一层答案（最容易理解的）

【答案A2】
深入一层（为什么这个答案有效）

【行动召唤】
读者现在可以做什么？（越具体越好）
```

## SCQA变体：短文版

用于朋友圈文案、微博、群发消息

```
S+C合并（1-2句）：
[一个现象] + [一个痛点]

Q+A合并（1-2句）：
[一个问题] → [一个解决方案]
```

**示例**：
```
很多人在问"AI能帮职场人做什么"。
真相是：不是AI不行，是你问问题的方式不对。
用"角色+任务+要求"的结构问AI，效率立刻翻倍。
```

---

## SCQA常见错误检查清单

写完之后，对着检查：

- [ ] S是否让人想说"对！我就是这样"
- [ ] C是否戳到真实的痛点，而不是泛泛的焦虑
- [ ] Q是否具体，收敛到一个核心问题
- [ ] A是否用一句话能说清楚核心观点
- [ ] A是否有具体的行动建议
- [ ] 全文是否超过3个核心观点（超过就拆）
