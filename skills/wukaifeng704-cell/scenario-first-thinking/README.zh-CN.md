# Scenario-First Thinking

> 让AI在正确的时间，用正确的工具，思考正确的问题。

**作者**: WH laowu
**版本**: 1.0.0
**定位**: 任务路由层思维框架

---

## 一句话说明

遇到任务时，先判断场景（6选1），再按优先级使用8种思维工具（2-3个），最后验证结论。

---

## 快速开始

### 安装

```bash
npx clawhub install scenario-first-thinking
```

### 触发

激活时机：任何需要思考和决策的任务输入。

### 使用

1. **场景路由**：根据任务类型自动匹配六场景之一
2. **工具选择**：按场景优先级使用2-3个工具
3. **效果验证**：检查结论是否可解释/可举例/无漏洞

---

## 六场景速查

| # | 场景 | 判断词 | 核心工具 |
|---|------|--------|---------|
| 1 | 方向/决策 | 创业/战略/转型 | 第一性原理 |
| 2 | 学习/内化 | 学不会/备考 | 费曼学习法 |
| 3 | 表达/说服 | 写/说/汇报 | SCQA |
| 4 | 效率/排序 | 太忙/理不清 | 四象限 |
| 5 | 危机/救火 | 紧急/要挂了 | 逆向思维 |
| 6 | 创意/探索 | 新方案/头脑风暴 | 费曼+第一性原理 |

---

## 典型使用案例

**案例1**：用户说"最近太忙了，任务理不清"

```
场景判断 → 场景4（效率/排序）
工具选择 → 四象限优先 + SCQA次之
执行 → 任务分入四个象限，输出优先级清单
验证 → 读者能否立刻知道先做哪个？
```

**案例2**：用户说"帮我写一篇公众号文章"

```
场景判断 → 场景3（表达/说服）
工具选择 → SCQA优先 + SMART次之
执行 → S共鸣建立 → C矛盾引入 → Q核心问题 → A解决方案
验证 → 读者能否产生共鸣？结论是否可操作？
```

## 设计原则

- **轻量**：SKILL.md < 500行，主逻辑极简
- **闭环**：场景判断 → 工具选择 → 效果验证，三步完整
- **高容错**：判断模糊时默认场景4，工具失败时降级为单工具
