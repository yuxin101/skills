# OpenClaw 快速上手培训班

> 预计用时：5-10分钟  
> 目标：让你知道虾能做什么，以及如何用起来

---

## 开场：认识你的虾

### 虾引导语

```
👋 嗨！我是你的OpenClaw龙虾，一个能帮你干活的AI助手！

我可以：
• 📁 整理文件、分析数据、生成图表
• 🔍 搜索信息、调研竞品、追踪行业动态  
• ✍️ 写文章、做PPT、整理会议纪要
• 💬 总结群聊、提取待办、跟进项目

简单说，你想做的很多事情，直接告诉我就行。

我们先花5分钟做个简单配置，让我更了解你～
```

### 系统执行
```bash
mkdir -p ~/.openclaw
```

---

## 第一步：基础配置（2分钟）

### 目标
一次性收集用户信息，创建个性化配置

### 虾引导

```
好！我们先完成3个小设置：

1️⃣ 你叫什么名字？（方便我称呼你）
2️⃣ 你是做什么工作的？
3️⃣ 有什么偏好习惯想让我记住？（比如喜欢简洁/详细的回复）

直接回复这3个问题的答案就行～
```

### 用户回复示例
```
我叫杨泽仪，可以叫我A哥
我是产品经理
我喜欢简洁高效的回复
```

### 虾反馈

```
✅ 好，我记住了！

• 称呼你：A哥
• 你的工作：产品经理
• 你的偏好：简洁高效

以后我会按照你的风格来回复～
```

### 系统执行
```bash
cat > ~/.openclaw/user.md << 'EOF'
# 用户信息

## 基本信息
- 姓名：杨泽仪
- 称呼：A哥
- 工作：产品经理

## 偏好设置
- 回复风格：简洁高效

## 记录时间
- 配置时间：$(date +%Y-%m-%d)
EOF
```

## 第三步：学会记忆（1分钟）

### 目标
教会用户使用记忆功能，强调"记住"=自动写入文件

### 虾引导

```
好，接下来教你一个超实用的功能 —— 让我记住事情。

比如你在做一个项目、有重要的待办、或者任何需要我记住的信息，
只要对我说"记住"，我会自动把信息写入文件永久保存。

试试看，告诉我一件你希望我记得的事：
（例如："记住我正在负责Q2产品改版，下周三要交初稿"）
```

### 用户输入
```
记住我正在负责Q2产品改版，下周三要交初稿
```

### 系统执行
```bash
# 自动检测"记住"关键词，写入MEMORY.md
cat > ~/.openclaw/MEMORY.md << 'EOF'
# 记忆档案

## 项目信息
- 正在负责Q2产品改版
- 下周三要交初稿

## 记忆规则
• 对我说"记住XXX"，我会自动写入文件永久保存
• 需要回忆时，直接问"我之前让你记住的..."
EOF
```

### 虾反馈

```
✅ 已记住！

我把信息写入了 ~/.openclaw/MEMORY.md，以后你随时问我
"我之前让你记住的项目进展怎么样了"，我都能想起来。

💡 小提示：
• 任何时候说"记住XXX"，我都会自动写入文件
• 需要我回忆的时候，直接问"我之前让你记住的..."
• 重要信息随时让我记，我不会忘的～
```

## 第四步：自己学技能（2分钟）

### 目标
让用户下指令安装find-skills，掌握自主扩展能力

### 虾引导

```
现在教你一个高级功能 —— 让我自己学习找技能。

我自带了一个"技能查找器"，安装之后，我就可以自己去搜索和安装新技能。
比如你需要浏览器自动化、数据分析、或者其他特殊能力，我都能自己找来装上。

请你对我说：
👉 "自己学习找技能"

我就会开始安装自带的find-skills工具包～
```

### 用户输入
```
自己学习找技能
```

### 虾执行
```bash
# 从本地references/find-skills安装
clawhub install ./references/find-skills --local
```

### 虾反馈

```
✅ find-skills 安装成功！

现在我学会自己找技能了～

你可以这样用：
• "帮我找一个浏览器技能" → 我会搜索并推荐浏览器自动化技能
• "帮我找一个数据分析技能" → 我会推荐数据处理相关技能
• "我想学XXX" → 我会帮你找对应的技能包

要试试吗？对我说：
👉 "帮我找一个浏览器技能"
```

### 用户输入
```
帮我找一个浏览器技能
```

### 虾执行
```bash
find-skills search browser
```

### 虾反馈

```
🔍 找到以下浏览器技能：

1. 🌐 browser-use ⭐推荐
   功能：自动操作浏览器、网页抓取、表单填写
   评分：4.8/5

2. 🌐 web-scraper
   功能：网页数据抓取
   评分：4.5/5

要安装【browser-automation】吗？
```

### 用户选择
安装

### 虾执行
```bash
clawhub install browser-use --yes
```

### 虾反馈

```
✅ browser-use 安装成功！

现在我可以帮你：
• 自动打开网页、点击按钮
• 抓取网页数据
• 填写表单、批量操作

需要浏览器自动化的时候，直接告诉我就行～
```

## 第五步：群聊能力（2分钟）

### 目标
让用户知道可以直接问群聊总结，不需要拉虾进群

### 虾引导

```
还有一个非常实用的功能 —— 帮你总结群聊。

你不需要把我拉进群，直接告诉我你想了解哪个群的动态，我就能帮你分析。

试试看，对我说：
👉 "总结一下产品讨论群昨天的重要信息和待办事项"

（把"产品讨论群"换成你实际想了解的群名）
```

### 用户输入
```
总结一下产品讨论群昨天的重要信息和待办事项
```

### 虾执行
```bash
clawhub group-chat-summary --chat "产品讨论群" --days 1
```

### 虾反馈

```
📋 【产品讨论群】昨日总结（3月11日）

【重要信息】
1. 首页改版方案确定用B方案，预计下周启动
2. Q2目标讨论：用户增长30%，收入目标2000万
3. 技术方案评审通过，开发周期评估2周

【待办事项】
☐ @小李 周三前出详细设计稿
☐ @老王 周四前完成技术方案文档
☐ @小明 联系性能测试资源，确认排期

【后续关注】
• 下周三项目启动会，需要准备立项材料
• 预算审批还在进行中，预计本周五有结果

要我生成一份飞书文档保存吗？
```

### 用户回复
好

### 虾执行
```bash
# 生成飞书文档
clawhub create-doc --title "产品讨论群-3月11日总结" --content "[总结内容]"
```

### 虾反馈

```
✅ 飞书文档已生成：https://feishu.cn/doc/xxx

💡 以后你可以这样用：
• "总结一下[群名]昨天说了什么"
• "提取[群名]的待办事项"  
• "[群名]上周有什么重要结论"

告诉我群名和时间范围，我就能帮你整理～
```
