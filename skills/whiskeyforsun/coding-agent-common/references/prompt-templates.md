# 任务提示词模板

## 功能开发

```
在 [模块名] 中实现 [功能描述]：

功能需求：
1. [需求1]
2. [需求2]
3. [需求3]

技术要求：
- 使用 [框架/库]
- 遵循 [规范]
- 添加 [测试/文档]

验收标准：
- [标准1]
- [标准2]

完成后运行测试验证。
```

## Bug 修复

```
修复 [问题描述]：

问题现象：
[描述具体表现]

复现步骤：
1. [步骤1]
2. [步骤2]

期望行为：
[描述期望结果]

相关信息：
- 错误日志：[如有]
- 相关文件：[如有]
- 最近变更：[如有]

请先分析根本原因，再实施修复。
```

## 代码审查

```
审查 [范围] 的代码：

审查重点：
- [ ] 代码质量
- [ ] 安全问题
- [ ] 性能问题
- [ ] 测试覆盖
- [ ] 文档完整

输出格式：
1. 发现的问题（按严重程度排序）
2. 改进建议
3. 整体评价
```

## 重构

```
重构 [模块/功能]：

重构目标：
- [目标1]
- [目标2]

约束条件：
- 保持 API 兼容
- 不改变外部行为
- 现有测试必须通过

重构范围：
- [文件1]
- [文件2]

请先说明重构计划，确认后再执行。
```

## 文档生成

```
为 [模块/功能] 生成文档：

文档类型：[API文档/README/架构文档]

要求：
- 包含使用示例
- 说明关键概念
- 列出配置选项
- 包含常见问题

输出格式：Markdown
```

## 测试编写

```
为 [模块/功能] 编写测试：

测试类型：
- [ ] 单元测试
- [ ] 集成测试
- [ ] E2E 测试

覆盖范围：
- 正常场景
- 边界条件
- 异常处理

测试框架：[Jest/Vitest/PyTest/etc.]

要求：
- 遵循 AAA 模式（Arrange-Act-Assert）
- 测试命名清晰
- 每个测试只验证一个行为
```

## 完成通知模板

对于长时间任务，在提示词末尾添加：

```
完成后运行以下命令通知我：
openclaw system event --text "完成: [任务摘要]" --mode now
```
