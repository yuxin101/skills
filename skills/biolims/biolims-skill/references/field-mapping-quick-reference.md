# Field Mapping Quick Reference

## Important Principles

1. **Strictly use English display names** - Do not translate to Chinese
2. **Patient name is a single field `name`** - No more firstName/lastName distinction
3. **One-to-one mapping** - Each API field has a unique display name
4. **Only display fields with values** - Do not display null or empty strings
5. **Field names must be accurate when creating orders** - Using wrong field names will cause data loss

---

## ⚠️ Create/Update Order Field Names (Extremely Important)

Field names in the JSON structure must be strictly accurate when creating or updating orders:

### Correct Field Names

```json
{
  "order": { ... },              ✅ Order main table
  "sampleOrderItem": [ ... ],    ✅ Sample information (must be this name!)
  "crmConsumeRmarkets": [ ... ]  ✅ Fee information (must be this name!)
}
```

### Wrong Field Names (Will Cause Data Loss)

```json
{
  "order": { ... },
  "samples": [ ... ],           ❌ Wrong! Sample data will not be saved
  "sampleItems": [ ... ],       ❌ Wrong! Sample data will not be saved
  "items": [ ... ],             ❌ Wrong! Sample data will not be saved
  "fees": [ ... ],              ❌ Wrong! Fee data will not be saved
  "payments": [ ... ],          ❌ Wrong! Fee data will not be saved
  "charges": [ ... ]            ❌ Wrong! Fee data will not be saved
}
```

### ⚠️ Patient Name Field (Extremely Important)

```json
{
  "order": {
    "name": "John Smith"          ✅ Correct! Patient name is a single field
  }
}
```

```json
{
  "order": {
    "firstName": "John",   ❌ Wrong! This system does not use firstName
    "lastName": "Smith"    ❌ Wrong! This system does not use lastName
  }
}
```

**Key reminders**:
- `sampleOrderItem` - Field name for sample information (cannot be simplified or renamed)
- `crmConsumeRmarkets` - Field name for fee information (cannot be simplified or renamed)
- Using any other name will prevent that section of data from being saved to the system!

## Custom Fields

Custom fields require calling an API to get the mapping:

| flag Parameter | Description | Storage Location |
|---------------|-------------|-----------------|
| `104-mainTable` | Order main table custom field mapping | `order.customFieldValue` |
| `104-sampleTable` | Order sample sub-table custom field mapping | `sampleOrderItem[n].customFieldValue` |

```bash
# Get order main table custom field mapping
bash "$SCRIPT" get-custom-fields 104-mainTable

# Get order sample sub-table custom field mapping
bash "$SCRIPT" get-custom-fields 104-sampleTable
```

## Common Error Examples

### ❌ Wrong Approach

```
Name: Zhang San        ← Wrong: Translated to Chinese
firstName: John   ← Wrong: This system does not use firstName/lastName
lastName: Smith   ← Wrong: This system does not use firstName/lastName
```

### ✅ Correct Approach

```
Patient Name: Zhang San
Patient Name: John Smith
Gender: Female
```

## Required Fields Summary

**Required when creating order (within order object)**:
- `name` - Patient name
- `productId` / `productName` - Test items

**Additional required when submitting order**:
- `confirmerUsername` / `confirmerNickname` - Approver

**Required sample detail fields (each sampleOrderItem)**:
- `slideCode` - Barcode (must be unique, no Chinese characters)
- `sampleTypeId` / `sampleType` - Sample type
- `productId` / `productName` - Test items (also needed for each sample)
