# session-sync-detect 🔍

跨会话同步检测与询问技能

## 功能

当用户谈到当前 session 没有的事情时，自动检测并询问是否进行跨会话同步。

**✨ 智能检测 + 用户确认**
- 自动分析用户话语，判断是否涉及跨 session 内容
- 必须得到用户允许后才执行同步
- 所有操作透明可追溯

## 使用场景

### 自动触发询问

当检测到以下信号时，会询问用户：

**时间引用：**
- "昨天的那个事..." → 当前 session 没有昨天的上下文
- "上周说的会议" → 当前 session 没有相关记录

**平台引用：**
- "微信里我发的消息" → 当前是飞书
- "邮件里提到的" → 跨平台引用

**内容引用：**
- "蒙老师的报告时间你记得吧？" → 当前 session 没提到过
- "你查一下我说的" → 暗示需要查找

### 询问示例

```
🔍 检测到您可能在谈论其他 session 的内容：

"蒙老师的报告时间你记得吧？"

我可以在以下范围查找：
- ✅ 记忆文件 (memory/YYYY-MM-DD.md, MEMORY.md)
- ✅ 最近活跃的 session (过去 24 小时)

是否需要我帮您同步这些信息？
```

### 用户回复

- ✅ "是" / "好的" / "查一下" → 执行同步
- ❌ "不用" / "跳过" / "不是那个" → 不执行
- 🤔 "先看看记忆文件" → 只查 memory 文件

## 工作流程

### 1. 检测信号

分析用户话语中的跨 session 信号：
- 时间引用（昨天/上周/之前）
- 平台引用（微信/飞书/邮件）
- 内容引用（那个/你说的/我说的）
- 行为模式（帮我查/找找看/你记得吧）

### 2. 询问确认

向用户展示检测到的信号，询问是否执行同步

### 3. 执行检索

按优先级执行：
1. 先查 memory 文件 - 最快，跨 session 共享
2. 再查 session 历史 - 完整但较慢
3. 最后查 session 列表 - 元数据快速扫描

### 4. 展示发现

向用户展示找到的信息，包括来源和上下文

### 5. 同步记忆

询问用户是否将发现写入共享 memory 文件

## 安全设计

- ✅ **检测 + 询问 + 执行** - 必须得到用户确认
- ✅ **透明操作** - 所有文件读写都会向用户报告
- ✅ **隐私保护** - 敏感信息不写入共享 memory
- ✅ **最小权限** - 仅执行用户确认后的操作
- ✅ **可追溯** - 所有操作都有来源记录

## 技术细节

- 使用 `sessions_list` 获取 session 元数据
- 使用 `sessions_history` 读取完整对话
- 使用 `grep/find` 搜索 memory 文件
- 使用 `write/edit` 同步发现的信息

## 最佳实践

- **检测要敏感** - 宁可多问，不要漏掉
- **询问要清晰** - 告诉用户要查什么、在哪查
- **检索要高效** - 先查 memory，再查 session
- **展示要简洁** - 只展示与用户请求相关的内容
- **同步要确认** - 写入前必须得到用户允许
- **隐私要保护** - 敏感信息不写入共享 memory

## 注意事项

- ⚠️ 检测信号要准确，避免频繁询问打扰用户
- ⚠️ 每个 session 的对话历史是隔离的，只有 memory 文件是共享的
- ⚠️ 读取 session 历史会消耗 tokens，控制 limit 参数
- ⚠️ 不要将敏感信息（密码、密钥）写入共享 memory 文件
