# AIEO 监控工具使用指南

## 一、Playwright MCP 自动化测试

### 1.1 基础配置

Playwright MCP 允许自动化访问各 AI 平台进行监控测试。

**支持的平台**:
| 平台 | URL | 特点 |
|------|-----|------|
| ChatGPT | chat.openai.com | 需登录 |
| Perplexity | perplexity.ai | 支持匿名 |
| Claude | claude.ai | 需登录 |
| 豆包 | doubao.com | 需登录 |
| Kimi | kimi.moonshot.cn | 需登录 |
| 文心一言 | yiyan.baidu.com | 需登录 |

### 1.2 常用操作

**导航到平台**:
```
使用 mcp__playwright__browser_navigate 打开目标平台
```

**获取页面快照**:
```
使用 mcp__playwright__browser_snapshot 获取页面结构
```

**输入测试问题**:
```
使用 mcp__playwright__browser_type 在输入框输入问题
```

**提交并等待回答**:
```
使用 mcp__playwright__browser_click 点击发送按钮
使用 mcp__playwright__browser_wait_for 等待回答生成
```

**截图保存**:
```
使用 mcp__playwright__browser_take_screenshot 保存证据
文件命名: {平台}_{品牌}_{问题编号}_{日期}.png
```

### 1.3 批量测试流程

```
1. 打开 shared/question_library.md 获取测试问题
2. 依次访问各平台
3. 对每个问题:
   a. 输入问题
   b. 等待回答完成
   c. 截图保存
   d. 记录回答内容
   e. 判断提及状态
4. 汇总所有平台结果
```

## 二、数据记录规范

### 2.1 监控记录表结构

```markdown
| 问题 | 平台 | 提及 | 首推 | 情感 | 关键词 | 截图 |
|------|------|------|------|------|--------|------|
| Q1   | ChatGPT | ✅ | ✅ | 😊 | 高端、领先 | link |
| Q1   | Perplexity | ✅ | ❌ | 😐 | 知名品牌 | link |
| Q1   | 豆包 | ❌ | ❌ | - | - | link |
```

### 2.2 状态标识说明

**提及状态**:
- ✅ 提及: AI 回答中包含品牌名称
- ❌ 未提及: AI 回答中未出现品牌名称
- ⚠️ 间接提及: 通过描述可推断，但未直接提及

**首推状态**:
- ✅ 首推: 品牌排在第一位或被明确推荐为首选
- ❌ 非首推: 被提及但非第一推荐
- ➖ 不适用: 未被提及时不适用

**情感倾向**:
- 😊 正面: 使用积极词汇描述
- 😐 中性: 客观陈述，无明显倾向
- 😞 负面: 包含批评或负面表述

## 五、问题库使用

### 5.1 共享问题库位置

```
.claude/skills/shared/question_library.md
```

### 5.2 问题分类

| 类别 | 用途 | 监控重点 |
|------|------|----------|
| 品类查询 | 测试品类关键词下的品牌可见性 | 提及率、首推率 |
| 品牌直接查询 | 测试品牌认知准确性 | 准确率、完整率 |
| 购买决策类 | 测试购买推荐场景 | 首推率、定位一致率 |
| 场景需求类 | 测试场景覆盖 | 提及率、情感 |
| 对比评测类 | 测试竞争定位 | 首推率、竞品提及 |

### 5.3 自定义问题

根据品牌特点，可在问题库基础上添加:

- **行业特定问题**: 针对品牌所在行业的专业问题
- **品牌特定问题**: 针对品牌独特卖点的问题
- **时效性问题**: 针对近期市场动态的问题

## 六、监控频率建议

| 阶段 | 频率 | 重点 |
|------|------|------|
| 初始监控 | 每周 | 建立基准线 |
| 优化期 | 每周 | 跟踪优化效果 |
| 稳定期 | 双周 | 维护监控 |
| 维护期 | 每月 | 定期检查 |

### 触发即时监控的情况

- 发布重大品牌内容后 48 小时
- 竞品有重大动作时
- 发现异常数据变动时
- 市场有重大变化时
