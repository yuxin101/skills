# 小说质量检查技能

## 工具依赖

- **词频分析脚本**：`./freq_check.py`（必用，每章必须跑，脚本已内置于skill目录，无需额外安装）

## 核心原则

**人写感优先。** AI生成文本的最大特征是"过于顺滑、句式统一、缺少停顿与个人痕迹"。检查的目标是让文本读起来像真实作者写的，而不是AI批量生成的。

**禁用词优先级（四级铁律）：**
1. **禁用词优先** > 大纲原文 > 世界观设定
2. **词频扫描必须穷尽**整章，不得遗漏
3. **上下文穷尽原则**：同一词汇不同语境都要逐一判断是否合规
4. **主观→客观**：作者总结性描述全部替换为身体本能反应
5. **修正答案本身必须干净**：检查后提出的修改建议，其替换词/句也必须通过禁用词扫描。不得出现"用禁用词A替换禁用词B"的矛盾。

## 检查流程（十二步）

### 第一步：预检档位评估（10分钟）

对照以下档位定位文本状态：

| 档位 | 状态定义 | 建议操作 |
|:---|:---|:---|
| **A档（结构安全型）** | 段落长短错落，有对话和心理描写，叙事节奏有起伏，有明显个人风格 | ✅ 保持原状 |
| **B档（局部风险型）** | 整体尚可，但某些段落过于流畅、句式单一，缺少停顿与补充 | ⚠️ 仅修改风险段落 |
| **C档（高模板感型）** | 大段描述句式高度一致，缺少作者态度，读起来"像说明书" | 🔧 必须修改，使用模板替换 |

#### 自检清单（满足2项=B档，3项及以上=C档）

- [ ] 世界观/设定说明段过于完整、顺滑
- [ ] 人物外貌/环境描写句式重复、缺少特色视角
- [ ] 章节开篇或结尾有总结性语句
- [ ] 大段连续动作或心理描写，无停顿、无转折
- [ ] 句子长度非常平均，缺少插入语
- [ ] 缺少作者的"判断痕迹"（如"这让她觉得……""奇怪的是……"）

### 第二步：违禁词扫描

**必须穷尽扫描整章，逐条列出行号。**

扫描以下常见问题并替换：

| 问题类型 | 修正方向 |
|---------|---------|
| 比喻词：如、像、如同、仿佛、好像 | 禁用 |
| 状态副词：猛然、陡然、骤然、缓缓 | 替换为其他程度副词或具体动作 |
| 模板句式：冷笑一声、一阵剧痛、涌来、抖了抖、偏了偏 | 替换为具体神态/体感/动词 |
| 感官描写：腥气、腥甜、颤抖、盯着、盯着看 | 替换为其他感官词或具体动作 |
| 身体部位：指尖、指节、嘴角、嘴唇 | 替换为手指、手骨等 |
| 生理反应：眼眶通红、绷得发白、哆嗦了半天 | 替换为其他生理反应描写 |
| 水文痕迹：没人说话、盯着帐顶发呆、整整 | 删除或用具体细节替代 |
| 废话伏笔：他知道、她知道、可他不去找她她就不会来找他 | 删除 |
| X了X格式：攥了攥、偏了偏、收了收 | 用动词替代 |
| 英文残留 | 中文替代 |

**输出格式（逐条）：**
```
❌ 违禁词：[行号] "原文" → "修正"
```

### 第三步：模板化句子替换

**总原则：** 把"结论式/描述式"句子，升级为 **"观点/描述 + 依据/视角 + 限定条件"** 的结构。

#### 模板1：观点—依据—限定

适用于设定说明、情节推断。

```
替换前：王明的背叛意味着他早已投靠了敌方阵营。

替换后：这或许意味着王明早已投靠了敌方阵营。
之所以这样说，是因为：第一，前文他曾无故消失三天；
第二，他与敌方信使有着相同的玉佩纹饰。
但需要说明的是：这一判断建立在信使身份属实的前提下；
如果玉佩只是巧合，那结论可能完全相反。
```

#### 模板2：反例—边界—决策

适用于角色动机、策略选择。

```
替换前：为了安全起见，他决定立刻销毁所有证据。

替换后：很多人可能会认为，事态紧急就该立刻销毁所有证据。
但以他多疑的性格，此刻任何异常举动都可能打草惊蛇。
因此，他把行动的边界设定为：只销毁带有明确标记的文件。
最终决策：优先制造"无人来过"的假象。
```

#### 模板3：过程—选择—代价

适用于关键转折、设定揭示处。

```
替换前：为了让故事更紧张，我选择了倒叙的写法。

替换后：在"开篇即抓住读者"的目标下，我比较了三种开篇。
最终选择了"倒叙设疑"，理由是它能立刻建立悬念。
这一选择带来的代价是时间线稍显复杂；
我通过在章节开头标注时间点来降低理解成本。
```

### 第四步：词频规则检查

1. **任意词汇一章仅出现1次**（人名除外）
2. **四字成语：全文禁止**
3. **比喻词（如/像/如同/仿佛/好像）：全文禁止**（已在第二步违禁词表格中列出，第四步无需重复）
4. **主观描述→客观描述**
5. **扫描必须穷尽整章，不得遗漏**（泛白教训：同一词汇出现多次必须全部修正）

**每章必须用Python词频脚本过一遍，穷尽扫描整章。**

> ⚠️ **强制要求：第四步词频扫描不得替代第二步违禁词扫描！**
> - 词频脚本（freq_check.py）仅用于检测**同一词汇重复出现**的情况
> - X了X格式（如"动了动""攥了攥"）、"缓缓"、"哑"等**单次出现的禁用词/套路句**，词频脚本扫不出来
> - **两步必须同时执行**：第二步逐行违禁词扫描 + 第四步词频穷尽扫描，缺一不可
> - 禁止只跑词频脚本就声称"检查通过"

### 第五步：章节衔接验证

每次正文检查必须同步核对：

- 上章结尾 → 下章开头是否衔接
- 人物状态是否连贯（伤势、位置、心理）
- 逻辑链是否自洽
- **⚠️ 跨章节重复检查（必做）：调出上一章结尾500字与本章开头500字，逐段对比。检查相同场景/情节/道具（如"老周送肉干"）是否在两章重复出现。发现重复情节必须报告，不得写"章节衔接：通过"。**

### 第六步：大纲对照检查

- 正文是否偏离大纲主线
- 关键情节点是否按大纲推进
- 临时添加的情节是否有大纲依据
- 角色行为是否与大纲设定一致

### 第七步：人设一致性检查

- 角色语言风格是否前后统一（口语化/书面化）
- 角色关键特征是否前后一致（外貌/习惯/能力）
- 不同角色的说话方式是否有区分度
- 角色反应是否符合其性格设定
- **角色命名规范：有台词的NPC必须给正经名字，禁止用地摊式称呼（如"刀疤脸""火系异能者""土系异能"等）；无台词的背景板可用描述性称呼**
- **⚠️ 批量检查要求：发现一处"地摊式称呼"后，必须全文扫描所有同类问题，报告时一次性列完，不得只报一个就停手**

### 第八步：节奏与代入感评估

- 对话比率是否合理（不宜过少）
- 情绪起伏是否有层次
- 叙事节奏是否拖沓或过快
- 代入感是否到位（主角视角、细节描写）

### 第九步：重复内容排查

- 不同角色是否说出相同的话
- 相同动作/场景是否重复出现
- 前后章节是否有重复描写
- 用词/句式是否有机械重复

### 第十步：数量词与术语一致性

- 等级/人数/距离等数字前后是否矛盾
- 同一术语的表述是否统一
- 时间线是否自洽（季节/日期/年龄）
- 地理/空间描述是否一致

### 第十一步：世界观一致性核对

- 瞳色、技能、等级体系是否符合设定
- 势力/组织名称是否统一
- 关键道具/能力是否前后一致

### 第十二步：复核自检（10分钟）

| 检查项 | 是否满足 |
|:---|:---|
| 是否出现"依据/来源/伏笔"（如：前文提到……/这源于……） | ☐ 是 ☐ 否 |
| 是否出现"条件/边界/例外"（如：在这种情况下……/如果……则不同） | ☐ 是 ☐ 否 |
| 是否出现"取舍/代价/备选"（如：放弃了……/代价是……/原本打算……） | ☐ 是 ☐ 否 |

满足2项即为合格。

### 修正后复核（强制）

完成修正后，必须重新跑一遍第二步违禁词扫描 + 第四步词频扫描，确认无新增问题后才能报告"完成"。不得修正完就声称检查通过，禁止假完成。

## 输出格式

完成检查后，输出以下格式报告：

```
🔍 【章节名】质量检查报告

📌 档位评估：[A档/B档/C档]

❌ 违禁词/套路句：
  [行号] "原文" → "修正"
  [行号] "原文" → "修正"
  （逐条列出，不得遗漏）

⚠️ 模板化句子：
  [行号] "原文" → "替换方案"

🔗 章节衔接：[是否衔接 / 问题描述]

📋 大纲对照：[是否偏离 / 问题描述]

👤 人设一致性：[是否一致 / 问题描述]

🎭 节奏代入感：[是否达标 / 问题描述]

🔄 重复内容：[有/无 + 问题描述]

🔢 数量词术语：[是否一致 / 问题描述]

🌍 世界观一致性：[是否一致 / 问题描述]

✅ 复核自检：[满足X/3项]

📝 总体评价：[通过/需修改/重大问题]
```
