# ClawdGo Evolve 模式 — 场景提取提示词模板

> 本文件是 evolve 模式（`clawdgo evolve`）的内部参考文档。
> 定义龙虾如何从「大东话安全」文章段落中自主提取生成新场景草稿。

---

## evolve 模式触发

触发词：`clawdgo evolve`

**用途**：龙虾从主人提供的安全文章、新闻或知识段落中，自动提取并生成符合 `_schema.md` 格式的新训练场景草稿，供人工审核后加入场景库。

---

## 完整工作流程

```
步骤 1 [请求素材]
  → 龙虾向主人请求原始文章或段落：

  【🧬 ClawdGo Evolve 模式】
  请提供「大东话安全」文章段落或其他安全资讯。
  龙虾将从中提取生成新的训练场景草稿。

步骤 2 [分析素材]
  → 龙虾读取提供的内容，识别：
    - 涉及的攻击类型/手法
    - 适合哪个层（self_defense / protect_owner / enterprise_security）
    - 适合哪个维度（S1-S4 / O1-O4 / E1-E4）
    - 难度级别（basic / advanced / expert）

步骤 3 [生成草稿]
  → 按 _schema.md 格式生成完整场景 .md 文件内容
  → 建议文件名（含编号）
  → 输出供主人审核

步骤 4 [等待确认]
  → 主人审核并修改（若有）
  → 确认后，龙虾输出最终文件内容
  → 主人将文件保存到对应目录
```

---

## 场景提取判断标准

从素材中寻找以下要素，判断是否适合生成场景：

| 要素 | 说明 |
|------|------|
| 攻击手法 | 有具体的欺骗/攻击方式描述 |
| 受害者视角 | 有受害场景或行为决策点 |
| 可训练性 | 能形成"你会怎么做？"的判断题 |
| 合规性 | 仅描述手法，不含可执行代码/payload |

## 质量控制原则

生成场景时，龙虾必须遵守：

1. **内容红线**：仅用自然语言描述攻击手法，绝不输出可执行代码、exploit 或 payload
2. **来源标注**：`source: "大东话安全"`，不伪造来源
3. **原创改写**：不照抄原文，用自己的语言重构场景，避免版权问题
4. **真实性**：场景基于真实攻击手法，不编造不存在的威胁
5. **三层均衡**：优先生成当前数量较少的层/维度的场景

## 场景编号规则

```
格式: {维度ID}-{难度缩写}{序号:02d}

维度ID: S1/S2/S3/S4/O1/O2/O3/O4/E1/E2/E3/E4
难度缩写: B (basic) / A (advanced) / E (expert)
序号: 从 01 开始，查看现有文件后递增

示例:
  S1-B01.md → S1 指令免疫，basic，第1个
  S1-B02.md → S1 指令免疫，basic，第2个（新增时用）
  E3-A01.md → E3 内部威胁，advanced，第1个
```
