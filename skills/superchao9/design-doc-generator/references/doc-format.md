# 设计文档格式详细规范

## 文件命名
- 文档：`<系统名>-<模块名>-设计文档-YYYY-MM-DD.docx`
- 输出目录：`workspace/outputs/<模块>-design-doc-<日期>/`
- 截图目录：`outputs/<模块>-design-doc-<日期>/screenshots/`

## Word 文档字体规范

| 元素 | 字体 | 大小 | 样式 |
|------|------|------|------|
| 封面大标题（系统名） | 宋体/仿宋 | 18pt | 加粗居中 |
| 封面副标题（模块名） | 宋体/仿宋 | 20pt | 加粗居中 |
| 封面信息（时间/说明） | 宋体/仿宋 | 11pt | 居中 |
| 正文 | 宋体/仿宋 | 12pt | 常规 |
| Heading 1 | 宋体/仿宋 | 16pt | 加粗 |
| Heading 2 | 宋体/仿宋 | 14pt | 加粗 |
| Heading 3 | 宋体/仿宋 | 12pt | 加粗 |
| 图注 | 宋体/仿宋 | 10pt | 居中 |
| 流程步骤列表 | 宋体/仿宋 | 12pt | 常规，• 开头 |

## 图片规范
- 宽度：420pt（约 14.8cm）
- 图注格式：`图N-M 描述文字`（N=章节号，M=图序号，居中 10pt）
- 缺图处理：输出 `【缺图】文件名` 占位

## 表结构表格规范

7列固定表头（按此顺序）：
`字段名 | 说明 | 类型 | 是否必填 | 默认值 | 约束 | 备注`

填写规范：
- 是否必填：`是` / `否`
- 默认值：`NULL`（无默认值）/ `AUTO_INCREMENT` / 具体值如 `0`
- 约束：`PRIMARY KEY` / `UNIQUE` / `INDEX` / 空字符串
- 备注：枚举值用空格分隔，如 `0草稿 1审核中 2已通过 3已驳回`；外键关联写 `关联xxx表`

## 流程步骤写法

每个步骤用 `• ` 开头，箭头用 ` -> ` 连接，格式：
```
• 用户进入列表页 -> 通过条件筛选查询数据
• 用户点击"新增" -> 填写主表信息 -> 填写子表信息 -> 点击保存 -> 系统校验必填项 -> 保存成功返回列表
• 用户点击"提交审核" -> 系统弹出确认框 -> 确认后触发审批流 -> 记录 process_instance_id
• 审批人在审批流中审批 -> 通过后状态变更为"已通过" / 驳回后状态变更为"已驳回"
• 用户可在列表查看审批状态 -> 点击"查看审批"跳转审批详情
```

## 目录
- 使用 Word TOC 域自动生成（`TOC \o "1-3" \h \z \u`）
- 封面后分页，目录后分页，再开始正文
- 提示：打开 Word 后右键目录选择"更新域"

## 生成方式
使用 Python `python-docx` 库，基于已有模板 `.docx`（保留样式）或从头创建。
参考脚本：`scripts/build_design_doc.py`
