# Good-Txt-To-Hwreader AI 增强设计方案

## 一、设计目标

在现有规则引擎基础上，引入 AI 能力处理以下三类问题：

1. **非标准广告识别** — 识别规则无法覆盖的广告模式
2. **复杂乱码智能修复** — 修复规则无法处理的乱码
3. **智能章节识别** — 优化章节标题提取和结构化

## 三、模块一：非标准广告识别

### 3.1 问题分析

规则引擎能处理的标准广告：
- 固定格式水印（如"本文来自XXX"）
- 方括号广告（如【下载APP】）
- URL 链接

规则引擎无法处理的非标准广告：
- 变体广告（故意添加干扰字符）
- 伪装成正文的内容推广
- 新平台/新格式的广告
- 上下文相关的软广

### 3.2 AI 识别策略

#### 3.2.1 分段检测

```
输入文本 (按段落分割)
    │
    v
┌─────────────────────────────────────┐
│  对每个段落进行广告概率评估          │
│                                     │
│  Prompt:                            │
│  "判断以下段落是否为广告/推广内容：  │
│   [段落内容]                        │
│   返回: {is_ad: bool, confidence:   │
│   float, reason: string}"           │
└─────────────────────────────────────┘
    │
    v
高置信度广告段落 -> 标记删除
低置信度段落 -> 保留但记录
```

#### 3.2.2 Prompt 设计

```markdown
# 角色
你是一个专业的文本广告识别专家，擅长识别各种形式的广告和推广内容。

# 任务
判断以下段落是否为广告/推广/水印内容。

# 判断标准
1. 包含推广链接、二维码提示
2. 引导下载APP、关注公众号、加入群聊
3. 平台水印、来源声明
4. 作者推广、打赏引导
5. 变体广告（故意添加干扰字符避开检测）
6. 软广（伪装成正文但实际是推广）

# 输出格式
{
  "is_ad": true/false,
  "confidence": 0.0-1.0,
  "ad_type": "watermark|promotion|soft_ad|author_promo|other",
  "reason": "判断理由"
}

# 待检测段落
{paragraph}
```

#### 3.2.3 批量处理优化

为避免过多 API 调用，采用批量处理：

```python
def batch_detect_ads(paragraphs, batch_size=10):
    """批量检测广告段落"""
    results = []
    for i in range(0, len(paragraphs), batch_size):
        batch = paragraphs[i:i+batch_size]
        prompt = build_batch_prompt(batch)
        response = llm_call(prompt)
        results.extend(parse_batch_response(response))
    return results
```

## 四、模块二：复杂乱码智能修复

### 4.1 问题分析

规则引擎能处理的乱码：
- 已知编码错误映射（如 鏈枃 → 本文）
- 固定字符替换

规则引擎无法处理的乱码：
- 未知编码组合产生的乱码
- 多次编码错误叠加
- 部分字符乱码（上下文可推断）
- 特殊字符集的乱码

### 4.2 AI 修复策略

#### 4.2.1 上下文推断

```
输入：包含乱码的句子
    │
    v
┌─────────────────────────────────────┐
│  LLM 根据上下文推断正确内容          │
│                                     │
│  Prompt:                            │
│  "以下句子包含乱码，请根据上下文     │
│   推断并修复：                      │
│   原文：[句子]                      │
│   返回修复后的句子"                 │
└─────────────────────────────────────┘
    │
    v
输出：修复后的句子
```

#### 4.2.2 Prompt 设计

```markdown
# 角色
你是一个专业的文本修复专家，擅长根据上下文推断和修复乱码字符。

# 任务
修复以下文本中的乱码字符。

# 修复原则
1. 根据上下文语义推断正确字符
2. 保持句子通顺和语义完整
3. 无法确定的内容标记为 [无法识别]
4. 不要修改非乱码的正常字符

# 常见乱码特征
- 鏈、銆、鈥、锛、熲 等生僻字通常是编码错误
- å、æ、ç 开头的字符通常是 UTF-8 编码错误
- 连续的"拷"字通常是"锟斤拷"问题
- 乱码通常出现在引号、标点、特殊字符位置

# 输出格式
{
  "original": "原文",
  "fixed": "修复后的文本",
  "changes": [
    {"position": 位置, "before": "乱码", "after": "修复后", "reason": "推断理由"}
  ],
  "confidence": 0.0-1.0
}

# 待修复文本
{text}
```

#### 4.2.3 分级处理策略

```
┌────────────────────────────────────────────────────┐
│                    乱码修复流程                     │
├────────────────────────────────────────────────────┤
│                                                    │
│  Level 1: 规则引擎快速修复                         │
│  - 已知乱码映射 -> 直接替换                        │
│  - 修复率统计                                      │
│                                                    │
│  Level 2: AI 智能修复                              │
│  - 对规则无法处理的乱码段落                        │
│  - 使用 LLM 进行上下文推断                         │
│  - 返回修复建议                                    │
│                                                    │
│  Level 3: 人工确认                                 │
│  - 低置信度修复提示用户确认                        │
│  - 记录用户选择用于规则学习                        │
│                                                    │
└────────────────────────────────────────────────────┘
```

#### 4.2.4 学习机制

```python
class MojibakeLearner:
    """从 AI 修复结果中学习新规则"""
    
    def learn_from_fix(self, original, fixed, confidence):
        if confidence > 0.9:
            # 高置信度修复 -> 提取模式
            pattern = self.extract_pattern(original, fixed)
            if pattern:
                # 添加到规则库
                self.add_to_rules(pattern)
                # 持久化保存
                self.save_rules()
```

## 五、模块三：智能章节识别

### 5.1 问题分析

规则引擎能处理的章节格式：
- 标准格式（第X章、Chapter X）
- 数字编号（1. 标题）

规则引擎无法处理的章节格式：
- 非标准章节标题
- 无编号的章节分隔
- 特殊格式的章节（如古风小说）
- 章节标题被广告干扰

### 5.2 AI 识别策略

#### 5.2.1 章节结构分析

```
输入：全文内容

    v

│   返回：章节标题格式、章节分隔方式   │
│         章节列表"                   │
└─────────────────────────────────────┘
    │
    v
输出：章节结构信息
```

#### 5.2.2 Prompt 设计


### 6.4 处理流程

```python
class AIEnhancedTxtCleaner:
    """AI 增强的 txt 清理器"""
    
    def __init__(self, config):
        self.rule_engine = RuleEngineCleaner()
        self.ad_detector = AIAdDetector(config)
        self.mojibake_fixer = AIMojibakeFixer(config)
        self.chapter_parser = AIChapterParser(config)
    
    def clean(self, text):
        # 1. 规则引擎预处理
        text = self.rule_engine.pre_clean(text)
        
        # 2. AI 广告识别
        paragraphs = self.split_paragraphs(text)
        ad_results = self.ad_detector.detect(paragraphs)
        text = self.remove_ads(text, ad_results)
        
        # 3. AI 乱码修复
        mojibake_results = self.mojibake_fixer.fix(text)
        text = self.apply_fixes(text, mojibake_results)
        
        # 4. AI 章节识别
        chapters = self.chapter_parser.parse(text)
        text = self.normalize_chapters(text, chapters)
        
        # 5. 规则引擎后处理
        text = self.rule_engine.post_clean(text)
        
        return text, self.generate_report()
```

```
│  高优先级：规则引擎                                 │

│  - 速度快，成本低                                  │
│                                                    │
│  中优先级：AI 批量处理                              │
│  - 处理规则无法覆盖的情况                          │
│  - 批量调用减少 API 请求                           │
│                                                    │
│  低优先级：AI 单独处理                              │
│  - 处理复杂边界情况                                │
│  - 可能需要用户确认                                │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 7.2 缓存策略

```python
class SmartCache:
    """智能缓存管理"""
    
    def __init__(self):
        self.cache = {}
        self.hit_count = 0
        self.miss_count = 0
    
    def get_ad_result(self, paragraph_hash):
        """获取广告检测结果缓存"""
        return self.cache.get(f"ad:{paragraph_hash}")
    
    def get_mojibake_result(self, text_hash):
        """获取乱码修复结果缓存"""
        return self.cache.get(f"mojibake:{text_hash}")
    
    def set(self, key, value, ttl=3600):
        """设置缓存"""
        self.cache[key] = {
            "value": value,
            "expire_at": time.time() + ttl
        }
```

### 7.3 成本控制

| 策略 | 说明 |
|------|------|
| 规则优先 | 先用规则处理，减少 AI 调用 |
| 批量处理 | 合并多个请求，减少 API 调用次数 |
| 缓存复用 | 相同内容不重复调用 |
| 置信度过滤 | 低置信度结果不采纳 |
| 用户确认 | 边界情况让用户决定 |
