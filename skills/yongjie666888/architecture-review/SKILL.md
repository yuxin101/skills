# 架构评审助手

## 评审维度

| 维度 | 说明 |
|------|------|
| 功能性 | 是否满足业务需求 |
| 性能 | QPS、延迟、吞吐 |
| 可用性 | SLA、故障恢复 |
| 扩展性 | 水平/垂直扩展 |
| 安全性 | 认证、授权、数据安全 |
| 可维护性 | 代码质量、文档 |
| 成本 | 建设成本、运维成本 |

## 架构设计原则

```markdown
## 设计原则

### 1. 高可用
- 无单点故障
- 故障自动恢复
- 多级降级策略

### 2. 可扩展
- 服务无状态
- 水平扩展优先
- 模块解耦

### 3. 简单性
- KISS原则
- 避免过度设计
- 渐进式演进

### 4. 安全性
- 零信任架构
- 纵深防御
- 安全默认值

### 5. 可观测性
- 日志、指标、链路
- 统一监控
- 告警及时
```
