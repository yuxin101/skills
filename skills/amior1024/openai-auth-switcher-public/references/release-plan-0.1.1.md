# Release plan 0.1.1

## Status

`0.1.1` is no longer the preferred next milestone.

After evaluating first-run usability, the recommended next release is `0.2.0` because the product needs a web-first onboarding and one-shot install experience rather than minor polish only.

## What 0.1.1 would have covered

1. further reduce root-specific path examples in docs
2. improve packager discovery diagnostics when `package_skill.py` cannot be found
3. add a small troubleshooting section for ClawHub publish auth/rate-limit confusion
4. refine tutorial page wording and screenshots if needed
5. optionally add a non-destructive `doctor --strict-publish` mode

## Recommendation

Fold the worthwhile polish items into `0.2.0` unless a hotfix-only release becomes necessary.
