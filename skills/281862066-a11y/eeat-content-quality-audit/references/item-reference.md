## Dimension Explanations

### CORE Dimensions (Content Quality)

#### C — Contextual Clarity
Evaluate whether content is clear, accurate, and directly answers user questions.

**Core Check Items**:
- C01 Intent Alignment: Does content match search intent
- C02 Direct Answer: Does it provide direct answer at beginning
- C03 Query Coverage: Does it cover related queries
- C04 Definition First: Does it define key terms first
- C05 Topic Scope: Does it clearly define topic scope
- C06 Audience Targeting: Does it clearly identify target audience
- C07 Semantic Coherence: Is content logically coherent
- C08 Use Case Mapping: Does it cover specific use cases
- C09 FAQ Coverage: Does it answer common questions
- C10 Semantic Closure: Does it fully close the topic

#### O — Organization
Evaluate content structure, hierarchy, and navigability.

**Core Check Items**:
- O01 Heading Hierarchy: Clear H1-H6 hierarchy
- O02 Summary Box: Has content summary box
- O03 Data Tables: Uses tables to organize data
- O04 List Format: Uses lists to improve readability
- O05 Schema Markup: Has structured data markup
- O06 Content Chunking: Reasonably chunks content
- O07 Visual Hierarchy: Good visual hierarchy
- O08 Anchor Navigation: Has anchor navigation
- O09 Information Density: Appropriate information density
- O10 Multimedia Structure: Appropriate multimedia use

#### R — Referenceability
Evaluate content's citation value and credibility.

**Core Check Items**:
- R01 Data Precision: Data is precise
- R02 Citation Density: Citations are sufficient
- R03 Source Hierarchy: Sources are authoritative
- R04 Evidence-Claim Mapping: Claims have evidence support
- R05 Methodology Transparency: Methodology is transparent
- R06 Timestamp & Version: Has timestamp and version info
- R07 Entity Precision: Entity naming is precise
- R08 Internal Link Graph: Internal links are reasonable
- R09 HTML Semantics: HTML is semantic
- R10 Content Consistency: Content is consistent (veto item)

#### E — Exclusivity
Evaluate content's unique value and originality.

**Core Check Items**:
- E01 Original Data: Contains original data
- E02 Innovative Framework: Proposes innovative framework
- E03 Preliminary Research: Contains preliminary research
- E04 Counter Viewpoint: Contains counter viewpoints
- E05 Proprietary Visual Materials: Has proprietary visual materials
- E06 Fills Gaps: Fills content gaps
- E07 Practical Tools: Provides practical tools
- E08 Depth Advantage: Has depth advantage
- E09 Synthesis Value: Has synthesis value
- E10 Forward-Looking Insights: Provides forward-looking insights

### EEAT Dimensions (Source Credibility)

#### Exp — Experience
Evaluate whether author demonstrates actual usage experience.

**Core Check Items**:
- Exp01 First-Person Narrative: Uses first-person narrative
- Exp02 Sensory Details: Describes sensory experiences
- Exp03 Process Documentation: Documents usage process
- Exp04 Substantive Proof: Has substantive proof
- Exp05 Usage Duration: States usage duration
- Exp06 Problems Encountered: Describes problems encountered
- Exp07 Before-After Comparison: Has before-after comparison
- Exp08 Quantified Metrics: Has quantified metrics
- Exp09 Repeated Testing: Conducts repeated testing
- Exp10 Limitation Acknowledgment: Acknowledges limitations

#### Ept — Expertise
Evaluate whether author demonstrates professional knowledge and skills.

**Core Check Items**:
- Ept01 Author Identity: Author identity is clear
- Ept02 Credential Display: Displays professional credentials
- Ept03 Professional Vocabulary: Uses professional vocabulary
- Ept04 Technical Depth: Has sufficient technical depth
- Ept05 Methodology Rigor: Methodology is rigorous
- Ept06 Edge Case Awareness: Considers edge cases
- Ept07 Historical Background: Provides historical background
- Ept08 Reasoning Transparency: Reasoning process is transparent
- Ept09 Cross-Domain Integration: Integrates cross-domain knowledge
- Ept10 Editorial Process: Has strict editorial process

#### A — Authority
Evaluate content source's authority (mostly site-level).

**Core Check Items**:
- A01 Backlink Profile: Has quality backlinks
- A02 Media Mentions: Has media mentions
- A03 Industry Awards: Has industry awards
- A04 Publishing Record: Has consistent publishing record
- A05 Brand Recognition: Brand has recognition
- A06 Social Proof: Has social proof
- A07 Knowledge Graph Presence: In Knowledge Graph
- A08 Entity Consistency: Entity information is consistent
- A09 Partner Signals: Has partner signals
- A10 Community Reputation: Community reputation

#### T — Trust
Evaluate content's trustworthiness.

**Core Check Items**:
- T01 Legal Compliance: Legally compliant
- T02 Contact Information Transparency: Contact info transparent
- T03 Security Standards: Meets security standards
- T04 Disclosure Statement: Has disclosure statement (veto item)
- T05 Editorial Policy: Has editorial policy
- T06 Correction & Update Policy: Has correction/update policy
- T07 Ad Experience: Ads don't affect experience
- T08 Risk Disclaimer: Has risk disclaimer
- T09 Review Authenticity: Reviews are authentic
- T10 Customer Support: Has customer support

## Scoring Standards

### Pass (10 points)
- Fully meets the standard
- Exceeds basic requirements, demonstrates best practices

### Partial (5 points)
- Partially meets the standard
- Has obvious room for improvement
- Meets basic requirements but not perfect

### Fail (0 points)
- Does not meet the standard
- Completely missing the element
- Has obvious problems or errors

## Veto Item Explanation

The following three are "one-vote veto items"; once triggered regardless of total score, content is considered to have serious issues:

1. **T04: Disclosure Statement** — Must disclose any conflicts of interest, sponsorships, or partnerships
   - Trigger Condition: Content is sponsored or has conflicts of interest but not disclosed

2. **C01: Intent Alignment** — Content must match user search intent
   - Trigger Condition: Title promises don't match actual content

3. **R10: Content Consistency** — Content must be internally consistent
   - Trigger Condition: Data, claims, or conclusions have contradictions

## Usage Recommendations

1. **First Audit**: Execute all 80 checks to establish baseline score
2. **Regular Audits**: Re-audit quarterly to track score changes
3. **Comparative Audits**: Audit simultaneously with competitors to identify gaps
4. **Iterative Improvements**: Improve item by item based on audit report priorities
5. **Re-Audit Verification**: Re-audit after improvements to verify effectiveness

## Notes

- This reference table provides standard check items; actual application can be flexibly adjusted based on specific situations
- Site-level items (A01-A10, T01-T03, T05, T07, T10) may not be evaluable during single-page audits
- GEO-First items (marked with 🎯) are critical for AI citation optimization
- Different content type weights reflect their core value dimensions
