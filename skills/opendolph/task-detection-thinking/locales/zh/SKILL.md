# 任务检测 + 主动思考技能

> 自动检测任务异常，智能分析原因，主动提出解决方案

---

## 触发条件

| 场景 | 触发方式 |
|------|----------|
| 自动触发 | Heartbeat 检测时自动执行 |
| 手动触发 | 用户输入「检查任务」「任务状态」「异常检测」 |
| 定时触发 | 每6小时自动扫描一次 |

---

## 任务管理文件规范

### HEARTBEAT.md - 全局任务看板模板

```markdown
## 全局任务看板
| 任务ID | 任务名称 | 状态 | 进度 | 截止时间 | 最后更新 | 阻塞原因 |
|--------|----------|------|------|----------|----------|----------|
| T001 | 初始化三层记忆 | Done | 100% | 2026-03-10 | 2026-03-10 | 无 |
| T002 | 集成向量检索 | Active | 60% | 2026-03-15 | 2026-03-12 | 等待API密钥 |
| T003 | 测试主动思考 | Waiting | 0% | 2026-03-20 | 2026-03-11 | 依赖T002完成 |
```

**状态定义：**
| 状态 | 含义 |
|------|------|
| Queue | 等待开始 |
| Active | 进行中 |
| Waiting | 阻塞/等待 |
| Done | 已完成 |
| Aborted | 已取消 |

### WORKING.md - 项目级任务规则

所有子任务必须标注：
- **状态**：Queue/Active/Waiting/Done
- **进度**：百分比（0-100%）
- **负责人**：任务责任人
- **依赖项**：此任务依赖的任务ID

**自动检测规则：**
- Active 状态停滞超24小时 → 自动标记为"异常" 🔴
- 阻塞任务必须填写："阻塞原因" + "所需支持"
- 每日23:00 → 自动汇总所有任务状态到 HEARTBEAT.md

## 检测规则（自动执行）

### 1. 扫描 HEARTBEAT.md

| 检测项 | 条件 | 标记 |
|--------|------|------|
| 停滞任务 | Active 状态 + 最后更新 > 24h | 🔴 停滞 |
| 待确认阻塞 | Waiting 状态 + 阻塞原因为空 | 🟡 待确认 |
| 超期任务 | 进度 < 100% + 截止时间 < 当前时间 | 🔴 超期 |
| 无进度任务 | Active 状态 + 进度 = 0% 超过 48h | 🟡 无进度 |
| 异常任务 | Active + 超过24小时未更新 | 🔴 异常 |

### 2. 扫描 WORKING.md

| 检测项 | 条件 | 标记 |
|--------|------|------|
| 依赖阻塞 | 上游任务未完成导致下游阻塞 | 🔗 依赖阻塞 |
| 重复任务 | 任务名称/内容相似度 > 80% | ⚠️ 重复 |
| 僵尸任务 | Done 状态但无完成时间 | 👻 僵尸 |
| 无意义任务 | 标题为空或内容 < 10 字 | 🗑️ 待清理 |
| 必填字段缺失 | 无状态/进度/负责人 | ⚠️ 不完整 |

## 主动思考逻辑

### 对「停滞任务」

**分析步骤：**
1. 检查最后更新内容 → 判断停滞类型
2. 扫描相关记忆 → 寻找上下文
3. 生成3个推进方案

**自动尝试（无需人工）：**
- 补充缺失信息（从记忆中提取）
- 重试失败的接口/命令
- 更新任务状态为 Waiting（如需等待）

**需人工介入：**
- 生成详细提醒，通过 Feishu 推送

### 对「阻塞任务」

**分析步骤：**
1. 识别阻塞类型（资源/依赖/决策/外部）
2. 判断是否可自动解决
3. 记录解决方案到温记忆

**自动解决场景：**
- 依赖任务已完成 → 自动解除阻塞
- 资源已可用 → 自动重试
- 信息已补充 → 自动推进

### 对「超期任务」

**分析步骤：**
1. 评估剩余工作量
2. 判断延期影响
3. 生成调整方案

**自动调整：**
- 更新截止时间为合理值
- 调整下游任务排期
- 标记为高优先级

## 配置参数

```yaml
task_detection:
  # 检测频率
  heartbeat_check: true      # Heartbeat 时检测
  cron_schedule: "0 */6 * * *"  # 每6小时
  
  # 阈值设置
  stale_threshold_hours: 24   # 停滞阈值
  overdue_check: true         # 检查超期
  dependency_check: true      # 检查依赖链
  
  # 自动修复
  auto_fix_enabled: true      # 启用自动修复
  auto_fix_max_attempts: 3    # 最大自动尝试次数
  
  # 通知设置
  feishu_alert: true          # 关键告警推送
  alert_on_overdue: true      # 超期告警
  alert_on_blocking: true     # 严重阻塞告警
```

```

    ↓
扫描 HEARTBEAT.md + WORKING.md
    ↓
识别异常任务（停滞/阻塞/超期/重复）
    ↓

    ↓
生成解决方案（3个选项）
    ↓

    ↓
写入检测结果 + 思考日志
    ↓
关键告警推送 Feishu
```
