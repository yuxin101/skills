### ⚠️ 搜索基础设施限制声明 (Search Infrastructure Disclaimer)

> **本框架依赖公开网络搜索作为数据源。分析师和投资者必须理解以下系统性限制：**

| 数据类型 | 公开搜索可获取性 | 质量评估 | 应对策略 |
| --- | --- | --- | --- |
| **板块/概念指数走势** | 🟡 中等 | 公开财经站点通常可查，但精确日度数据常不可用 | 用近似月度趋势替代，标注精度限制 |
| **行业 CAGR/市场规模** | 🟡 中等 | 通常来自咨询公司二手转述，偏差可达 ±50% | 交叉验证 2+ 来源，标注数据范围 |
| **产能利用率/库存数据** | 🔴 低 | 通常滞后 3-6 个月（来自上季财报或过时研报） | **触发数据时效熔断**，明确标注数据时间戳 |
| **企业成本结构** | 🔴 低 | 非公开信息，仅少数研报含估算值 | 用行业平均/同业对比替代，标注为 [推演] |
| **机构持仓/资金流向** | 🟡 中等 | 龙虎榜/基金季报可查，但滞后且不完整 | 仅作辅助参考，降权处理（公开的≠真实的） |
| **实时交易数据** | 🔴 不可用 | 需要 Bloomberg/Wind 终端 | 使用收盘数据和趋势分析替代 |

**⛔ 强制规则：当搜索中某类数据完全不可获取时：**
1. **报告中标注** `[数据受限：XX — 使用替代方法]`，不得用推测值伪装成事实
2. **使用替代数据降级分析**——如产能利用率不可用，用龙头盈亏状况、行业景气指数等替代指标，并标注为 `[推演]`
3. **仅当核心逻辑的全部支撑数据均不可获取时**，才触发整体置信度降级——单一维度数据缺失不应封顶整体结论

> **核心理念：搜索引擎不是 Bloomberg 终端。但"数据受限"≠"不能分析"——机构分析师也常用替代指标和交叉验证。关键是诚实标注数据来源和精度，让投资者自行判断。**

#### D. Institutional Forensics (机构级取证) - 代理指标扫描 ⭐

**⚠️ 核心原则：通过客观数据推演主力意图，而非搜索主观评论。**

> 💡 **业内共识**: _"主力出货"是意图（Intent），不是事实（Fact）。真正的机构在出货时会配合发布利好消息。直接搜"主力出货"只会搜到散户哀嚎和营销号臆测——这些是噪音，甚至是反向指标。_

> **⚠️ 公开数据局限性声明 (Public Data Limitation Disclaimer) — 新增：**
> 本节所有数据来源均为**公开渠道**（网页搜索、公告、财报）。真正的 Smart Money（主权基金、顶级对冲基金）建仓行为通常**不会留下公开痕迹**。能在公开渠道搜到的"机构动向"具有以下局限：
>
> 1. **滞后性**：公开披露的持仓数据（如 13F、基金季报）滞后 1-3 个月
> 2. **幸存者偏差**：只能看到"被披露的"动作，看不到"刻意隐藏的"动作
> 3. **误导风险**：部分公开信息可能是故意做给散户看的（Painted Tape）
>
> **因此，本节结论的权重在最终投资建议中不应超过 20%。它是辅助验证工具，不是决策依据。**

**二阶思维 (Second-Order Thinking)：**

- 散户看新闻："利好！买入！"
- 分析师看反应："利好为什么不涨？谁在卖？"

> 💡 **业内共识**: _Identifying a trend is easy; Identifying a trap is where the money is saved._ (识别趋势容易，识别陷阱才能保命。)

#### E. Cost Structure Analysis (成本结构分析) ⭐

**⚠️ 核心原则：利润 = 售价 - 成本。仅分析售价（商品价格）而忽略成本端，是初级分析师最常犯的致命错误。**

> 💡 **业内共识**: _Revenue tells you the story; Margin tells you the truth. A company selling at lower prices but with even lower costs is a profit expansion story, not a decline story._
> （收入讲故事，利润率讲真话。一家售价下跌但成本下跌更快的公司，是利润扩张的故事，不是衰退的故事。）

**强制搜索指令（周期型/成熟型赛道不可跳过）：**

| 维度              | 搜索指令                                              | 机构视角                 |
| ----------------- | ----------------------------------------------------- | ------------------------ |
| **主要原料价格**  | `[概念] 主要原材料价格 走势`, `[概念] 原料成本 同比`  | 成本端是否在改善？       |
| **能源/运输成本** | `[概念] 运输成本`, `[概念] 能源成本 变化`             | 隐性成本项变化           |
| **龙头毛利率**    | `[概念] 龙头 毛利率 变化`, `[概念] 头部企业 成本优势` | 利润剪刀差是否打开？     |
| **行业成本曲线**  | `[概念] 行业 平均成本`, `[概念] 完全成本 vs 售价`     | 行业是否处于亏损线以下？ |

**利润剪刀差判定矩阵：**

| 售价变化 | 成本变化      | 利润剪刀差        | 对股价的影响          |
| -------- | ------------- | ----------------- | --------------------- |
| 📉 下跌  | 📉📉 **暴跌** | ✅ **剪刀差打开** | 🟢 利好（利润扩张）   |
| 📉 下跌  | ➡️ 持平       | ❌ 剪刀差恶化     | 🔴 利空（利润压缩）   |
| ➡️ 持平  | 📉 下跌       | ✅ **剪刀差打开** | 🟢 利好（静默增利）   |
| 📈 上涨  | 📈📈 **暴涨** | ❌ 剪刀差恶化     | 🔴 利空（增收不增利） |

> 💡 **猪肉案例**：2025 年猪价下跌 17%，但豆粕（饲料核心原料）价格下跌 20%+。利润剪刀差打开 → 龙头毛利率反而改善 → 解释了"猪价跌、猪股涨"的悖论。

**输出变量：** `COST_TREND`, `MARGIN_SPREAD_STATUS`, `COST_STRUCTURE_SUMMARY`

#### F. Earnings Quality Analysis (盈利质量分析) ⭐ 新增 — 投委会要求

> **设计原因：** 利润剪刀差"打开"并不等于企业在创造价值。政府补贴、应收账款堆积、资本化处理等都会制造"纸面利润"。一份合格的分析必须区分"报表利润"和"真实盈利能力"。

**强制搜索指令（当 Step 3.E 显示利润改善时必须执行）：**

| 维度 | 搜索指令 | 检查目的 |
| --- | --- | --- |
| **现金流匹配** | `[龙头企业] 经营性现金流 vs 净利润`, `[龙头企业] 现金流量表` | 利润是否有真金白银支撑？ |
| **非经常性损益** | `[龙头企业] 非经常性损益 政府补贴`, `[龙头企业] 扣非净利润` | 利润是靠补贴还是靠经营？ |
| **应收账款** | `[龙头企业] 应收账款 变化`, `[概念] 行业 回款周期` | 是否在"赊账做收入"？ |
| **资本回报率** | `[龙头企业] ROE ROIC`, `[概念] 行业 资本回报率` | 赚的钱是否超过资本成本？ |

**盈利质量红绿灯：**

| 评级 | 特征 | 对投资结论的影响 |
| --- | --- | --- |
| 🟢 **高质量** | 经营现金流 ≥ 净利润 + 扣非利润占比 > 80% + 应收账款稳定 | 利润剪刀差结论可信 |
| 🟡 **中等质量** | 现金流略低于净利润 或 有一定补贴依赖 | 利润改善需打折理解，标注风险 |
| 🔴 **低质量** | 现金流远低于净利润 或 主要靠补贴/一次性收益 或 应收暴增 | **利润剪刀差结论不可靠**，降低整体置信度 |

**输出变量：** `EARNINGS_QUALITY` (🟢/🟡/🔴)

### 3.5 Iterative Search Validation (迭代搜索验证) ⭐ 关键步骤

**⚠️ 核心原则：每轮搜索后必须验证信息可靠性，不可靠则继续挖掘，最多执行 5 轮搜索。**

#### 验证循环流程

```
┌─────────────────────────────────────────────────────────────┐
│  第N轮搜索完成                                               │
│       ↓                                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 信息可靠性评估 (Reliability Check)                   │   │
│  │ □ 数据来源是否权威？(研报/财报/官方 vs 自媒体/论坛)   │   │
│  │ □ 关键数据是否有多个独立信源交叉验证？                │   │
│  │ □ 时效性是否足够？(数据是否过时)                      │   │
│  │ □ 是否存在明显矛盾或存疑信息？                        │   │
│  │ □ 核心结论是否有具体数据支撑？                        │   │
│  └─────────────────────────────────────────────────────┘   │
│       ↓                                                     │
│  ┌──────────────┐    ┌──────────────────────────────────┐  │
│  │ 全部✅通过    │───→│ 结束搜索，进入下一分析步骤        │  │
│  └──────────────┘    └──────────────────────────────────┘  │
│       ↓ 存在❌                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ 生成下一轮搜索关键词 (基于信息缺口)                    │  │
│  │ → 针对存疑点设计验证性搜索                            │  │
│  │ → 针对缺失数据设计补充性搜索                          │  │
│  │ → 尝试不同关键词组合挖掘更多细节                      │  │
│  └──────────────────────────────────────────────────────┘  │
│       ↓                                                     │
│  N < 5? ──Yes──→ 执行第N+1轮搜索 ──→ 返回顶部              │
│       ↓ No                                                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │ 达到5轮上限，输出当前最佳结论 + 标注置信度             │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

#### A. 信息可靠性定性评估 (v2.0 — 定性分级替代伪量化评分)

> **⚠️ 设计变更说明：** 原版使用 15 分制评分（每维度 0-3 分），但阈值（12分/8分/7分）缺乏统计校准依据，且等权假设不合理（一条财报数据远比三条自媒体交叉验证有价值）。现改为定性 A/B/C 分级，更符合"模糊的正确"原则。

**五维度逐一评估，每维度标注 ✅/⚠️/❌：**

| 评估维度 | ✅ 充分 | ⚠️ 存疑 | ❌ 不足 |
| --- | --- | --- | --- |
| **信源权威性** | 研报/财报/官方公告/头部财经媒体 | 行业媒体/知名博主 | 匿名论坛/营销软文/小道消息 |
| **数据时效性** | 3 个月内（参考 A2 保鲜期标准） | 3-12 个月 | 超过 1 年 |
| **交叉验证** | 3+ 个独立信源一致 | 2 个信源一致 | 仅单一信源 |
| **数据具体性** | 有具体数字/图表/来源 | 有定性描述但缺数字 | 纯观点无数据 |
| **矛盾程度** | 无明显矛盾 | 存在小矛盾但可解释 | 存在重大矛盾 |

**⚠️ 权重差异化规则（非等权）：**
- **信源权威性** 和 **数据时效性** 是核心维度——如果这两个维度都是 ✅，即使其他维度有 ⚠️，整体仍可评为 B 级
- 反之，如果核心维度有 ❌，即使其他维度全 ✅，整体也不应超过 B 级

**综合定性分级：**

> **⚠️ 声明：以下分级衡量的是"信息完备性"（搜索结果的质量与充分性），不是"投资确信度"（该不该买/卖）。信息完备不等于结论正确。**

| 分级 | 判定条件 | 含义 | 行动 |
| --- | --- | --- | --- |
| **A 级** (信息充分) | 核心维度均 ✅ + 无 ❌ | 可进入下一步分析 | 继续 |
| **B 级** (基本可用) | 核心维度至少 1 个 ✅ + ❌ ≤ 1 个 | 可分析但需在结论中标注信息缺口 | 补搜或标注 |
| **C 级** (严重不足) | 核心维度有 ❌ 或 ❌ ≥ 2 个 | 信息不足以支撑分析 | 必须继续搜索 |

#### A2. 数据时间戳感知与熔断机制 ⭐⭐ 新增 — 高盛尽职调查要求

> **业内共识原则**: _Public web data has severe latency bias. Capacity data from quarterly reports is 3-6 months stale. Using stale fundamentals to explain real-time prices creates dangerous attribution errors._
> （公开网络数据存在严重的时滞偏差。季报中的产能数据滞后 3-6 个月。用过时基本面解释实时股价会产生危险的归因谬误。）

**核心原则：不同类型的数据有不同的"保鲜期"。超过保鲜期的数据不是无用的，但必须标注过期状态，并降低其在结论中的权重。**

**数据保鲜期分级标准：**

| 数据类型                                | 保鲜期    | 过期后处理                         | 说明                   |
| --------------------------------------- | --------- | ---------------------------------- | ---------------------- |
| **股价/ETF 资金流向**                   | 实时-1 天 | 过期几乎不可能（搜索结果通常实时） | 市场价格类数据通常最新 |
| **新闻/政策/事件**                      | 7 天      | 标注"[时效待确认]"，搜索最新进展   | 政策可能已变化         |
| **月度行业数据** (PMI/CPI/销量)         | 30 天     | 标注"[数据滞后]"，降低权重         | 快变量                 |
| **季度财报数据** (营收/利润/库存)       | 90 天     | 标注"[季报数据，滞后~X 月]"        | 基本面滞后指标         |
| **年度结构数据** (CAGR/市占率/产能格局) | 180 天    | 仍可使用（慢变量），但标注日期     | 结构性数据变化慢       |
| **行业研报预测**                        | 90 天     | 标注发布日期，注意预测假设是否已变 | 预测随时可能被修正     |

**⚠️ 数据过期熔断规则：**

| 熔断级别      | 触发条件                                                                     | 强制行为                                                                                                             |
| ------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **🟡 软熔断** | 核心结论依赖的关键数据已过保鲜期                                             | 在结论旁标注 `[⏰ 数据滞后X月]`，降低该结论的置信度                                                                  |
| **🔴 硬熔断** | **股价归因分析**中使用的基本面数据全部超过 90 天，且无任何 30 天内的验证数据 | **禁止输出确定性归因结论**。改为输出：`"⚠️ 可用数据过期，基本面归因不可靠。建议等待最新财报/月度数据发布后再评估。"` |

**输出要求：在每条关键数据后标注时间戳：**

```markdown
- 牧原完全成本 13 元/kg **(来源: 2025Q3 财报, 滞后~4 月)** ⏰
- 生猪均价 12.13 元/kg **(来源: 行业监测, 2025-12-31)** ✅
- 能繁母猪 3961 万头 **(来源: 农业农村部, 2025 年末)** ✅
```

**⛔ 禁止行为：**

- ❌ 禁止不标注数据日期就直接引用（如"产能利用率为 60%"但不说这是何时的数据）
- ❌ 禁止用 3 个月前的库存数据作为"当前"状态的唯一证据
- ✅ 当数据过期时，必须用"搜索最新动态"来补充，或明确标注不确定性
