# Step 1B: 前瞻性竞争优势评估 (Forward-Looking Competitive Advantage Assessment) ⭐ 核心增量模块

> **目的**：识别公司**正在形成但尚未被市场充分认知**的新竞争优势。这是超额收益（alpha）的核心来源。
> **设计原理**：Morningstar 护城河框架是回顾性的——它只评估已有壁垒。但投资的本质是前瞻性判断：NVIDIA 在 2022 年底已有 GPU 设计的宽护城河，但真正改变命运的是 AI 算力需求对 CUDA 生态的垄断式锁定——这一"正在形成中的超级护城河"直到 2023 年 ChatGPT 爆发后才被市场集中定价（股价从 $150 涨至 $500+）。**这是已被历史验证的经典案例：前瞻性优势在早期被忽视，兑现期被集中定价——识别这个时间差正是超额收益的核心来源。**
>
> **理论依据**：
> - Andy Grove 的**战略拐点理论** (Strategic Inflection Points, *Only the Paranoid Survive*) — 当 10X 力量改变竞争规则时，适应者获得不对称优势
> - Clayton Christensen 的**颠覆式创新** (*The Innovator's Dilemma*) — 新能力往往从边缘市场开始，最终重塑主流
> - Carlota Perez 的**技术革命周期** (*Technological Revolutions and Financial Capital*) — 技术部署阶段的先行者享有超额收益
> - Geoffrey Moore 的**跨越鸿沟** (*Crossing the Chasm*) — 技术采用生命周期中的关键拐点决定赢家
> - Michael Mauboussin 的**预期投资** (*Expectations Investing*) — 超额回报来自市场预期与现实之间的差距

**前置依赖**: Step 0（行业趋势）+ Step 1（商业模式、收入结构、护城河）的共享上下文必须已写入。

## 1B.1 新兴优势信号捕捉 (Emerging Advantage Signal Detection)

**⚠️ Step 0 → Step 1B 桥接规则（强制）：**

> **前瞻性优势的信号检测必须锚定在 Step 0 已识别的行业趋势上，而非凭空想象。**
> Step 0 中识别的"行业景气驱动因素"和"行业生命周期变化方向"是本步骤搜索的起点。

**搜索指令（必须执行）：**

```
搜索 E0: "[Step 0 识别的行业趋势/变化关键词] + [公司名] 布局 进展 竞争力"  ← 锚定 Step 0
搜索 E1: "[公司名] 新业务 新产品 战略布局 [当前年份]"
搜索 E2: "[公司名所在行业] 范式转变 技术革命 新趋势 [当前年份]"
搜索 E3: "[公司名] [识别到的新趋势关键词] 布局 竞争力"
```

> **搜索 E0 是强制的第一步**——它确保后续信号扫描的起点来自已验证的行业事实，而非分析师的主观联想。如果 Step 0 未识别到显著的行业趋势变化，则搜索 E0 可标注"行业趋势平稳，无显著范式信号"并继续。

**六大信号类型扫描（逐一检查）：**

| 信号类型 | 定义 | 典型场景 | 检查方法 |
| --- | --- | --- | --- |
| 🔬 **技术范式转变** | 底层技术革命重新定义行业竞争规则 | 新通用技术对传统行业的渗透/替代 | 公司是否具备关键技术/平台/数据/专利？ |
| 📐 **商业模式迁移** | 行业价值链重构，新模式取代旧模式 | 订阅化替代一次性交付、直营替代经销 | 公司是否率先/有效转型？ |
| 🏛️ **政策红利窗口** | 监管变化创造结构性机会 | 产业扶持政策、准入门槛变化、合规要求升级 | 公司是否处于政策受益核心？ |
| 🌊 **需求结构性变化** | 终端需求发生不可逆的升级/迁移 | 人口结构变化、生活方式迁移、代际消费差异 | 公司是否占据新需求的入口？ |
| 🔗 **生态/平台锁定** | 用户/开发者生态形成正反馈飞轮 | 开发者工具链生态锁定（如 CUDA 生态）、应用平台生态 | 公司是否在构建难以替代的生态？ |
| 🏭 **供给侧重构** | 产能/技术/人才门槛大幅提升，淘汰竞争者 | 制造工艺代际跃升、行业准入标准提升 | 公司是否处于门槛之上而竞争者之下？ |

**信号检测输出：**

```
检测到的新兴优势信号:
- 信号类型: [六大类型之一]
- 信号描述: [一句话描述正在发生什么变化]
- 公司关联: [公司如何与这个变化产生关联]
- 竞争者状态: [同行业其他公司在这个维度的布局]
```

**⚠️ 如果扫描后未检测到任何有意义的新兴优势信号，直接标注"未检测到前瞻性竞争优势"并跳过后续子步骤。这是正常情况——大多数公司在任何时点都不具备前瞻性优势。强制寻找不存在的前瞻性优势 = 确认偏差。**

### 1B.1b 概念纯度评估 (Concept Purity Filter)

> **设计原理**：在 A 股等主题驱动市场中，一家公司常因"沾边"某个热门赛道而获得估值溢价，但其实际收入中来自该赛道的比例可能极低。概念纯度将这种定性标签转化为定量过滤器，防止"93% 的收入与概念无关"的公司获得不当的前瞻性评级。

**概念纯度 = 前瞻性优势相关收入 / 总收入**

> 所需数据来源：Step 1.1 收入结构（已搜索获取）+ Step 1B.1 信号检测中识别到的相关业务线

| 概念纯度 | 分类 | 对前瞻性评估的影响 |
| --- | --- | --- |
| **>50%** | 纯正标的 | 前瞻性优势与公司核心业务深度绑定，评估权重正常 |
| **20-50%** | 混合型 | 前瞻性优势有实质基础，但非核心业务，评估时需注明"非主业驱动" |
| **5-20%** | 概念成分偏重 | 前瞻性优势仍处早期，**成熟度上限锁定为 L3**（即使局部数据支持 L4，整体收入贡献不足以"确立"） |
| **<5%** | ⚠️ 纯概念标签 | **前瞻性成熟度上限锁定为 L2**，不得上调护城河展望或影响评级。仅作为催化剂监控 |

**概念纯度的搜索验证：**

```
搜索 E-purity: "[公司名] [前瞻性优势相关业务] 收入 占比 [最近年度/季度]"
```

**⚠️ 概念纯度是硬性过滤器**——如果无法从公开信息中获取相关收入数据，则默认归类为"<5% 纯概念标签"。举证责任在于证明纯度足够高，而非假设足够高。

**必须输出：**

```
概念纯度: [X%] ([相关收入] / [总收入]) [数据来源+时间]
纯度分类: [纯正/混合/概念成分偏重/纯概念标签]
成熟度上限约束: [无/L3封顶/L2封顶]
```

### 1B.1c 范式量级承接与权重校准 (Paradigm Magnitude Calibration)

> **设计原理**：1859 年 Drake 打出第一口油井时（L1），没人预见 25 年后 90% 的捕鲸业将消亡。标准 L1-L4 纪律在"正常竞争"中是正确的——防止概念炒作污染分析。但在范式级颠覆（S3）中，这套纪律可能让投资者保住了短期本金，却错过了一个时代。本步骤从 Step 0.5 承接范式量级评估，校准后续前瞻性分析的权重。
>
> **理论依据**：Carlota Perez 技术革命周期理论——"安装期"早期信号与"部署期"全面展开之间存在系统性认知延迟；Roy Amara 定律——短期高估、长期低估的非对称性。

**从 Step 0.5 读取：**

```
范式量级: [Step 0.5 输出的 S1/S2/S3/不适用]
公司角色: [Step 0.5 输出的 🗡️/🔄/☠️/👀/不适用]
```

**范式量级对后续子步骤的权重校准规则：**

| 子步骤 | S1 增量型 | S2 显著型 | S3 范式型 |
| --- | --- | --- | --- |
| **三支柱评估 (1B.2)** | 标准权重 | 时序(B)权重上调——窗口期判断更关键 | 时序(B)+能力(A)权重显著上调 |
| **成熟度分级 (1B.3)** | 标准 L1-L4 | L3+ 可适度上调增量 | **L2 获得"监控溢价"**（详见 1B.4 修订矩阵） |
| **迁移速度 (1B.3b)** | 标准判定 | 关注🚀快速上行的催化效应 | **迁移速度的投资含义放大**——S3+🚀 = 可能的"石油时刻" |
| **竞争图谱 (1B.4a)** | 标准排他性 | 关注转型质量差异 | **颠覆方 vs 被颠覆方的生死分化**成为核心 |

**⚠️ 核心铁律（不因范式量级改变）：**
- **L1 在任何量级下都不影响评级**——没有产品、没有收入、没有用户 = 纯概念
- **概念纯度约束不因范式量级松动**——收入占比 <5% 就是 <5%，即使处于 S3 范式中
- 这两条铁律的作用是：即使在"AI 是石油级变革"的共识下，仍然防止"任何沾边 AI 的公司都获得溢价"

**必须输出：**

```
承接范式量级: [S1/S2/S3/不适用]
承接公司角色: [🗡️/🔄/☠️/👀/不适用]
权重校准影响: [无/时序权重上调/L2监控溢价/战略级权重调整]
铁律确认: L1不影响评级 ✓ / 概念纯度约束不松动 ✓
```

### 1B.1d 单位经济学评估 (Unit Economics Assessment) ⭐ 条件触发

> **理论依据**：General Atlantic 成长型投资标准 — 理解一家公司的单位经济学是判断其商业模式可持续性和规模化潜力的基础。CAC/LTV 比率、毛利回收期、边际经济性等指标比宏观财务数据更能揭示业务的内在健康度。
>
> **触发条件（满足任一即触发，否则跳过）：**
> 1. Step 1.1 识别的商业模式为 **SaaS/订阅型、平台型、互联网/科技型、消费直营（DTC）** 之一
> 2. 公司处于**高增长期**（Step 2.5 成长性为 🟢高成长）且尚未实现稳定盈利
> 3. Step 1B.1 检测到的前瞻性优势信号类型为 **📐商业模式迁移** 或 **🔗生态/平台锁定**

**搜索指令（触发后必须执行）：**

```
搜索 UE-1: "[公司名] 获客成本 CAC 客户获取成本"
搜索 UE-2: "[公司名] 用户留存率 复购率 LTV 客户终身价值"
搜索 UE-3: "[公司名] 单位经济 毛利 每单利润 ARPU"
```

**按商业模式类型选择评估维度：**

| 商业模式 | 核心单位经济指标 | 健康阈值 | 危险信号 |
| --- | --- | --- | --- |
| **SaaS/订阅型** | LTV/CAC、月/年留存率、NDR（净收入留存率）、ARR 增速 | LTV/CAC >3x，NDR >100%，年留存 >85% | LTV/CAC <1.5x，NDR <90% |
| **平台/市场型** | Take Rate、GMV 增速、买卖双方留存、单位交易成本 | Take Rate 稳定/上升，双边留存增长 | Take Rate 下降，单边流失 |
| **消费直营 (DTC)** | CAC、复购率、客单价趋势、LTV | 复购率 >30%，CAC 回收期 <12月 | 复购率下降 + CAC 上升 |
| **广告模型** | ARPU、DAU/MAU 比、用户时长、广告填充率 | ARPU 增长 + 用户时长稳定 | 用户时长下降 + ARPU 依赖提价 |
| **制造/零售** | 坪效/人效、存货周转、毛利率/单位、产能利用率 | 效率指标改善趋势 | 效率指标持续恶化 |

**单位经济学健康度评估（必须输出）：**

| 维度 | 当前值 | 趋势 (近 3-4 季度) | 评级 | 数据来源 |
| --- | --- | --- | --- | --- |
| [核心指标 1] | | [📈改善/➡️稳定/📉恶化] | [🟢/🟡/🔴] | |
| [核心指标 2] | | [📈改善/➡️稳定/📉恶化] | [🟢/🟡/🔴] | |
| [核心指标 3] | | [📈改善/➡️稳定/📉恶化] | [🟢/🟡/🔴] | |

**单位经济学综合判断（必须输出）：**

```
单位经济学健康度: [🟢健康(可持续规模化) / 🟡待验证(部分指标不确定) / 🔴不健康(烧钱无回报)]
核心发现: [一句话——如 "LTV/CAC=5.2x 且持续改善，获客效率强"]
规模化风险: [低/中/高] — [规模扩张是否会恶化单位经济性？]
```

**⚠️ 数据获取约束**：单位经济学数据（尤其 CAC、LTV）通常不在标准财报中披露。如果搜索无法获取具体数值，使用以下替代：
- **CAC 替代**: 销售费用 / 新增客户数（如可获取）；或 销售费用增速 vs 营收增速的比较
- **LTV 替代**: ARPU × 估算留存期；或 复购率 × 平均客户周期 × 客单价
- **无法获取时**: 标注 "数据不可得" 并跳过该指标，不伪造数据

> **传导**: 单位经济学健康度供 Step 2 成长性评估参考（高增长但单位经济恶化 = 低质量增长）；🔴不健康 → 在 Step 5 反面审查中作为攻击维度。

## 1B.2 三支柱可信度评估 (Three-Pillar Credibility Assessment)

> **设计原理**：检测到信号 ≠ 真实优势。必须通过三个独立维度的交叉验证来评估可信度。每个支柱独立打分，然后综合。

**搜索指令（必须执行）：**

```
搜索 E4: "[公司名] [新兴优势关键词] 产品进展 落地效果 客户反馈"
搜索 E5: "[新兴趋势] 市场规模 渗透率 竞争格局 [当前年份]"
```

**支柱 A：能力基础 (Capability Foundation)**

> 公司是否具备将新兴优势转化为现实壁垒的**底层能力**？

| 检查维度 | 🟢 强 | 🟡 中 | 🔴 弱 |
| --- | --- | --- | --- |
| **核心技术/资源匹配度** | 已有关键技术/专利/数据/牌照 | 有部分相关能力，需补强 | 几乎无相关基础，靠蹭概念 |
| **团队胜任力** | 核心团队有该领域深度经验 | 有通用管理能力但缺专业纵深 | 纯跨界，无相关经验 |
| **资源投入力度** | 研发/资本开支明确向该方向倾斜 | 有投入但规模不大 | 仅停留在口头/PPT 层面 |
| **存量业务协同** | 新优势与现有业务强协同（数据/渠道/客户复用） | 有一定协同 | 新旧业务割裂 |

**支柱 B：市场时序 (Market Timing)**

> 新兴优势所依赖的市场/技术趋势，处于什么阶段？太早和太晚都不是好信号。

| 阶段 | 特征 | 投资含义 |
| --- | --- | --- |
| ⏳ **过早 (Too Early)** | 技术不成熟/需求未启动/标准未定 | 投入可能"烧空"，先行者未必是赢家 |
| ✅ **窗口期 (Sweet Spot)** | 技术刚过拐点/需求启动/但竞争格局未定 | **最佳投资窗口** — 优势正在固化 |
| ⚠️ **中期 (Mid-Game)** | 需求已明确/竞争者已入场/格局正在形成 | 已有先发优势的公司受益，后来者困难 |
| 🔴 **过晚 (Too Late)** | 格局已定/赢家已清晰/市场已充分定价 | 优势已计入估值，无预期差 |

**支柱 C：执行证据 (Execution Evidence)**

> 是"说到"还是"做到"？早期证据比宏大叙事更有价值。

| 证据强度 | 示例 | 可信度 |
| --- | --- | --- |
| **💎 硬数据 (Hard Evidence)** | 新产品收入数据、用户数、签约订单、专利授权 | 🟢 高 |
| **📋 软数据 (Soft Evidence)** | 产品发布公告、合作意向、行业评测、试点结果 | 🟡 中 |
| **📢 叙事 (Narrative Only)** | 管理层口头承诺、PPT 愿景、概念炒作 | 🔴 低 |

**三支柱综合打分表（必须输出）：**

| 支柱 | 评分 | 变化方向 | 依据（必须有搜索证据） |
| --- | --- | --- | --- |
| A. 能力基础 | [🟢强/🟡中/🔴弱] | [📈增强/➡️持平/📉被追赶] | [事实] [具体证据] |
| B. 市场时序 | [⏳过早/✅窗口期/⚠️中期/🔴过晚] | [⏩加速打开/➡️稳定/⏪正在关闭] | [事实] [市场数据] |
| C. 执行证据 | [💎硬/📋软/📢叙事] | [🚀加速兑现/➡️匀速/🐌停滞/📉退化] | [事实] [具体证据] |

> **变化方向**：三支柱不仅是某一时点的静态快照，更需要捕捉**变化率**。能力在增强还是被追赶？窗口在打开还是关闭？执行证据在加速还是停滞？变化方向直接影响成熟度的迁移速度预判（见 1B.3b）。

## 1B.3 成熟度分级 (Maturity Classification)

> **设计原理**：新兴竞争优势不是二元的（有/无），而是沿着成熟度阶梯逐步演进。不同成熟度对投资判断的影响完全不同。

| 级别 | 名称 | 定义 | 对投资判断的影响 | 判定标志 | 经典历史验证案例 |
| --- | --- | --- | --- | --- | --- |
| **L1** | 🌫️ 概念期 (Concept) | 仅有战略宣示/行业趋势联想，无产品无收入 | 不计入护城河评估，仅作为"可能性监控" | 管理层口头提及/行业研报概念联想/无实质产品 | 2021 多数"元宇宙概念"公司 |
| **L2** | 🔬 验证期 (Validation) | 产品已存在但市场反馈未出/效果未证明 | 标注"前瞻性优势待验证"，不修改静态护城河评级但影响催化剂评估 | 产品已发布或试点中/但无可量化的市场效果数据 | NVIDIA 2022Q4：CUDA+AI训练能力已具备，但数据中心收入尚未爆发 |
| **L3** | 📊 兑现期 (Realization) | 早期市场数据可见（收入贡献/用户增长/订单可查证） | **上调护城河展望**，可影响估值中对增长部分的评估 | 新业务收入开始出现在财报中/订单量可验证/用户规模可追踪 | NVIDIA 2023H1：数据中心收入同比翻倍，AI算力需求得到财报验证 |
| **L4** | 🏆 确立期 (Establishment) | 竞争格局已形成，优势已被财务数据验证 | **并入静态护城河评估**（此时它已不再是"前瞻性"而是"已有"优势） | 新优势贡献已占收入/利润显著比例/竞争者难以复制已得到验证 | NVIDIA 2024：AI算力垄断地位确立，数据中心占收入>75% |

**⚠️ 分级纪律**：
- 只有 **L3 及以上** 才允许对估值和综合评级产生正面影响
- **L1-L2** 仅作为监控和催化剂评估的输入，**不得**用于提升护城河评级或合理化高估值
- 这条规则防止"概念炒作"污染基本面分析
- **概念纯度约束叠加**：如果 1B.1b 概念纯度 <5%，成熟度上限锁定 L2；5-20% 锁定 L3。此约束优先于上表

### 1B.3b 成熟度迁移速度 (Maturity Migration Velocity)

> **设计原理**：当前成熟度（L1-L4）是静态快照——它告诉你"在哪里"，但投资者更需要知道"往哪去"以及"多快到达"。迁移速度决定了 L2 是走向 L3 还是退回 L1，而这对催化剂时间线和估值判断至关重要。

**迁移速度判定（必须输出）：**

| 迁移速度 | 标识 | 判定标准 | 投资含义 |
| --- | --- | --- | --- |
| **快速上行** | 🚀 | 过去 2 个季度内从 Lx 实质升至 L(x+1)，有可验证的里程碑事件 | 催化剂兑现在即，关注估值是否已反映 |
| **正常上行** | ➡️📈 | 过去 4 个季度内有从 Lx 向 L(x+1) 的显著进展 | 正常演进轨道，匹配行业节奏 |
| **停滞** | 🐌 | >4 个季度无实质性里程碑推进 | ⚠️ 警告信号——"伪验证"风险，可能永远卡在 L2 |
| **退化** | 📉 | 有从 L(x) 退回 L(x-1) 的迹象（收入下滑/产品下架/客户流失/竞争者超越） | 🔴 前瞻性优势正在消解，需立即下调增量评估 |

**搜索验证（必须执行）：**

```
搜索 E-velocity: "[公司名] [前瞻性优势关键词] 最新进展 [近2-4个季度时间范围]"
```

**迁移速度对评估的影响规则：**

1. **🚀 快速上行 + L2** → 在催化剂日历中标注"可能在 1-2 个季度内升至 L3"，密切关注
2. **🐌 停滞 + L2** → 标注"验证停滞风险"，前瞻性增量评估降一级（📋→⚪）
3. **📉 退化（任何级别）** → **强制下调**当前成熟度一级，并在 逻辑失效 中标注"前瞻性优势退化"
4. **停滞 + 概念纯度 <20%** → ⚠️ 双重警告：概念炒作 + 执行停滞，前瞻性增量直接归零

**必须输出：**

```
当前成熟度: L[x]
迁移速度: [🚀快速上行/➡️📈正常上行/🐌停滞/📉退化]
速度判据: [具体里程碑事件 + 时间线]
迁移速度对评估的影响: [具体影响描述]
```

## 1B.4 前瞻性护城河增量评估 (Forward Moat Delta)

**将三支柱评分 + 成熟度 + 范式量级 综合为"前瞻性护城河增量"：**

**标准权重矩阵（S1 增量型 或 范式量级不适用时使用）：**

| 成熟度 | 三支柱均🟢/✅/💎 | 部分🟢/✅ | 多数🟡或🔴 |
| --- | --- | --- | --- |
| **L4 确立** | 🔺🔺 强增量 → 并入静态护城河 | 🔺 中增量 | — (矛盾，需复核) |
| **L3 兑现** | 🔺 中增量 → 上调护城河展望 | 🔺⚠️ 弱增量 → 标注观察 | — (数据存疑) |
| **L2 验证** | 📋 监控中 → 纳入催化剂日历 | 📋 监控中 | ⚪ 无增量 |
| **L1 概念** | ⚪ 无增量 → 仅记录 | ⚪ 无增量 | ⚪ 无增量 |

**范式校准权重矩阵（S2 显著型 / S3 范式型时使用）：**

| 成熟度 | S2 显著型 | S3 范式型 |
| --- | --- | --- |
| **L4 确立** | 🔺🔺 强增量 + 估值溢价 | 🔺🔺🔺 战略级增量 + "时代红利"溢价 |
| **L3 兑现** | 🔺🔺 加强上调（比标准矩阵多一级） | 🔺🔺🔺 **战略级上调**：可触发"转型投资豁免"（见 Step 2）★★★ |
| **L2 验证** | 📋 监控 + 催化剂日历标注⭐高影响力 | 📋 **监控溢价**：写入论题"高期权价值"段，催化剂标注⭐⭐最高影响力 ★★ |
| **L1 概念** | ⚪ 无增量（**铁律不变**） | ⚪ 无增量（**铁律不变**） ★ |

> ★ **L1 铁律**：无论范式量级多大，只要还在概念阶段（没有产品、没有收入、没有用户），就不影响评级。这防止了"AI 概念股"式的全面炒作。
>
> ★★ **S3+L2 的"监控溢价"**：不改变当前评级，但在投资论题中增加"如果 L2→L3 跃迁成功，评级可能大幅上调"的**期权性描述**，并在催化剂日历中标注为⭐⭐最高影响力催化剂。
>
> ★★★ **S3+L3 的"战略级上调"**：前瞻性增量直接提升至🔺🔺🔺，且当公司出现短期财务噪音（FCF 转负、费用率上升）时，可触发 Step 2 的"转型投资豁免评估"——区分"战略性烧钱"和"经营性恶化"。

**⚠️ 前瞻性增量的"反面检查"（Anti-Hype Check）：**

```
反面搜索 E6: "[公司名] [新兴优势关键词] 失败 风险 竞争对手 更强"
反面搜索 E7: "[新兴优势关键词] 竞争格局 龙头 主要公司 市场份额"
```

### 1B.4a 竞争景观图谱 (Competitive Landscape Map)

> **设计原理**：前瞻性优势不应在真空中评估。"这家公司有 AI 能力"没有意义——只有"这家公司的 AI 能力在竞争者中处于什么位置"才有意义。强制输出竞争图谱，防止孤立分析。

**搜索指令（必须执行）：**

## Gate Check (验收门禁)

```
□ ⭐ 前瞻性竞争优势六大信号已逐一扫描（锚定 Step 0 行业趋势）
□ ⭐ 如检测到信号 → 概念纯度已计算（收入占比 + 纯度分类 + 成熟度上限约束）
□ ⭐ 如触发单位经济学评估 → 核心指标已按商业模式选取 + 健康度已评估 + 规模化风险已判断
□ ⭐ 如检测到信号 → 三支柱评估已完成（含变化方向） + 成熟度已分级 + 增量已判定
□ ⭐ 如检测到信号 → 迁移速度已判定（🚀/➡️📈/🐌/📉）
□ ⭐ 如检测到信号 → 竞争景观图谱已输出（≥3 家可比公司）+ 排他性已判定
□ ⭐ 如检测到信号 → 反面检查已执行（排他性 + 定价 + 失败条件 + 估值纯度错配）
□ ⭐ 前瞻性评估汇总表已输出（含概念纯度、迁移速度、竞争排他性）
□ 如果概念纯度 <5% → 确认成熟度已锁定 L2 上限
□ 已执行 ≥6 次搜索（含 E0 锚定搜索 + E-purity + E-velocity + E-comp + ≥1 次反面搜索）
□ 共享上下文已更新（前瞻性优势信息已写入）
□ ⭐ 预结论辩证已执行：关键假设已在结论形成前挑战，裁定结果已记录（见 SKILL.md 三步辩证协议）
```

> **预结论辩证提示**：在给出成熟度分级/迁移速度之前——暂停。问自己"我即将判定成熟度为 [LX]，这最依赖什么假设？"然后执行反面搜索。常见需挑战的假设：技术路线是唯一可行方案、市场规模预测可信、公司执行力可持续、竞争对手不会快速追赶。如果反面搜索发现有力证据 → **修改结论**。
