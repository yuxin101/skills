# Step 9: 报告生成 (Output Generation)

> **目的**：将全流程分析结果整合为结构化 Markdown 研究报告。
> **核心原则**：结论驱动、数据为证、简洁有力。投资者关注结论和依据，不要让格式淹没内容。

## 9.2 [事实] vs [推演] 标注规则 — ⛔ 致命级

| 标签     | 含义                             | 示例                                              |
| -------- | -------------------------------- | ------------------------------------------------- |
| **[事实]** | 来自搜索结果/财报/公告的确证数据 | "**[事实]** 2025Q3 营收同比 +15%（来源：季报）"       |
| **[推演]** | 基于事实的逻辑推导或预测         | "**[推演]** 预计 2026 年毛利率改善（逻辑：成本下降）" |

规则：

1. 每段关键分析至少 1 个标注
2. 定量数据必须标 **[事实]** + 数据来源
3. 未来预期必须标 **[推演]** + 推理依据
4. ⛔ 禁止将推演伪装为事实
5. ⛔ 标注必须加粗：输出为 `**[事实]**`、`**[推演]**`，不允许不加粗的 `[事实]`、`[推演]`

## 9.3 来源归因（内联标注，不设独立章节）

| 数据类型   | 归因要求      | 格式                         |
| ---------- | ------------- | ---------------------------- |
| 财务数据   | 来源 + 时间   | "营收 ¥X亿（年报, YYYYQX）"  |
| 股价数据   | 来源 + 日期   | "收盘价 ¥X（YYYY-MM-DD）"    |
| 分析师预测 | 机构名 + 日期 | "预测 X%（XX证券, YYYY-MM）" |
| 行业数据   | 来源 + 时间   | "市场规模 X亿（来源, YYYY）" |

**⛔ 来源可信度约束：**

- 禁止使用低可信度转载平台（豆丁网、道客巴巴、百度文库等）作为唯一来源
- 如仅从转载平台获取数据，必须标注"（转引自[原始机构]，未能验证原文）"
- 优先使用：官方财报/公告 > 券商研报 > 权威财经媒体 > 行业机构报告

## 9.4 报告结构模板

### ⛔ 精简输出原则 — 强制执行

> 投资者需要的是**结论+关键证据**，而非分析过程的完整记录。

1. **结论前置**: 每章第一行直接给核心结论，不做铺垫
2. **苏格拉底引导（简化版）**: 每章结论后紧跟 1 行斜体关键问题（引导读者独立判断）。格式：`*关键问题：...？*`。不使用引用块 `>`，仅用斜体。
3. **禁止元信息**: 不输出"生成时间"、"分析模型"等元数据头
4. **禁止规范性标题**: 章节标题简洁，不加"（仅文字呈现）"等说明
5. **表格优先**: 同类数据用表格呈现，不写成段落
6. **删除套话**: "从宏观角度来看"、"综合以上分析"等修饰语全部删除
7. **不设独立的来源归因/信息验证章节**: 来源在正文中内联标注
8. **纠错透明度**: 报告末尾（免责声明前）输出 1 行纠错状态声明
9. **字数控制**: 总报告 3500-5000 字，宁短勿长
10. **禁止伪精确**: 情景矩阵中股价表现用涨跌幅区间（如"+20%-30%"），禁止给出精确目标价

### 视觉规范 — 必须遵守

- **决策仪表盘**: 每行评估列必须含状态 icon（🟢🟡🔴⚪📈←↗等）
- **章节编号**: 使用中文数字序号 `## 一、` `## 二、`，子节用阿拉伯数字 `### 1.` `### 2.`
- **护城河矩阵**: 逐项 ✅存在/❌无 检查，附量化指标
- **财务红旗**: 使用 🚩 前缀，严重度标注
- **反面审查裁决 icon**: ✅辩护成立 / ⚠️部分成立 / 🔴质疑成立
- **催化剂**: 类型用 icon（📊财报/🚀产品/🏭产能/📜政策/🌍国际），方向用 🟢正/🔴负
- **逻辑失效**: 表格化（类型+条件+验证方法）
- **表格中关键公司/指标**: 用 **粗体** 突出
