# Step 0: 宏观环境与行业定位（轻量快照）

> **目的**：在深入个股之前，建立宏观和行业坐标系。避免在"水位下降时捞鱼"。
> **设计原则**：个股分析的宏观模块应轻量但不可缺失。Sector-analysis 做深度行业分析，本步骤做"快照定位"。
> **理论依据**：Peter Lynch 的自上而下框架 — "不要在经济衰退期买周期股"；Howard Marks 的"了解你在哪里"（《周期》）。

---

## 0.1 宏观环境快照

**搜索指令（必须执行）：**

```
搜索 1: "[目标市场] 宏观经济 货币政策 [当前年份]"
搜索 2: "[目标市场] 利率 流动性 [当前季度]"
```

**宏观环境评级矩阵：**

| 评级 | 条件 | 对个股影响 |
| --- | --- | --- |
| 🟢 宽松 | 降息周期/流动性充裕/经济扩张 | 估值扩张空间大，成长股受益 |
| 🟡 中性 | 利率稳定/经济平稳 | 个股 alpha 为主，beta 有限 |
| 🔴 紧缩 | 加息周期/流动性收紧/经济放缓 | 估值压缩风险，防御股相对受益 |
| ⚫ 危机 | 系统性风险/流动性枯竭/黑天鹅 | **⛔ 终止分析** → 输出风险警告 |

**宏观对个股估值的传导路径（必须评估）：**

```
利率环境 → 无风险收益率 → 折现率 → 估值中枢
经济周期 → 企业盈利 → EPS 增速 → 估值支撑
流动性 → 风险偏好 → 估值溢价 → PE 扩张/收缩
```

## 0.2 行业景气度定位

**搜索指令（必须执行）：**

```
搜索 3: "[公司名] 所属行业 景气度 [当前年份]"
搜索 4: "[行业名] 增速 市场规模 竞争格局 [当前年份]"
```

**行业景气度评级：**

| 评级 | 特征 | 对个股含义 |
| --- | --- | --- |
| 📈 上行 | 行业增速加快/需求扩张/供给优化 | 行业 beta 正向，龙头更受益 |
| 🔁 稳定 | 行业增速平稳/供需平衡 | 纯 alpha 博弈，看公司竞争力 |
| 📉 下行 | 行业增速放缓/产能过剩/需求萎缩 | 慎选，除非公司有逆势份额提升逻辑 |

**行业生命周期快速定位：**

| 阶段 | 特征信号 | 典型估值特征 |
| --- | --- | --- |
| 朝阳期 | 渗透率<20%，增速>30%，玩家少 | 高 PS，不看 PE |
| 成长期 | 渗透率 20-50%，增速 15-30%，龙头初现 | PE 30-50x，看增速 |
| 成熟期 | 渗透率>50%，增速<15%，格局稳定 | PE 15-25x，看分红和回购 |
| 衰退期 | 负增长，替代品兴起，退出者增多 | 低 PE 可能是价值陷阱 |

## 0.2b 行业概念发展脉络与衍生概念追踪

> **设计原理**：个股的中长期价值锚定于其所处行业概念的"发展阶段"和"衍生路径"。概念不是静态标签，而是一棵不断分裂的树——AI 从"大模型训练"分裂出"推理加速""AI 应用""AI 终端"等衍生概念，每个衍生概念对应不同的公司受益窗口和商业化节奏。
>
> **为什么是基本面级别**：
> 1. 概念的成熟阶段决定行业估值中枢（萌芽期看 PS，爆发期看增速 PE，成熟期看 FCF/分红）
> 2. 衍生概念的分裂路径决定哪些公司真正受益（上游 vs 下游 vs 应用层）
> 3. 概念演进的时间节奏决定催化剂日历和兑现窗口
>
> **典型案例（经典历史验证）**：
> - 2022年底 "AI 算力" 概念 → 衍生出 "大模型训练""推理部署""AI 应用""AI 终端" → 英伟达处于"算力"核心节点，概念成熟度从萌芽快速进入爆发期
> - 2021年 "元宇宙" 概念 → 衍生出 "VR/AR 硬件""虚拟社交""数字人""3D 引擎" → 多数衍生概念停滞在萌芽期，未能向爆发期迁移

**搜索指令（必须执行）：**

```
搜索 C1: "[公司所属行业] 核心概念 发展趋势 [当前年份]"
搜索 C2: "[公司核心概念关键词] 产业链 衍生 细分赛道 [当前年份]"
搜索 C3: "[衍生概念关键词] 商业化进展 落地 [当前年份]"
```

**概念发展脉络映射（必须输出）：**

│   └── 发展阶段: [萌芽/导入/爆发/成熟/衰退]
├── 衍生概念 B: [名称] — 与公司关联度: [强/中/弱]
│   └── 发展阶段: [萌芽/导入/爆发/成熟/衰退]
├── 衍生概念 C: [名称] — 与公司关联度: [强/中/弱]

| --- | --- | --- | --- | --- |

**概念发展五阶段定义：**

| 阶段 | 信号特征 | 典型估值特征 | 对公司的含义 |
| --- | --- | --- | --- |
| 🌱 萌芽期 | 技术论文/实验室/早期融资 | 概念股炒作，无业绩锚 | 纯叙事驱动，无法验证 |
| 🌿 导入期 | 首批商业产品/标杆客户/政策鼓励 | 看 PS/用户数/渗透率 | 关注公司是否已布局 |
| 🔥 爆发期 | 渗透率加速（10→30%）/多家盈利/融资密集 | 看增速 PE，估值容忍度高 | 最大受益窗口，拼速度和份额 |
| 🏭 成熟期 | 渗透率>50%/增速趋缓/格局稳定 | 回归 PE/FCF/分红 | 看份额和壁垒 |
| 📉 衰退期 | 负增长/替代品兴起/退出者增多 | 低估值可能是价值陷阱 | 警惕 |

步骤 2C — 跨板块概念全扫描 ⭐ 必填：

> **设计原理**：A 股市场的定价逻辑不受一级行业分类限制。一家光伏公司可能因为航天概念被归入商业航天赛道炒作；一家化工公司可能因为氢能源概念获得新能源估值溢价；一家养殖公司可能因为合成生物概念获得科技板块关注。**只看主业板块的衍生概念，等于只看了投资者定价逻辑的一半。**
>
> **核心规则**：**概念扫描不能被主业板块限制。必须搜索公司关联的所有市场概念，包括跨行业、跨板块的题材联动。** 如果报告只关注主业板块内部的概念脉络，会完全遗漏跨板块驱动的股价运动，导致"基本面看空但股价看涨"的归因失败。

**搜索指令（⛔ 必须执行 — 不可跳过）：**

```
搜索 CS1: "[公司名/代码] 概念 题材 板块"
搜索 CS2: "[公司名/代码] 近期上涨原因" 或 "[公司名/代码] 近期下跌原因"
搜索 CS3: "[发现的跨板块概念] [公司名] 关联"（对每个新发现的跨板块概念执行）
```

**跨板块概念识别与分类（必须输出）：**

| 概念名称 | 所属板块 | 与主业关系 | 公司实质关联度 | 市场热度 | 对股价影响 |
| --- | --- | --- | --- | --- | --- |
| [概念A] | [板块名] | [主业内/跨板块] | [实质/概念/纯炒作] | [🔥高/🟡中/⚪低] | [显著/辅助/微弱] |
| [概念B] | [板块名] | [主业内/跨板块] | [实质/概念/纯炒作] | ... | ... |
| ... | ... | ... | ... | ... | ... |

**公司实质关联度的三级判定：**

| 级别 | 判定标准 | 分析影响 |
| --- | --- | --- |
| **实质关联** | 公司已有相关业务/产品/收入/技术储备，有可验证的事实依据 | 纳入概念脉络核心分析，影响前瞻性竞争优势评估 |
| **概念关联** | 公司主业具备延伸可能性但尚无实质业务，市场以"想象空间"定价 | 纳入概念脉络但标注为 `[推演]`，影响"市场在定价什么"的归因分析 |
| **纯炒作** | 公司与该概念无实际关联，仅因名称/业务表面相似被市场误关联 | 标注为"炒作风险"，在反面审查中作为反面证据 |

**⛔ 关键规则：**
1. **即使概念关联为"纯炒作"，只要它正在显著影响股价，报告也必须讨论**——因为投资者需要知道"市场在定价什么"
2. **跨板块概念如果被评定为"实质关联"或"概念关联"，必须纳入 Step 0.2b 的概念发展树**
3. **如果搜索 CS2 发现近期股价运动由跨板块概念驱动，而 Step 0.2b 的概念树中未包含该概念 → 概念树不完整，必须补充**

**必须输出：**

```
跨板块概念扫描结果: [发现 X 个跨板块关联 / 未发现跨板块概念]
关键跨板块概念: [名称] — 关联度: [实质/概念/纯炒作] — 市场热度: [🔥/🟡/⚪]
近期股价驱动力归因: [主业基本面 / 跨板块概念炒作 / 宏观流动性 / 混合]
```

步骤 2D — 需求本质检查（世俗力量三问）：

> **设计原理**：避免投资"柯达"或"诺基亚"——行业景气度是周期性指标，但"下一代人是否还需要这个东西"是世俗性判断，决定了周期底部是否真正是底部。

**必须搜索并回答以下三个核心问题：**

| 核心问题 | 思考逻辑 | 搜索关键词 |
| --- | --- | --- |
| **受众迁移** | 下一代消费者/用户是否会比这一代更需要/更认同公司的核心产品？ | `[公司核心产品/行业] 年轻人 态度`, `[行业] 用户年龄分布 变化` |
| **范式转换** | 是否存在根本性的替代方案正在改变游戏规则？（与 Step 0.5 范式威胁呼应但聚焦公司层面） | `[公司核心产品] 被替代`, `[替代品] 增长率 vs [传统产品]` |
| **需求本质** | 公司满足的需求是"永恒的人性"还是"特定时代的产物"？ | `[行业] 需求变化 趋势`, `[公司核心产品] 长期前景` |

**需求本质判定：**

| 判定 | 含义 | 对投资论题影响 |
| --- | --- | --- |
| 📈 **上升** | 需求在扩散到新人群，没有替代威胁，解决的是永恒问题 | 长期论题基础坚实 |
| ➡️ **稳态** | 核心需求不随时代变化，受众稳定 | 看重当前竞争力和估值 |
| 📉 **下降** | 核心用户群在老去且新生代不接棒，或被范式转换替代 | ⚠️ **世俗性风险**——即使当前财务健康，长期论题有硬伤 |

**必须输出：**

```
需求本质判定: [📈上升/➡️稳态/📉下降]
受众迁移: [一句话结论]
范式转换威胁: [一句话结论]
需求永恒性: [一句话结论]
```

> **⚠️ 如果需求本质 = 📉下降，此信息必须写入共享上下文，并在 Step 5 反面审查中被强制引用。**

## 0.4 监管环境评估

**搜索指令（必须执行）：**

```
搜索 6: "[公司名/行业] 监管政策 [当前年份]"
```

**监管评级（五级制 — 区分"规范化"与"毁灭级"监管）：**

> **设计原理**：大多数 AI 分析工具一遇到"监管"二字就恐慌性降级，但"出台管理办法"往往是行业被官方认可的标志。中国市场尤其需要区分"管而不禁"（规范化发展）与"一刀切"（行业级风险）。

| 评级 | 含义 | 对投资论题影响 |
| --- | --- | --- |
| 🟢 友好 | 政策鼓励/补贴/产业基金支持 | 估值享受政策溢价 |
| 🟡 中性 | 无重大政策变化 | 不影响投资论题 |
| 🟡中低 规范化 | 出台管理办法/行业标准/牌照制度 | **这通常是利好信号**：行业被官方认可→劣币出清→龙头受益。合规成本是短期成本，市场集中是长期利好 |
| 🟠 中高 | 针对性整治/罚款/限制部分商业模式 | 降低盈利预期，注意合规成本上升，但行业仍可存续 |
| 🔴 毁灭级 | 行业被取缔/核心商业模式被禁止/准入完全封死 | **论题可能无效**。教培双减、P2P清退级别 |

**⚠️ 监管判定四要点（防止过度恐慌）：**
1. "出台管理办法" ≠ 收紧 → 可能是 🟡中低 规范化
2. "治理整顿部分违规参与者" ≠ 行业级风险 → 龙头可能受益（劣币出清）
3. "短期股价下跌因监管消息" ≠ 长期风险 → 区分市场情绪反应和实质影响
4. 只有"彻底禁止核心商业模式"才能触发 🔴 毁灭级

## 0.5 范式威胁排查 (Paradigm Threat Check)

> **设计原理**：石油替代鲸油、互联网重构传媒、电商颠覆零售——每一次范式级变革中，最大的投资风险不是"买错公司"，而是"站在历史的错误一边"。1870 年的捕鲸公司财报漂亮、估值合理——但行业已注定消亡。
> **本步骤目的**：当 0.2 中识别到行业存在显著技术/模式变革趋势时，评估目标公司相对于这一变革的位置。

**触发条件**：0.2 行业景气度/生命周期评估中出现以下任一信号即触发：
- 行业生命周期出现"转型/再成长"迹象
- 识别到新技术/新模式对传统业态的潜在替代
- 行业出现跨界竞争者或颠覆性产品

**搜索指令（触发后必须执行）：**

```
搜索 P1: "[行业关键技术/趋势] 对 [传统行业/业态] 冲击 替代 影响"
搜索 P2: "[公司名] 在 [新趋势] 中的角色 布局 定位"
```

**范式量级三级分类：**

| 量级 | 定义 | 判定标准 | 历史类比 |
| --- | --- | --- | --- |
| **S1 增量型** | 现有范式内的效率提升或产品改良 | 优化现有流程，不改变行业结构 | 更快的 CPU、更好的 UI |
| **S2 显著型** | 行业内的新方法替代旧方法，但不改变行业边界 | 核心价值传递方式改变，但终端需求不变 | 电动车替代燃油车、流媒体替代 DVD |
| **S3 范式型** | 跨行业颠覆，重新定义价值链，旧行业面临存亡级威胁 | 必须同时满足 ≥3 条 S3 判定标准（见下） | 石油→捕鲸消亡、互联网→传媒重构 |

**S3 范式型判定标准（必须同时满足 ≥3 条）：**

| 判定条件 | 检验方法 | 搜索验证要求 |
| --- | --- | --- |
| **技术通用性** | 该技术是否可应用于 ≥3 个不同行业？ | 搜索跨行业应用案例 |
| **成本曲线颠覆** | 是否能将某项核心功能的成本降低 ≥10 倍？ | 搜索成本对比数据 |
| **价值链重构** | 是否改变了行业价值从创造到传递到捕获的基本链条？ | 搜索行业价值链变化分析 |
| **监管/制度响应** | 政府是否已为该技术制定专门政策框架？ | 搜索政策文件 |
| **资本密度信号** | 全球对该领域的投资是否在 3 年内增长 ≥5 倍？ | 搜索投资/融资数据 |

**⚠️ S3 误判防护：**
1. 必须有 **≥2 个独立搜索来源**确认范式量级
2. 如果仅有媒体炒作而无产业数据支撑 → 降级为 S2 或 S1
3. S3 判定每次分析**独立评估**，不继承历史判定

**公司角色定位（范式量级 ≥S2 时必须评估）：**

| 公司角色 | 定义 | 对后续分析的影响 |
| --- | --- | --- |
| 🗡️ **颠覆方** | 公司是新范式的推动者/核心受益者 | Step 1.6 前瞻性优势评估适用，范式量级作为权重调节器 |
| 🔄 **转型方** | 传统公司正在向新范式主动迁移 | 触发"转型投资豁免评估"（Step 2），需评估转型质量 |
| ☠️ **被颠覆方** | 公司核心业务面临新范式的存亡级威胁 | **⚠️ 评级上限约束**：S3+☠️ → 最高🟡谨慎乐观，即使当前财务健康 |
| 👀 **旁观者** | 范式变革与公司无直接关联 | 无影响，标准流程 |

**⚠️ 被颠覆方强制标注规则：**
- S2 + ☠️ → 报告中标注 `[⚠️ 行业转型风险]`
- S3 + ☠️ → 报告中标注 `[☠️ 范式威胁警告]`，评级上限锁定🟡，且必须在投资论题中写入"核心业务面临范式级存亡威胁"

**必须输出：**

```
范式量级: [S1增量型/S2显著型/S3范式型/不适用]
(如 S3) 满足的判定标准: [列出 ≥3 条 + 搜索证据]
公司角色: [🗡️颠覆方/🔄转型方/☠️被颠覆方/👀旁观者/不适用]
对后续分析的影响: [具体说明]
```
