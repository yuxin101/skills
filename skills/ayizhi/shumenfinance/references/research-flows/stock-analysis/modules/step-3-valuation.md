# Step 3: 多维度估值分析

> **目的**：通过多种估值方法的交叉验证，判断当前股价相对于内在价值是低估、合理还是高估。
> **核心原则**：**禁止输出单一精确目标价**。一切估值以**区间**表达。
> **理论依据**：Aswath Damodaran 的估值三角（内在价值/相对价值/或有价值）；Benjamin Graham 的安全边际原则；Greenwald 的 EPV (Earnings Power Value) 方法论；均值回归假设。

---

## 3.1 估值方法选择矩阵

**根据 Step 1 确定的商业模式类型，选择适用的估值方法：**

| 商业模式类型 | 主要方法 | 辅助方法 | 不适用方法 |
| --- | --- | --- | --- |
| 🏭 制造型 | EV/EBITDA + PB | PE（正常化）+ DCF 感知 | PS |
| 💻 平台型 | PS + EV/Revenue | PE（如已盈利）+ 用户价值 | PB |
| 🔄 订阅型 (SaaS) | ARR 倍数 + Rule of 40 | EV/Revenue + DCF 感知 | PB |
| 🛍️ 消费品牌型 | PE + EV/EBITDA | DCF 感知 + 品牌溢价 | PB（轻资产） |
| 🏦 金融型 | PB + 股息率 | PE + ROE-PB 矩阵 | EV/EBITDA |
| ⛏️ 资源型 | EV/储量 + 成本曲线 | PB（NAV）+ 周期正常化 PE | PS |
| 💊 研发驱动型 | Pipeline 概率加权 | PS（如有收入）+ EV/R&D | PE（未盈利时） |

**⚠️ 至少使用 2 种方法进行交叉验证。单一方法结论不可作为最终判断。**

## 3.2 可比公司估值 (Comparable Company Analysis — 可比公司估值) ⭐ 华尔街标准

> **理论依据**：Citi 权益研究标准 可比公司估值 方法论 — 可比公司分析的核心不是简单对比 PE，而是通过系统性筛选同业、多维倍数对比、增速调整，定位目标公司在同行估值分布中的合理位置。

**搜索指令（必须执行，迭代搜索直至获取充分数据）：**

```
搜索 1: "[公司名/代码] PE EV/EBITDA 估值 [当前年份]"
搜索 2: "[行业] 上市公司 估值 PE EV/EBITDA 对比 排名"
搜索 3: "[公司名/代码] 主要竞争对手 同行业公司 市值"
```

### Peer Group 筛选方法论

**筛选标准（按优先级排序）：**

| 优先级 | 筛选维度 | 说明 |
| --- | --- | --- |
| P1 | **业务相似度** | 核心业务/收入结构与目标公司高度相似 |
| P2 | **规模可比性** | 市值在目标公司 0.3x-3x 范围内（宽泛可接受） |
| P3 | **增长阶段** | 处于相似的生命周期阶段（高增长 vs 成熟 vs 下行） |
| P4 | **地域/市场** | 同一国家或可比市场 |

**⚠️ Peer 数量要求：至少 5 家，目标 8 家。如行业内可比公司不足 5 家，可扩展至相邻子行业，但需标注。**

> **百分位解读**：目标公司在可比组中处于 [XX%] 百分位，意味着 [XX%] 的可比公司估值更低。

### 估值溢价/折价深度分析

| 目标 vs 中位数 | 含义 | 合理性校验 |
| --- | --- | --- |
| 显著溢价 (>30%) | 市场认为目标更优质或增速更高 | **必须验证**：增速差异是否匹配溢价？ROE 是否领先？护城河是否更宽？ |
| 小幅溢价 (0-30%) | 轻微看多 | 通常合理，确认护城河/增速 |
| 折价 (<0%) | 市场有顾虑 | **必须调查**：折价来源是风险（正确定价）还是被忽视（Alpha 机会）？ |

**增速调整估值（PEG 校验，必须输出）：**

```
目标公司 PEG = PE(Fwd) / 未来1年净利增速(%) = [X]
可比组 PEG 中位数 = [Y]
→ PEG < 1: 增速支撑估值 / PEG 1-2: 合理 / PEG > 2: 估值透支增长
→ 目标 PEG vs 可比组 PEG: [低于(可能低估)/接近(合理)/高于(可能高估)]
```

**⚠️ 可比估值的局限性**：可比法假设市场对同行业定价合理。如果整个行业处于泡沫/恐慌期，可比法会系统性失效。必须结合历史分位（3.3）和 DCF-Lite（3.4）交叉验证。

**历史分位判断表：**

| 当前分位 | 含义 | 投资含义 |
| --- | --- | --- |
| 0-20% 分位 | 历史估值底部区域 | 可能被低估（排除基本面恶化） |
| 20-40% 分位 | 偏低 | 有安全边际 |
| 40-60% 分位 | 中枢附近 | 合理区间 |

| 陷阱 | 说明 | 对策 |
| --- | --- | --- |

## 3.4 DCF-Lite 估值 (Simplified DCF Valuation) ⭐ 华尔街标准

> **设计理念**：传统 DCF 需要精确的 Beta、WACC、Terminal Growth Rate，公开搜索难以获取。DCF-Lite 采用**行业基准 WACC + Step 2.2b 前瞻预测数据**进行简化但结构化的内在价值估算，同时保留"隐含增长率反推"作为交叉验证。
>
> **理论依据**：Aswath Damodaran 的 DCF 哲学 — "I would rather be vaguely right than precisely wrong"（宁要模糊的正确，不要精确的错误）。DCF-Lite 追求的是方向性正确的估值区间，而非伪精确的点估值。

**搜索指令（必须执行）：**

```
搜索 5: "[公司名/代码] 自由现金流 FCF [最近年份]"
搜索 6: "[公司名/代码] 市值 当前 企业价值 EV"
搜索 7: "[公司名所在行业] WACC 折现率 资本成本"
```

### Step A: WACC 估算（行业基准法）

> 无法精确计算 WACC 时，使用行业基准 + 风险调整。

| 行业类型 | 基准 WACC 区间 | 风险调整 |
| --- | --- | --- |
| 消费品/公用事业 | 7-9% | 稳定型，低风险溢价 |
| 制造业/工业 | 8-10% | 标准风险 |
| 科技/互联网 | 9-12% | 增长型，较高不确定性 |
| 生物医药/研发驱动 | 11-15% | 高度不确定 |
| 金融 | 8-11% | 杠杆型，系统风险 |
| 资源/周期 | 9-12% | 价格波动大 |

**WACC 风险调整因子：**
- 小市值（<100 亿）：+1-2%
- 财务健康度 🟠/🔴（Step 2.4）：+1-2%
- 治理/信息透明度低：+1%

```
选用 WACC = [X%] (行业基准 [Y%] + 风险调整 [Z%])
```

### Step B: 3 年 FCF 预测 + 终值

**从 Step 2.2b 承接 FCF 预测：**

| 年份 | FCF (亿元) | 折现因子 (1/(1+WACC)^n) | 现值 |
| --- | --- | --- | --- |
| Y+1 | [Step 2.2b] | | |
| Y+2 | [Step 2.2b] | | |
| Y+3 | [Step 2.2b] | | |
| **预测期现值合计** | | | **[A]** |

**终值 (Terminal Value) 计算：**

```
终值 = Y+3 FCF × (1 + g) / (WACC - g)
其中 g = 永续增长率（通常 2-3%，不超过 GDP 长期增速）
终值现值 = 终值 / (1+WACC)^3 = [B]
```

**企业价值 = [A] + [B] = [EV_dcf]**
**股权价值 = EV_dcf - 净负债 + 现金 = [Equity_dcf]**
**每股内在价值 = Equity_dcf / 总股本 = [¥X]**

**⚠️ DCF-Lite 诚实声明**：此估值高度依赖 FCF 预测和 WACC 假设。±1% WACC 或 ±1% 永续增长率可能导致估值变动 20%+。因此输出**估值区间**而非点值：

```
DCF-Lite 估值区间: ¥[下限] - ¥[上限]
(下限 = 悲观 FCF + 高 WACC, 上限 = 乐观 FCF + 低 WACC)
当前股价: ¥[X] → [低于/处于/高于] DCF 区间
```

### Step C: 隐含增长率反推（交叉验证）

> 反向检验：当前股价隐含了什么增长预期？该预期是否合理？

```
当前市值 = [搜索获取]
近 12 个月 FCF = [搜索获取]
→ 隐含 FCF 永续增长率 = WACC - (FCF / 市值) = [X%]
```

| 隐含增长率 vs 历史/预测增速 | 判断 |
| --- | --- |
| 隐含增长 < 历史增速 且 < Step 2.2b 预测增速 | 市场预期保守，可能被低估 |
| 隐含增长 ≈ Step 2.2b 预测增速 | 市场定价合理 |
| 隐含增长 > 预测增速 × 1.5 | 市场预期激进，估值泡沫风险 |
| 隐含增长 > 行业天花板增速 | ⚠️ 过度乐观，除非有颠覆性创新 |

### Step D: DCF 场景矩阵

| 场景 | FCF 增速假设 | WACC | 永续 g | 估值 (每股) | vs 当前股价 |
| --- | --- | --- | --- | --- | --- |
| 🐂 乐观 | Step 2.2b 乐观 | 基准-1% | 3% | ¥[X] | [低估X%/高估X%] |
| 📊 基准 | Step 2.2b 基准 | 基准 | 2.5% | ¥[Y] | [低估X%/高估X%] |
| 🐻 悲观 | Step 2.2b 悲观 | 基准+1% | 2% | ¥[Z] | [低估X%/高估X%] |

## 3.5 特殊估值场景处理

**周期股正常化估值：**

- **不使用当期 PE**（利润高位时 PE 看似便宜=陷阱）
- 使用**过去完整周期的平均盈利**作为正常化利润基础
- 参考 EV/EBITDA 或 PB + 历史 ROE 带

```
搜索: "[公司名/代码] 周期 利润 历史 平均"
```

**高增长股 PEG 校验：**

- PEG = PE / 盈利增速（%）
- PEG < 1 = 增速支撑估值
- PEG > 2 = 估值透支增长

**亏损/微利公司估值：**

- 使用 PS + 参考 SaaS/生物医药 行业惯例
- 关注亏损来源（投资期亏损 vs 业务恶化亏损）

## 3.5b 分部估值 (Sum-of-the-Parts Valuation — SOTP) ⭐ 条件触发

> **理论依据**：Evercore 重组顾问标准 SOTP 方法论 — 当公司拥有多个独立经营的业务分部时，整体估值可能无法反映各分部的真实价值。拆分估值再汇总，可识别"集团折价"或"隐藏价值"。
>
> **触发条件（满足任一即触发，否则跳过）：**
> 1. 公司明确拥有 **≥2 个独立业务分部**（不同行业/商业模式），且各分部在财报中有独立披露
> 2. Step 1.1 收入结构显示**非主业收入占比 >25%**
> 3. 公司为**控股集团/综合企业**（如复星、中信等）

**搜索指令（触发后必须执行）：**


## 3.7 敏感性与情景分析 (Sensitivity & Scenario Analysis) ⭐ 华尔街标准

> **理论依据**：UBS 风险管理标准 — 单点估值是危险的。投资决策的核心不是"值多少钱"，而是"在什么假设下值多少钱，如果假设变了会怎样"。敏感性分析将估值从一个点展开为一个面，暴露估值对关键变量的依赖程度。
