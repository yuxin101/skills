# Alerts

## Summary
Set alerts to catch rate limit issues, spikes in usage, or webhook failures.

## Guidance
- Alert on high error rates or `429` spikes.
- Alert on webhook delivery failures.
- Use multiple channels (email, Slack, PagerDuty) if available.
