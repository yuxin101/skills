# 专业级行程输出（亲子 / family-trip-travel）

与 `SKILL.md` 中「图 + 链」「能力边界」并列使用：本节约束**攻略式结构、信息可信度、对比逻辑与合规表述**，使回答读起来像 **专业旅行杂志 / 深度攻略**（能照着排行程、能点链预订），而非闲聊或内部交付文档。

## 1. 信息架构（攻略体 · 建议顺序）

按用户问题复杂度选用；**简单一问一答可压缩，但不得打乱「先帮读者定方向、再给依据与动作」**。

**对用户显示的标题**须用攻略语（下表「用户可见标题示例」）；下表「代理构思要点」可与 SKILL 内部表述对应，**勿把左列原词照抄给用户**。

| 用户可见标题示例 | 代理构思要点 |
|------------------|----------------|
| **这篇怎么帮你选** / **快速选方向** | 导语 2～5 句：适合谁、首推哪条线、还有哪些要读者补充；可加一句场景钩（周末/低龄/怕下雨等）。 |
| **你的行程条件** | 已给条件表格化；未给的关键项（年龄、日期、预算）标 **建议补充** 或 **行前确认**。 |
| **目的地怎么选** / **几条线怎么挑** | 主方案 + 备选；**矩阵或表格**对比，避免只有散文。 |
| **推荐这样玩** / **一日（多日）行程** | 按半日/全日或 Day1/Day2；每段：**活动、时长量级、车程或步行、带娃备注**。 |
| **预订参考** | SKILL 必选：**先图、后链**；再写名称、价格（仅数据里有的写，无则「以预订页为准」）、评分同理；每条可缀 **一句适用场景**。 |
| **出发前核对** | 票规、退改、证件、安全、过敏、天气 Plan B；**不编造**具体票价与政策条文。 |

英文用户：标题可用自然攻略语（如 *How this guide helps*, *Your trip at a glance*, *What to book*），规则等价。

## 2. 攻略文风与用语

- **编辑感**：段落长短错落；重要信息用 **小标题、表格、短清单** 呈现，方便手机扫读。
- **克制**：避免夸张营销词（「必去」「震撼」「绝了」）；改用可验证表述（「4A 景区」「含演出场次」需来自数据或官方描述）。
- **精确**：地名、景区名与数据 / 官方一致；区县级地址可引用 POI `address`。
- **单位**：车程用「约 X 小时 / 分钟」、步行用「约 X 公里或步数级」；避免模糊「不远」「很近」除非同时给量级。
- **不确定显式标注**：对推断内容使用 **「通常」「一般」「建议行前确认」**；与 **平台/列表里写明的信息** 分开写（对用户勿写「JSON」）。

## 3. 可信度与来源

| 内容类型 | 写法要求 |
|----------|----------|
| 来自 `flyai` JSON 的字段 | 可直接引用（名称、链接、图片、部分价格/票名）。 |
| JSON 未返回的价格、儿童规则、退改 | 统一写 **以预订页 / 航司 / 景区当日公示为准**。 |
| 通识性建议（午休、推车、排队） | 可写，但勿伪装成某酒店/景区的官方承诺。 |

禁止：捏造评分细则、捏造库存、捏造「含儿童早餐」等套餐细节（除非 JSON 或用户粘贴的条款中有）。

## 4. 比选矩阵（示例列）

对 2～3 个候选目的地或产品，至少包含：

- **单程交通 / 体力**（低中高）
- **天气敏感度**（户外为主 / 可室内替代）
- **适龄**（婴儿车友好度、身高项目限制—无数据则写「行前核对」）
- **行程弹性**（是否适合午休、单人带娃）

## 5. 儿童与特殊需求

- **年龄与票种**：只陈述「常见需核对项」，不代填具体身高票价。
- **过敏、医疗、无障碍**：提示用户向酒店 / 航司 / 景区**书面或电话确认**；不做医学承诺。
- **单人带娃**：在日程中标注 **动线简单 / 换乘少** 等可执行点。

## 6. Markdown 与版式

- 同级问题用同级标题；避免跳级 `###` → `#####`。
- 步骤用有序列表；并列项用无序列表或表格。
- 链接使用有意义的锚文（「预订」「详情」），与 SKILL 的图链顺序一致。

## 7. 与「必选图 + 链」的关系

攻略专业度**不降低**图链要求：每条来自检索的推荐仍须 **有图显图、有链显链**。图链上方用 **攻略式一行导语**（类型 + 适合谁），例如：「水乐园 + 住宿套餐 · 想省心放电、可接受周末人潮的家庭」，提升可扫读性。

## 8. 用户可见文案（非技术读者）

- **禁止**在向用户输出的标题与正文里出现：`flyai`、`fliggy-fast-search`、`search-poi`、其他子命令名、`JSON`、`jq`、字段名、「检索 JSON」等（与 `SKILL.md`「用户可见文案」一致）。
- **可信度与来源**（见第 3 节）在写给用户时改用：**「平台页面显示」「预订页公示」**，勿写「JSON 里有/没有」。
- 代理内部仍可按 skill 用命令与字段解析；**内外分层**，不把内部术语泄漏到用户界面。
