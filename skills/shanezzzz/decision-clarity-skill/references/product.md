# Product

Use this file for feature prioritization, MVP scope, onboarding, user-job analysis, and product simplification.

## Core orientation

Most product confusion comes from mixing these layers:

- user outcome
- workflow
- interface
- feature
- implementation

Do not assume the requested feature is the real solution.

## Key lenses

Examine:

- user job
- current friction
- desired outcome
- minimum capability
- feature assumption load
- complexity added by each layer
- time-to-value

## Strong questions

- What user outcome is actually blocked?
- Is this feature the smallest intervention that solves it?
- Are we building this because users need it, or because the product is unfocused elsewhere?
- What can be removed while preserving the core result?
- What is the smallest product shape that still creates real value?

## Typical outputs

- tighter MVP scope
- fewer compensatory features
- clearer product focus
- simpler onboarding path
- better now-vs-later sequencing
