# Operations

Use this file for workflows, SOPs, approvals, handoffs, internal processes, and efficiency redesign.

## Core orientation

Most operational heaviness comes from:

- inherited approvals
- team-boundary friction
- duplicate information entry
- meetings compensating for poor information flow
- outdated process patches that were never revisited

## Key lenses

Examine:

- desired outcome
- trigger
- transformations
- approvals
- handoffs
- control points
- failure modes
- monitoring needs

## Strong questions

- If this process were designed today, would it still look like this?
- Which steps create value, and which steps only manage history or anxiety?
- What truly requires human review?
- Which handoffs are caused by org design rather than task logic?
- What step can be removed first without unacceptable risk?

## Typical outputs

- fewer steps
- fewer handoffs
- less approval drag
- simpler process design
- lower coordination cost
