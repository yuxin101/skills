# Output Patterns

Use these patterns when a structured answer improves clarity.

## Pattern 1: Quick reframe

- Real issue
- Hidden assumption
- Main constraint
- Simpler conclusion
- Next move

## Pattern 2: Structured decision analysis

- Real goal
- Assumptions
- Basic facts
- Hard constraints
- Soft constraints
- Simplified options
- Recommendation
- First test

## Pattern 3: Reasoning audit

- Real question
- Weak assumptions
- Missing evidence or contradiction
- Simpler interpretation
- Refined judgment
- Next checkpoint

## Pattern 4: Product or business choice

- Real decision
- Current assumptions
- Facts that matter
- Complexity drivers
- Better options
- Best current move

## Pattern 5: Workflow simplification

- Desired outcome
- Existing complexity
- What is essential
- What can be removed
- Simplified design
- First removal or test
