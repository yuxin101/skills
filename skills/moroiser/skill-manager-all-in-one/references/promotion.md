# Promotion Guide | 宣传指南

## De-identification checklist | 去标识化检查清单

Before promoting a skill, check that the published content does not expose:  
在宣传技能前，检查已发布内容不要暴露以下信息：
- city, province, or country details  
  城市、省份、国家等位置信息
- personal names  
  个人姓名
- API keys, passwords, or credentials  
  API 密钥、密码或凭据
- device identifiers, IPs, or internal paths  
  设备标识、IP 或内部路径

## Promotion style rules | 宣传文案规则

- Use English first, Chinese after.  
  英文在前，中文在后。
- Keep copy short, polished, and clean.  
  文案保持简短、精致、干净。
- Use exact links.  
  使用准确链接。
- Put install commands on their own line.  
  安装命令单独成行。
- Use the current published version, not a guessed one.  
  使用当前已发布版本，不要猜版本号。
- Keep the result public-facing and professional.  
  保持对外、专业、体面。

## Suggested format | 推荐格式

```markdown
**Skill Name | 技能名** (vX.Y.Z)
Short English sentence. 简短中文说明。
https://clawhub.com/skills/<slug>
clawhub install <slug>
```

## Reminder | 提醒

Promotion copy may be more lively than changelog copy, but changelog text itself should stay formal.  
宣传文案可以比 changelog 略活一些，但 changelog 本身必须保持正式。
