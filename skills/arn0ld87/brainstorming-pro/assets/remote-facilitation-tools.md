## 2. Whiteboard-Tools

### 2.1 Miro

**Features:**
- Unlimited Boards (Free: 3)
- Templates Library
- Real-time Collaboration
- Integrations (Jira, Slack, etc.)
- Offline Mode (iPad)

**Brainstorming-Templates:**
- Mind Map
- Brainwriting
- SCAMPER
- Impact/Effort Matrix
- Kanban

**Tipps:**
```
✅ Sticky Notes in batches erstellen (Shift+Klick)
✅ Templates vor Session vorbereiten
✅ Frame für jeden Teilnehmer erstellen
✅ Timer als Widget einfügen
✅ Laser Pointer für Fokus nutzen

❌ Nicht zu viele Elemente auf einmal
❌ Nicht zu kleine Sticky Notes
❌ Nicht zu bunt (杂乱 = unübersichtlich)
```

### 2.2 Mural

**Features:**
- Templates spezifisch für Design Thinking
- Facilitator Tools (Guided Sessions)
- Integrations (Zoom, Teams)
- Timer built-in
- Sticky Note clustering

**Best Practices:**
```
✅ Guided Sessions: Teilnehmer sehen nur aktuellen Schritt
✅ Vote-Counter: Dot-Voting automatisch
✅ "Facilitator Only" Layers für private Notizen
✅ Templates: Design Sprint, Retrospective, Brainstorm

❌ Zu viele Stickies (max 50 pro Cluster)
❌ Nicht alle Features gleichzeitig nutzen
```

### 2.3 FigJam (Figma)

**Features:**
- Native in Figma (nicht extra App)
- Widgets (Timer, Vote, Forms)
- Design-System Integration
- Low learning curve für Figma-User

**Best Practices:**
```
✅ Für Design-Teams ideal (beste UI-Features)
✅ Rapid Prototyping Integration
✅ Einfaches Sharing via Figma Links

❌ Kein Offline Mode
❌ Performance bei >100 Elementen
```

## 3. Video-Call Setup

### 3.1 Optimaler Setup

```
Kamera:
□ Position: Augenhoehe
□ Licht: Frontal (kein Gegenlicht)
□ Hintergrund: Neutral oder virtueller BG

Audio:
□ Headset preferred (keine AirPods für多人)
□ Mikrofon testen vor Start
□ Stummschalten wenn nicht redet

Display:
□ Dual Monitor (ideal)
□ Presentation Mode für Screen-Sharing
□ Gallery View für Participants
```

### 3.2 Breakout-Rooms

```
Für Sessions >6 Personen:
□ 3-4 Personen pro Room
□ 5-7 Min Zeit für Subgroup-Diskussion
□ Klare Aufgabe für Subgroup
□ Rapporteur definiert (berichtet afterwards)

Facilitation:
□ Breakout vorbereiten (Frage/Task)
□ Timer setzen (5-7 Min)
□ ALL-BACK Ankündigung 1 Min vor Ende
□ Summary im Haupt-Room einsammeln
```

### 3.3 Remote-Regeln Etablieren

```
Script: Zu Beginn der Session

"Okay, bevor wir starten - ein paar Regeln für Remote:

1. Kamera ist an (wenn möglich) - hilft Engagement
2. Mikro aus wenn nicht redet (Reduce noise)
3. Hand heben (oder Chat nutzen) zum Unterbrechen
4. Wir nutzen Miro für Ideen - Link kommt gleich
5. Timer wird geteilt - wir halten uns an Zeiten

Noch Fragen zu den Regeln?"

→ Dokumentiert als Miro-Sticky oder Slide
```

## 4. Remote Brainstorming Techniques

### 4.1 Anonymized Brainwriting (Remote)

```
Tools: Miro + Zoom Breakout Rooms

Ablauf:
1. In Breakout Rooms (2-3 Min): 
   - Jeder schreibt 3-5 Ideen auf Miro-Stickies
   - Nicht den Namen drauf
   
2. Zurück im Haupt-Room:
   - Miro-Stickies einsammeln (Facilitator)
   - Mixen/reihenfolge randomizen
   
3. Diskussion:
   - Facilitator liest vor (anonym)
   - Team diskutiert ohne Bias

→ Verhindert: "Die Idee vom Chef ist besser"
```

### 4.2 Chat-Based Ideation

```
Tools: Zoom Chat, Slack, or Mentimeter

Ablauf:
1. Question stellen
2. Alle tippen Ideen in Chat (2 Min)
3. Facilitator screen-shared:
   - Chat kopieren in Miro
   - In Spalten/Cluster sortieren
   
4. Nicht-Getipptes fragen:
   - "Wer hat noch Ideen die nicht getippt sind?"

→ Für: Introvertierte, asynchrone Teilnehmer
```

### 4.3 Voice-Based Brainstorming

```
Tools: Zoom, Teams (kein Video nötig)

Ablauf:
1. "Frei Schnauze" (5 Min):
   - Alle reden, Facilitator notiert
   - NON-STOP - keine Unterbrechung
   - Timer sichtbar

2. Pause (30 Sek)

3. Clarifying Questions (5 Min):
   - Jeder kann Fragen stellen
   - Keine Kritik, nur Klärung

4. Dot-Voting (3 Min):
   - "Bitte in Chat eure Top-3"

→ Für: Kreative Sessions, Brainstorming-alt
```

### 4.4 Async Brainstorming

```
Tools: Slack, Email, or Miro (zeitversetzt)

Ablauf:
1. Vor Session (24h vorher):
   - Agenda + Context verschicken
   - Pre-Work Assignment: "Bitte 3 Ideen vorbereiten"
   
2. Während Session:
   - Nur Priorisierung und Diskussion
   - Pre-Work wird nicht nochmal vorgestellt
   
3. Nach Session:
   - Miro-Board bleibt offen für 48h
   - Late-Entry-Posting erlaubt

→ Für: Globale Teams, Zeitzonen-übergreifend
```

## 5. Remote Facilitation Scripts

### 5.1 Starting a Remote Session

```
Script:

"Okay, willkommen alle! Wir sind komplett um [Zeit] - danke fürs Kommen.

Ich bin [Name], Facilitator heute.
[Optional: Co-Facilitator für Chat-Monitoring]

AGENDA:
- 5 Min: Zielklärung
- 30 Min: Brainstorming
- 10 Min: Pause
- 20 Min: Priorisierung
- 5 Min: Wrap-Up

TOOL: Wir nutzen Miro - Link kommt gleich in Chat.
Ich teile den Link jetzt. Bitte kurz öffnen und schauen ob alles geht.

[Link in Chat posten]

Hat jeder den Miro-Board offen? Perfekt.

Let's go!"
```

### 5.2 Managing Participation

```
Problem: Stille in Remote

Script:

"Hallo [Name], ich sehe dich noch nicht contribute.
Magst du kurz was teilen? Keine langen Sachen - nur ein Gedanke."

Oder:

"Lasst mich kurz die Kamera-Check machen.
[Name], kannst du kurz deine Kamera anmachen?"
(Damit Person integriert wird, nicht isolated)

Oder:

"Wir machen eine kurze Round-Robin.
Jeder kommt kurz dran - 30 Sekunden.
Los geht's bei [Name]."
```

### 5.3 Time Warnings

```
Script:

[Bei 10 Min übrig von Phase]:

"Okay, wir haben noch 10 Minuten für diesen Teil.
Die Top-Sachen die noch kommen sollten: [Erinnerung]

[Bei 5 Min übrig]:

"Noch 5 Minuten - last push!
Was muss noch auf den Miro-Board?"

[Bei 1 Min übrig]:

"1 Minute! Bitte finalisiert eure Einträge.
Ich zähle hoch wenn wir bei 0 sind."
```

### 5.4 Ending a Remote Session

```
Script:

"Okay, wir sind bei Zeit. 

ERGEBNIS-ZUSAMMENFASSUNG:
- [Top 3 Ideen mit kurzer Begründung]

ACTION ITEMS:
| Was | Wer | Bis |
|-----|-----|-----|

OFFENE FRAGEN:
- [Liste]

NEXT STEPS:
- Doku kommt per Email
- Follow-Up: [Datum]

Danke fürs Mitmachen! Ich beende jetzt die Session.
Bei Fragen: [Slack/Email]"

[Session beenden]
```

## 6. Hybrid Sessions (Remote + In-Person)

### 6.1 Setup-Empfehlungen

```
Falls Hybrid (some remote, some in-person):

□ Kamera auf In-Person-Gruppe gerichtet
□ separates Mikrofon für In-Person (nicht Laptop)
□ Miro/Whiteboard für alle sichtbar (TV/Beamer)
□ Remote-Teilnehmer equal behandeln (nicht vergessen)
□ Extra Facilitator für Remote-Monitoring (Chat, Tech)
```

### 6.2 Scripts für Hybrid

```
Challenge: Remote fühlen sich excluded

Script:
"Okay, bevor wir starten - eine Bitte an die In-Person-Leute:

Die Remote-Kollegen können nicht so leicht "rum" sehen.
Wenn wir was auf dem Board diskutieren, bitte 
explizit sagen was wir meinen:
NICHT: 'Da, die rote Idee'
SONDERN: 'Die rote Idee oben links, unter der Kategorie Innovation'

Danke!"

Challenge: Parallele Diskussionen

Script:
"Wir haben gerade parallele Gespräche - die Remote-Leute hören nur eine Seite.

Bitte nur eine Person zur Zeit. 
Ich mache die Moderation: [Name] redet, alle anderen hören.

Kurze Ausnahme für die Remote-Leute: 
Ihr könnt jederzeit 'Hand heben' per Chat oder emoji."
```

## 7. Troubleshooting Remote

### 7.1 Tech-Probleme

| Problem | Solution |
|---------|----------|
| Audio-Probleme | Headset wechseln, Laptop-Mic testen |
| Miro laggy | Weniger Elemente, Browser-Refresh |
| Video-Call disconnect | Backup: Telefon-Einwahl bereithalten |
| Screen-Share funktioniert nicht | Alternative: Miro-Screenshot teilen |

### 7.2 Engagement-Probleme

| Symptom | Solution |
|---------|----------|
| Alle sind still | Warm-Up, Pair-Diskussion, Brainwriting |
| Eine Person dominiert | "Let's hear from others", Brainwriting |
| Multitasking | Kamera-An, Pausen mit Interaktion |
| Timezone-Konflikt | Async-Voting, Miro offen lassen |

### 7.3 Timezone-Probleme

```
Challenge: Team in 3+ Zeitzonen

Lösungen:

1. Rotation: "Jede Woche andere Zone hostet"
2. Async-Participation: Pre-Work + Vote-Async
3. Core-Hours: Nur 2h overlap, fokussiert nutzen
4. Recordings: Session aufnehmen + replay für andere
5. Champions: Lokale Lead in jeder Zeitzone

Script für Worst-Case-Zeit:
"Okay, die Session-Zeit ist suboptimal für [Region].
Ich entschuldige mich dafür.
Als Kompensation:
- Doku ist super detailliert
- 30-Min Follow-Up für Timezone-benachteiligte
- Ihr bekommt extra Input-Time in Async"
```

## 9. Quick-Start Guide

### Für 30-Min Remote Brainstorm

```
1. Vorher (5 Min):
   □ Agenda verschicken
   □ Miro-Board vorbereiten (Link in Email)
   □ Pre-Work: "3 Ideen mitbringen"
   □ Timer-App bereit (z.B. Timer+)

2. Start (3 Min):
   □ Kamera-Check
   □ Regeln etablieren
   □ Miro-Link in Chat
   □ Warm-Up (30 Sek: "Was nervt dich an Meetings?")

3. Brainstorm (15 Min):
   □ 10 Min: Stille Ideen-Generation (Miro-Stickies)
   □ 5 Min: Vorstellen (Facilitator liest, 30 Sek pro Idee)

4. Pause (2 Min):
   □ Hydration, Bathroom

5. Konvergenz (10 Min):
   □ Dot-Voting (3 Votes pro Person)
   □ Top 3 identifizieren
   □ Action Items mit Ownern

6. Wrap-Up (3 Min):
   □ Zusammenfassung
   □ Doku ankündigen
   □ Danke + Close
```
