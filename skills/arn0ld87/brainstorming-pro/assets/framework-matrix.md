# Framework-Matrix — Decision-Tree für Brainstorming-Techniken

**Use:** Vor Session start. Richtige Technik für Kontext wählen.

---

## 1. Pre-Session Checklist

**Before choosing framework:**

- [ ] **Ziel geklärt** (zielklaerung-template.md durchlaufen)
- [ ] **Stakeholder identifiziert** (Wer entscheidet?)
- [ ] **Constraints bekannt** (Budget, Zeit, Compliance)
- [ ] **Team-Größe klar** (1, 2-6, 6+)
- [ ] **Zeitrahmen gesetzt** (<10 Min, 10-30 Min, 30-60 Min)

---

## 2. Primary Decision: Divergent vs. Convergent

```
Session-Phase?
├── Ideen generieren → DIVERGENT (divergent-techniques.md)
└── Ideen priorisieren → CONVERGENT (convergent-frameworks.md)
```

**Rules:**
- **Niemals** convergent vor divergent (kills creativity)
- **Immer** explizit wechseln ("We switch from divergent to convergent now")
- **Time-Box:** 60% Divergent, 40% Convergent

---

## 3. Divergent Framework Selection

### Decision-Tree

```
Team-Größe?
│
├── 1 Person
│   ├── Zeit <10 Min → Rapid Ideation
│   ├── Zeit 10-30 Min → Mind Mapping
│   └── Zeit 30+ Min → SCAMPER (self-facilitated)
│
├── 2-6 Personen
│   ├── Problem bekannt → 6-3-5 Methode
│   ├── Team blockt → Brainwriting Pool
│   ├── Dominante Personen → Brainwriting Pool
│   ├── Branchen-blind → Analogy Thinking
│   └── Perfektionismus → Worst Possible Idea
│
└── 6+ Personen
    ├── Zeit <30 Min → SCAMPER (facilitated)
    ├── Zeit 30-60 Min → Role Storming
    └── Remote → Mind Mapping (digital tool)
```

### Kontext-Exceptions

| Situation | Override |
|-----------|----------|
| **CEO im Raum** | Role Storming (rollen-basiert → weniger Hierarchie) |
| **Compliance-sensitive** | Brainwriting (anonym → keine self-censorship) |
| **Müdes Team** | Worst Possible Idea (Humor öffnet) |
| **Remote** | Digital Mind Mapping (Miro/Mural) |
| **Kunde dabei** | Starbursting (strukturiert → professionell) |

## 4. Convergent Framework Selection

### Decision-Tree

```
Stakeholder-Typ?
│
├── Demokratisch (Team-Entscheid)
│   ├── Ideen ≤20 → Dot-Voting
│   ├── Ideen 20-50 → 100-Dollar-Test
│   └── Ideen 50+ → Dot-Voting (top 20 first)
│
├── Executive (C-Level)
│   ├── Strategy-Focus → Value vs. Complexity
│   ├── Roadmap-Focus → Now / Next / Later
│   └── Budget-Focus → RICE Scoring
│
├── Data-Driven (Product)
│   ├── Metriken verfügbar → RICE
│   ├── Metriken fehlend → Weighted Scoring
│   └── A/B-Tests möglich → Decision-Matrix
│
└── Customer-Focus (UX)
    ├── Known Needs → Kano (Basis/Performance)
    ├── Unknown Needs → Kano (Excitement)
    └── Competitor-Vergleich → Decision-Matrix (Pugh)
```

### Kontext-Exceptions

| Situation | Override |
|-----------|----------|
| **Zeit <15 Min** | Dot-Voting (schnellste) |
| **Stakeholder-Conflict** | Weighted Scoring (objektiv) |
| **Scope-Verhandlung** | MoSCoW (explizit Out-of-Scope) |
| **Roadmap-Planning** | Now/Next/Later (Horizonte) |
| **Investor-Pitch** | RICE (ROI-transparent) |

## 5. Hybrid-Sessions (Divergent + Convergent)

### 60-Minuten-Template

```
00-05 Min: Zielklärung (zielklaerung-template.md)
05-35 Min: Divergent (SCAMPER oder 6-3-5)
35-40 Min: Break (5 Min)
40-55 Min: Convergent (Impact/Effort oder Dot-Voting)
55-60 Min: Next Steps (output-template.md)
```

### 90-Minuten-Template

```
00-10 Min: Zielklärung
10-50 Min: Divergent (Role Storming oder Mind Mapping)
50-55 Min: Break
55-80 Min: Convergent (RICE oder Weighted Scoring)
80-90 Min: Handoff (handoff-protocol.md)
```

### 120-Minuten-Template

```
00-10 Min: Zielklärung
10-70 Min: Divergent (SCAMPER + Brainwriting Pool)
70-80 Min: Break
80-110 Min: Convergent (Kano + Now/Next/Later)
110-120 Min: Documentation + Handoff
```

## 6. Remote-Specific Adjustments

### Tool-Mapping

| Framework | Recommended Tool |
|-----------|-----------------|
| Mind Mapping | Miro, Mural, FigJam |
| 6-3-5 | Google Docs (shared) |
| Brainwriting Pool | Miro Board (anonymous cards) |
| SCAMPER | Mural (facilitated) |
| Dot-Voting | Miro (sticky votes) |
| Impact/Effort | Miro (2x2 Matrix) |
| RICE | Google Sheets (formula) |

### Remote-Facilitation-Tipps

- **Camera-On** für Divergent (Energie sichtbar)
- **Chat-Only** für Convergent (faster voting)
- **Breakout-Rooms** bei 6+ Personen (2-3 pro room)
- **Timer-Sharing** (alle sehen Countdown)
- **Anonymous-Mode** bei Hierarchie (Miro hide names)

---

## 7. Escalation Path

**Wenn Framework nicht passt:**

1. **Team blockt in Divergent** → Switch to Worst Possible Idea (Humor)
2. **Stakeholder-Conflict in Convergent** → Switch to Weighted Scoring (objektiv)
3. **Zeit läuft weg** → Switch to Dot-Voting (5 Min)
4. **Remote-Tool fails** → Switch to verbal Brainstorming (fallback)
5. **Dominante Person** → Switch to Brainwriting (anonym)

---

## 8. Success-Metriken pro Framework

| Framework | Success-Metric | Target |
|-----------|---------------|--------|
| SCAMPER | Ideen gesamt | 50+ |
| 6-3-5 | Ideen gesamt | 108 (6×3×6) |
| Brainwriting | Participation rate | 100% (alle schreiben) |
| Mind Mapping | Äste + Zweige | 5 Hauptäste, 20+ Zweige |
| Dot-Voting | Top 3 klar | ≤5 Items |
| Impact/Effort | Quick Wins | 3-5 Items |
| RICE | Top 5 scored | ≥20 Score |
| Kano | Basis identifiziert | Alle Must-Haves |

---

## 9. Anti-Patterns

### ❌ Falsche Framework-Wahl

| Symptom | Cause | Fix |
|---------|-------|-----|
| Team silent | Divergent mit Hierarchie | Brainwriting (anonym) |
| 5 Ideen nach 30 Min | Falsche Divergent-Technik | SCAMPER (strukturierter) |
| Stakeholder fight | Convergent ohne Daten | Weighted Scoring (objektiv) |
| Alles ist P0 | Kein MoSCoW | Must-Hen ≤60% erzwingen |
| Remote disengaged | Falsches Tool | Miro (interactive) |

### ❌ Timing-Fehler

- Divergent zu kurz (<30 Min für 50 Ideen)
- Convergent zu früh (before idea saturation)
- Break fehlt (nach 40 Min kognitiv müde)
- No documentation (last 5 Min skip)
