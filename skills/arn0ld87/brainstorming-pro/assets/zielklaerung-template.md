# Zielklärungs-Template — Stakeholder-Interview

**Use:** Vor divergenter Phase, um Scope + Success-Criteria zu klären

---

## 1. Kontext (5 Min)

```markdown
**Ausgangssituation:**
[Welches Problem/Chance wird adressiert?]

**Stakeholder:**
- Primary: [Wer ist entscheidungsberechtigt?]
- Secondary: [Wer ist betroffen?]
- Observer: [Wer beobachtet nur?]

**Zeitrahmen:**
- Deadline: [Wann muss Ergebnis vorliegen?]
- Follow-up: [Wann wird nachgesteuert?]
```

---

## 2. Zieldefinition (10 Min)

### SMART-Check

| Kriterium | Frage | Antwort |
|-----------|-------|---------|
| **S**pecific | Was genau soll erreicht werden? | |
| **M**easurable | Woran messen wir Erfolg? | |
| **A**chievable | Ist es mit Ressourcen machbar? | |
| **R**elevant | Warum ist es wichtig jetzt? | |
| **T**ime-bound | Bis wann muss es fertig sein? | |

### Success-Criteria

```
Must-Haves (P0):
1. [Nicht verhandelbar]
2. [Nicht verhandelbar]

Should-Haves (P1):
1. [Verhandelbar]
2. [Verhandelbar]

Nice-to-Haves (P2):
1. [Optional]
```

---

## 3. Constraints (5 Min)

### Hard Constraints

- **Budget:** [€/Personen/Tage]
- **Ressourcen:** [Verfügbar vs. benötigt]
- **Compliance:** [Regulatorische Anforderungen]
- **Technical:** [Architektur-Entscheidungen]

### Soft Constraints

- **Stakeholder-Erwartungen:** [Explizit vs. implizit]
- **Organisational:** [Politische Faktoren]
- **Cultural:** [Team-Dynamik]

---

## 4. Scope-Abgrenzung (5 Min)

### In-Scope

```
✅ [Thema 1]
✅ [Thema 2]
✅ [Thema 3]
```

### Out-of-Scope

```
❌ [Thema A] — Grund: [Zeit/Budget/Priority]
❌ [Thema B] — Grund: [Nicht relevant jetzt]
❌ [Thema C] — Grund: [Anderer Owner]
```

### Grey-Zone (Review nötig)

```
⚠️ [Thema X] — Entscheidung bis [Datum]
⚠️ [Thema Y] — Abhängig von [Constraint]
```

---

## 5. Output-Erwartung (5 Min)

### Format

- [ ] **Ideenliste** (priorisiert)
- [ ] **Action-Items** (mit Ownern)
- [ ] **Decision-Log** (mit Begruendung)
- [ ] **Follow-up-Plan** (mit Dates)

### Detailgrad

- [ ] **High-Level** (1-2 Sätze pro Idee)
- [ ] **Medium** (Kontext + Rationale)
- [ ] **Deep-Dive** (Implementation-Skizze)

---

## 6. Eskalations-Pfad

Wenn Zielklärung blockt:

1. **Unklare Success-Criteria** → Product-Owner eskalieren
2. **Widersprüchliche Constraints** → Sponsor-Decision required
3. **Scope-Creep** → Change-Request initiieren
4. **Stakeholder-Conflict** → Mediation-Session ansetzen

---

## 7. Übergabe an Divergente Phase

**Checkliste vor Start:**

- [ ] SMART-Criteria dokumentiert
- [ ] Must-Haves priorisiert
- [ ] Constraints kommuniziert
- [ ] Scope abgegrenzt (In/Out/Grey)
- [ ] Output-Erwartung geklärt
- [ ] Stakeholder aligned

**Trigger für nächste Phase:**
> "Alle P0-Punkte geklärt → Starte Divergente Phase mit [Technik]"
