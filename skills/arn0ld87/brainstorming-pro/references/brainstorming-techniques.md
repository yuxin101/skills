# Brainstorming Techniques Deep-Dive

**Level:** Intermediate-Advanced  
**Time:** 20-30 Min Lesen  
**Use:** Als Facilitator fortgeschrittene Techniken verstehen

---

## 1. SCAMPER Deep-Dive

### 1.1 Vollständige SCAMPER Checklist

**S — Substitute:**
- Was kann ich ersetzen?
  - Material, Komponenten, Prozess
  - Andere Ressourcen
  - Andere Menschen (Stakeholder, Kunden)
  - Regeln, Policies, Standards

**C — Combine:**
- Was kann ich kombinieren?
  - Features, Services, Produkte
  - Teams, Abteilungen
  - Technologien
  - Prozesse mit anderen

**A — Adapt:**
- Was kann ich adaptieren?
  - Best Practices aus anderen Branchen
  - Historische Lösungen
  - Competitor-Ansätze
  - Nature-Lösungen (Biomimicry)

**M — Modify:**
- Was kann ich modifizieren?
  - Größe, Form, Struktur
  - Speed, Frequenz, Timing
  - Farbe, Design, Packaging
  - Prozess-Schritte

**P — Put to other uses:**
- Wie kann ich es anders nutzen?
  - Andere Zielgruppen
  - Andere Use Cases
  - Andere Märkte
  - Nebenprodukte/Abfall

**E — Eliminate:**
- Was kann ich entfernen?
  - Komplexität reduzieren
  - Steps eliminieren
  - Features streichen
  - Hierarchie abflachen

**R — Reverse:**
- Was kann ich umkehren?
  - Prozess umdrehen
  - Inside-out denken
  - Rollen tauschen
  - Tradition brechen

### 1.2 SCAMPER Workshop-Ablauf (60 Min)

```
00:00 - 5 Min: Problem vorstellen
         - Was ist die Challenge?
         - Constraints definieren
         
05:00 - 5 Min: S + C
         - Jeder schreibt 5 Min
         - 2 Min Sharing pro Person
         
15:00 - 5 Min: A + M  
         - 5 Min schreiben
         - 2 Min Sharing
         
25:00 - 5 Min: P + E
         - 5 Min schreiben
         - 2 Min Sharing
         
35:00 - 5 Min: R
         - 5 Min schreiben
         - 2 Min Sharing
         
45:00 - 10 Min: Dot-Voting
         - Top 3 per Voting
         
55:00 - 5 Min: Wrap-Up
         - Was nehmen wir mit?
         - Next Steps
```

## 2. 6-3-5 Deep-Dive

### 2.1 Was ist 6-3-5?

- **6** Personen
- **3** Ideen pro Person
- **5** Runden
- **108** mögliche Ideen (6×3×6)

### 2.2 Regeln

```
1. Jeder schreibt 3 Ideen in 5 Minuten
2. Blatt wird im Uhrzeigersinn weitergegeben
3. Nächste Person baut auf Ideen auf (nicht ersetzen)
4. Keine Kritik, keine Diskussion
5. Nach 6 Runden: Alle Ideen gesammelt
```

### 2.3 Vor-/Nachteile

| Vorteile | Nachteile |
|----------|-----------|
| Anonym (keine Dominanz) | Braucht 6+ Personen |
| Quantity hoch | Zeitintensiv (30+ Min) |
| Cross-Pollination |纸张-basiert (oder digital) |
| Keine early commitment | Kann repetitiv werden |

### 2.4 Digital Alternative: Miro Template

```
1. 6 Karten/Person erstellen
2. 5-Min Timer setzen
3. Personen schreiben in eigene Karte
4. Karte weitergeben (automatisch in Miro)
5. Repeat 6×
6. Alle Karten in Pool sammeln
```

## 3. Mind Mapping Deep-Dive

### 3.1 Basics

```
Zentrum: Hauptthema (1 Wort, Bild)
Äste: 5-7 Hauptkategorien
Zweige: Freie Assoziationen
Details: Sub-Ideen, Beispiele
```

### 3.2 Mind Mapping Regeln

```
1. Zentrum zuerst (2 Min)
   - Bild oder großes Wort
   - Was ist das Kernthema?
   
2. Hauptäste (5 Min)
   - Was sind die Hauptkategorien?
   - Nicht mehr als 7 (kognitive Grenze)
   
3. Zweige (15-20 Min)
   - Freie Assoziation
   - Alles aufschreiben
   - Nicht filtern
   
4. Review (5 Min)
   - Was fällt auf?
   - Verbindungen zeichnen
   - Cluster identifizieren
```

| Coggle | Free | Simple, Beautiful |
| MindMeister | Paid | GSuite Integration |
| XMind | Paid | Cross-Platform |
| Scapple | Paid | Free-form (nicht hierarchisch) |

## 4. Reverse Brainstorming

### 4.1 Wie es funktioniert

**Normal:** "Wie lösen wir Problem X?"  
**Reverse:** "Wie verschlimmern wir Problem X?"

### 4.2 Warum es funktioniert

```
Psychologie:
- Druck raus (erlaubt "schlechte" Ideen)
- Humor öffnet Kreativität
- Constraints werden explizit
- Umkehrung zeigt oft Root-Causes
```

### 4.3 Workshop-Ablauf (30 Min)

```
00:00 - 5 Min: Problem vorstellen
         "Wir wollen Problem X verschlechtern!"
         
05:00 - 10 Min: Reverse-Ideen generieren
         "Wie machen wir X noch schlimmer?"
         → 10+ schlechte Ideen sammeln
         
15:00 - 10 Min: Flip
         Jede Idee umkehren:
         "Wie machen wir das Gegenteil?"
         → Aus schlecht = gut
         
25:00 - 5 Min: Top 3 auswählen
         Welche "Lösungen" sind wirklich gut?
```

### 4.4 Beispiel

```
Problem: Checkout-Conversion niedrig

Reverse: "Wie senken wir Conversion?"
- Komplexe Formulare erstellen
- Preise verstecken
- Support abschalten
- Checkout auf 10 Schritte machen

Flip: "Was ist das Gegenteil?"
- Formulare vereinfachen → SIMPLE Checkout
- Preise transparent → Preisübersicht prominent
- Support proaktiv → Chatbot + Human
- Checkout verschlanken → 1-Click Checkout
```

## 5. Worst Possible Idea

### 5.1 Konzept

1. Teams sammelt absichtlich die SCHLECHTESTEN Ideen
2. Ideen werden präsentiert (mit Humor)
3. Analyse: "Was ist eigentlich gut an der schlechten Idee?"
4. Die "guten Kerne" werden zu echten Lösungen

### 5.2 Warum es funktioniert

```
- Perfektionismus blockiert → Worst Ideas erlaubt alles
- Humor senkt Hemmschwelle
- Kreativität ohne Bewertung
- Oft: schlechteste Idee hat überraschend guten Kern
```

### 5.3 Beispiel

```
Challenge: Remote Team Engagement

Worst Idea: "Lass uns alle 5 Minuten status updaten!"

Analyse:
✅ Mikromanagement erkannt (gut erkannt!)
✅ Bedürfnis nach Connection (echtes Bedürfnis)
✅ Frequenz ist Problem (5 Min = zu viel)

Lösung: Weekly "Coffee Connect" - 15 Min freiwilliges Video-Checkin
```

## 6. Role Storming

### 6.1 Konzept

Teilnehmer schlüpfen in andere Rollen und brainstormen aus deren Perspektive.

### 6.2 Rollen-Beispiele

```
Klassiker:
- CEO (Kosten, Strategie)
- Kunde (Value, Experience)
- Investor (ROI, Growth)
- Competitor (Angst, FOMO)
- Technologie (Machbarkeit)

Ungewöhnlich:
- 14-jähriger TikTok-User
- 80-jährige Großmutter
- Außerirdischer (Was ist hier seltsam?)
- Kind (Naive Fragen)
- Gegner (Anti-Idee)
```

### 6.3 Workshop-Ablauf (45 Min)

```
00:00 - 5 Min: Challenge vorstellen

05:00 - 5 Min: Rollen erklären
         - Kärtchen mit Rollen vorbereiten
         - 3-5 Rollen für das Thema

10:00 - 15 Min: Role-Switch + Brainstorm
         - Jeder zieht Rolle (oder wählt)
         - 3 Min in Rolle "leben"
         - 12 Min Ideen aus Rolle generieren

25:00 - 10 Min: Präsentation
         - "Als [Rolle] würde ich..."
         - Andere rate nicht wer wer ist
         
35:00 - 10 Min: Diskussion
         - Was war überraschend?
         - Welche Perspektive fehlt?
         - Was nehmen wir mit?
```

## 7. Brainwriting Pool

### 7.1 Konzept

Anstatt zu reden, schreiben alle auf Karten. Karten werden getauscht und weitergearbeitet.

### 7.2 Regeln

```
1. Jeder bekommt 5 leere Karten
2. 5 Min: 5 Ideen schreiben (1 pro Karte)
3. Karten in Mitte ("Pool")
4. Jeder zieht 3 fremde Karten
5. 5 Min: Auf fremden Ideen aufbauen
6. Karten zurück in Pool
7. Repeat 3-4×
```

### 7.3 Vorteile

```
- Anonym (keine Hierarchie)
- Alle kommen zu Wort
- Cross-Pollination
- Schriftlich = dokumentiert
- Schneller als reine Diskussion
```

## 8. Starbursting

### 8.1 Die 6 Fragen

```
Who?    — Wer ist betroffen?
What?   — Was wird gebraucht?
Where?  — Wo wird es eingesetzt?
When?   — Wann ist Deadline?
Why?     — Warum ist es wichtig?
How?    — Wie wird es umgesetzt?
```

### 8.2 Wann nutzen

- Anforderungen komplett erfassen
- Stakeholder-Perspektiven verstehen
- Audit/Viability-Check
- User Story Writing vorbereiten

### 8.3 Beispiel

```
Feature: AI-Chatbot für Kundenservice

Who?    — Kunden (jung/alt/technisch/nicht)
         — Support-Agenten
         — IT/Ops
         — Management

What?   — 24/7 Verfügbarkeit
         — Schnelle Antworten
         — Human Escalation
         — Multi-Language

Where?  — Website Widget
         — Mobile App
         — WhatsApp
         — Social Media

When?   — Launch: Q2 2026
         — Peak: Wochenenden, Nachts
         — SLA: <30 Sekunden

Why?     — Kosten senken
         — Kundenzufriedenheit
         — Skalieren ohne Headcount

How?     — LLM-API (GPT-4, Claude)
         — RAG für Knowledge Base
         — Human Handoff Workflow
         — Analytics Dashboard
```

## 9. Analogy Thinking

### 9.1 Konzept

"Idee X aus Branch Y kann uns inspirieren für Branch Z"

### 9.2 Branchen-Mapping

```
Branche A → Branche B → Insight

Amazon Logistics → Healthcare:
- Same-Day Lieferung → Notfall-Medikamente
- Locker Pickup → Klinik-Abholung
- Returns Policy → Test-Results Abholung

Netflix Recommendation → E-Learning:
- Watch Next → "Learn Next"
- Binge-Watching Pattern → Kurs-Fortschritt
- Personalization → Skill-Gaps

Airbnb Experience → Corporate Training:
- Local Experiences → Internal Experts
- Reviews → Peer-Ratings
- Superhost → Top-Trainer Zertifizierung
```

### 9.3 Workshop-Ablauf (30 Min)

```
00:00 - 5 Min: Problem vorstellen

05:00 - 5 Min: Analogie-Branchen wählen
         - 4-5 Branchen für Mapping
         - Tech, Healthcare, Retail, Nature, Gaming

10:00 - 15 Min: Analogien generieren
         "Wie löst [Branche] ähnliche Probleme?"
         - Pro Branche 2-3 Analogien
         
25:00 - 5 Min: Top-Analogien auswählen
         - Vote: Welche sind most inspiring?
         
30:00 - 0 Min: Wrap-Up
         - Transfer: Was lernen wir für unser Problem?
```

## 10. Technik-Kombinationen

### 10.1 Hybride Sessions

**60-Min Session mit 2 Techniken:**

```
00-10 Min: Zielklärung (zielklaerung-template.md)
10-35 Min: SCAMPER (divergente Phase)
35-40 Min: Pause
40-55 Min: Impact/Effort Matrix (konvergente Phase)
55-60 Min: Wrap-Up (output-template.md)
```

**90-Min Session mit 3 Techniken:**

```
00-10 Min: Zielklärung
10-20 Min: Mind Mapping (divergent)
20-35 Min: 6-3-5 (auf Mind Map aufbauend)
35-40 Min: Pause
40-55 Min: Reverse Brainstorming
55-80 Min: Dot-Voting + Konvergenz
80-90 Min: Wrap-Up
```

### 10.2 Technik-Flow Empfehlungen

| Phase | Technik | Dauer |
|-------|---------|-------|
| Warm-Up | Worst Possible Idea | 10 Min |
| Divergent | SCAMPER oder Mind Mapping | 30-40 Min |
| Cross-Pollination | Brainwriting Pool | 20 Min |
| Convergent | Impact/Effort | 20 Min |
| Final | Dot-Voting | 10 Min |

## 11. Troubleshooting

### Problem: Technik funktioniert nicht

```
1. Zurück zum Problem: Ist die Challenge klar?
2. Energie-Check: Müdigkeit? Pause einlegen
3. Technik-Wechsel: Andere Technik probieren
4. Kleiner werden: Subgroups statt Full Group
```

### Problem: Zu wenige Ideen

```
1. Warm-Up: Erst 2-Min Warm-Up Exercise
2. Quantitätsdruck: "Ziel: 50 Ideen in 10 Min!"
3. Wild Ideas: "Heute ist alles erlaubt!"
4. Timer sichtbar: Countdown erzeugt Druck
```

### Problem: Zu viele ähnliche Ideen

```
1. Perspektiv-Wechsel: Role Storming
2. Analogien: "Wie macht das Amazon/Netflix/etc.?"
3. Reverse: "Wie verschlechtern wir X?"
4. Kategorisieren: Clustern zuerst, dann mergen
```
