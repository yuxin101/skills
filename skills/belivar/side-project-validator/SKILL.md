# 副业 Idea 验证器 v2.0

帮你验证一个副业想法能不能做、值不值得做。**带真实案例和数据支撑，不是空对空。**

## 触发条件

用户表达以下意图时触发：
- "帮我看看这个想法能不能做"
- "这个副业方向怎么样"
- "这个产品有人做吗"
- "这个idea能赚钱吗"
- "帮我调研下 XXX 有没有市场"
- 类似的产品/副业验证需求

---

## 验证流程（5步法）

### Step 1: 需求验证 ⭐⭐⭐⭐⭐

**核心问题**：真的有人需要吗？

**必须做的**：
1. 搜索关键词，看讨论热度
2. 找到目标用户群，问真话
3. 记录用户抱怨/痛点

**判断标准**：
| 热度 | 结论 |
|------|------|
| 大量讨论、很多人问 | 需求存在 ✓ |
| 没人讨论 | 伪需求 ✗ |
| 都在抱怨现有方案 | 强痛点 ✓✓✓ |

**工具**：使用 `scripts/research-demand.sh` 自动调研

---

### Step 2: 竞品分析 ⭐⭐⭐⭐

**核心问题**：别人做了吗？做得怎样？

**必须做的**：
1. 列出直接竞品（做一样产品的）
2. 列出间接竞品（解决同样问题的）
3. 分析竞品优缺点
4. 找到你的差异化

**判断标准**：
| 竞品情况 | 结论 |
|----------|------|
| 竞品多且活得好 | 市场验证 ✓，但要差异化 |
| 竞品少 | 可能是蓝海，也可能是坑 |
| 竞品都死了 | 谨慎，可能是伪需求 |

**工具**：使用 `scripts/find-competitors.sh` 快速查找

---

### Step 3: 市场规模 ⭐⭐⭐

**核心问题**：天花板有多高？

**必须做的**：
1. 查关键词热度（Google Trends/百度指数）
2. 估算目标用户数
3. 估算潜在收入

**判断标准**：
| 市场规模 | 结论 |
|----------|------|
| >10亿 | 大市场，可以搞 |
| 1-10亿 | 中等，细分切入 |
| <1亿 | 小市场，要专注细分 |

**工具**：使用 `scripts/search-trend.sh` 查询热度

---

### Step 4: 变现模式 ⭐⭐⭐⭐⭐

**核心问题**：怎么赚钱？

**必须回答**：
1. 你想怎么赚钱？
2. 参考案例是谁？
3. 能不能规模化？

**常见变现模式**（详见 `references/monetization-patterns.md`）：

| 模式 | 适合 | 难度 |
|------|------|------|
| SaaS订阅 | 工具类产品 | ⭐⭐⭐ |
| 付费下载 | 模板/素材 | ⭐⭐ |
| 广告/联盟 | 内容产品 | ⭐⭐ |
| 电商卖货 | 有供应链 | ⭐⭐⭐⭐ |
| 咨询服务 | 有专业技能 | ⭐⭐ |
| 开源赞助 | 开发者工具 | ⭐⭐⭐ |

---

### Step 5: 实施难度 ⭐⭐⭐⭐

**核心问题**：你能做吗？来得及吗？

**必须评估**：
1. 技术难度（你需要学什么？）
2. 启动成本（要多少钱/时间？）
3. 冷启动（第一批用户从哪来？）
4. 风险（最大的坑是什么？）

**判断标准**：
| 因素 | 好信号 | 坏信号 |
|------|--------|--------|
| 技术 | 你能搞定 | 要学很多 |
| 成本 | <5000 | 要投入很多 |
| 启动 | 有渠道 | 找不到人 |
| 风险 | 已知风险 | 未知太多 |

## 使用示例

### 示例 1：验证"开奶茶店"

```
你：帮我看看开奶茶店能不能做

我：
=== 需求验证 ===
- 搜索热度：高（但负面内容多）
- 结论：需求有，但供给过剩

=== 竞品分析 ===
- 头部：蜜雪冰城 3 万家、喜茶、奈雪
- 竞争度：极度红海
- 结论：没有差异化空间

=== 变现分析 ===
- 单店模型：投入 20-30 万，回本 2-3 年
- 60% 奶茶店亏损
- 结论：变现困难

=== 实施难度 ===
- 资金门槛：高
- 经验门槛：高（需要餐饮经验）
- 结论：不适合程序员

=== 推荐指数 ===
⭐⭐ 谨慎
```

### 示例 2：验证"程序员副业咨询"

```
你：我想做程序员副业咨询，能做吗

我：
=== 需求验证 ===
- 搜索热度：中（知乎、B站有大量讨论）
- 目标用户：想赚外快的程序员
- 结论：需求真实

=== 竞品分析 ===
- 直接竞品：程序员之家、极客时间、掘金
- 间接竞品：B站技术 UP 主
- 差异化：可以专注"副业"这个细分
- 结论：有机会

=== 变现模式 ===
- 1v1 咨询：500-2000/时
- 训练营：3000-8000/期
- 付费社群：199-499/年
- 结论：变现可行

=== 实施建议 ===
- 从知乎/公众号内容起步
- MVP：先做 1v1 咨询，验证付费意愿
- 风险：需要持续输出内容
- 结论：推荐 ⭐⭐⭐⭐
```

### 示例 3：验证"殡葬行业"

```
你：帮我看看殡葬行业能不能做

我：
=== 需求验证 ===
- 市场规模：2023年 3358 亿元
- 老龄化：60岁以上 3.1 亿，占比 22%
- 结论：需求明确

=== 竞品分析 ===
- 传统殡葬：有福寿园等，但数字化程度极低
- 互联网+殡葬：几乎空白
- 结论：蓝海

=== 实施建议 ===
- 建议做"白事服务信息平台"（轻资产）
- 不碰遗体相关业务
- 需要资质，注意政策风险
- 结论：可以做，但有门槛
```

## 深度分析技巧

### 1. 问"如果不做会怎样"

用户如果不用你的产品，会怎么样？
- 忍痛继续用现有方案 → 有需求
- 无所谓 → 伪需求

### 2. 问"用户现在怎么做"

用户现在怎么解决这个问题？
- 有成熟方案 → 市场存在
- 没办法/硬扛 → 痛点强

### 3. 问"愿意付多少钱"

直接问用户：
- "如果这个产品收费 X 元，你会买吗？"
- 沉默/回避 = 不愿意

### 4. 算最小收入

目标：月入 1 万
- 客单价 100 元 → 需要 100 个付费用户
- 转化率 5% → 需要 2000 个潜在用户
- 能做到吗？

## 重要原则

1. **数据说话**：不要"我觉得"，要"数据是"
2. **案例参考**：类似想法的成败案例
3. **动手优先**：验证完赶紧做，不要完美主义
4. **迭代思维**：想法会变，验证是迭代过程
5. **最小成本**：用最低成本验证，再放大
