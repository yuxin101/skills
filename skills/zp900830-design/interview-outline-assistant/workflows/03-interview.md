# 访谈阶段：提问技巧与模拟演练

## 执行流程

```
开场破冰 → 场景还原 → 深度挖掘 → 假设验证 → 开放收尾
```

---

## 提问策略选择

根据受访者情况选择提问结构：

| 策略 | 适用场景 | 特点 |
|------|---------|------|
| **漏斗式**（从宽到窄） | 探索型访谈 / 首次了解用户 | 先发散再聚焦，适合挖掘未知痛点 |
| **倒漏斗式**（从窄到宽） | 时间有限 / 专业用户 | 先确认事实再延伸，效率高 |

- [ ] 已根据受访者类型选择合适的提问策略 ⭐⭐⭐

---

## 开场破冰（前5分钟）

**目标**：建立信任，确认授权，进入状态

- [ ] 自我介绍（姓名 + 角色 + 本次目的） ⭐⭐⭐
- [ ] 征得录音/记录授权 ⭐⭐⭐
- [ ] 说明"没有对错"，引导对方说真实想法 ⭐⭐⭐
- [ ] 请对方简述职责和典型工作日 ⭐⭐

**开场话术示例**：
```
"今天访谈大概30分钟，我想了解您在工作中使用系统的真实场景。
没有标准答案，我们特别想听您说一些觉得不顺手的地方，
这对我们改进产品非常有价值。请问我可以录音方便回顾吗？"
```

## 核心提问技巧

### 技巧1：场景还原（最重要）⭐⭐⭐

**原则**：不问"你觉得"，问"你上次"

| 错误问法 ❌ | 正确问法 ✅ |
|-----------|-----------|
| 你觉得报表功能好用吗？ | 你上次查报表是什么场景？能描述一下整个过程吗？ |
| 审批流程复杂吗？ | 你最近一次发起审批，从开始到结束经历了哪些步骤？ |

- [ ] 提问时聚焦具体事件而非主观评价 ⭐⭐⭐

---

### 技巧2：梯子追问（挖隐性需求）⭐⭐⭐

从表面需求逐层追问，找到真实动机：

```
用户说："希望有个导出功能"
↓ 追问："导出之后，你一般拿这个数据做什么？"
用户说："发给我的领导看"
↓ 追问："领导通常需要看哪些内容？格式上有要求吗？"
用户说："他要在周报里用，所以需要 Excel 格式的汇总"
↓ 真实需求：定期汇总报表 + Excel 导出，而非简单导出功能
```

- [ ] 遇到功能需求请求，至少追问3层"为什么/拿来做什么" ⭐⭐⭐

---

### 技巧3：绕开敏感问题 ⭐⭐⭐

| 敏感直问 ❌ | 侧面绕问 ✅ |
|-----------|-----------|
| 你有没有把账号借给别人？ | 你们团队是怎么分配系统操作权限的？ |
| 你们有没有绕过审批流程？ | 当审批流程比较长时，你们一般怎么处理紧急情况？ |
| 你觉得系统有 Bug 吗？ | 有没有遇到过系统表现不符合你预期的情况？ |

- [ ] 涉及合规/安全话题时使用侧面问法 ⭐⭐⭐

---

### 技巧4：抽象具象化 ⭐⭐

遇到"挺好的""还可以"等模糊回答，立刻量化追问：

```
用户说："这个功能用起来还行"
↓ 追问："你一周大概用几次？每次大概花多少时间？"
↓ 追问："有没有哪次用起来特别顺手/特别麻烦的经历？"
```

- [ ] 收到模糊正面反馈时，必须用频次/时间/具体事件追问 ⭐⭐⭐

---

### 技巧5：控场与拉回主线 ⭐⭐

用户跑题时，用"总结+转场"公式：

```
"您说的这个 [X] 很有价值，我记下来了。
我们回到刚才的 [核心话题]，您说到 [关键点]，
能再展开说说 [具体问题] 吗？"
```

- [ ] 每次拉回主线前，先肯定对方当前话题 ⭐⭐

---

### 技巧6：对比性问题测试偏好 ⭐⭐

```
"如果有两种方案：
A方案是 [描述A]
B方案是 [描述B]
在你的日常工作场景里，你会倾向于哪个？为什么？"
```

- [ ] 对比题时，两个方案描述对等、无明显倾向性 ⭐⭐⭐

---

## 实时记录要点

- [ ] 记录用户原话（含情绪词汇、叹气、停顿） ⭐⭐⭐
- [ ] 标注"高痛点"（用户多次提及或情绪激动处） ⭐⭐⭐
- [ ] 区分"用户诉求"与"用户提出的解决方案" ⭐⭐⭐
- [ ] 记录用户提到的**工具替代**（如用微信/Excel替代系统） ⭐⭐

## 模拟演练说明

**当用户需要对练时**，我将扮演受访者角色：

1. 告知我：角色（操作层/管理层/决策层）+ 所在行业 + 核心使用场景
2. 我会模拟真实B端用户的回答风格（包含含糊、跑题、抱怨、敷衍等情况）
3. 你来提问，我来回应，之后一起复盘你的提问技巧

## 常见错误速查 ❌

| 错误行为 | 正确行为 |
|---------|---------|
| 接受"挺好的"就跳过 | 追问具体场景和频次 |
| 问"你觉得应该怎么设计" | 只问问题，不找用户要方案 |
| 问完就记录，不追问 | 用梯子理论至少追3层 |
| 帮用户补全回答 | 沉默等待，让用户自己说完 |
| 时间到了仍不收尾 | 提前5分钟控场，留结尾问题 |
