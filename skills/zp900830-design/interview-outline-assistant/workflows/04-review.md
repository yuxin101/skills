# 复盘阶段：记录整理与报告生成

## 执行流程

```
转录整理 → 痛点提取 → 机会点识别 → 体验地图 → 报告撰写
```

---

## Step 1：访谈记录整理

**在访谈结束后24小时内完成**，记忆最清晰。

- [ ] 已将录音/笔记转录为结构化文本 ⭐⭐⭐
- [ ] 每位受访者单独建立记录文件（含：角色、公司规模、使用场景） ⭐⭐
- [ ] 标注**情绪词汇**（"很麻烦"、"根本用不了"、"还不错"） ⭐⭐⭐
- [ ] 标注**工具替代行为**（用Excel/微信/纸质代替系统的情况） ⭐⭐⭐

### 记录模板（单人访谈）

```
## 受访者档案
- 姓名/代号：[A1]
- 角色：[操作层 - 采购专员]
- 公司规模：[500人]
- 使用时长：[8个月]
- 访谈时间：[2026-03-01 14:00]

## 核心使用场景
[描述用户主要用系统做什么]

## 痛点记录（原话）
1. "[原话1]" → 标签：#效率 #重复操作
2. "[原话2]" → 标签：#数据查找 #高频
3. "[原话3]" → 标签：#权限 #协作

## 工具替代行为
- 用 Excel 手动汇总 XX 数据（系统无法导出）
- 用微信发图确认审批（审批流通知不及时）

## 正面反馈
[用户提到的满意点]

## 用户提出的方案（仅记录，不采纳为需求）
[用户说"希望有XX功能"]
```

---

## Step 2：痛点提取与分级

### 痛点分级矩阵

| 级别 | 条件 | 处理优先级 |
|------|------|-----------|
| P0 严重痛点 | ≥50%受访者提及 + 有替代行为 | 立即排期 |
| P1 高频痛点 | ≥30%受访者提及 | 下个迭代 |
| P2 中频痛点 | ≥2人提及 | 纳入规划 |
| P3 低频痛点 | 仅1人提及 | 观察积累 |

- [ ] 已对所有痛点按频次统计排序 ⭐⭐⭐
- [ ] 已按 P0-P3 完成痛点分级 ⭐⭐⭐
- [ ] 已区分"用户诉求（痛点）"与"用户方案（要求）" ⭐⭐⭐

---

## Step 3：设计机会点识别

**从痛点推导机会点**，而不是直接把用户方案变成需求：

```
痛点：用户每次查历史订单要翻很多页，"太难找了"（P0，8/12人提及）
用户方案：希望加个搜索框
↓ 深层分析：
用户真实需求：快速定位特定订单（按条件过滤）
设计机会点：订单列表增加多维度筛选（时间范围/状态/金额区间/关键词）
```

- [ ] 每个P0/P1痛点都已推导出对应的设计机会点 ⭐⭐⭐
- [ ] 机会点描述的是"要解决的问题"，而非"具体方案" ⭐⭐⭐

---

## Step 4：用户体验地图（简版）

针对核心功能链路，逐节点标注：

| 节点 | 用户操作 | 情绪状态 | 痛点 | 机会点 |
|------|---------|---------|------|--------|
| 登录 | 输入账号密码 | 😐 中性 | - | - |
| 找到功能入口 | 在导航中搜索 | 😕 困惑 | 入口层级深 | 常用功能置顶 |
| 填写表单 | 逐个录入字段 | 😤 烦躁 | 字段过多，无历史填充 | 上次记录自动带入 |
| 提交审批 | 点击提交 | 😌 完成 | 不确定是否成功 | 提交后状态明确反馈 |
| 等待结果 | 手动查询进度 | 😤 焦虑 | 无通知，要主动刷新 | 审批进度实时推送 |

- [ ] 已覆盖目标功能的完整操作链路 ⭐⭐⭐
- [ ] 已在每个节点标注情绪状态 ⭐⭐
- [ ] 情绪低谷节点（😤😕）均对应了痛点和机会点 ⭐⭐⭐

## 报告质量检查清单

- [ ] 摘要清晰，3-5句话可独立传递核心结论 ⭐⭐⭐
- [ ] 痛点有频次数据支撑（非"有人反映"，而是"X/X人提及"） ⭐⭐⭐
- [ ] 有原话引用，增强说服力 ⭐⭐⭐
- [ ] 设计机会点描述的是"问题"而非"方案" ⭐⭐⭐
- [ ] 建议分优先级，不是一锅粥 ⭐⭐⭐
- [ ] 体验地图覆盖情绪低谷节点 ⭐⭐
- [ ] 报告结论与访谈数据一致（无主观臆断） ⭐⭐⭐

---

## 一票否决

- 报告只有用户原话，没有提炼和洞察
- 直接把用户说的"想要XX功能"写进设计建议
- 优先级全部是P0（说明没有真正分析）
- 数据来源不透明（X/X人比例必须标注）
