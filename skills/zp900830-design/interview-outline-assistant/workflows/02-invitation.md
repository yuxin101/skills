# 约访阶段：受访者筛选与邀约

## 执行流程

```
确定配额 → 筛选候选人 → 发送邀约 → 确认回复 → 准备材料
```

---

## Step 1：受访者配额规划

### 角色分层配额表

| 角色层 | 建议人数 | 筛选优先级 |
|--------|---------|-----------|
| 操作层（高频用户） | 5-6人 | 最近30天活跃，使用核心功能≥3次/周 |
| 管理层（中频用户） | 5-6人 | 负责团队流程审批或数据查看 |
| 决策层（低频用户） | 3-4人 | 采购决策者或产品续费关键人 |

- [ ] 已确定各角色访谈人数配额 ⭐⭐⭐
- [ ] 操作层优先覆盖高频用户（最有真实痛点） ⭐⭐⭐
- [ ] 决策层单独约访，不与操作层混访 ⭐⭐

### 特殊用户价值极高，优先纳入

| 用户类型 | 价值 |
|---------|------|
| 近期流失/降频用户 | 暴露严重痛点 |
| 内部积极反馈者 | 验证正向假设 |
| 刚入职的新用户 | 发现新手卡点 |
| 同时使用竞品的用户 | 对标竞品差距 |

- [ ] 已加入≥1类特殊用户 ⭐⭐

---

## Step 2：筛选渠道

- [ ] 平台数据：从后台筛选符合行为标签的用户 ⭐⭐⭐
- [ ] 内部推荐：客成/销售团队推荐有代表性的客户 ⭐⭐⭐
- [ ] 社群/渠道：从产品社群、用户微信群定向邀约 ⭐⭐
- [ ] 避免只访谈"愿意配合"的用户（幸存者偏差风险） ⭐⭐⭐

---

## Step 3：邀约话术模板

### 模板A：正式邮件/钉钉（适合决策层）

```
主题：诚邀参与 [产品名] 用户体验访谈（30分钟）

[对方姓名] 您好，

我是 [公司名] 产品团队的 [你的姓名]。

我们正在开展一次用户体验研究，目的是了解您在 [核心场景] 方面的真实使用
体验，以便我们在下一个版本中重点优化您最关心的功能。

希望邀请您参与一次 30 分钟的线上访谈：
- 时间：[提供2-3个时间选项]
- 形式：视频通话（腾讯会议 / 飞书）
- 内容：聊聊您的日常工作场景，没有标准答案

本次访谈结果仅用于产品改进参考，不涉及商务内容，全程保密。

期待您的回复，感谢您的支持！
```

### 模板B：微信/企微（适合操作层，简短友好）

```
您好 [姓名]！

我是 [公司] 的产品同学。最近我们在做一个用户调研，想了解下大家日常
使用 [产品] 的真实体验～

时间不长，大概30分钟，主要是聊聊您工作中用到系统的场景，
帮我们找找哪些地方还可以做得更好。

方便的话，这几个时间您哪个合适？
① [时间1]
② [时间2]
③ [时间3]

感谢！🙏
```

---

## Step 4：邀约质量检查

- [ ] 话术中已说明**访谈目的**（改进产品，非商务） ⭐⭐⭐
- [ ] 话术中已说明**信息用途**（保密，仅用于产品研究） ⭐⭐⭐
- [ ] 提供了**明确的时间选项**（减少对方决策成本） ⭐⭐
- [ ] 说明了**访谈时长**（30-45分钟为佳） ⭐⭐
- [ ] 话术语气与受访者职级匹配（决策层正式，操作层亲切） ⭐⭐
- [ ] 未透露具体访谈问题（避免提前准备影响真实性） ⭐⭐⭐

---

## Step 5：访谈前准备清单

- [ ] 已发送日历邀请并附视频会议链接 ⭐⭐⭐
- [ ] 已准备好录音授权告知（访谈开始时征得同意） ⭐⭐⭐
- [ ] 已备好对应角色的访谈提纲（参考 `workflows/01-preparation.md`） ⭐⭐⭐
- [ ] 已了解受访者基础信息（公司规模、岗位、使用时长） ⭐⭐
- [ ] 已设置 30 分钟访谈结束前的提醒（控制时间） ⭐⭐
