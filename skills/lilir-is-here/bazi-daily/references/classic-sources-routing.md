# Classic Sources Routing

本文件定义 `bazi-daily` 在三本经典之间的职责边界，确保“结构、调候、原理”分层清晰，不混库使用。

## Source Registry

- A（原理层）：`references/classics/A_滴天髓.txt`
- B（结构层）：`references/classics/B_渊海子平.txt`
- C（调候层）：`references/classics/C_穷通宝鉴.txt`

若上述 txt 文件不可读，立即报错"经典文本文件缺失，无法完成分析"，不得使用其他路径或内置知识替代。

## Routing Rules

### 1) B《渊海子平》优先判结构

适用问题：
- 是否成格、何格为主、格局是否破坏
- 用神框架与十神结构关系

输出要求：
- 给出“主格/兼格/不成格”结论
- 给出结构层用神框架（主次序）
- 标记 `[B-结构]`

### 2) C《穷通宝鉴》执行调候校正

适用问题：
- 月令寒暖燥湿偏性
- 是否需要扶抑与调候并用

输出要求：
- 明确“寒暖燥湿”偏向与校正动作
- 说明是否改变结构层用神次序
- 标记 `[C-调候]`

### 3) A《滴天髓》用于气机解释

适用问题：
- 为什么这样定取用
- 如何解释旺衰、生克、流通的方向

输出要求：
- 对结构+调候后的最终结论做体系化解释
- 强调趋势与条件，不做宿命断语
- 标记 `[A-原理]`

## Conflict Resolution

当三源出现张力时，按以下顺序决策：

1. 锁定 `B` 的结构边界（不可越界取用）
2. 用 `C` 做季节与体感层校正（可调主次，不可颠覆边界）
3. 用 `A` 解释最终取舍逻辑与风险条件

## Minimal Output Contract

最终回答至少包含以下三个证据段：

1. `结构结论`（含 `[B-结构]`）
2. `调候校正`（含 `[C-调候]`）
3. `气机解释`（含 `[A-原理]`）

并在最后提供可执行建议：
- 宜（2-4 条）
- 忌（2-4 条）
- 风险提示（1 句）
