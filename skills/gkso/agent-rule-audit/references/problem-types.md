# Problem types for agent-rule-audit

Use these categories when auditing an agent's behavior layer.

## 1. Structure confusion
Rules with different roles are mixed together.
Examples:
- working style mixed with temporary trial rules
- persona mixed with operational workflow
- hard boundaries buried inside soft style guidance

## 2. Repetition / redundancy
The same rule exists multiple times with little added value.
This makes the behavior layer longer without making it stronger.

## 3. Rule conflict
Two files push behavior in different directions.
Examples:
- one file rewards warm responsiveness while another demands act-first execution
- a local rule and a shared rule subtly disagree

## 4. Focus drift
The most important rule is no longer the clearest rule.
This often happens when the file keeps growing and older priorities stay in place.

## 5. Behavior dilution
There are too many weak rules around the important rule, so the key behavior no longer stands out.

## 6. Symptom-vs-root-cause confusion
The rule set focuses on visible symptoms instead of the real failure mode.
Examples:
- trying to stop “progress-sounding” messages when the real issue is lack of execution
- trying to improve tone when the real issue is broken follow-through

## 7. Style overpowering execution
The rules strongly reward sounding helpful, responsive, warm, or smooth, but do not equally reward actually doing the work.

## 8. Stale guidance
Old rules remain after the goal or working method has changed.
The files look updated, but older guidance still pulls behavior backward.

## 9. Trial rule not truly live
A self-improvement goal exists, but the live behavior-layer files were never updated enough for the trial to matter.

## 10. Evidence mismatch
Daily examples, learnings, or user corrections point one way, but the core rules still emphasize something else.
