# Logo 自动核验说明

## 核验目标

确保每个下载的 Logo 图片确实属于对应公司，不出现张冠李戴的情况（如下载到的是搜索结果排名靠前的无关图片）。

## 核验流程

```
下载 Logo
   ↓
① 图片质量检查 → 不通过 → 标记失败
   ↓
② OCR 文字识别（提取图片中的文字）
   ↓
③ 关键词匹配评分（中文名 + 英文名）
   ↓
分数 ≥ 0.3 → 通过
分数 < 0.3 且有文字 → 失败，重新搜索
分数 = 0（无文字） → 视为图形Logo，通过（置信度中等）
```

## 图片质量检查规则

| 规则 | 判断标准 |
|------|---------|
| 文件大小 | < 1KB → 拒绝 |
| 图片尺寸 | 宽或高 < 30px → 拒绝 |
| 尺寸过大 | 宽或高 > 5000px → 可能是截图页面，拒绝 |
| 颜色单一 | 大图（>800px）且颜色方差 < 50 → 可能是空白图，拒绝 |

## OCR 匹配评分

**分数含义**：

| 分数 | 含义 |
|------|------|
| 0.8 | 图片中识别到公司中文名关键字 |
| 0.7 | 图片中识别到公司英文名关键字 |
| 0.5 | 图片中无文字（图形类Logo），不能靠OCR判断，给中等置信度 |
| < 0.3 | 图片中有文字但没有公司名关键字，可能错误 |

**中文关键词提取**（去掉常见后缀再匹配）：
- 去除：有限公司、股份有限公司、集团、控股、企业、科技、管理、咨询、代理、银行、航空、酒店

**英文关键词提取**（取前2个实义词）：
- 去除：The, Limited, Holdings, Company, Group, Management, Services, Co, Ltd, Inc, Corp

## 重新搜索策略

核验失败时，按以下顺序尝试不同关键词（最多3次）：
1. `{中文名} logo 官方`
2. `{英文名} logo official`
3. `{中文名} {英文名} 品牌标志`

## OCR 安装方法

```bash
# macOS
brew install tesseract tesseract-lang
pip install pytesseract pillow

# Ubuntu/Debian
sudo apt-get install tesseract-ocr tesseract-ocr-chi-sim
pip install pytesseract pillow
```

若未安装 tesseract，verify_logos.py 会退化为仅做图片质量检查（不做 OCR），此时应生成预览 HTML 页面供人工抽查。

## 已知局限性

1. **纯图形Logo**（无文字）：如 Apple、耐克对号等无法通过 OCR 核验，会被标为"中等置信度通过"
2. **搜索结果噪声**：百度图片搜索有时会返回介绍该公司的文章配图，OCR 可能识别到公司名但图片并非官方 Logo
3. **Logo 语言**：部分国际公司在中国只有中文Logo，搜索到的可能与英文官方Logo不同
