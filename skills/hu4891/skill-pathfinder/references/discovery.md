# 全网发现与安全评估模块 (Discovery & Evaluation)

本模块在本地能力不足时生效，指导你如何向外部进行拓展，并确保引入技能的质量与安全。

## 1. 外部渠道检索触发机制与优先级
当你在执行意图匹配时，确信**本地技能库中没有任何一个匹配项**时，自动触发向外的检索。

**请按照以下优先级顺序，利用搜索工具在指定的站点内进行定向检索：**

### 第一梯队平台（优先）
1. **[ClawHub](https://clawhub.ai/)**：OpenClaw 的官方技能仓库。
2. **[MCP Market](https://mcpmarket.com/)**：专注于 MCP 插件的市场。
3. **[Smithery](https://smithery.ai/)**：丰富且成熟的 AI Agent 扩展包生态。
4. **[Glama](https://glama.ai/)**：发现各类前沿 AI 技能模块的平台。

### 第二梯队（兜底补充）
如果在以上四大平台没有找到结果，可前往 **Twitter / X 等社交平台** 执行发散性搜索，寻找开发者社区中的实验性项目。

## 2. 安全检查（双轨策略）

当找到候选技能后，在推荐给用户之前需要进行安全检查：

### 基础检查（内置，永远可用）
使用内置的 `python scripts/security_check.py <目标地址或 slug>` 进行基础合规扫描。该脚本为纯 Python 实现，零外部依赖，检查项包括：URL 格式校验、代码中的可疑调用模式、敏感路径访问等。

### 深度扫描（如果用户已安装 AgentGuard）
如果用户环境中已安装 `@goplus/agentguard` 插件，可额外调用其深度扫描能力（24 类检测规则）。如果未安装，跳过此步，依赖基础检查结果即可。

### 安检结果处理
- 如果检查**通过** → 进入汇报环节
- 如果检查**未通过** → 记录原因，继续寻找替代方案，最终在汇报中如实告知用户

## 3. 简洁汇报与安装授权

向用户汇报发现结果时，**请使用简洁的段落格式**，只包含以下四个要素：

### 汇报模板
1. **任务拆解**：简明列出每一步的安排（Pipeline）
2. **匹配技能**：每步推荐使用的 Skill 名称
3. **适配判断**：一句话说明该 Skill 是否合适
4. **索要授权**：是否允许安装

**禁止输出以下内容**：
- ❌ 平台检索结果表格
- ❌ 候选技能多维对比矩阵
- ❌ 安全扫描内部细节
- ❌ Markdown 表格排版

### 汇报示例
> 📋 **任务安排**
>
> 第一步：搜索 ArXiv 论文 → 推荐使用 **arxiv-watcher**，专门用于论文检索与摘要提取，非常适合。
>
> 第二步：写入 Notion → 推荐使用 **notion**，支持 Notion API 页面创建，完全匹配。
>
> 基础安检均已通过。是否允许我为您安装这两个技能？

### 授权等待
汇报输出后请暂停，等待用户明确回复同意后再执行安装操作。请避免在未获得用户授权的情况下执行下载或安装。
