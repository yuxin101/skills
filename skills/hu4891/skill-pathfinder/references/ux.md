# 用户交互与兜底保障 (UX & Fallback)

确保整个"调度-寻找-安装-执行"的链路对用户是友好的、透明的，以及最重要的——**受控且匹配用户语言环境的**。

## 1. 全局多语言适配机制 (Language Mirroring)
本元技能在对用户进行所有的话术反馈（包括状态栏、告警提示、询问授权）时，**请避免硬编码为固定的中文或英文**。
- **自动镜像**：侦测并跟随用户在本次会话中发起的主控语言进行同步。例如，如果用户用英文写 "Find a tool"，你所有的进度更新和询问词都应切换为英文；若用户说中文，同理跟从。
- **Emoji 辅助**：凡是进入等待确认、出现提醒或系统降级的状态时，推荐混合使用国际通用 Emoji（如 ⚠️, 🌐, ✅, 📥）来增强视觉提示。

## 2. 状态透明与进度反馈
在执行复杂的编排链路时，如果预期等待时间较长，应向用户反馈当前的运行进程状态：
- **任务规划反馈**：当拆解复杂意图后，开场即告诉用户计划。
  *范例*："✅ **任务编排已确立**：`[查找股市研报]` -> `[财务图表生成]` -> `[总结并抄送邮件]`。开始执行第一步..."
- **外网检索反馈**：
  *范例*："🌐 **正在搜索**：本地库暂无匹配功能，正在接入 ClawHub / MCP Market 进行寻源..."
- **安全检查反馈**：在执行安全扫描时，建议向用户展示进度。
  *范例*："正在为您对 `excel-parser` 进行安全检查，请稍候..."

## 3. 授权确认节点 (Authorization Checkpoints)
为了确保安全，系统在特定节点应暂停等待用户确认：
- **环境补齐建议后**：当基线扫描探测到缺少推荐技能并给出建议时。
- **外部技能安装前**：当从外部获取到新技能并通过安全检查后，展示简洁汇报并等待用户授权。

在这两个节点，请暂停所有下载/部署操作，向用户确认："是否允许我为您安装？"，在获得用户明确同意后再继续执行。

## 4. 多意图消歧追问
- 用户的语言很多时候是不精确的。当需求指向不明，或检索到多个能力重叠的 Skill 时，请不要盲目猜测。
- 应当主动询问，给出具体选项让用户做选择。
   *范例*："您说的'帮我发那份报告'，是想通过【内网 OA】还是【外部邮件】发送呢？"

## 5. 分阶段优雅降级 (Graceful Degradation)
如果整个流程走到死胡同：
1. 本地没有可用技能。
2. 跨网搜索也没有符合条件的结果。
3. 搜索到了选项，但用户拒绝安装。

**此时，请做到优雅降级：**
- 不要生硬报错。
- 礼貌地退回到系统最基础的原生对话模式，尝试给出一个参考性的探讨结果。
- **记录需求缺口**：将本次未满足的需求记录下来，作为未来改进的参考线索。
