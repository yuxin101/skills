# 执行标准与行为准则 (Execution Operations)

## 0. 安全执行原则
在处理所有**外部资源引入**的任务时（特别是涉及 `npm install`、`git clone` 或直接向硬盘写入新文件），请在执行前向用户展示完整命令并获得确认。
- 只有在用户明确回复同意后，才执行安装操作。

## 1. 甄选目标与黄金准则 (The Target Standard)

本模块旨在让用户的 Agent 持续拥有最新的硬核生产力扩展，同时保持极高的推荐纯净度与交互优雅。

大模型在考虑向用户推荐任何新 Skill 之前，请对目标进行以下标尺测算：
> 值得推荐给用户的组件，应具有明显的**"生产力突变"**、能大幅扩展现有 Agent 的**物理作业边界**（如：直接操控本地文件系统、能链接或控制外部重型 API、打通复杂工作流）。
> ❌ **排除**：纯文字游戏、算命娱乐、简单的套壳 API、重造的轮子。

## 2. 互补型全网多渠道雷达 (Multi-channel Discovery)
系统不再被动等待，而是定期或在用户表现出"探索新玩意"意图时，启用双线并行的嗅探机制：
- **前沿阵地（极客风暴）**：使用搜索能力前往 **Twitter (X)** 等技术社群，检索 `#AI #AgentSkills #ClawHub #MCP` 等相关话题，捕获热度正在飙升的工具。
- **正规军火库（市场榜单）**：前往 **ClawHub、MCP Market、Smithery** 等核心源站，抓取精选或近期下载量突增的组件。

## 3. UI 交互：纯净段落式推荐陈列 (Clean Text-only Formatting)
向用户展示推荐时，请遵循以下排版规范：
🚨 **禁止使用 Markdown 表格排版输出** 🚨

使用清爽的"一物一段"形式。每推荐一个组件，格式为：

**【组件名称】**，一到两句话的痛点说明。
🔗 直达源：`[链接]`

*(每个组件空一行，结构为：加粗名字 + 人话说明 + 短链接。)*

## 4. 每日精选推荐定时任务 (Daily Recommendation Schedule)

### 触发条件
当用户在安装/更新后的 `[AGENT POST-INSTALL HOOK]` 流程中同意开启每日推荐时，在 Memory 中写入：
`Skill-Pathfinder 每日推荐定时任务: 已启用, 推送时间: 用户所在地区每日 10:00`

### 推送规则
- **推送时间**：用户偏好地区的**每天早上 10:00**
- **推送数量**：每次只推荐 **1 个**精选组件
- **推送格式**：使用上述第 3 节的段落式格式
- **甄选标准**：严格遵循第 1 节的黄金准则
- **不重复**：已推过的组件不再重复推送
- **静默撤回**：如果当天没找到符合标准的组件，不输出任何消息

### 示例推送

> 📡 **今日精选推荐**
>
> **【data-visualizer】**，支持在终端中直接生成交互式图表和仪表盘。非常适合您最近频繁进行的数据分析工作，省去了导出到第三方工具的步骤。
> 🔗 直达源：`https://clawhub.ai/xxx/data-visualizer`
>
> 是否安装？回复 Y 即可。

### 关闭机制
用户可以随时说"关闭每日推荐"或类似意图，此时在 Memory 中更新为已关闭状态。
