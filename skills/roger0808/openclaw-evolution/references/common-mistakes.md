# Common Mistakes

Things that trip up almost every new OpenClaw user.

---

## ❌ "I want to set up everything on Day 1"

**What happens:** You spend 4 hours configuring skills, cron jobs, multiple channels, complex AGENTS.md rules... and then get overwhelmed when something breaks.

**Do instead:** Minimal setup → use for a few days → add one thing → repeat. You don't know what you need until you start using it.

---

## ❌ "I wrote SOUL.md once and never touched it again"

**What happens:** The agent's personality feels "off" but you can't pinpoint why. You get frustrated and blame the AI.

**Do instead:** Treat SOUL.md as a living document. After 2-3 days of use, revisit it. Add things you liked, remove things that didn't work, be more specific about style. The 10th version will feel completely different from the 1st.

---

## ❌ "I skipped the memory system"

**What happens:** Every conversation starts from zero. The agent asks the same questions, forgets decisions, can't build on previous work. It feels like a fancy chatbot.

**Do instead:** Set up daily notes (memory/YYYY-MM-DD.md) from Day 2. Even minimal notes ("Decided to use React for the project. User prefers dark mode.") compound over time.

---

## ❌ "I copied someone else's entire config"

**What happens:** The agent behaves in ways that make sense for someone else but not for you. Rules conflict with your workflow. You end up fighting the configuration instead of using it.

**Do instead:** Start with the default template. Read other people's configs for inspiration, but adapt to YOUR workflow. Your agent should fit you like a glove, not someone else's glove.

---

## ❌ "I treat the agent like a search engine"

**What happens:** You only ask factual questions or give simple commands. The agent never develops personality or context. It stays at Level 1 forever.

**Do instead:** Share context. Say why you're asking, not just what. "I'm preparing for a meeting with my boss about this project, can you help me think through the risks?" vs "List project risks."

---

## ❌ "I automated too much too fast"

**What happens:** Multiple cron jobs firing, heartbeats checking things you don't care about, notifications you didn't expect. You turn everything off in frustration.

**Do instead:** One automation at a time. Run it for a week. If it's genuinely useful, keep it. If not, remove it. The goal is fewer distractions, not more.

---

## ❌ "I never gave feedback"

**What happens:** The agent does things in a mediocre way forever because it doesn't know what you like or dislike.

**Do instead:** When the agent does something well, say so specifically ("I liked how you structured that summary"). When it does something wrong, explain what you wanted instead. This is training through conversation, and it works.

---

## ❌ "I expected it to be perfect immediately"

**What happens:** First interaction isn't amazing, so you conclude "it's not that useful" and abandon it.

**Do instead:** Give it a week. The first day is setup. The first week is calibration. The real value shows up in week 2-3 when the agent has memory, knows your preferences, and starts being proactive.
