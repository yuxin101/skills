## 📊 评分说明

### 待办质量评分（5 星）

| 星级 | 说明 | 标准 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 高价值 | 具体 + 可执行 + 重要 + 完整 |
| ⭐⭐⭐⭐ | 中价值 | 较具体 + 较可执行 + 较重要 |
| ⭐⭐⭐ | 一般 | 一般具体 + 一般可执行 |
| ⭐⭐ | 低价值 | 模糊 + 依赖外部因素 |
| ⭐ | 待优化 | 不明确 + 不可执行 |

**评分维度**：
- 具体性（25%）：是否明确具体
- 可执行性（25%）：是否可以立即行动
- 重要性（25%）：影响范围和紧急程度
- 完整性（25%）：是否有上下文信息

### 想法价值评分（5 星）

| 星级 | 说明 | 标准 |
|------|------|------|
| 💎💎💎💎💎 | 高价值 | 创新 + 可行 + 高收益 |
| 💎💎💎💎 | 有价值 | 较创新 + 较可行 |
| 💎💎💎 | 一般 | 一般创新 + 一般可行 |
| 💎💎 | 低价值 | 缺乏创新 |
| 💎 | 待完善 | 不可行 |

**评分维度**：
- 创新性（30%）：是否新颖
- 可行性（30%）：是否可实现
- 价值度（20%）：带来的收益
- 关联度（20%）：与当前项目的相关性

## ❓ 常见问题

### Q1: 为什么总结的内容不完整？
**A**: 可能原因：
- 对话历史超过 200 条 → 只会获取最近 200 条
- 包含昨天的对话 → 自动过滤，只保留今天
- 反馈"有遗漏" → 我会学习并改进

**解决**：
- 提供具体内容让我总结
- 说"总结关于 XXX 的讨论"指定范围
- 反馈遗漏内容，我会补充

### Q2: 评分是否准确？
**A**: 评分基于 4 个维度综合计算，初期可能不够准确。

**改进**：
- 提供反馈（"评分不合理"）
- 我会学习你的偏好
- 持续优化评分算法

### Q3: 可以总结昨天的对话吗？
**A**: 当前版本只支持总结**今天**的对话。

**原因**：
- 聚焦当天内容，更有针对性
- 避免信息过载

**变通**：
- 查看 memory 目录的历史记录
- 提供昨天的对话内容让我总结

### Q4: 保存的文件在哪里？
**A**: 保存在 `memory/{date}.md`

**查看**：
```bash
ls memory/
cat memory/2026-03-25.md
```

### Q5: 如何删除已保存的内容？
**A**: 手动删除文件：
```bash
rm memory/2026-03-25.md
```
