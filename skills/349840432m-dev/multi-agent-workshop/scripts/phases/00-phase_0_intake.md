# 阶段 0：任务澄清 — `phase_0_intake`

## 为何需要阶段 0

用户常用「如何写出高质量 X」「帮我做一篇 Y」等**模糊表述**：可能要 **方法论/SOP**，也可能要 **完整成稿**。若不先对齐**交付物类型**，后续容易把「研讨方案」做成「直接交文章」，或相反。

## 导演检查清单

- [ ] **期望交付物类型**已在 `state.md` 勾选或写明（见 `templates/state.example.md`「阶段 0」）
- [ ] **非目标**写清（本 session 明确不交付什么；例：不要默认产出全文成稿）
- [ ] **成功标准**一句话可验收（阶段 6 对照用）
- [ ] **与用户原话对齐**：用户原话摘要 + 导演对交付物类型的理解（有歧义须追问，**不得猜**）
- [ ] 若用户坚持「都要」→ 写 **混合** 并拆分：哪部分止于方案、哪部分止于成稿、先后顺序
- [ ] `orchestrator.py set <sid> deliverable_type <type>` + `set <sid> success_criteria "..."` → `orchestrator.py advance <sid>`
