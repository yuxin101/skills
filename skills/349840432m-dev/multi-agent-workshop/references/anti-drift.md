# 防走样（导演必读）

从 `SKILL.md` 拆出的详细规则。主文档保留三条摘要 → 本文件。

---

## `director_monologue` 与「压缩模式」关系

- **`mode: director_monologue`**：由导演**一人执笔**模拟多角色发言（无独立 subagent 痕迹）。  
- **「压缩模式」**：省略**部分**规定动作（如少一轮互评、跳过迫问、省略 subagent）。  
- 二者**可叠加**：常见为「独白 + 压缩」。**须在 `state.md` 与 `plan.md` 风险同时披露**。

## 建议**不要**仅用压缩/独白（须全量 subagent + 完整迫问优先）

- 对外**法律意见、监管报送、可审计的多方责任结论**  
- 用户明确要求「每个角色独立发言留痕」或「用于诉讼/合规备案」  

若用户仍坚持压缩：在 `plan.md` **风险**写清**证据效力与审计边界**。

## 其余情况

- **禁止**在同一条回复里用同一口吻输出「多角色全文」却不标 `director_monologue` / 压缩。  
- **默认**：阶段 4 **每角色至少一次独立 subagent**（或等价独立调用）；在 `transcript.md` 或 `state.md` 用标题区分（如 `### 观点轮 — 〈角色〉`）。  
- 省略 subagent：须在 `state.md` 与 `plan.md`「风险」写 **`压缩模式：已省略分角色调用`**。

## 迫问清单（`director-forcing-questions.md`）跳过格式

- **禁止**仅用「已内化」「略」等模糊表述。  
- **必须**写：**「未使用 `director-forcing-questions.md`，原因：〈一句具体原因，如用户要求 10 分钟内出稿 / 任务仅为格式转换〉」**。
