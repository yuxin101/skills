# 从招聘 JD 构建角色卡

阶段 2 确定角色后，阶段 3 构建角色卡时可用此方法：**从招聘平台抓取对应岗位 JD，提炼为角色卡**。比起凭经验编写，JD 提供了真实的职责边界、技能要求和考核关注点，角色之间的张力更接近真实团队。

## 适用场景

- 角色不在 `sample-roles/`（ops/product/tech）覆盖范围内
- 需要行业或垂直领域的专业视角（如医疗合规、跨境电商运营、数据分析师）
- 希望角色卡的"禁止项"和"关注点"更贴近真实岗位边界

## 操作步骤

### 1. 搜索 JD

根据阶段 2 确定的角色名称，在招聘平台搜索 **3～5 份** 相近岗位的 JD：

| 平台 | 适合 |
|------|------|
| BOSS直聘 | 互联网 / 创业公司，JD 简洁直接 |
| 猎聘 | 中高端岗位，职责描述更结构化 |
| 拉勾 | 技术类岗位描述较详细 |
| 脉脉 | 可看到同岗位的真实吐槽，补充"禁止项"灵感 |
| LinkedIn | 外企 / 国际化团队 |

**搜索技巧**：用角色名 + 行业关键词（如"内容运营 短视频"、"数据分析师 电商"），优先选与当前任务**行业匹配**的 JD。

### 2. 提取 → 映射

从 JD 中提取以下四类信息，对应角色卡骨架的各节：

| JD 中的内容 | 映射到角色卡 | 提取要点 |
|-------------|-------------|----------|
| **岗位职责** | **立场** | 这个角色"最坚持什么"——从核心职责中提炼 1～2 句立场声明 |
| **任职要求 / 必备技能** | **发言要求** | 转化为"发言时必须体现的专业能力"——如"必须用数据论证"、"必须考虑合规风险" |
| **加分项 / 软技能** | **发言要求（补充）** | 跨部门协作、汇报能力等 → 转化为"与其他角色互动时的行为约束" |
| **岗位边界（暗示或显式的"不负责"）** | **禁止** | JD 常写"不负责 XX"或从职责范围反推；也可从"汇报对象"判断该角色不该越权的领域 |

### 3. 补充"默认怀疑"

JD 不会直接写"我怀疑谁"，但可从以下线索推断：

- **考核指标**：KPI 与其他角色的 KPI 是否存在天然冲突（如运营追求速度 vs 技术追求稳定）
- **协作对象**：JD 写"与 XX 部门紧密配合"→ 说明这两个角色之间有交接摩擦
- **失败案例**（脉脉、知乎搜"XX岗位踩坑"）：真实的跨角色冲突场景

将这些转化为角色卡的 `默认怀疑：……` 句式。

### 4. 绑定当前任务

**关键步骤**：JD 提取的内容是"通用岗位画像"，必须在角色卡首行写明 `针对任务：〈本次任务一句话〉`，并在立场和发言要求中加入**任务特定的侧重点**。

示例——如果任务是"评审一个直播带货方案"：

- 通用 JD 立场："负责直播间整体运营策略"
- 任务绑定后："**针对本次直播带货方案评审**，重点关注 GMV 预估的合理性、主播话术的合规性、以及排品节奏是否与流量节点匹配"

### 5. 精简为角色卡

最终角色卡仍遵循 `role-card-skeleton.md` 骨架。来自 JD 的内容经过提炼后不应超过骨架的篇幅——角色卡是给 subagent 的**指令**，不是岗位说明书。

## 示例：从 JD 到角色卡

假设阶段 2 确定需要一个"合规法务"角色，任务是评审一份用户协议改版方案。

**搜到的 JD 片段**（合规法务专员）：

> 职责：审核公司产品、运营活动的合规性；跟踪监管动态；出具法律意见书。
> 要求：熟悉《个人信息保护法》《数据安全法》等；3年以上互联网公司法务经验。
> 汇报：向法务总监汇报，不直接参与产品设计决策。

**转化后的角色卡**：

```markdown
# 角色：合规法务（针对任务：评审用户协议改版方案）

## 立场
- 关注：**合规底线、监管风险、用户知情同意的完整性、条款可执行性**。
- 默认怀疑：产品和运营为追求转化率，可能弱化用户告知义务或模糊数据使用范围。

## 发言要求
1. 逐条对照 **《个保法》《数安法》** 及行业监管动态，指出条款的合规风险等级（高/中/低）。
2. 对每个风险点给出 **具体修改建议**（不能只说"有风险"不给方案）。
3. 区分"必须改"（法律硬性要求）和"建议改"（行业最佳实践），便于产品做优先级取舍。

## 禁止
- 不参与产品功能设计决策；只对法律合规面发表意见。
- 不替代业务方做风险收益权衡；风险判断归法务，接受与否归业务。
```

## 与现有流程的衔接

- 本方法是阶段 3 角色卡编写的**可选增强**，不替代 `role-card-skeleton.md` 骨架
- 如果 `sample-roles/` 已有语义匹配的角色且任务简单，可直接复用 + 加任务绑定段，不必每次都查 JD
- JD 搜索可在阶段 2 选角时同步进行，缩短阶段 3 耗时
