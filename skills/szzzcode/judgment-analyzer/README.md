# 判决书分析技能

分析判决书文档，提取关键信息，生成结构化分析报告。

## 设计思路

本技能采用 **「规则提取 + AI 智能分析」** 的混合模式：

```
Python 脚本 ──提取文本──> .txt 文件 ──AI 读取──> 智能分析 ──> 报告
```

### 为什么这样设计？

1. **扬长避短**
   - Python 适合：读取文件、批量处理、结构化提取（案号/日期等）
   - AI 适合：理解语义、识别案由、提炼争议焦点、归纳法律要点

2. **避免硬编码**
   - 传统做法：在代码里写死"如果是金融案件就分析xxx，如果是劳动案件就分析yyy"
   - 问题：案由类型上百种，根本写不完，而且分析逻辑一变就要改代码
   - 本技能做法：把分析要求写到 SKILL.md，AI 根据实际案由动态决定分析重点

3. **更灵活的分析**
   - 同一份判决书，合同纠纷和侵权纠纷的分析角度不同
   - AI 可以根据案由自动调整分析侧重点

## 核心特色

| 特色 | 说明 |
|------|------|
| 批量分析 | 支持文件夹批量处理，一次分析多份判决书 |
| 结构化提取 | 提取要求固定模板，输出格式统一规范 |
| 本地文件支持 | 直接指定本地文件或文件夹路径 |
| 混合架构 | Python 提取 + AI 分析，分工明确 |
| 案由自适应 | AI 根据案件类型自动选择分析要点 |
| 纯净文本提取 | 先将 PDF/Word 转为统一 txt，避免格式干扰 |

### 为什么先转 txt？

判决书通常是 PDF 或 Word 格式，内部结构复杂、编码多样，直接给 AI 读取容易出现：
- 乱码、换行错位
- 表格、印章等元素干扰
- 编码不一致导致的信息丢失

本技能先用 Python 提取纯净文本，得到：
- **统一格式**：不管源文件是 PDF 还是 Word，输出都是纯文本
- **干净无噪**：无多余排版符号、无乱码
- **便于 AI 理解**：AI 读取和分析更稳定，信息提取更准确
- **便于存档**：提取后的 txt 可长期保存复用

## 支持的案由类型

AI 可自动识别并分析以下类型的案件：

- 合同纠纷（买卖、租赁、借款、委托理财等）
- 侵权纠纷（人身损害、财产损害、医疗事故等）
- 劳动争议（劳动关系、赔偿金、加班费等）
- 知识产权（专利、商标、著作权等）
- 婚姻家庭（离婚、抚养权、财产分割等）
- 交通事故
- 其他民商事案件

## 使用方案

### OpenClaw 中使用（智能体模式）

在 OpenClaw 平台配置为智能体技能：

```
技能名称：判决书分析
触发词：分析判决书、批量分析、生成判决书报告
```

配置后，用户发送「分析判决书 /path/to/case.pdf」即可自动执行完整流程。

### Claude Code 对话中使用

用户可以直接在 Claude Code 对话中发送指令：

```
分析判决书 ~/Desktop/case.pdf
批量分析 ~/Desktop/cases/
生成判决书综合报告
```

助手会自动：
1. 调用 Python 脚本提取文本
2. 读取提取的文本
3. 按照 SKILL.md 中的格式要求生成分析报告

### 命令行直接调用

```bash
# 提取文本
python3 ~/.claude/skills/judgment-analyzer/scripts/analyzer.py <路径>

# 单文件
python3 ~/.claude/skills/judgment-analyzer/scripts/analyzer.py ~/Desktop/case.pdf

# 文件夹（批量）
python3 ~/.claude/skills/judgment-analyzer/scripts/analyzer.py ~/Desktop/cases/
```

## 工作流程

```
1. 用户指定判决书路径（单文件或文件夹）
2. Python 脚本批量提取文本
3. AI 读取文本，按 SKILL.md 要求分析
4. 生成逐案摘要 + 综合对比报告
```

## 文件结构

```
judgment-analyzer/
├── SKILL.md           # 技能定义（分析要求、输出格式）
├── scripts/
│   ├── analyzer.py    # 文本提取脚本
│   └── requirements.txt
├── references/
│   └── output-template.md
└── README.md          # 本文件
```
