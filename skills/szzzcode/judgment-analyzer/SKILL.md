# 判决书分析技能

分析本地判决书文档，提取案件关键信息，生成结构化的摘要报告和综合分析报告。

## 工作流程

### 步骤 1：接收输入

用户指定判决书路径，支持：
- **单文件**：`/path/to/judgment.pdf`
- **文件夹**：`/path/to/cases/`（批量处理所有 PDF/Word 文件）
- 支持格式：PDF、Word (.docx/.doc)

### 步骤 2：文本提取（Python）

运行 `scripts/analyzer.py` 提取原始文本：

```bash
python3 ~/.claude/skills/judgment-analyzer/scripts/analyzer.py <输入路径>
```

脚本会：
1. 读取每个判决书文件（PDF 或 Word）
2. 提取纯文本内容
3. 将文本保存到 `输入路径/摘要/` 文件夹，文件名为 `原文件名.txt`

### 步骤 3：生成摘要（AI）

1. 读取 `摘要/` 文件夹中的所有 txt 文件
2. 按照输出格式模板，对每个案件生成结构化摘要
3. 将摘要保存为 `案件名_摘要.txt`
4. 批量处理时，额外生成 `综合分析报告.txt`

## 信息提取要求

### 通用信息（所有案件必须提取）

| 字段 | 说明 | 提取指导 |
|------|------|----------|
| 案号 | 案件编号 | 匹配格式：`（年份）字第N号` 或 `（年份）民终N号` |
| 法院 | 审理法院 | 提取法院全称 |
| 案由 | 案件类型 | 判决书中"案由"后的内容 |
| 裁判日期 | 判决时间 | 支持格式：`2023年1月1日`、`2023-01-01` |
| 当事人 | 原告/被告 | 原告/被告 或 上诉人/被上诉人 |
| 审理程序 | 一审/二审/再审 | 根据内容判断 |
| 判决结果 | 判决主文 | "判决如下"后的具体内容 |

### 案由相关分析（根据实际案由调整）

**金融委托理财/合同纠纷类**
- 合同性质与效力（委托理财vs借贷vs合伙）
- 适当性义务是否履行（了解客户/了解产品/适当销售）
- 保底承诺条款的效力
- 损失认定与计算
- 刚兑承诺的法律后果

**侵权责任纠纷类**
- 侵权行为的认定（作为/不作为）
- 过错的判断标准
- 因果关系的认定
- 责任比例的划分
- 损害赔偿范围

**劳动争议类**
- 劳动关系认定
- 法律责任主体
- 赔偿项目计算

**知识产权类**
- 权利有效性
- 侵权认定标准
- 赔偿数额计算

**婚姻家庭类**
- 财产分割原则
- 子女抚养安排
- 债务承担认定

> 提示：根据实际案由，选择相关的分析要点，不必面面俱到。
> 金融类案件重点关注：适当性义务、举证责任倒置、保底条款效力。

### 关键段落识别

按以下顺序查找并提取关键段落：
1. **案件事实**（"经审理查明"、"本院查明"）
2. **争议焦点**（"争议焦点"、"本案焦点"）
3. **法院论证**（"本院认为"、"本院审理认为"）
4. **判决结果**（"判决如下"）

### 输出格式模板

每个案件摘要必须包含以下八个部分，**每部分都要有具体内容描述**，不能只有标题：

```
================================================================================
                            案 件 摘 要
================================================================================

【基本信息】
案号：（年份）粤/京/鲁...民终/民申/民再N号
法院：XX省/市XX中级人民法院/高级人民法院/最高人民法院
案由：具体案由名称
程序：一审/二审/再审 | 日期：YYYY-MM-DD

【当事人】
上诉人/原告（原审身份）：当事人名称
被上诉人/被告（原审身份）：当事人名称
[第三人]

【案件事实】
详细描述：当事人于何时、因何事、采取何种行为、产生何种后果。
要包含关键时间、金额、行为过程等要素。（至少3-5句话）

【争议焦点】
1. 第一个争议焦点是什么？
2. 第二个争议焦点是什么？
[根据案件情况列出1-3个核心争议点]

【法院认定】
针对每个争议焦点，法院的具体分析和认定理由。
要引用法院的核心观点和论证逻辑，不能省略。

【判决结果】
最终判决主文：支持/驳回哪些请求，责任如何划分，
具体金额或比例是多少。

【法律要点】
✓ 第一个法律要点（核心裁判规则）
✓ 第二个法律要点
[提炼2-5个与案由相关的法律裁判规则]
```

**内容充实度要求**：
- 【案件事实】：必须包含时间、人物、行为、结果四要素
- 【争议焦点】：每个焦点都要具体说明争的是什么
- 【法院认定】：要引用法院的原话或核心论证
- 【法律要点】：要提炼出可直接引用的裁判规则

## 综合分析报告要求

批量分析时，生成综合报告：

### 报告结构

**一、案件概览**
以表格形式呈现：
| 案号 | 案由 | 法院 | 程序 | 判决结果 |
|------|------|------|------|----------|

统计信息：
- 案由分布：[列出每种案由及数量]
- 法院分布：[列出涉及的法院及数量]
- 时间范围：[最早日期] 至 [最晚日期]
- 程序分布：[一审/二审/再审 各多少件]

**二、案件类型分析**
根据主要案由类型，分类归纳该类案件的：
- 共同特征
- 核心争议焦点
- 裁判规则

**三、法律要点总结**
以表格形式对比各案：
| 案件 | 争议焦点 | 裁判规则 | 责任比例 |
|------|----------|----------|----------|

**四、结论与建议**
- 案件总体特征
- 风险提示
- 实践建议

## 使用示例

### 触发方式
- 「分析判决书 /path/to/case.pdf」
- 「批量分析 /Users/xxx/Desktop/cases/」
- 「生成判决书综合报告」
- 「对比分析这几份判决书」

### 完整流程

```
用户：批量分析 ~/Desktop/casetest/ 文件夹

助手：
1. 运行文本提取脚本
2. 读取提取的文本内容
3. 对每个案件进行 AI 智能分析
4. 生成逐案摘要
5. 生成综合分析报告
6. 展示分析结果统计
```

## 注意事项

- 脚本路径：`~/.claude/skills/judgment-analyzer/scripts/analyzer.py`
- 首次使用需安装依赖：`pip install -r ~/.claude/skills/judgment-analyzer/scripts/requirements.txt`
- 批量处理时，自动跳过非判决书文件（.pdf, .docx, .doc）
- 摘要文件夹创建在输入文件夹同级目录
- 优先提取文本，如文本质量差可标记"扫描件，需OCR"
