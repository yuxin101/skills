# Examples

## Example 1 — Statistics page

### User asks
Design a department performance statistics page for a CRM admin system.

### Correct classification
**Statistics page**

### Correct structure
```text
Page title
Filter/search/export bar
Optional org tree or left hierarchy panel
Time switching tabs
KPI grid
2-column chart area
Detail table or summary section
```

### What not to do
- do not style it like an AI hero page
- do not replace charts with decorative cards only
- do not overuse gradients

---

## Example 2 — AI analytics page

### User asks
Create an AI customer insights page that helps managers identify high-value customers, silent customers, and risk customers.

### Correct classification
**AI analytics page**

### Correct structure
```text
Title + subtitle
Time switching + scope filters
KPI insight cards
Customer segmentation insight cards
Trend charts
Detail table
Action recommendation section
```

### What not to do
- do not make it a pure numbers dashboard
- do not remove the recommendation layer
- do not design it like a marketing AI landing page

---

## Example 3 — List page

### User asks
Design a customer list page for daily operations with filters, batch actions, and pagination.

### Correct classification
**List page**

### Correct structure
```text
Page title
Filter bar
Action bar
Table
Pagination
```

### What not to do
- do not force KPI/dashboard blocks onto the page unless the task really needs them
- do not reduce table density for style reasons
- do not overcomplicate row actions

---

## Example 4 — Configuration page

### User asks
Design an AI analysis settings page with left navigation, rule parameters, upload area, and config table.

### Correct classification
**Configuration / tool page**

### Correct structure
```text
Left functional menu
Right content panel
Config form
Tooltip-assisted parameters
Upload area
Config table
Save / submit actions
```

### What not to do
- do not design this like a dashboard
- do not replace forms with storytelling cards
- do not hide important parameters behind overly complex interactions

---

## Example output pattern

When responding to a user request, a strong answer should usually include:

1. page type
2. page goal
3. main users
4. information architecture
5. section breakdown
6. Ant Design component mapping
7. design constraints / what not to do
8. optional AI-generation prompt
