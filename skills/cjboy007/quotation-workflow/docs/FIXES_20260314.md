## 📋 问题清单

### 1. HTML 总金额框太小 ❌ → ✅

**问题描述：**
- HTML 版本中总金额（Total Amount）的框宽度固定，在不同屏幕尺寸下显示不佳

**原因：**
- 使用了固定宽度类 `w-1/2 md:w-1/3`

**修复方案：**
```html
<!-- 修复前 -->
<div class="w-1/2 md:w-1/3 ...">

<!-- 修复后 -->
<div class="w-full md:w-1/2 lg:w-1/3 print:w-1/2 ...">
```

**效果：**
- 📱 手机：100% 宽度（充分利用空间）
- 💻 平板：50% 宽度
- 🖥️ 桌面：33% 宽度
- 📄 打印：50% 宽度（避免 PDF 导出时过度放大）

**额外优化：**
- 调整字体大小，打印时适当放大
- `text-xl print:text-2xl` - 确保 PDF 中清晰可见

### 2. PDF 导出时总金额自动放大 ❌ → ✅

**问题描述：**
- HTML 在浏览器查看正常，但导出 PDF 时 Total 金额部分被异常放大

**原因：**
- 浏览器打印引擎自动缩放
- 没有针对打印的专门优化

**修复方案：**

**A. 添加打印专用样式**
```css
@media print {
    .print\:w-1/2 {
        width: 50% !important;
    }
    .print\:text-lg {
        font-size: 1.125rem !important;
    }
    .print\:text-2xl {
        font-size: 1.5rem !important;
    }
}
```

**B. 优化打印颜色保真**
```css
@media print {
    body {
        -webkit-print-color-adjust: exact;
        print-color-adjust: exact;
    }
    .bg-slate-50 {
        background-color: #f8fafc !important;
        -webkit-print-color-adjust: exact;
    }
}
```

**效果：**
- ✅ PDF 导出时字体大小正常
- ✅ 背景颜色正确保留
- ✅ 布局与 HTML 一致

### 3. Excel 导出 PDF 撕裂/分页错误 ❌ → ✅

**问题描述：**
- Excel 导出 PDF 时内容被切断，表格跨页显示不完整

**原因：**
- 没有设置打印区域
- 没有设置页边距
- 没有优化分页

**修复方案：**

**A. 设置打印区域**
```python
ws.print_area = f'A1:F{row}'
```

**B. 设置页面适配**
```python
ws.page_setup.fitToWidth = 1  # 所有列 fit 到一页宽度
ws.page_setup.fitToHeight = 1  # 所有行 fit 到一页高度（如果可能）
ws.page_setup.paperSize = 9  # A4 纸张
```

**C. 设置页边距**
```python
ws.page_margins.left = 0.25   # 英寸
ws.page_margins.right = 0.25
ws.page_margins.top = 0.5
ws.page_margins.bottom = 0.5
```

**D. 优化内容布局**
- 在签名区域前留出空白行用于自然分页
- 避免内容正好在页面边界

**效果：**
- ✅ PDF 内容完整，无撕裂
- ✅ 所有列在一页内
- ✅ 边距合理，美观
