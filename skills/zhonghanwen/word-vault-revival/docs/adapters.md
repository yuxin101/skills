# 平台适配说明

## 核心流程

本技能的推荐流程是：

**OpenClaw 浏览器打开页面 → 检查登录状态 → 抓取页面内容或补充缓存 → 本地同步脚本入库**

手动导出文本不是主流程，只是在浏览器侧无法直接完成抓取时的备用方案。

## 平台统一步骤

每个平台都遵循相同的高层顺序：

1. `open`：打开收藏页
2. `detect-login`：确认登录状态
3. `detect-page`：确认当前页面是否为正确的收藏页 / 单词本页
4. `sync-page-content`：抓取当前页内容（必要时补充更多页），再交给本地脚本入库

## Google Translate Saved

### 目标页面

- `https://translate.google.com/saved?sl=en&tl=zh-CN&op=translate`

### 预期顺序

1. 打开 Google Saved 页面
2. 若需要登录，则停止并提示先登录
3. 确认页面包含“已保存”等收藏页标记与词条列表
4. 抓取当前页内容
5. 若存在稳定流程，则继续翻页并补充缓存
6. 将抓取结果交给同步脚本处理
7. 更新统一词库

### 当前识别依据

- `已保存`
- `英语 → 中文（简体）`
- `第 1-10 个短语（共 235 个）`
- 可识别的收藏词条结构

## 有道单词本

### 字段规则

- `word` 保存原语言词条
- `meanings` 保存释义或翻译

### 目标页面

- `https://dict.youdao.com/wordbook/`

### 预期顺序

1. 打开有道单词本页面
2. 若需要登录，则停止并提示先登录
3. 确认页面包含单词本 / 收藏页标记
4. 抓取当前页内容
5. 若存在稳定流程，则继续翻页或继续加载并补充缓存
6. 将抓取结果交给同步脚本处理
7. 更新统一词库

### 当前识别依据

- `有道`
- `单词本`
- `生词本`
- `收藏`
- 单词 / 释义行模式

## 当前边界

- 长期目标是支持更完整的多页抓取；当前仓库内稳定交付的是基于页面缓存的入库流程
- 登录必须人工完成
- 页面改版时，可靠识别状态比乐观解析更重要
- 直接浏览器抓取失败时，可以退回文本提取作为恢复路径
