## 核心指标

- **日活功能使用次数（DAU_Feature_Use）** — 每日触发该功能的独立用户数，反映功能的基础热度
- **功能渗透率（Feature Adoption Rate）** — 使用该功能的DAU占总DAU的比例，衡量功能的覆盖广度
- **功能留存率（Feature Retention D1/D7/D30）** — 功能上线后第N日继续使用的用户占比，衡量粘性
- **功能满意度（NPS/CSAT）** — 用户对该功能的评分或推荐意愿，衡量体验质量
- **功能转化贡献率（Conversion Contribution）** — 该功能对核心转化路径的贡献权重
- **人均使用次数（Avg Use Count）** — 每日人均触发该功能的次数，反映使用深度
- **功能卸载/关闭率（Feature Drop Rate）** — 主动关闭或停止使用该功能的用户比例
- **功能故障率（Feature Error Rate）** — 功能使用时发生异常/崩溃的比例

## 图表详细定义

### 1. 功能热度排行榜

- **图表类型**: horizontal_bar（水平条形图）
- **数据格式**:
```python
{
    "categories": ["搜索功能", "消息通知", "个人中心", "支付功能", "分享功能", "收藏功能", "扫码功能", "客服入口", "活动弹窗", "推荐feed"],
    "series": [
        {"name": "使用次数(万)", "data": [1250, 980, 860, 720, 540, 430, 380, 290, 210, 150]},
        {"name": "日均DAU渗透率(%)", "data": [92, 85, 78, 68, 52, 41, 38, 28, 22, 15]}
    ]
}
```
- **简洁解读**: Top功能使用次数和渗透率双维度对比，快速定位核心功能和潜力功能
- **洞察规则**: 渗透率>70%且使用次数高→核心必备功能；渗透率低但使用次数高→流量入口优化；渗透率低次数低→考虑下线

---

### 2. 渗透率趋势（折线图）

- **图表类型**: line（折线图）
- **数据格式**:
```python
{
    "xAxis": ["第1周", "第2周", "第3周", "第4周", "第5周", "第6周", "第7周", "第8周"],
    "series": [
        {"name": "新功能A渗透率", "data": [35, 42, 48, 51, 54, 55, 56, 56]},
        {"name": "新功能B渗透率", "data": [28, 30, 31, 30, 29, 27, 25, 24]},
        {"name": "对照组(旧版)", "data": [25, 24, 23, 22, 21, 20, 19, 18]}
    ]
}
```
- **简洁解读**: 追踪功能上线后渗透率爬坡曲线，评估功能吸引力
- **洞察规则**: 渗透率持续上升→功能价值被认可；快速达峰后下滑→功能价值未达预期；横盘不动→入口过深或引导不足

---

### 3. 功能满意度气泡图

- **图表类型**: bubble（气泡散点图）
- **数据格式**:
```python
{
    "series": [
        {"name": "功能满意度", "data": [
            [85, 92, 120, "搜索功能"],
            [78, 88, 95, "消息通知"],
            [72, 75, 80, "个人中心"],
            [65, 70, 60, "支付功能"],
            [55, 82, 45, "分享功能"],
            [82, 65, 70, "收藏功能"],
            [60, 72, 35, "扫码功能"],
            [48, 55, 28, "客服入口"],
            [70, 42, 55, "活动弹窗"],
            [62, 58, 40, "推荐feed"]
        ]}
    ],
    "xAxis": "满意度评分(0-100)",
    "yAxis": "使用频率(次/人/天)",
    "zAxis": "使用人数规模(万)"
}
```
- **简洁解读**: 四象限定位：高分高频大规模=明星功能，高分低频=潜力功能，低分高频=急需改进，低分低频=考虑下线
- **洞察规则**: 第一象限（高满意度+高频）→重点维持；第二象限（低满意度+高频）→紧急优化；右下角高分低频→加强曝光

### 5. 功能使用漏斗（漏斗图）

- **图表类型**: funnel（漏斗图）
- **数据格式**:
```python
{
    "stages": ["功能曝光(曝光UV)", "功能点击(进入页)", "功能首次使用", "功能重复使用(≥3次)", "功能依赖(每日使用)"],
    "data": [100000, 68000, 42000, 18000, 8500],
    "conversion_rates": [100, 68, 41.8, 18, 8.5]
}
```
- **简洁解读**: 从曝光到依赖的完整转化漏斗，定位流失最严重的环节
- **洞察规则**: 曝光→点击转化率<60%→入口曝光文案/位置问题；点击→首次使用<50%→功能加载/引导问题；首次→重复<40%→功能价值不足

---

### 6. 功能留存曲线（折线图）

## 深度洞察

1. **功能生命周期阶段判断**：通过渗透率曲线的斜率变化判断功能所处阶段——快速爬坡期（需持续监控衰减风险）、稳定期（需提升体验细节）、衰退期（需决策优化或下线）

2. **A/B实验效果量化**：对比实验组与对照组的留存率差异，计算功能优化带来的日活增量价值，评估是否值得灰度扩大

3. **功能价值分层**：依据满意度气泡图四象限分布，将产品内功能划分为必备功能（需稳定保障）、增长功能（需加强曝光）、修复功能（需紧急优化）、淘汰功能（需评估下线）

4. **用户行为路径断点分析**：通过热力图发现用户从A功能跳转到B功能时存在异常流失，可定位页面跳转的体验断点

5. **功能渗透率预测**：基于历史爬坡数据拟合S曲线，预测新功能何时达到增长天花板，辅助版本规划决策

---

## 运营建议

1. **P0 - 优化功能入口**：对渗透率>50%但增速放缓的核心功能，优化入口位置和引导文案，目标渗透率再提升15%。优先级来源：渗透率趋势图横盘节点即行动信号。

2. **P0 - 修复低满意度高频功能**：优先处理满意度<60但使用频率>5次/人的功能（气泡图第二象限），建立功能体验问题快速响应机制，目标满意度提升至75+。

3. **P1 - 推进功能下线评估**：对连续4周渗透率<15%且留存率<10%的功能，启动下线评审流程，避免维护成本消耗。

4. **P1 - 建立功能实验标准**：参照字节A/B实验平台规范，制定每个功能迭代的最小样本量、观察周期（≥7天）和显著性判断标准，减少实验误判。

5. **P2 - 完善功能数据埋点**：对尚未纳入追踪的新功能，确保埋点覆盖曝光→点击→使用→复用的全链路，关键节点定义标准化。

6. **P2 - 定期输出功能健康度报告**：每周自动汇总功能渗透率、满意度、留存率三维评分，形成功能健康度排行榜，推动产品迭代数据驱动化。

7. **P3 - 探索功能关联推荐**：基于热力图数据，当用户使用某功能时智能推荐关联功能，提升功能间的协同使用率，目标关联跳转率提升10%。
