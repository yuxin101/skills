## 核心指标

| 指标名称 | 英文缩写 | 定义 | 单位 | 达标线 |
|---------|---------|------|------|-------|
| 5G覆盖率 | Coverage Rate | MR覆盖率（RSRP≥-100dBm占比） | % | ≥95% |
| 无线掉线率 | Drop Rate | 无线连接异常释放次数/总连接次数 | % | ≤0.5% |
| 网络投诉量 | Complaints | 月度网络相关投诉工单总量 | 件 | ≤500 |
| 投诉解决率 | Resolution Rate | 已解决投诉/总投诉 | % | ≥95% |
| 基站容量利用率 | Capacity Util | 忙时PRB利用率均值 | % | ≤70% |
| 网络时延 | Latency | 端到端用户面时延（P99） | ms | ≤30ms |
| 切换成功率 | HO Success Rate | 切换成功次数/切换尝试次数 | % | ≥98.5% |
| 故障平均修复时长 | MTTR | 告警确认到故障恢复的平均时长 | 分钟 | ≤120min |

## 图表详细定义

### 图表1：网络质量评分地图
- **标题**：网格级网络质量综合评分分布
- **图表类型**：GIS热力地图
- **数据格式**：
  ```json
  {
    "granularity": "网格(500m×500m)",
    "regions": [
      {"grid_id": "G001", "lat": 31.23, "lng": 121.47, "score": 92, "coverage": 97.2, "drop_rate": 0.32},
      {"grid_id": "G002", "lat": 31.24, "lng": 121.48, "score": 78, "coverage": 91.5, "drop_rate": 0.68}
    ],
    "color_scale": {"red": "<70", "yellow": "70-85", "green": ">85"}
  }
  ```
- **解读**：以网格粒度展示网络质量空间分布，红色区域为质量洼地，需优先处理。
- **洞察规则**：连续2周评分<70的网格标记"持续劣化"；评分周环比下降超10分的网格标记"突发劣化"。

### 图表2：投诉热力时序图
- **标题**：月度每日投诉量分布（日历热力图）
- **图表类型**：日历热力图
- **数据格式**：
  ```json
  {
    "year": 2026,
    "month": 3,
    "daily_values": {
      "01": 12, "02": 8, "03": 15, "04": 22, "05": 45, "06": 38,
      "07": 11, "08": 9, "...": "..."
    },
    "color_thresholds": {"low": 10, "medium": 25, "high": 40}
  }
  ```
- **解读**：以日历形式展示投诉量，深色日期为投诉高峰日，便于回溯事件关联。
- **洞察规则**：单日投诉量超月均2倍标记"异常投诉日"；连续3天投诉上升趋势标记"持续性问题"。

### 图表3：故障分类分布图
- **标题**：网络故障类型占比分析
- **图表类型**：环形图
- **数据格式**：
  ```json
  {
    "categories": [
      {"name": "无线链路故障", "value": 35, "trend": "↑"},
      {"name": "传输光缆中断", "value": 22, "trend": "→"},
      {"name": "电源异常", "value": 18, "trend": "↓"},
      {"name": "核心网拥塞", "value": 12, "trend": "↑"},
      {"name": "设备硬件故障", "value": 8, "trend": "→"},
      {"name": "其他", "value": 5, "trend": "→"}
    ]
  }
  ```
- **解读**：展示故障类型构成，识别主要故障源。
- **洞察规则**：某类故障占比连续2月上升且>30%时触发"系统性风险"预警；设备硬件故障MTBF低于阈值触发"设备更新"预警。

### 图表4：容量预警趋势图
- **标题**：TOP20高负荷基站容量利用率趋势
- **图表类型**：面积图+阈值线
- **数据格式**：
  ```json
  {
    "x_axis": ["周一", "周二", "周三", "周四", "周五", "周六", "周日"],
    "series": [
      {"name": "基站A(商业区)", "values": [82, 78, 85, 83, 88, 75, 68]},
      {"name": "基站B(高铁站)", "values": [91, 89, 92, 90, 93, 85, 82]}
    ],
    "thresholds": {"warning": 70, "critical": 85}
  }
  ```
- **解读**：展示高负荷基站的容量变化趋势，红色虚线为预警阈值。
- **洞察规则**：利用率连续3天超85%标记"扩容紧急"；周峰值与低谷差值>30pp标记"潮汐效应明显"（建议载波共享或微站补盲）。

### 图表5：掉线率与投诉关联散点图
- **标题**：网格级掉线率与投诉量关联分析
- **图表类型**：散点图（含回归线）
- **数据格式**：
  ```json
  {
    "points": [
      {"grid": "G001", "drop_rate": 0.32, "complaints": 3},
      {"grid": "G002", "drop_rate": 0.68, "complaints": 12},
      {"grid": "G003", "drop_rate": 1.15, "complaints": 28}
    ],
    "trend_line": {"slope": 25.3, "r_squared": 0.82}
  ```
- **解读**：揭示掉线率与投诉量的相关性，高于回归线的网格说明投诉还受其他因素影响。
- **洞察规则**：R²>0.7说明投诉主要由网络质量驱动；偏离回归线上方的网格需排查非网络因素（如营业厅服务态度）。

## 深度洞察

1. **劣化网格集中化趋势**：若质量评分<70的网格在空间上呈现聚集效应（如同一商圈或交通走廊），通常指向共性根因（如传输链路瓶颈或频段干扰），而非偶发故障。此时应从"单点修复"转向"区域专项整治"。

2. **容量与覆盖的跷跷板效应**：高密度城区常面临覆盖充足但容量不足的矛盾，而郊区则是覆盖不足但容量富余。同一套网络参数优化方案无法同时解决两类问题，需按场景差异化制定策略。

3. **投诉量与实际质量偏差**：当散点图中R²<0.5时，说明用户感知投诉与网络客观质量脱节。常见原因包括：投诉入口门槛低导致"非网络投诉混入"、特定区域用户容忍度更低、或竞品营销制造了不合理的期望落差。

4. **故障修复效率衰减**：MTTR连续3个月上升趋势需关注——可能是备件库存不足、外协团队响应变慢、或故障复杂度上升（如从单站故障升级为区域级故障），需分层诊断。

## 运营建议

| 优先级 | 建议内容 | 预期效果 |
|-------|---------|---------|
| P0 | 对连续2周评分<70的网格启动"72小时修复专项行动"，调配应急通信车覆盖临时盲区，同步安排射频优化团队驻场 | 7天内劣化网格清零 |
| P0 | 容量利用率>85%的TOP10基站紧急扩容，优先采用载波聚合+CA+负荷均衡方案，其次考虑新增小微基站 | 忙时拥塞率下降50% |
| P1 | 建立投诉工单与网络告警的自动关联系统，实现"投诉即派单"，省去人工研判环节，压缩从投诉到修复的全链路时长 | MTTR从120分钟压缩至60分钟内 |
| P1 | 针对传输光缆中断故障（占比22%），启动"光缆隐患排查"专项行动，重点检查施工工地周边、河道桥梁等高危段落，预置应急光纤 | 传输类故障下降30% |
| P2 | 部署AI驱动的网络自愈系统，对常见无线链路故障（占比35%）实现自动检测-自动隔离-自动恢复闭环，减少人工介入 | 无线故障自愈率提升至60% |
