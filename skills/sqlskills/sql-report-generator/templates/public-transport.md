# 公共交通运营看板

适用于：公交/地铁/出租车公司运营监控、交通局大数据分析、智慧交通调度
对标：高德交通大数据平台、深圳巴士集团、成都地铁运营监控

## 模板布局

```
┌─────────────────────────────────────────────────────────────┐
│  BANNER: 公共交通运营看板 + 日期/时段 + 线路/公司筛选       │
├─────────────────────────────────────────────────────────────┤
│  KPI: 日客运量 | 准点率 | 满载率 | 事故数 | 投诉量 | 运营里程│
├─────────────────────────────────────────────────────────────┤
│  客运量趋势（面积图）            │  线路负荷热力图          │
├────────────────────────────────────┼────────────────────────┤
│  准点率仪表盘                    │  时段分布（折线图）     │
├────────────────────────────────────┼────────────────────────┤
│  投诉分析（饼图）                │  站点客流排行（表格）    │
├────────────────────────────────────┴────────────────────────┤
│  深度洞察 + 运营建议                                      │
└─────────────────────────────────────────────────────────────┘
```

## 核心指标

- **日客运量** — 当日总乘车人次 — 目标：同比增长 +5%
- **准点率** — 按时到站比例 — 目标：> 90%
- **满载率** — 实际载客/额定载客 — 目标：早晚高峰 < 85%，平峰 > 30%
- **事故数** — 安全事故（含险兆）— 目标：0 重大事故
- **投诉量** — 乘客投诉工单数 — 目标：< 50 单/百万客运量
- **运营里程** — 车辆总行驶里程 — 基准：用于成本核算

## 图表详细定义

### 1. 客运量趋势
- **图表类型**: area（面积图）
- **数据格式**:
```python
{"categories": ["周一","周二","周三","周四","周五","周六","周日"],
 "series": [{"name": "本周客运量(万人次)", "data": [85, 88, 90, 92, 110, 75, 68]},
             {"name": "上周客运量", "data": [80, 83, 86, 88, 105, 72, 65]},
             {"name": "目标", "data": [82, 85, 88, 90, 108, 78, 70]}]}
```
- **简洁解读**: 追踪客运量周内波动，识别工作日通勤高峰和周末低谷
- **洞察规则**: 周五客运量最高（通勤+休闲）；周日为周内最低；与目标差距 >5% 需分析原因

### 2. 线路负荷热力图
- **图表类型**: heatmap（热力图）
- **数据格式**:
```python
{"x_labels": ["线路1","线路2","线路3","线路4","线路5","线路6","线路7","线路8"],
 "y_labels": ["7-9时","9-11时","11-13时","13-15时","15-17时","17-19时","19-21时"],
 "z_values": [[85,60,45,50,55,90,70],
               [80,55,40,42,48,88,65],
               [30,35,40,38,35,30,25],
               [35,40,45,42,40,35,30],
               [50,55,60,58,65,80,55],
               [88,62,50,52,60,92,72],
               [60,45,35,38,42,58,40]]}
```
- **简洁解读**: 精确定位各线路各时段负荷，Z值=满载率%，识别拥挤和空载时段
- **洞察规则**: 满载率 >85% 时段需加密班次；<30% 时段可考虑减班或调整车型

### 3. 准点率仪表盘
- **图表类型**: gauge（仪表盘）
- **数据格式**:
```python
{"title": "今日准点率", "value": 92.5, "min": 0, "max": 100,
 "ranges": [{"from": 0, "to": 80, "color": "#FF4444"},
            {"from": 80, "to": 90, "color": "#FFB300"},
            {"from": 90, "to": 100, "color": "#00C853"}]}
```
- **简洁解读**: 实时监控准点率，与行业标准（90%）对标
- **洞察规则**: 准点率 <85% 需排查堵点和调度问题；恶劣天气可适当放宽至 80%

### 4. 时段客流分布
- **图表类型**: line（折线图）
- **数据格式**:
```python
{"categories": ["6时","7时","8时","9时","10时","11时","12时","13时","14时","15时","16时","17时","18时","19时","20时"],
 "series": [{"name": "客流(万人次)", "data": [2, 8, 15, 12, 6, 8, 10, 7, 6, 9, 15, 18, 12, 8, 5]},
             {"name": "班次", "data": [200, 400, 600, 550, 350, 400, 450, 380, 350, 420, 580, 650, 500, 350, 250]}]}
```
- **简洁解读**: 精确到小时的客流分布，识别早/晚高峰和平峰
- **洞察规则**: 17-19时为晚高峰（最大客流），需最大运力保障

### 5. 投诉分析
- **图表类型**: donut（环形图）
- **数据格式**:
```python
{"labels": ["准点性差","服务态度","车内环境","票价问题","安全投诉","线路规划","其他"],
 "values": [30, 18, 15, 12, 8, 10, 7]}
```
- **简洁解读**: 识别投诉主因，指导服务改进优先级
- **洞察规则**: 准点性占比 >35% 说明调度优化空间大；服务态度 >20% 需加强培训

### 6. 站点客流排行
- **图表类型**: table（表格）
- **数据格式**:
```python
{"columns": ["站点名称","线路","日均客流","高峰客流","环比","满载率"],
 "rows": [["火车站", "1/3号线", 35000, 55000, "+8%", "88%"],
          ["商业中心", "2号线", 28000, 42000, "+5%", "82%"],
          ["区政府", "1号线", 22000, 35000, "+3%", "78%"],
          ["大学城", "3号线", 18000, 30000, "+12%", "75%"],
          ["医院", "2/5号线", 15000, 25000, "-2%", "70%"]]}
```
- **简洁解读**: 识别高客流枢纽站点，指导线路和班次优化
- **洞察规则**: 站点客流环比增长 >10% 需加密班次；满载率 >85% 影响乘客体验

## 深度洞察

1. **周五客流峰值显著**：周五客运量比周均值高 25%，但周末降幅达 20%，通勤刚需稳定
2. **早晚高峰超载明显**：17-19时满载率持续 >90%，车厢拥挤影响乘客体验
3. **大学城站增长最快**：客流环比 +12%，与高校开学季相关，需关注长期增长趋势
4. **准点性是投诉主因**：占投诉 30%，与高峰拥堵强相关，需优化信号优先

## 运营建议

1. 早晚高峰（7-9时、17-19时）增加班次 15%，目标满载率降至 80% — 高优先级
2. 协调交通信号优先，在主要拥堵路段设置公交信号优先 — 高优先级
3. 针对大学城站制定开学季专项保障方案 — 中优先级
4. 建立投诉实时预警：当日投诉 >30 单自动触发调度复盘 — 高优先级
5. 平峰期（11-15时）适当减班，节省运营成本 — 中优先级
6. 推出实时到站查询 APP 功能，减少乘客等候焦虑 — 低优先级
