# Novel Studio 📖

> 专为中文网文打造的 AI 创作系统 — 16个专业智能体，覆盖从构思到发布的完整工作流。

---

## 为什么需要这个

写网文最难的不是写，是**管理**。

你写到第50章，忘了第3章埋的伏笔。你写了30万字，发现主角的能力前后矛盾。你换了个风格写了几章，回头看发现和前面完全不像同一个人写的。

Novel Studio 解决的就是这三个问题：

- **记忆问题** → 记忆管理者维护实时事实数据库，每章写作前自动生成启动包，把上章状态、人物位置、未解悬念全部打包给写手
- **一致性问题** → 编辑/校对在每章完成后强制比对人物卡和事实数据库，发现矛盾立即标记
- **文风问题** → 文风调教师制定规范，初稿写手每次写作前必读，不读不写

## 工作流程

### 流程一：从零开始写新书

这是完整的构思到动笔流程，16个智能体按顺序协作。

```
第一阶段：市场定位
└── 创意顾问
    ├── 分析目标平台（番茄/起点/晋江）的当前热门类型
    ├── 识别你的故事在哪个细分赛道有竞争力
    ├── 提炼核心卖点（一句话能说清楚的那种）
    └── 输出：市场分析报告 + 给后续智能体的方向建议

第二阶段：世界观构建
└── 世界观架构师
    ├── 读取市场分析报告，确认故事类型和卖点方向
    ├── 构建世界基础设定（时代/地点/特殊规则）
    ├── 设计力量体系（仙侠/玄幻必须，都市可简化）
    ├── 规划势力分布和历史背景
    └── 输出：世界观设定.md

第三阶段：人物设计
└── 角色塑造师
    ├── 读取世界观设定，确保人物生活在这个世界里
    ├── 设计主角（外貌/性格/背景/能力/口头禅）
    ├── 设计女主/重要配角（性格反差/说话风格示例）
    ├── 设计反派（动机必须合理，不能只是"为了坏而坏"）
    └── 输出：人物卡.md

第四阶段：故事规划
└── 大纲规划师
    ├── 读取世界观 + 人物卡
    ├── 规划全书结构（分几部/卷，每部核心任务）
    ├── 制定章节规划（至少提前20章，重要转折点必须预先规划）
    ├── 设计爽点/虐点分布和伏笔计划
    └── 输出：大纲规划.md + 完整故事梗概.md

第五阶段：文风定调
└── 文风调教师
    ├── 读取人物卡 + 市场分析报告
    ├── 确定整体风格（小白文/老白文/文青文）
    ├── 制定句子节奏标准（正常/紧张/温馨场景各不同）
    ├── 列出禁止写法和推荐写法（附示例段落）
    ├── 为每个主要角色定制说话风格示例
    └── 输出：文风规范_[风格].md

第六阶段：初始化记忆
└── 记忆管理者
    ├── 读取所有设定文件，录入角色初始状态
    ├── 创建事实数据库（人物位置/道具状态/未解悬念）
    ├── 生成第1章写作启动包
    └── 输出：continuity/fact_database.md + chapters/context_第1章.md
```

### 流程二：写每一章（核心循环）

这是写作阶段每章都要走的流程，缺一步都会出问题。

```
Step 1：生成启动包（记忆管理者）
├── 读取事实数据库，了解上章结尾状态
├── 读取大纲/梗概，找到本章规划内容
├── 生成本章启动包，包含：
│   ├── 上章结尾状态（人物位置/情绪/未解悬念）
│   ├── 本章需要推进的信息点（2-3个）
│   ├── 本章涉及人物的当前状态
│   ├── 连续性注意事项（上章遗留事项必须处理）
│   ├── 本章梗概（来自大纲）
│   └── 结尾钩子建议
└── 输出：chapters/context_第N章.md

Step 2：写正文（初稿写手）
├── 必读：project.md → 启动包 → 事实数据库 → 文风规范 → 人物卡 → 梗概
├── 按启动包中的信息点推进剧情
├── 严格遵循文风规范（不读不写）
├── 执行自查清单：
│   ├── 文风是否符合规范
│   ├── 本章推进了哪些信息点
│   ├── 结尾是否有悬念钩子
│   ├── 人物行为是否符合人物卡
│   └── 上章出现的人/事/物是否都有合理去向
└── 输出：chapters/第N章_[标题].md（2000字下限）

Step 3：质量审查（编辑/校对）[必须]
├── 读取：project.md → 事实数据库 → 人物卡
├── 检查错别字（的地得/形近字/拼音错误）
├── 检查语病（逻辑语病/重复表达/衔接问题）
├── 检查逻辑漏洞：
│   ├── 时间线是否矛盾
│   ├── 实力表现是否前后一致
│   ├── 角色行为是否符合人设
│   └── 因果关系是否成立
├── 按P0/P1/P2优先级标注问题
└── 输出：校对报告（必须修改P0，建议修改P1）

Step 4：文笔润色（润色师）[可选]
├── 读取：project.md → 文风规范 → 人物卡
├── 句式优化（长短句结合，节奏感）
├── 描写增强（画面感/代入感）
├── 情绪渲染（紧张/温馨/悲伤场景各有写法）
├── 保持原有文风，不改变剧情和人物
└── 输出：润色报告 + 修改建议

Step 5：更新记忆（记忆管理者）[必须]
├── 读取刚完成的章节正文
├── 提取本章关键事实变化：
│   ├── 人物状态变化（位置/情绪/关系）
│   ├── 新揭露的秘密（读者知道了什么）
│   ├── 道具变化（位置/持有者）
│   ├── 死亡/离场角色
│   └── 新增悬念
├── 检测事实冲突（发现冲突标记，不自行修改）
├── 更新事实数据库
├── 生成下一章启动包
└── 更新project.md中的项目状态
```

---

### 流程三：审查已有章节

适用于检查已写好的章节，或者发现问题后的修复流程。

```
编辑/校对 → 逐句检查，输出问题清单（P0/P1/P2分级）
     ↓
润色师   → 在校对基础上提升文笔（可选）
     ↓
记忆管理者 → 如果章节有修改，重新更新事实数据库
```

---

### 流程四：卡壳了不知道怎么写

```
情节设计师
├── 读取：大纲 + 事实数据库 + 本章启动包
├── 设计具体冲突（人与人/人与天/人与自己）
├── 规划高潮节奏（小高潮/卷末大高潮/全书高潮）
├── 设计转折点（必须有伏笔，不能凭空出现）
└── 输出：具体情节方案，交给初稿写手执行
```

## 智能体完整列表

### 规划层（构思阶段）

| 智能体 | 核心职责 | 输出文件 |
|--------|----------|----------|
| 创意顾问 | 平台市场分析、核心卖点提炼、风险评估 | 市场分析报告 |
| 世界观架构师 | 世界设定、力量体系、势力分布、历史背景 | 世界观设定.md |
| 角色塑造师 | 人物外观/性格/背景/能力/对话风格 | 人物卡.md |
| 大纲规划师 | 全书结构、章节规划、爽虐点分布、伏笔计划 | 大纲规划.md + 完整故事梗概.md |
| 文风调教师 | 风格定位、句子节奏、禁止/推荐写法、角色说话风格 | 文风规范_[style].md |

### 执行层（写作阶段）

| 智能体 | 核心职责 | 触发时机 |
|--------|----------|----------|
| 情节设计师 | 具体情节/冲突/高潮/转折点设计 | 写作前细化，或卡壳时 |
| 初稿写手 | 根据启动包写正文，执行自查清单 | 每章写作 |
| 记忆管理者 | 维护事实数据库，生成章节启动包 | 每章前后各一次 |

### 质量层（审查阶段）

| 智能体 | 核心职责 | 是否必须 |
|--------|----------|----------|
| 编辑/校对 | 错别字/语病/逻辑漏洞/格式，P0-P2分级 | 必须 |
| 润色师 | 句式优化/描写增强/情绪渲染 | 可选 |
| 过审顾问 | 平台规则合规检查，安全写法建议 | 发布前可选 |
| 版权顾问 | 查重、侵权风险评估 | 发布前可选 |

### 类型专家（按需调用）

| 智能体 | 适用类型 | 专长 |
|--------|----------|------|
| 悬疑专家 | 悬疑/推理/惊悚 | 推理逻辑、悬念铺设、线索管理 |
| 言情专家 | 言情/都市 | 情感线节奏、CP互动、虐点设计 |
| 科幻专家 | 科幻/赛博朋克 | 硬核设定、未来科技、世界观自洽 |
| 仙侠专家 | 仙侠/玄幻/修真 | 修炼体系、灵宝秘境、境界设计 |

## 项目文件结构

```
你的项目目录/
├── project.md              ← 所有智能体的配置入口（必须）
├── 世界观设定.md            ← 从 templates/ 复制填写
├── 人物卡.md               ← 从 templates/ 复制填写
├── 大纲规划.md              ← 从 templates/ 复制填写
├── 完整故事梗概.md          ← 从 templates/ 复制填写
├── 文风规范_[风格].md       ← 从 templates/ 复制填写
├── continuity/
│   └── fact_database.md   ← 记忆管理者自动维护，不要手动改
└── chapters/
    ├── context_第1章.md   ← 记忆管理者自动生成
    ├── 第1章_[标题].md    ← 初稿写手写作
    ├── context_第2章.md
    ├── 第2章_[标题].md
    └── ...
```

## 快速开始

### 新书从零开始

**第一步：复制模板**

把 `templates/` 里的文件复制到你的项目目录并填写：

```
templates/project.md      → 你的项目/project.md
templates/世界观设定.md    → 你的项目/世界观设定.md
templates/人物卡.md        → 你的项目/人物卡.md
templates/大纲规划.md      → 你的项目/大纲规划.md
templates/完整故事梗概.md  → 你的项目/完整故事梗概.md
templates/文风规范.md      → 你的项目/文风规范_[你的风格].md
```

**第二步：初始化**

```
我的项目目录是 [路径]，project.md 已填好，请初始化记忆系统
```

**第三步：开始写**

```
请写第1章
```

### 已有设定，直接写章节

```
我的项目在 [路径]，请为第N章生成启动包，然后写第N章
```

### 审查已有章节

```
请审查第N章，检查错别字、逻辑漏洞和设定一致性
```
