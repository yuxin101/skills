# 合规摘要与报告导出 SOP

## 适用场景
当用户希望：
- 查看某个合规框架或多个合规框架的通过率；
- 对比框架下规则总数、通过规则数、不适用规则数；
- 导出正式合规报告；
- 追踪合规报告是否已生成完成；
应优先使用本 SOP。

## 已验证事实
以下链路已在真实环境中验证通过：
1. `ruleStandardsAndRegulation/list` 可返回合规框架列表；
2. 使用某个框架 ID 调用 `compliance/summary` 可返回摘要列表；
3. `reports/compliance` 可成功返回 `reportId` 与 `pollingUrl`；
4. `reports/{reportId}` 在轮询中可进入 `SUCCESS`，并带有下载链接字段。

## 标准步骤
1. 先查询合规框架列表；
2. 选择一个或多个框架 ID；
3. 调用合规摘要接口获取通过率、规则总数、不适用规则数等指标；
4. 若用户需要正式文件，发起报告导出；
5. 轮询报告详情直到状态完成；
6. 若报告文件可下载，可作为外部交付物之一返回给用户；
7. 若还需要解释性输出，再将摘要结果整理成 HTML 报告或文本结论。

## 报告组织建议
可按以下结构组织：
- 选定框架与摘要指标总览
- 多框架比较表
- 通过率与未通过风险说明
- 导出报告状态与链接信息

## 注意事项
- `standardAndRegulationsIds` 必须为非空数组；
- 报告导出成功不等于文件立刻可下载，应轮询详情状态；
- 下载链接字段可能存在，但示例中不得展示真实环境链接；
- 若只需要分析结论，不必强制导出正式报告。
