# 按规则维度下钻风险 SOP

## 适用场景
当用户的查数问题聚焦于“哪条规则产生了最多风险”“某个高风险规则影响了哪些资产”“按规则解释风险分布”时，优先使用本 SOP。

## 已验证事实
- `group-by/rule-id` 可返回规则聚合桶；
- 使用聚合得到的规则 ID 再查询 `risks/list`，返回记录中的规则映射能够与该规则 ID 匹配；
- 风险明细中的 `rule` 与 `riskMapping` 已足够支持规则维解释，无需先依赖独立规则列表接口。

## 标准步骤
1. 先调用规则维聚合接口获取规则桶；
2. 选择风险数较高或用户关注的规则 ID；
3. 使用 `risks/list` 并传入 `ruleId` 做小页下钻；
4. 从明细中观察受影响资产、风险状态、严重度和规则说明；
5. 对重点条目使用 `riskMapping.id` 获取详情；
6. 在报告中形成“规则 -> 资产 -> 风险详情”的解释链。

## 报告组织建议
可按以下顺序展示：
- 规则聚合概览
- 该规则下的风险明细表
- 受影响资产统计或透视结果
- 重点风险详情说明

## 注意事项
- 下钻时优先小页查询，不要默认需要一次获取大量明细；
- 若用户需要规则原始定义的更完整维表，可再考虑补查规则列表接口，但当前默认链路不依赖它；
- 技能示例中必须使用 mock 规则名称、mock 资产名称与 mock 风险记录。
