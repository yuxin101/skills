# 规则列表补充查询 SOP

## 适用场景
当用户已经从风险主干中定位到某条规则，且需要补充规则维表信息、规则定义、规则类型、严重度或分类标签时，使用本 SOP 作为默认风险主干的补充步骤。

## 已验证事实
- `rules/list` 可稳定返回规则记录；
- 返回记录中包含 `uuid` 与 `severity` 等关键字段；
- 通过锚点规则名称与严重度定向查询，能够精确命中风险主干中的真实规则 UUID。

## 标准步骤
1. 从风险明细中取得锚点规则名称、规则 UUID、严重度；
2. 先按规则名称与严重度查询规则列表；
3. 若命中目标规则，则把规则列表结果作为风险主干的补充维表；
4. 若未命中，再考虑缩短关键词做模糊匹配；
5. 将规则列表结果与风险明细结果拼接，用于解释规则背景、分类、标签与参考信息。

## 适用边界
- 规则列表是默认风险主干的补充层，不是默认起点；
- 不要假设规则列表首页一定覆盖风险明细中的规则；
- 先用定向条件检索，再使用结果做补充解释。
