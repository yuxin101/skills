# 默认风险查数 SOP

## 适用场景
当用户提出“看整体风险情况”“帮我看一下有哪些风险资产”“查当前高风险分布”“从总览逐步下钻到具体风险”“生成风险分析报告”之类的需求时，优先使用本 SOP。

## 已验证前提
以下链路已在真实环境中跑通或验证成立：
- 探针与枚举接口可达；
- 严重度聚合可用；
- 风险分页明细可用；
- `riskMapping.id` 可稳定用于详情查询；
- `severities`、`ruleId`、`cloudInstanceId`、`instanceHashId` 可作为已验证过滤键优先使用。

## 标准步骤
1. 先获取探针与枚举信息：云厂商、云账号、严重度、规则类型；
2. 使用风险聚合接口获取总览，优先查看严重度、规则类型、规则 ID、账号维等聚合维度；
3. 选择一个最有价值的维度作为下钻入口；
4. 使用 `risks/list` 做小页明细查询；
5. 若需要解释单条风险，提取 `riskMapping.id` 调用详情接口；
6. 若需要交付结果，同步生成 HTML 报告；
7. 若当前任务属于概览性风险查询，必须使用 `assets/anduoduo_risk_report.html` 作为固定模板骨架生成 HTML 彩页报告。

## 过滤优先级
推荐优先级从高到低：
1. `ruleId`
2. `severities`
3. `cloudInstanceId`
4. `instanceHashId`

`cloudAccountId` 当前不能作为默认可靠过滤主键，应谨慎使用。

## 输出组织建议
- 总览：聚合统计、风险等级分布、主要规则类型
- 明细：风险记录表，按资产信息、规则信息、风险映射信息拆列
- 单条解释：对重点风险补充详情说明
- HTML 报告：包含总览、表格、透视分析、文本结论，明细默认最多展示前 300 条

## 概览报告的强制约束
- 严重度必须完整展示“严重 / 高危 / 中危 / 低危”四档，即使某一档数量为 0 也不能省略；
- 类型分布中若遇到 `Issue` / `issues` / `issus` / `问题`，统一对外展示为 `风险`；
- 概览类报告至少包含：风险严重度分布、规则类型分布、受影响实例类型 TOPN；
- 若数据支持，应补充云账号维分布、高风险规则 TOPN、风险根因/主因分布等图表或表格；
- AI 解读必须依据真实结果生成，至少解释总体态势、0 值严重度、主导风险类型、重点高风险对象及处理建议。

## 禁忌
- 不要默认不同 `pageSize` 的首屏结果顺序稳定；
- 不要直接大页宽范围扫全量风险；
- 不要把账号过滤当作稳定锚点；
- 不要在示例中展示真实环境返回；
- 不要把模板中的静态示例统计、案例和图表值原样带入最终交付。
