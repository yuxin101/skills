# 文件交付策略

## 1. 交付决策顺序
当任务生成 HTML、CSV、JSON、DOCX、ZIP 等文件时，按以下顺序判断：
1. 是否具备对象存储上传能力；
2. 若无上传能力，是否可直接把文件发送给用户；
3. 若两者都不可用，则返回本地存储路径。

## 2. 默认策略
- 最终交付给用户的有价值资产：优先上传并返回链接；
- 内部验证材料、真实响应原文、调试日志：仅保留本地，不对外提供；
- 面向用户的文件必须是去敏后的定稿版本。

## 3. 风险控制
- 任何包含真实账号名、实例名、真实下载链接、真实环境标识的文件都不得直接对外；
- 打包前应先进行内容检查，确认示例已 mock 化；
- 原始验证 JSON 仅作为内部证据，不进入最终技能包压缩文件。
