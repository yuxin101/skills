# 默认 HTML 报告生成规则

## 1. 适用范围
当用户提出默认查数、风险分析、规则排查、风险资产盘点、生成查数报告等需求时，应同步生成 HTML 报告。

其中，凡是以风险总览、风险资产概览、整体风险情况、高风险分布为主的概览性查询，必须使用固定模板：`assets/anduoduo_risk_report.html`。

## 2. 固定模板使用原则
- 固定模板指固定版式骨架、样式分区和图表位置；
- 不得保留模板中的静态示例统计、案例、时间、图表数据；
- 生成报告时，必须用当前查询结果替换模板示例值；
- 若模板中存在 `Issue` / `issues` / `issus` / `问题` 等展示文本，统一改为 `风险`。

## 3. 最小组成元素
概览类固定模板报告至少应包含：
- 指标总览卡片；
- 风险严重度分布图；
- 规则类型分布图；
- 受影响实例类型 TOPN 图（若明细字段稳定支持）；
- 关键文本解读；
- 明细数据表（默认最多展示前 300 条）。

## 4. 强制展示规则
- 严重度必须始终包含“严重 / 高危 / 中危 / 低危”四档；
- 即使某一档为 0，也必须保留展示位；
- 类型分布需要统一展示名，不直接把 `Issue` 英文原值暴露给最终用户；
- 若聚合数据存在缺桶，应在展示层补齐为 0，而不是省略。

## 5. 数据来源建议
- 总览：优先来自 `group-by/*`；
- 明细：优先来自 `risks/list`；
- 单条解释：优先来自 `risks/{riskId}`；
- 规则补充：来自 `rules/list`；
- 账号维补充：优先来自 `group-by/account-id`；
- 实例类型分布：可由 `risks/list` 明细汇总得到。

## 6. 图表与 AI 解读增强
若数据支持，优先补充：
- 云账号维风险分布；
- 高风险规则 TOPN；
- 风险根因/主因分布；
- 重点区域或云厂商分布；
- AI 洞察：总体态势、0 值严重度解释、主导风险类型、重点高风险对象、优先修复建议。

## 7. 非概览场景边界
- 合规导出链路中的正式报告文件不强制改造成该固定模板；
- 若合规场景仅需解释性摘要，可参考该模板组织解释性 HTML，但不要替代正式导出物；
- 示例报告必须使用 mock 数据，不能复现真实环境内容。
