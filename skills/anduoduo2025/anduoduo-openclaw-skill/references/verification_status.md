# 验证状态总表

## 一、已验证路径

### 1. 默认风险查数主干
- 探针与枚举接口可达；
- 风险聚合接口可返回总览；
- `risks/list` 可返回分页明细；
- `riskMapping.id` 可用于风险详情查询。

### 2. 已验证有效的过滤键
- `ruleId`
- `severities`
- `cloudInstanceId`
- `instanceHashId`

### 3. 规则维补充路径
- `group-by/rule-id` -> `risks/list(ruleId=...)` 成立；
- `rules/list` 通过规则名称与严重度可命中锚点规则。

### 4. 合规链路
- 合规框架列表可取；
- 合规摘要可返回；
- 合规报告导出可返回 `reportId` 与 `pollingUrl`；
- 报告详情轮询可进入 `SUCCESS`。

## 二、已知限制与谨慎项
- 不同 `pageSize` 下的首屏结果不应假设为稳定前缀；
- `cloudAccountId` 当前不能当作默认可靠风险过滤主键；
- 从任意资产直接正向回查风险不稳定，默认应先从风险明细反推资产；
- 真实环境返回含敏感信息，技能示例必须使用 mock 数据。

## 三、仅文档确认、尚未升级为已验证 SOP 的路径
- 扫描任务发起与扫描详情链路；
- 某些未单独实测的聚合接口维度；
- 云账号新增链路。
