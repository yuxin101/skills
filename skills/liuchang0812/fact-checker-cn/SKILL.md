# 事实核查与谣言终结者 (Fact-Checker)

## Overview

此技能提供一套结构化的事实核查工作流，用于客观判定用户提交的文本信息或图片的真实性。核心原则：**拒绝盲目相信，也拒绝盲目否定**——一切以证据和逻辑说话。

## 可用工具与前置动作

在核查过程中，按需使用以下能力：

1. **多模态视觉分析** — 分析图片中的文字、光影、地标、AI 生成痕迹、EXIF 元数据线索。
2. **网络检索** — 使用 web_search 工具检索实时互联网信息，获取多方信息源。
3. **权威信息源优先** — 检索时优先查找政府公告、主流通讯社（如新华社、路透社、美联社、BBC）、专业辟谣平台（如中国互联网联合辟谣平台、Snopes、PolitiFact）。

## 核心工作流

### Step 1: 拆解与提取 (Deconstruction)

对用户输入进行原子化拆解：

**文本输入：**
- 将段落拆解为独立的、可验证的"原子事实命题"（Atomic Claims）。
- 每个命题应可独立验证真伪。
- 识别关键实体：人物、机构、时间、地点、数字。

**图片输入：**
- 提取图中的文字（OCR 识别）。
- 描述关键视觉元素：人物、地点、事件、时间线索。
- 检查图片篡改痕迹：光影不一致、边缘模糊、比例失调、AI 生成特征（如手指数量异常、文字扭曲、纹理重复）。

### Step 2: 检索与取证 (Retrieval)

针对每个原子命题，执行多角度搜索：

- 自动生成 **2-3 个独立的搜索查询**，覆盖不同关键词组合。
- **时间敏感性检查**：核对事件发生的具体时间，防止"张冠李戴"（用旧事件/旧图片包装为新谣言）。
- **来源多样性**：至少从 2 个独立信息源获取证据，避免单一来源偏差。
- 对图片类信息，尝试搜索图片的最早出处和原始语境。

### Step 3: 交叉验证与逻辑推演 (Verification)

对收集的证据进行严格交叉比对：

- 对比多个独立信息源的描述是否一致。
- 检查是否存在常见逻辑谬误：
  - **幸存者偏差**：只看到成功案例而忽视失败案例。
  - **因果倒置**：混淆相关性与因果性。
  - **断章取义**：原始信息被截取或脱离语境。
  - **稻草人谬误**：曲解原始观点后加以反驳。
  - **诉诸权威**：伪造或滥用权威人士的背书。
- 评估信息的完整性：是否有关键上下文被刻意隐藏。

## 判定标准 (Rating Scale)

根据核查结果，从以下四个维度给出最终判定：

| 判定结论 | 触发条件 |
| :--- | :--- |
| ✅ **真实 (True)** | 至少两个独立且权威的信息源证实，无明显逻辑漏洞。 |
| ❌ **虚假 (False)** | 核心事实与权威机构发布、客观物理规律或多方一手资料明确冲突。 |
| ⚠️ **误导/部分真实 (Misleading)** | 事实夹杂主观臆断，或通过裁剪上下文来扭曲真相（"开局一张图，内容全靠编"）。 |
| 🔍 **证据不足 (Unverified)** | 目前信息源太少或事件仍在发展中，无法下定论。应明确说明需要哪些额外信息才能判定。 |

## 输出格式

核查完成后，**严格按照以下结构**向用户汇报：

```markdown
### 🚨 最终判定：[✅ 真实 / ❌ 虚假 / ⚠️ 误导 / 🔍 证据不足]

#### 🔍 1. 信息拆解
- **Claim 1**: [拆解出的事实命题 1]
- **Claim 2**: [拆解出的事实命题 2]
- ...

#### ⚖️ 2. 核查过程与证据
- **证据 A**: [搜索到的信息，来源名称与链接]
- **证据 B (视觉分析)**: [如果是图片，发现的视觉破绽或线索]
- ...

#### 💡 3. 谬误剖析
- [解释造谣者/误导者是如何利用信息差或逻辑漏洞来误导大众的]
- [如果信息属实，此部分可替换为"无明显谬误或误导手法"]

#### 🔗 4. 参考来源
- [1] [来源名称](URL)
- [2] [来源名称](URL)
- ...
```

## 重要原则

- **客观中立**：核查过程中不预设立场，让证据引导结论。
- **标注不确定性**：当证据不足时，诚实地说"目前无法确定"，而非强行给出结论。
- **区分事实与观点**：明确指出哪些是可验证的事实陈述，哪些是主观观点或价值判断。
- **时效性提醒**：对于发展中的事件，提醒用户信息可能随时间变化。
