# 🎯 任务闭环机制 - 触发标准

> **原则**：不是所有事情都需要完整的闭环追踪，但重要任务必须有保障

---

## ✅ 需要触发闭环机制的任务

### 1️⃣ 周期性任务（必须）

**特征**：按固定频率重复执行

**示例**：
| 任务 | 频率 | 优先级 |
|------|------|--------|
| 每日新闻推送 | 每天 7:00 | 中 |
| 系统健康检查 | 每天 8:00, 20:00 | 高 |
| 每小时任务扫描 | 每小时 | 中 |
| Git 自动推送 | 每 5 分钟 | 高 |
| 每日本地备份 | 每天 22:00 | 高 |
| 每周统计报告 | 每周日 | 低 |

**原因**：需要追踪执行历史、成功率、故障模式

---

### 2️⃣ 关键操作（必须）

**特征**：影响系统稳定性或数据安全

**示例**：
- ✅ 系统配置修改（crontab、systemd 服务）
- ✅ 数据同步（Git、坚果云、Obsidian）
- ✅ 备份/恢复操作
- ✅ 系统升级/回滚
- ✅ 服务重启/停止

**原因**：失败可能导致系统不可用或数据丢失

---

### 3️⃣ 对外交互（必须）

**特征**：涉及与外部系统或人员的交互

**示例**：
- ✅ 邮件发送（必须用户批准 + 追踪结果）
- ✅ 消息推送（飞书、微信等）
- ✅ API 调用（外部服务）
- ✅ 文件上传/下载

**原因**：需要确认送达、失败重试、审计追踪

---

### 4️⃣ 用户明确委托的任务（必须）

**特征**：Alfred 直接交代的任务

**示例**：
- ✅ "帮我写一篇文章"
- ✅ "检查系统状态"
- ✅ "整理考核资料库"
- ✅ "分析这个文件"

**原因**：需要反馈完成结果，建立信任

---

### 5️⃣ 长时间运行任务（必须）

**特征**：执行时间 > 1 分钟

**示例**：
- ✅ 大批量文件处理
- ✅ 数据迁移
- ✅ 全文索引构建
- ✅ 视频/图片处理

**原因**：需要进度追踪、超时保护、断点续传

---

## ⚠️ 可简化处理的任务

### 6️⃣ 简单查询（可选）

**特征**：只读操作，无副作用

**示例**：
- 📝 读取文件内容
- 📝 查看系统状态（不记录）
- 📝 搜索信息

**处理方式**：
- 不需要完整闭环
- 但需要记录到 `memory/日期.md`

---

### 7️⃣ 即时对话（不需要）

**特征**：纯聊天、问答

**示例**：
- 💬 "今天天气怎么样"
- 💬 "你觉得这个怎么样"
- 💬 闲聊

**处理方式**：
- 不需要任务登记
- 正常对话即可

---

### 8️⃣ 探索性工作（可选）

**特征**：尝试性、研究性

**示例**：
- 🔍 "帮我查一下这个技术"
- 🔍 "看看有没有更好的方案"

**处理方式**：
- 可登记为简单任务
- 不需要完整的阶段追踪

---

## 📊 决策流程图

```
收到任务
   ↓
是否周期性任务？→ 是 → 完整闭环 ✅
   ↓ 否
是否关键操作？→ 是 → 完整闭环 ✅
   ↓ 否
是否对外交互？→ 是 → 完整闭环 ✅
   ↓ 否
是否用户委托？→ 是 → 完整闭环 ✅
   ↓ 否
是否长时间运行？→ 是 → 完整闭环 ✅
   ↓ 否
是否简单查询？→ 是 → 简化处理 📝
   ↓ 否
即时对话/闲聊 → 不需要闭环 💬
```

---

## 🎯 优先级判定

| 优先级 | 判定标准 | 示例 |
|--------|----------|------|
| **高** | 影响系统/数据/安全 | 备份、配置、同步 |
| **中** | 影响用户体验/日常工作 | 新闻推送、健康检查 |
| **低** | 可选功能、优化类 | 统计报告、清理任务 |

---

## 📝 闭环粒度

### 完整闭环（推荐用于高优先级）

```bash
task_init → task_start → task_stage × N → task_complete/task_fail
```

**包含**：
- ✅ 任务登记
- ✅ 阶段追踪
- ✅ 详细日志
- ✅ 结果反馈
- ✅ 故障分析

### 简化闭环（用于中优先级）

```bash
task_init → task_start → task_complete/task_fail
```

**包含**：
- ✅ 任务登记
- ✅ 结果反馈
- ⚠️ 阶段追踪（可选）

### 最小记录（用于低优先级）

```bash
# 仅在 memory/日期.md 中记录
```

**包含**：
- ✅ 简单记录

## 🛠️ 实施建议

### 新任务接入流程

1. **判断类型** → 参考上述分类
2. **确定优先级** → 高/中/低
3. **选择闭环级别** → 完整/简化/最小
4. **登记到注册表** → `memory/task-registry.json`
5. **执行并追踪** → 使用 `task-utils.sh`
6. **反馈结果** → 完成/失败都通知
