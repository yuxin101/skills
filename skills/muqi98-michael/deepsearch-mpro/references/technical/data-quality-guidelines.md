# 数据质量控制指南

## 概述

数据质量是咨询报告可信度的基石。本文档提供数据收集、验证、标注的标准流程。

---

## 数据质量维度

| 维度 | 说明 | 评估标准 |
|------|------|----------|
| **准确性** | 数据是否正确 | 与多个来源交叉验证 |
| **完整性** | 数据是否完整 | 是否覆盖所有必需字段 |
| **时效性** | 数据是否最新 | 数据时间范围是否满足需求 |
| **一致性** | 数据是否一致 | 不同来源的数据是否一致 |
| **可信度** | 来源是否可信 | 来源类型评估 |

---

## 数据收集标准

### 必需字段

每条数据必须包含：

```json
{
  "metric": "市场规模（亿元）",
  "value": 3450,
  "unit": "亿元",
  "time_range": "2024",
  "source": {
    "title": "2024年中国护肤品行业研究报告",
    "url": "https://example.com/report",
    "type": "行业报告",
    "date": "2024-01-15"
  },
  "confidence": "high",
  "verification": "verified"
}
```

### 数据类型分类

| 类型 | 说明 | 示例 |
|------|------|------|
| **定量数据** | 具体数值 | 市场规模、增长率、市场份额 |
| **定性数据** | 描述性信息 | 趋势描述、专家观点、案例研究 |
| **混合数据** | 包含定量和定性 | 消费者画像（含年龄分布 + 行为描述） |

## 数据验证流程

### 多源验证

对于 P0 级别的关键数据，必须从至少 2 个独立来源验证。

**验证步骤**：

1. **第一来源**：搜索并记录数据
2. **第二来源**：使用不同搜索引擎验证
3. **交叉对比**：对比两个来源的数据
4. **标注结果**：verified / conflicting / unverified

**验证示例**：

```markdown
# 数据：中国护肤市场规模 2024

## 来源1
- 数值：3450 亿元
- 来源：艾瑞咨询
- URL：https://example.com/report1

## 来源2
- 数值：3380 亿元
- 来源：欧睿国际
- URL：https://example.com/report2

## 验证结果
- 状态：verified（差异在 5% 以内）
- 采纳值：3450 亿元（优先艾瑞咨询，因为更权威）
```

### 数据冲突处理

当不同来源的数据存在冲突时：

| 冲突程度 | 处理方式 |
|---------|----------|
| **小差异（<10%）** | 选择最可信来源，或使用范围值 |
| **中等差异（10-30%）** | 记录所有来源，分析差异原因，标注为"数据存在争议" |
| **大差异（>30%）** | 深入调查，寻找第三来源，或标注为"数据不可靠" |

**冲突处理示例**：

```markdown
# 数据：护肤品市场增长率

## 来源A：8.5%
## 来源B：12.3%
## 来源C：9.1%

## 分析
差异原因：统计口径不同（来源A不含线上渠道，来源B含跨境电商）

## 处理
采纳：9.1%（来源C，统计口径最全面）
标注：数据存在统计口径差异
```

## 数据可信度评估

### 来源类型分级

| 来源类型 | 可信度等级 | 说明 |
|---------|-----------|------|
| **学术期刊（peer-reviewed）** | 最高 | 经同行评审，方法论严谨 |
| **政府统计** | 高 | 官方权威数据 |
| **上市公司财报** | 高 | 法定披露，经审计 |
| **行业报告（知名咨询公司）** | 中高 | 专业研究，但可能有商业倾向 |
| **主流新闻媒体** | 中 | 时效性好，但需交叉验证 |
| **专业博客** | 中低 | 专业性强，但缺乏审核 |
| **个人博客、论坛** | 低 | 仅作参考，需验证 |

### 可信度标注标准

| 等级 | 说明 | 标准 |
|------|------|------|
| **high** | 高可信度 | 3+ 个高可信度来源验证，或来源为学术期刊/政府统计 |
| **medium** | 中可信度 | 2 个中高可信度来源验证，或来源为主流媒体 |
| **low** | 低可信度 | 1 个来源，或来源为博客/论坛 |

## 数据去重与整合

### 去重原则

1. **URL 去重**：相同 URL 只保留一次
2. **内容去重**：相同内容的不同来源合并记录
3. **版本去重**：同一报告的不同版本只保留最新

### 数据整合

对于同一指标的多条数据：

```json
{
  "metric": "市场规模（亿元）",
  "values": [
    {
      "value": 3450,
      "year": 2024,
      "source": "艾瑞咨询",
      "confidence": "high"
    },
    {
      "value": 3380,
      "year": 2024,
      "source": "欧睿国际",
      "confidence": "high"
    }
  ],
  "consensus": {
    "method": "平均值",
    "value": 3415,
    "range": "3380-3450"
  }
}
```

## 常见数据质量问题

### 问题1：数据缺失

**处理方式**：
- 明确标注"数据暂不可得"
- 不要编造或估算
- 说明已尝试的搜索方法

**示例**：
```markdown
| 数据指标 | 数据值 | 状态 |
|---------|--------|------|
| 市场规模 | 3450亿元 | 已获取 |
| 增长率 | 数据暂不可得 | 已搜索3个来源，未找到可靠数据 |
```

### 问题2：数据冲突

**处理方式**：
- 记录所有来源
- 分析差异原因
- 选择最合理值或使用范围

**示例**：
```markdown
增长率：8.5%（艾瑞咨询）vs 12.3%（欧睿国际）
差异原因：统计口径不同
采纳：8.5-12.3%（范围值）
```

### 问题3：数据过时

**处理方式**：
- 标注数据时间
- 尝试寻找更新数据
- 必要时使用旧数据并标注

**示例**：
```markdown
市场规模：3200亿元（2023年数据，2024年数据暂未发布）
```

### 问题4：来源不明

**处理方式**：
- 追溯原始来源
- 标注为"引用未验证"
- 降低可信度等级

**示例**：
```markdown
增长率：10%（来源：某行业报告，原始链接失效，可信度：low）
```
