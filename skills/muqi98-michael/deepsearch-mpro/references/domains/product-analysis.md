# 产品竞争分析框架

## 适用场景

当研究主题涉及**单个产品的竞争分析**（如"用友BIP产品分析"、"金蝶云星空分析"、"Salesforce Sales Cloud分析"）时使用本框架。

**典型输入示例**：
- "分析用友BIP的产品竞争力"
- "金蝶云星空产品深度研究"
- "Salesforce vs HubSpot 产品对比"

**与企业分析的区别**：
- 产品分析：聚焦单个产品/产品线，深入功能、用户、定价、竞争
- 企业分析：分析整个公司的战略、业务组合、财务、组织

---

## 核心分析模型

### 推荐框架组合

```
产品概览 → PESTEL → 目标用户 → 竞品矩阵 → 商业模式画布 → SWOT → 核心功能 → 产品热点分析
```

| 顺序 | 模型 | 分析目的 |
|------|------|---------|
| 1 | 产品概览 | 产品定位、发展历程、市场地位 |
| 2 | PESTEL | 宏观环境影响 |
| 3 | 目标用户 | 用户画像、需求场景、痛点分析 |
| 4 | 竞品矩阵 | 与同类产品多维度对比 |
| 5 | 商业模式画布 | 产品商业模式分析 |
| 6 | SWOT | 产品优劣势与机会威胁 |
| 7 | 核心功能 | 功能模块、差异化功能、技术能力 |
| 8 | 产品热点分析 | 近一周产品发布、市场活动、中标、宣传等热点 |

---

## 分析维度

### 1. 产品概览

| 维度 | 关键信息 | 数据来源 |
|------|---------|---------|
| **产品名称** | 官方名称、版本号、代号 | 官网、产品页面 |
| **所属公司** | 母公司、业务线、团队 | 官网、财报 |
| **产品定位** | 目标市场、核心价值、差异化定位 | 官网、产品介绍 |
| **发展历程** | 发布时间、重大版本、里程碑 | 官网、新闻、技术博客 |
| **市场地位** | 市场份额、行业排名、客户数量 | 行业报告、官网案例 |
| **技术架构** | 云原生/AI化/微服务、技术栈 | 技术白皮书、技术博客 |

---

### 2. 宏观环境影响（PESTEL）

| 维度 | 分析要点 |
|------|---------|
| **政策（P）** | 行业政策、合规要求、信创适配 |
| **经济（E）** | 企业IT预算、采购意愿、经济周期 |
| **社会（S）** | 数字化转型、用户习惯、社会需求 |
| **技术（T）** | AI技术、云计算、新技术趋势 |
| **环境（E）** | 绿色IT、可持续发展 |
| **法律（L）** | 数据安全、隐私保护、知识产权 |

---

### 3. 目标用户分析

| 维度 | 分析内容 | 关键问题 |
|------|---------|---------|
| **用户画像** | 企业规模、行业、地区、角色 | 谁是目标客户？ |
| **需求场景** | 核心使用场景、业务流程 | 解决什么问题？ |
| **痛点分析** | 现有痛点、未满足需求 | 客户痛点是什么？ |
| **决策路径** | 购买决策流程、关键决策者 | 如何决策购买？ |
| **使用体验** | 用户评价、NPS、客户满意度 | 用户反馈如何？ |

---

### 4. 竞品对比矩阵

| 对比维度 | 分析内容 |
|---------|---------|
| **功能覆盖** | 核心功能、差异化功能、缺失功能 |
| **技术能力** | 技术架构、性能指标、扩展性 |
| **定价策略** | 定价模式、价格区间、性价比 |
| **用户体验** | 界面设计、易用性、学习曲线 |
| **生态整合** | API开放、第三方集成、生态伙伴 |
| **客户服务** | 实施服务、售后支持、培训体系 |
| **市场表现** | 客户数量、市场份额、增长趋势 |

**竞品选择原则**：
- 直接竞品（同功能、同市场）
- 间接竞品（替代方案、跨界产品）
- 国际标杆（行业领先产品）

**矩阵示例**：

| 产品 | 公司 | 功能覆盖 | 技术能力 | 用户体验 | 定价策略 | 生态整合 | 市场表现 |
|------|------|---------|---------|---------|---------|---------|---------|
| **产品A** | 公司A | ★★★★★ | ★★★★★ | ★★★★☆ | ★★★☆☆ | ★★★★★ | ★★★★★ |
| 产品B | 公司B | ★★★★☆ | ★★★★☆ | ★★★★★ | ★★★★☆ | ★★★★☆ | ★★★★☆ |
| 产品C | 公司C | ★★★☆☆ | ★★★☆☆ | ★★★★☆ | ★★★★★ | ★★★☆☆ | ★★★☆☆ |

---

### 5. 商业模式画布

| 模块 | 分析内容 | 关键问题 |
|------|---------|---------|
| **关键合作** | 技术伙伴、实施伙伴、渠道伙伴 | 核心合作伙伴是谁？ |
| **关键活动** | 产品研发、市场推广、客户服务 | 主要活动是什么？ |
| **关键资源** | 技术平台、团队、品牌、客户资源 | 核心资源是什么？ |
| **价值主张** | 核心功能价值、效率提升、成本节约 | 为客户创造什么价值？ |
| **客户关系** | 服务模式、客户成功、培训体系 | 如何服务客户？ |
| **客户细分** | 企业规模、行业、地区 | 目标客户是谁？ |
| **渠道通路** | 直销、渠道、线上、线下 | 如何触达客户？ |
| **成本结构** | 研发成本、销售成本、服务成本 | 成本主要在哪里？ |
| **收入来源** | 订阅费、实施费、增值服务 | 如何赚钱？ |

---

### 6. SWOT 分析

| 维度 | 分析内容 |
|------|---------|
| **优势（S）** | 核心功能优势、技术领先、品牌优势、客户资源、生态优势 |
| **劣势（W）** | 功能短板、技术不足、服务能力、市场覆盖、价格劣势 |
| **机会（O）** | 市场增长、技术变革、政策利好、客户需求升级 |
| **威胁（T）** | 竞品压力、技术替代、客户流失、价格战 |

---

| **模块B** | 核心功能描述 | vs竞品优势/劣势 | 用户反馈 |
| **模块C** | 核心功能描述 | vs竞品优势/劣势 | 用户反馈 |
| **差异化功能** | 独特功能 | 独特性说明 | 用户价值 |
| **技术亮点** | 技术创新点 | 技术优势 | 技术壁垒 |

### 8. 产品热点分析（近一周）

| 序号 | 热点类型 | 热点内容 | 来源 |
|------|---------|---------|------|
| 1 | 产品发布/市场活动/中标/宣传/官网发布 | 具体事件描述 | 新闻链接 |
| 2 | ... | ... | ... |
| ... | ... | ... | ... |
| 10 | ... | ... | ... |

**热点类型说明**：
- **产品发布**：新版本发布、功能更新、产品升级
- **市场活动**：发布会、展会、营销活动、客户活动
- **产品中标**：招投标中标、政府采购、大客户签约
- **产品宣传**：媒体报道、广告投放、软文推广
- **官网发布**：官网公告、产品动态、博客文章

**数据收集方法**：
```markdown
# 近期热点搜索
web_search(query="{产品名} 发布 新版本 2025", max_results=5)
web_search(query="{产品名} 中标 招投标 2025", max_results=5)
web_search(query="{产品名} 活动 发布会 展会 2025", max_results=5)
web_search(query="{产品名} 签约 合作 2025", max_results=5)
web_fetch(url="{产品官网}/news")  # 官网新闻
web_fetch(url="{产品官网}/blog")  # 官方博客
```

| **定价信息** | 定价模式、价格区间、付费方式 | P1 |

## 输出模板建议

**产品竞争分析报告结构**：

1. **产品概览** - 产品定位、发展历程、市场地位
2. **宏观环境影响** - PESTEL 分析
3. **目标用户分析** - 用户画像、需求场景、痛点
4. **竞品对比矩阵** - 与主要竞品多维度对比
5. **商业模式分析** - 商业模式画布九大模块
6. **SWOT 分析** - 优劣势与机会威胁
7. **核心功能分析** - 功能模块、技术能力、差异化
8. **产品热点分析** - 近一周产品发布、市场活动、中标、宣传等热点（10条）
9. **综合判断** - 产品竞争力评估与建议

- `swot-analysis-template.html` → SWOT 分析
- `core-features-template.html` → 核心功能
