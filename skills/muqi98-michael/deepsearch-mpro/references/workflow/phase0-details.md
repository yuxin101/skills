# 阶段 0：主题理解与确认 - 详细流程

## 目的

在开始分析前，**确保正确理解用户的研究需求**，避免方向偏差。系统将解析用户输入，识别研究主题、分析领域、搜索范围，生成确认提示，等待用户确认或修改。

---

## 详细工作流程

### Step 0.1：解析用户输入

**识别以下关键信息**：

| 识别项 | 说明 | 示例 |
|-------|------|------|
| **研究主题** | 用户要分析的核心对象 | DeepSeek 大模型技术能力、金蝶云星空市场表现 |
| **分析领域** | 自动推断的领域类型 | AI 大模型厂商、SaaS 厂商、市场分析 |
| **搜索范围** | 时间范围、地域范围、行业细分 | 2024-2025、中国、制造业 |
| **特定角度** | 用户关注的特定视角 | 技术能力、市场竞争、产品功能 |

**解析技巧**：

1. **提取关键词**：从用户输入中提取关键名词和动词
2. **识别实体**：识别公司名、产品名、技术名词
3. **提取约束**：识别时间、地域、行业等约束条件
4. **理解意图**：判断用户是要分析、学习还是比较

**示例解析过程**：

```markdown
用户输入："分析 DeepSeek 大模型的技术能力和市场表现"

解析结果：
- 研究主题：DeepSeek 大模型技术能力与市场表现
- 核心对象：DeepSeek（AI 大模型厂商）
- 分析领域：AI 大模型厂商分析
- 关键词：技术能力、市场表现
- 约束条件：无明确约束，使用默认值
```

---

### Step 0.2：推断分析领域

根据研究主题关键词，自动匹配分析领域：

| 关键词特征 | 推断领域 | 加载文档 |
|-----------|---------|---------|
| 包含 AI 厂商名（OpenAI、DeepSeek、豆包、智谱） | AI 大模型厂商 | `domains/ai-vendor-analysis.md` |
| 包含 AI 工具名（Cursor、ChatGPT、Midjourney）+ "学习/使用" | AI 工具学习指南 | `domains/ai-tool-learning-guide-framework.md` |
| 包含 AI 工具类别（文生图、文生 PPT、文生视频） | AI 工具产品 | `domains/ai-tools-analysis.md` |
| 包含 SaaS 厂商名（金蝶、用友、SAP、Oracle） | SaaS 厂商 | `domains/saas-vendor-analysis.md` |
| 包含"市场/竞争/行业"关键词 | 市场/竞争分析 | `domains/market-analysis.md` |
| 包含"财务/投资/估值"关键词 | 财务/投资 | `methodology/analysis-frameworks.md` |

**推断逻辑**：

1. **优先匹配特定实体**：如果用户输入包含特定的公司名或产品名，优先匹配对应领域
2. **关键词组合判断**：结合多个关键词综合判断
3. **默认回退**：如果无法明确判断，默认为"市场分析"领域

**示例推断过程**：

```markdown
场景 1："DeepSeek 大模型技术能力分析"
- 识别实体：DeepSeek → AI 厂商
- 推断领域：AI 大模型厂商
- 加载文档：domains/ai-vendor-analysis.md

场景 2："金蝶云星空在制造业的市场表现"
- 识别实体：金蝶、云星空 → SaaS 厂商 + 产品
- 识别约束：制造业
- 推断领域：SaaS 厂商分析 + 市场分析
- 加载文档：domains/saas-vendor-analysis.md + domains/market-analysis.md

场景 3："Cursor 学习指南"
- 识别实体：Cursor → AI 工具
- 识别意图：学习/使用 → 学习指南
- 推断领域：AI 工具学习指南
- 加载文档：domains/ai-tool-learning-guide-framework.md
```

```markdown
### 搜索范围
| 项目 | 内容 |
|------|------|
| **时间范围** | [默认：近1周，可修改] |
| **地域范围** | [默认：中国，可修改] |
| **行业细分** | [如适用，默认：不限] |
| **数据来源** | [默认：综合来源，可指定] |

### 分析重点
- [分析重点1]
- [分析重点2]
- [分析重点3]

---

**请确认或修改：**
- ✅ 确认无误，直接回复"确认"或等待 **10秒** 自动开始
- ✏️ 如需修改，请直接输入修改内容，例如："时间范围改为2023-2025"
- ❓ 如有疑问，请直接提问

```

**确认提示生成规则**：

1. **基本信息**：必须包含研究主题、分析领域、加载文档
2. **搜索范围**：提供默认值，明确可修改
3. **分析重点**：根据领域自动生成 3-5 个分析重点
4. **特殊需求**：如果是 AI 工具学习指南，需要询问用户角色

---

### Step 0.4：处理用户反馈

| 用户行为 | 系统处理 |
|---------|---------|
| **无响应（10秒）** | 自动进入阶段1，开始分析 |
| **回复"确认"** | 立即进入阶段1，开始分析 |
| **回复修改内容** | 更新分析参数，重新生成确认提示 |
| **提出疑问** | 解答疑问，等待确认 |

**修改处理逻辑**：

```markdown
用户修改："时间范围改为2023-2025"

处理步骤：
1. 解析修改内容：识别"时间范围" → "2023-2025"
2. 更新搜索范围：time_range = "2023-2025"
3. 重新生成确认提示，展示更新后的参数
4. 等待用户确认
```

**疑问处理逻辑**：

```markdown
用户疑问："为什么要加载这些文档？"

处理步骤：
1. 解析疑问内容
2. 根据分析领域解释文档的作用
3. 展示文档的主要内容概览
4. 等待用户确认
```

---

### Step 0.5：最终确认清单

进入阶段1前，确认以下信息：

- [ ] 研究主题已明确
- [ ] 分析领域已识别
- [ ] 搜索范围已确定（或使用默认值）
- [ ] 用户已确认或超时自动确认

**确认失败处理**：

如果无法确认以上信息，应：
1. 询问用户补充必要信息
2. 提供选择建议
3. 避免盲目开始分析

## 特殊场景处理

### 场景 1：用户输入过于简短

**用户输入**：
```
DeepSeek
```

**处理方式**：
```markdown
## 📋 研究主题确认

您提到了 "DeepSeek"，我识别出这是一个 AI 大模型厂商。请确认您的研究意图：

1. 分析 DeepSeek 的技术能力和市场表现
2. 学习如何使用 DeepSeek
3. 其他（请说明）

请回复数字 1、2 或描述您的具体需求。
```

### 场景 2：无法识别分析领域

**用户输入**：
```
帮我分析一下这个产品
```

**处理方式**：
```markdown
## 📋 需要更多信息

我需要更多信息来帮助您分析。请告诉我：

1. 您想分析的产品名称是什么？
2. 您关注的是哪个方面？（市场表现、技术能力、用户体验等）

例如："分析 Cursor 的市场表现" 或 "学习如何使用 ChatGPT"
```

### 场景 3：多个领域交叉

**用户输入**：
```
DeepSeek vs 豆包 vs 智谱 AI，哪个更适合企业采购？
```

```markdown
### 对比维度
- 技术能力（模型性能、API 能力）
```

## 总结

阶段0的核心目标是**确保理解正确**，避免方向偏差。关键要点：

1. **充分解析**：不要急于开始，确保理解用户意图
2. **明确范围**：时间、地域、行业等约束条件要明确
3. **交互确认**：给用户确认和修改的机会
4. **文档加载**：根据领域自动加载对应的参考文档
5. **特殊处理**：对简短输入、无法识别、多领域交叉等场景有明确处理方式
