# Industry Terms: Academic / Scholarly Research

> **Usage note:** This file is for cross-domain academic and scholarly work: publication, peer review, reproducibility, author response, and research communication. Use it alongside a technical domain file such as ML, systems, data management, or networking when the user's experience includes papers, experiments, benchmarks, submissions, or other academic workflows.

---

## Research Design & Evaluation

| User's description | Industry term |
|---|---|
| "compared against previous methods" | Baseline Comparison / Benchmark Evaluation |
| "removed parts to see what mattered" | Ablation Study |
| "tested whether results were statistically meaningful" | Statistical Significance Testing |
| "defined clear hypotheses before running experiments" | Hypothesis-driven Research Design |
| "validated results on several datasets or settings" | Cross-dataset Generalization / External Validation |
| "made the experiment easy to rerun" | Reproducibility / Experimental Rigor |
| "checked where the method fails" | Failure Mode Analysis |
| "separated what is novel from what is engineering" | Novelty Attribution / Contribution Framing |
| "made sure each claim was backed by actual results" | Claims Alignment / Evidence-backed Argumentation |
| "used stronger comparison methods instead of easy baselines" | Strong Baselines / Fair Comparison |
| "ran the method multiple times and reported the spread" | Multi-seed Evaluation / Variance Reporting |
| "wrote down how hyperparameters were chosen" | Hyperparameter Selection Protocol |
| "tested on several benchmarks instead of just one" | Multi-benchmark Evaluation / Breadth of Evaluation |
