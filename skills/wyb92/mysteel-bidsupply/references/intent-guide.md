## 意图识别流程

意图识别是技能执行的第一步，需要准确判断用户的查询类型：

```
用户输入
  ↓
特征提取（关键词、句式、语义）
  ↓
意图分类（招投标/供需）
  ↓
参数提取（基于意图类别）
  ↓
调用对应API
```

## 招投标查询意图

### 关键词特征

**核心关键词**：
- 招标、投标、开标、截标、公告、公示
- 项目、工程、采购项目、建设项目
- 中标、中标结果、招标文件

**衍生关键词**：
- 招投标、招投标项目、招投标信息
- 采购、招标采购、公开招标、邀请招标
- 竞标、投标报价、投标单位

### 典型表达

- "查最近的工程招标"
- "找钢材采购项目"
- "有哪些投标机会"
- "2024年的招标项目"
- "最近的中标信息"
- "正在招标的项目"

### 意图判断要点

- 关注"项目"、"招标"、"工程"等词汇
- 用户关注的是"机会"和"商机"
- 时间敏感性较强，通常有时间范围要求

## 供需查询意图

### 关键词特征

**核心关键词**：
- 供应、求购、现货、资源、库存
- 采购、销售、买、卖
- 品种、规格、材质、钢厂、仓库、地区

**具体品种**：
- 螺纹钢、线材、盘螺、圆钢
- 热轧板卷、冷轧板卷、中厚板
- 工字钢、H型钢、槽钢、角钢

### 典型表达

- "找螺纹钢供应商"
- "有热轧板卷求购吗"
- "查询上海的库存资源"
- "哪里有HRB400E现货"
- "想买一批钢材"
- "我有钢材要卖"

### 意图判断要点

- 关注"现货"、"资源"、"库存"等词汇
- 用户关注的是具体的货物信息
- 地区、规格、材质等参数较多

## 角色判断规则

当识别为供需查询意图后，必须进一步判断用户的角色，决定查询供应信息还是求购信息。

### 买家角色（查询供应信息，type=1）

**特征关键词**：
- 找、采购、买、需要、要、寻找

**行为特征**：
- 主动寻找货源
- 表达购买需求
- 询问哪里有货

**典型表达**：
- "找螺纹钢供应商" → type=1
- "我要采购HRB400E" → type=1
- "哪里有热轧板卷现货" → type=1
- "需要50吨螺纹钢" → type=1
- "想买一批钢材" → type=1

**参数说明**：
- type=1 表示查询供应信息（找货源）
- 智能体需要将用户的购买需求转化为供应查询

### 卖家角色（查询求购信息，type=2）

**特征关键词**：
- 卖、出售、销售、有货、求购

**行为特征**：
- 主动销售货物
- 寻找购买方
- 询问求购需求

**典型表达**：
- "我有热轧板卷要卖" → type=2
- "想出售一批螺纹钢" → type=2
- "有钢材求购信息吗" → type=2
- "哪里有人要买钢材" → type=2
- "找买家" → type=2

**参数说明**：
- type=2 表示查询求购信息（找买家）
- 智能体需要将用户的销售需求转化为求购查询

### 模糊表达处理

**问题表达**：
- "钢材资源"
- "钢材信息"
- "查一下钢材"

**处理方式**：
- 通过上下文和关键词进行智能判断，不直接询问用户
- 如果表达过于模糊，可以提供两种选项的示例，让用户选择
- 优先根据常见表达模式进行判断：提及"现货"、"库存"等通常为买家找资源（type=1），提及"求购"等通常为卖家找买家（type=2）
- 如果无法明确判断，可以默认查询供应信息（type=1），这是最常见的使用场景

## 参数提取规则

### 招投标查询参数

**必填参数**：
- `query`: 用户提问的自然语言文本

**可选参数**：
- `start_time`: 开始时间戳（毫秒），默认近3个自然年
- `end_time`: 结束时间戳（毫秒），默认当前时间
- `top_k`: 召回条数，默认10

**时间处理规则**：
- 用户未提供时间：自动使用近3个自然年
- 用户指定时间：验证是否在3个自然年内，超过则拒绝
- 时间提取：将"2024年"、"2024年1月"等转换为时间戳

### 供需查询参数

**必填参数**：
- `type`: 信息类型，1=供应信息，2=求购信息
- `shop_spot_limit`: 每个店铺最多取几条，必填，默认5

**可选参数**：
- `breed_name`: 品种名称（如：螺纹钢、热轧板卷）
- `spec`: 规格名称（如：HRB400E、Q235B）
- `material`: 材质名称（如：20#、45#）
- `steel_mill`: 钢厂名称（如：宝钢、沙钢）
- `warehouse_area`: 地区名称（如：上海、杭州）
- `warehouse_name`: 仓库名称

**参数提取规则**：
- 品种：优先从具体品种词提取（螺纹钢、热轧板卷等）
- 规格：从规格描述中提取（HRB400E、20mm、Φ12等）
- 地区：从地区描述中提取（上海、杭州、江苏等）
- 钢厂：从钢厂名称中提取（宝钢、沙钢、马钢等）

## 常见示例

### 示例1：明确买家意图

**用户输入**："找上海的螺纹钢供应商"

**处理流程**：
1. 意图识别：供需查询（"供应商"、"现货"）
2. 角色判断：买家（"找"、"供应商"）→ type=1
3. 参数提取：
   - type=1
   - breed_name="螺纹钢"
   - warehouse_area="上海"
4. 调用脚本：`python scripts/supply_demand_api.py --type 1 --breed_name 螺纹钢 --warehouse_area 上海`

### 示例2：明确卖家意图

**用户输入**："我有热轧板卷要卖"

**处理流程**：
1. 意图识别：供需查询（"热轧板卷"）
2. 角色判断：卖家（"要卖"、"有货"）→ type=2
3. 参数提取：
   - type=2
   - breed_name="热轧板卷"
4. 调用脚本：`python scripts/supply_demand_api.py --type 2 --breed_name 热轧板卷`

### 示例3：模糊供需意图

**用户输入**："钢材资源"

**处理流程**：
1. 意图识别：供需查询（"资源"）
2. 角色判断：模糊（无明确买/卖关键词）
3. 补充询问："您是想找货源（买家），还是找买家（卖家）？"
4. 根据用户回答设置type参数

### 示例4：招投标默认时间

**用户输入**："最近有哪些钢材采购的招标项目"

**处理流程**：
1. 意图识别：招投标查询（"招标"、"项目"）
2. 参数提取：
   - query="钢材采购招标项目"
   - start_time: 自动计算近3个自然年
   - end_time: 当前时间
   - top_k=10（默认）
3. 调用脚本：`python scripts/bidding_api.py --query "钢材采购招标项目"`

### 示例5：招投标指定时间

**用户输入**："查2024年的招标项目"

**处理流程**：
1. 意图识别：招投标查询（"招标"）
2. 参数提取：
   - query="招标项目"
   - start_time: 2024年1月1日时间戳
   - end_time: 2024年12月31日时间戳
3. 验证：2024年在近3个自然年内，允许查询
4. 调用脚本：`python scripts/bidding_api.py --query "招标项目" --start_time 1704038400000 --end_time 1735660799999`

### 示例6：招投标超出时间范围

**用户输入**："查2020年的招标项目"

**处理流程**：
1. 意图识别：招投标查询（"招标"）
2. 参数提取：时间戳为2020年
3. 验证：2020年早于3个自然年（2024-2026年），拒绝查询
4. 返回提示："对不起，无法查询超过3个自然年之前的招投标信息。请调整查询时间范围。"

## 注意事项

- 意图识别是第一优先级，必须先确定查询类型
- 角色判断是供需查询的关键，直接影响type参数设置
- 时间范围验证是招投标查询的强制检查
- 参数提取要灵活，支持多种表达方式
- 遇到模糊表达时，优先询问用户以避免误判
