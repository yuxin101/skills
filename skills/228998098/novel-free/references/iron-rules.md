# 核心铁律（优先级降序排列）

> 任何时候不得违反，所有 Agent 必须遵守。

---

## P0 用户边界违规（绝对禁止）

- 所有创作必须严格遵守用户指定的边界（R18/BE/特定元素）
- 用户未明确说明时，默认"适合全年龄、纯爱/正剧向、无露骨色情"
- 违规直接阻断写作，P0级事故

## P1 人设崩坏/OOC（严格禁止）

- 角色行为、语言、决策必须与角色圣经一致
- 风格锚定文档是写作宪法，违反视为P1事故
- "OOC"定义：角色行为/语言偏离角色圣经中定义的弧光、性格、动机

## P2 设定逻辑冲突（必须修复）

- 世界观内部逻辑自洽
- 战力/道具/时间线与追踪表一致
- 伏笔埋设与回收逻辑连贯

## P3 风格偏离（应当修复）

- 语言风格/叙事视角/禁用词表遵循风格锚定
- 对话风格/感官偏好与示范文本一致
- 对于项目已明确列为“模板化禁句”的句式，必须视为硬规则执行；一经发现应优先改写，不得以“偶尔顺手”保留
- **禁用 AI 味转折模板**：禁止使用“不是...，不是...，而是...”“不是...，而是...”这类对称转折套话；禁止用同构句式反复制造对比感，必须改写为更自然、更具体的叙述表达
- **禁用提示词腔**：避免“核心在于 / 重点是 / 需要注意的是 / 值得一提的是 / 既...又...”这类说明书味、总结腔、并列堆叠句；优先写具体动作、感受、场景和结果

## P4 节奏字数偏差（优化项）

- 字数应在目标±20%范围内
- 章节节奏应符合细纲规划（铺垫/上升/高潮/过渡）

---

## 其他铁律

- **结尾钩子**：每章正文结尾必须有强叙事钩子或情感留白
- **结构化文档**：项目所有文档必须结构化（Markdown优先）、版本化、可追溯
- **目标语言**：输出永远用流畅、自然、高文学性的目标语言（默认中文）
- **禁止擅自添加**：严禁擅自添加用户未授权的元素（OC、强行HE/BE等）
- **档案先行**：每次写新章节前必须查阅上章结尾状态、本章伏笔线、情感/关系演变目标。禁止凭空创造导致与前文矛盾
- **状态机先行**：每次续写、批量写作、自动推进恢复前，必须先执行恢复协议并校验 `meta/workflow-state.json`。禁止凭聊天记忆直接续写
- **Phase顺序**：严格按 Phase 0→1→2→3 顺序推进，不可跳跃
- **纯英文路径**：小说项目目录下所有目录和文件命名必须使用英文、数字、连字符，**禁止使用中文**。包括章节文件名（如 `ch001.md` 而非 `第一章.md`）
- **章节名称同步**：`chapters/chapter_name.md` 用于保存章节编号与中文名称的映射。每写完一章必须更新该文件，格式为 `| ch001 | 章节中文名称 |`。禁止在正文章节文件名中使用中文
- **风格锚定体积控制**：`meta/style-anchor.md` **必须控制在 1000 字以内**；Phase 1 Step 1-7 StyleAnchorGenerator 输出后，Coordinator 须立即统计字数，超出时要求 StyleAnchorGenerator 精简（保留：禁用词表 + 负面示范 + 1-2 段正文样本，删除：冗余说明、过多场景分类）。每章全文喂入 MainWriter，超体积直接增加固定 token 消耗
- **Tracker 合并写入**：每章闭环时，`state-tracker.md`、`relationship-tracker.md`、`plot-tracker.md` 三个 tracker 应在同一步骤内集中写入，避免分散写入导致部分遗漏；若写入中断，视为 `archiveSync = false`

---

## 冲突优先级

当不同文档之间出现矛盾时，按以下优先级裁决：

```
角色圣经 > 世界观 > 大纲 > 细纲
```

Coordinator 有权根据更高优先级文档修正矛盾，修正后记录到 `archive/archive.md`。
