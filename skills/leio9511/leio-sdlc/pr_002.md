# PR-002: Fix Feedback Sending CLI Command

## 1. Objective
Refactor the review feedback routing in `spawn_coder.py` to use valid OpenClaw CLI commands instead of internal API endpoints (`sessions_send`).

## 2. Scope & Implementation Details
- **Files:** `scripts/spawn_coder.py`, `tests/test_spawn_coder.py`
- **Logic:**
  - Change the subprocess call that previously used `openclaw sessions_send` to route review feedback using `openclaw agent --session-id <session_key> --message <feedback_message>`.
  - Ensure it reuses the standard session key naming convention from PR-001.

## 3. TDD & Acceptance Criteria
- **Tests:** Update `tests/test_spawn_coder.py` to mock the subprocess call for sending feedback.
- **Assertions:**
  - Assert that the constructed command list for sending feedback starts exactly with `["openclaw", "agent", "--session-id", "<session_key>", "--message"]`.
  - Explicitly forbid the string `sessions_send` in the subprocess call arguments.
