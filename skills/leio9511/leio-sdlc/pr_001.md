# PR-001: Fix Initial ACP Session Spawning CLI Command

## 1. Objective
Refactor the initial session creation logic in `spawn_coder.py` to use valid OpenClaw CLI commands instead of internal API endpoints (`sessions_spawn`). Also update the session key naming convention.

## 2. Scope & Implementation Details
- **Files:** `scripts/spawn_coder.py`, `tests/test_spawn_coder.py`
- **Logic:**
  - Change the subprocess call that previously used `openclaw sessions_spawn` to initialize the session using `openclaw agent --session-id <session_key> --message <task_string>`.
  - Ensure the session key string format strictly matches standard naming conventions (`sdlc_coder_<PR_ID>`).

## 3. TDD & Acceptance Criteria
- **Tests:** Update `tests/test_spawn_coder.py` to mock the subprocess call for initial session creation.
- **Assertions:**
  - Assert that the constructed command list starts exactly with `["openclaw", "agent", "--session-id", "<session_key>", "--message"]`.
  - Explicitly forbid the string `sessions_spawn` in the subprocess call arguments for session initialization.
