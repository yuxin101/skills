**日期：** 2026-03-15

## 📱 产品形态

**MVP 版本：** Web/H5 单页应用

**核心流程：**
```
用户输入问题 → 滚滚分析 → 输出判断 → 用户反馈
```

## 🛠️ 技术栈

### 前端
```yaml
框架：Vue 3 + Vite
UI 库：Tailwind CSS + Headless UI
状态管理：Pinia
HTTP: Axios
路由：Vue Router
打包：Vite
部署：Vercel/Netlify
```

### 后端
```yaml
框架：Node.js (Express) 或 Python (FastAPI)
数据库：SQLite (MVP) → PostgreSQL (v1.0)
AI 集成：OpenClaw API
缓存：Redis (可选)
部署：阿里云/腾讯云
```

## 📊 API 设计

### 1. 提交问题
```http
POST /api/v1/problems
Content-Type: application/json

{
  "problem": "问题描述",
  "type": "职业选择",
  "context": {
    "current_status": "当前状态",
    "options": ["方案 A", "方案 B"],
    "concerns": "顾虑",
    "preference": "偏好"
  }
}

Response:
{
  "problem_id": "prob_123",
  "status": "analyzing"
}
```

### 2. 获取判断结果
```http
GET /api/v1/problems/prob_123

Response:
{
  "problem_id": "prob_123",
  "status": "completed",
  "result": {
    "recommendation": "方案 B",
    "reasoning": "分析理由...",
    "comparison": {...},
    "action_plan": [...]
  }
}
```

### 3. 提交反馈
```http
POST /api/v1/problems/prob_123/feedback
{
  "satisfaction": 5,
  "adopted": true,
  "comment": "很有帮助"
}
```

## 💡 核心原则

1. **简洁第一** - 不要复杂功能，先跑通流程
2. **快速迭代** - 每周一个版本，快速试错
3. **用户反馈** - 每个判断都收集反馈
4. **数据驱动** - 用数据优化判断质量
