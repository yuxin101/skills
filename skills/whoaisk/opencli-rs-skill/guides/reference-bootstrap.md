# Reference Bootstrap

为新的 `references/<target>.md` 编写 target 卡时使用本文件。

## 写法原则

- 以 `opencli-rs <target>` 的真实命令为骨架。
- 只保留高频、稳定、可复用的命令模式。
- 不抄大段 `--help`，不重写完整说明书。
- 只补充真正非显然、会影响执行成败的最小说明。
- 不重复 `SKILL.md` 与上位 guide 已写明的通用规则。
- 不写该 target 的显然常识、平台简介或重复性的通用执行规则。
- 内容保持短、小、硬。

## 展示技巧写入边界

- 若某个 target 的返回结构、链接形态、图片资源或内容组织方式，会稳定影响特定渠道上的最终展示效果，可在 `最小说明` 中记录该 target 的最佳展示技巧。
- 仅记录与该 target 强相关、稳定复用、明显影响交付质量的最小技巧。
- 不把 target 卡写成通用渠道发送手册；与 target 无关的通用发送规则，仍应留在上位技能或对应渠道技能中。
- 若某条展示技巧只适用于一次性案例、临时 workaround，或与 target 本身无强关联，则不要写入 target 卡。

- <仅保留该 target 特有、非显然、会影响执行成败的提示>
</target-md-template>

## 可删节规则

- 标题使用用户可识别的展示名；必要时可带常见别名。
- 常用模式只保留最常用的 1–3 个。
- 若没有额外风险，`最小说明` 只保留 1–2 条即可。
- 若目标几乎没有额外说明点，可只保留极短提示。
- 若少量命令名或命令形态直接取自真实 `--help` 更简洁，可以最小引用；不要搬运说明性文字，更不要整段复制完整 `--help`。
- 若某 target 存在稳定复用、且明显影响交付质量的渠道展示技巧，可作为 target-specific 说明保留 1–2 条。

## 回读检查

- 是否符合统一模板
- 标题是否使用了合适的展示名
- 命令示例是否来自真实可用命令
- 是否混入了上位已定义的通用规则
- 是否写成了手册，而不是高频命令卡
- 是否只保留了真正有用的最小说明
