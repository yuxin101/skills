# 追问流程指南

## 何时追问

当用户需求包含以下特征时触发追问：

1. **模糊词汇**：大概、可能、类似、优化一下、改进
2. **缺乏细节**：未指定文件、未说明目标
3. **多解歧义**：同一需求有多种合理解法
4. **涉及复杂逻辑**：需要理解现有代码结构

## 追问轮次分配

### 第一轮：结构化快速确认

每次只问 1 个问题，提供 2-4 个选项：

```
1️⃣ 你的核心目标是什么？
   A) 功能实现（新增 XXX 功能）
   B) 性能优化（让 XXX 更快）
   C) 代码质量（让 XXX 更清晰）
   D) 其他：_______

2️⃣ 这个改动涉及范围多大？
   A) 单个文件
   B) 某个模块/目录
   C) 整个项目

3️⃣ 有无特殊约束？
   A) 需要兼容旧版本
   B) 需要保持 API 不变
   C) 无特殊要求
```

### 第二三轮：深入提问

针对第一轮未覆盖的模糊点：

```
4️⃣ 你提到的"优化"，具体是指？
   A) 执行速度更快
   B) 内存占用更低
   C) 代码更易维护
   D) 其他：_______

5️⃣ 这个函数目前有测试吗？
   A) 有，需要确保测试通过
   B) 没有，可以添加
   C) 不需要测试
```

### 第四五轮：查漏补缺

确认边界情况和潜在问题：

```
6️⃣ 修改后需要兼容之前的调用方式吗？
   A) 必须完全兼容
   B) 可以略有变化
   C) 无所谓

7️⃣ 有没有你希望避免的方式？
   A) 不想用 XXX 库
   B) 不想改变现有文件结构
   C) 没有限制
```

## 追问技巧

1. **一次一问**：避免一次抛出一堆问题
2. **选项具体**：避免"都可以"这种无效选项
3. **解释利弊**：复杂选项要说明 trade-off
4. **适时停止**：当信息足够时就停止追问
5. **记录理解**：每轮确认 AI 对需求的理解是否正确

## 示例对话

```
用户：帮我优化一下这个函数

AI：
收到！在开始之前，我需要澄清几个问题：

1️⃣ 你的"优化"主要指哪方面？
   A) 性能提升（执行速度、内存）
   B) 代码可读性（注释、结构）
   C) 两者都有

---

用户：C

AI：
明白，要兼顾性能和可读性。

2️⃣ 这个函数目前在哪个文件中？
   [等待用户回答]

---

用户：src/utils/helper.js

AI：
好的，找到了这个文件。

3️⃣ 你遇到的具体问题是什么？
   A) 执行太慢，有性能数据
   B) 代码太难懂，维护困难
   C) 功能有问题，行为不符合预期

---

用户：B

AI：
明白了，主要是代码可读性问题。

4️⃣ 你希望怎样让代码更易读？
   A) 添加详细注释
   B) 拆分成更小的函数
   C) 重构变量命名
   D) 以上都要

---

用户：D

AI：
清楚了！你需要：
- 目标：优化 helper.js 中的函数
- 方向：性能和可读性都要
- 范围：单文件 src/utils/helper.js

我来看一下代码，然后给出优化方案。
```

## 快速跳过

当需求足够明确时，可以快速确认并跳过追问：

```
用户：在 src/utils/helper.js 第 15 行的 foo 函数后面加一行注释 "// 计算完成"

AI：
（跳过追问，直接执行）
好的，在指定位置添加注释。

🤔 确认执行？
- 文件：src/utils/helper.js
- 操作：在第 15 行后添加注释
- 风险：低风险（纯文本添加）

[继续执行]
```
