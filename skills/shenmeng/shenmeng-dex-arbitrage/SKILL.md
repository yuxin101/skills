# DEX Arbitrage DEX搬砖套利

帮助用户发现和执行DEX之间的搬砖套利机会，最大化套利收益。

## 核心能力

1. **套利机会发现** — 监控多DEX价格差异
2. **收益计算** — 精确计算套利净利润（含Gas、滑点）
3. **策略设计** — 搬砖路径优化、时机选择
4. **自动化脚本** — 套利机器人模板、监控脚本
5. **风险管理** — MEV保护、三明治攻击防护
6. **跨链套利** — 桥接选择、资金调度

## 搬砖套利类型

### 1. 跨DEX套利（同一链）

**原理**：
```
同一代币对在不同DEX价格不同
A: Uniswap  ETH/USDC = $3,500
B: SushiSwap ETH/USDC = $3,520
差价：0.57%

操作：在A买ETH，在B卖ETH，赚USDC
```

**适合链**：
- 以太坊（流动性最大，差价机会多）
- BSC（Gas低，适合小额）
- Arbitrum/Optimism（L2 Gas低，速度快）

### 2. 跨链套利

**原理**：
```
同一资产在不同链价格不同
ETH主网 ETH = $3,500
Arbitrum ETH = $3,480
差价：0.57%

操作：在Arb买ETH，跨链到主网卖
注意：桥接时间和成本
```

**常用桥**：
- Across（速度快）
- Stargate（流动性好）
- Hop（费用低）
- 官方桥（最安全）

### 3. 三角套利

**原理**：
```
三种代币间价格不均衡
USDC → ETH → WBTC → USDC

如果循环后USDC变多，存在套利空间
需要同时考虑三个池子的滑点
```

**公式**：
```
套利收益 = 最终USDC - 初始USDC - Gas费用

条件：收益 > 0
```

### 4. 闪电贷套利

**原理**：
```
无本金套利模式
1. 闪电贷借入USDC
2. Uniswap买ETH
3. SushiSwap卖ETH
4. 还款 + 利息
5. 剩余为利润

风险：如果第4步不够还款，交易回滚
```

**闪电贷来源**：
- Aave（0.09%费用）
- Balancer（0%费用）
- dYdX（0%费用）
- Uniswap V3（0.05%费用）

### 5. CEX-DEX套利

**原理**：
```
中心化 vs 去中心化价格差异
Binance ETH = $3,500
Uniswap ETH = $3,520

操作：
1. 在Binance买ETH
2. 提到钱包
3. 在Uniswap卖ETH

注意：提币时间和手续费
```

## 搬砖收益计算框架

### 净利润公式

```
毛利润 = 卖出金额 - 买入金额

净利润 = 毛利润 - Gas费 - 滑点损失 - 协议费 - 桥接费

净利润率 = 净利润 / 投入本金
```

### 各项成本详解

**Gas费用**：
```
以太坊主网：
- 简单转账：~$5-20
- Swap操作：~$20-100
- 闪电贷套利：~$50-200

L2（Arbitrum/Optimism）：
- Swap操作：~$0.5-5

BSC：
- Swap操作：~$0.1-1
```

**滑点损失**：
```
取决于：
- 交易规模 vs 池子深度
- 交易对流动性

估算：
滑点 ≈ 交易金额 / (2 * 池子TVL)
```

**协议费用**：
```
Uniswap V3：0.05% / 0.3% / 1%
SushiSwap：0.3%
Curve：0.04%
Balancer：0.15% - 1%
```

**桥接费用**（跨链）：
```
Across：~$5-20 + 0.04%
Stargate：~$5-15 + 0.06%
官方桥：Gas费 + 时间成本
```

### 盈亏平衡点

```
最小可套利价差 = Gas费/本金 + 滑点 + 协议费

示例（本金$10K，主网）：
最小价差 = $50/$10000 + 0.3% + 0.3% = 1.1%

L2（本金$10K）：
最小价差 = $2/$10000 + 0.3% + 0.05% = 0.55%
```

## 搬砖操作流程

### 手动搬砖步骤

**步骤1：发现机会**
```
使用工具：
- DexScreener查看价格
- 1inch比较路径
- 自定义脚本监控
```

**步骤2：验证收益**
```
计算：
- 两边价格
- 滑点估算
- Gas成本
- 净利润

确认：净利润 > 0
```

**步骤3：执行交易**
```
1. 在低价DEX买入
2. 等待确认（1-2区块）
3. 在高价DEX卖出
4. 确认收到资金

注意：速度要快，价格会变动
```

**步骤4：记录复盘**
```
记录：
- 套利金额
- 实际收益
- Gas消耗
- 耗时

分析：
- 策略有效性
- 改进空间
```

## MEV与三明治攻击防护

### MEV（最大可提取价值）

**问题**：
```
套利交易被MEV机器人发现
- 被抢先交易（Front-running）
- 被套利者利用
- 利润被抢
```

**解决方案**：

**1. Flashbots（以太坊）**
```
使用私有内存池
交易直接发送给矿工
避免公开mempool暴露

集成：
from flashbots import flashbot

bundle = [
    w3.eth.send_transaction(buy_tx),
    w3.eth.send_transaction(sell_tx)
]
flashbot.send_bundle(bundle)
```

**2. 私有RPC**
```
- Eden Network
- Flashbots Protect
- MEV Blocker

将交易发送到私有节点
延迟公开广播
```

**3. 时间延迟**
```
大额套利分多笔执行
避免单个大额交易引起注意
```

### 三明治攻击防护

**攻击原理**：
```
攻击者看到套利交易
抢先买入推高价格
套利者以更高价成交
攻击者卖出获利
```

**防护方法**：
```
1. 设置滑点容忍度（如0.5%）
2. 使用MEV保护RPC
3. 分批执行
4. 避开高峰期
```

## 搬砖风险管理

### 主要风险

**1. 价格变动风险**
```
买入后价格下跌
卖出前价格上涨
导致亏损

控制：
- 快速执行
- 设置止损
- 小资金测试
```

**2. Gas价格风险**
```
Gas暴涨导致成本超预期
交易卡住无法执行

控制：
- 预设最大Gas
- 使用EIP-1559
- 准备应急资金
```

**3. 流动性风险**
```
池子深度不足
大额交易滑点过大

控制：
- 检查TVL
- 分批交易
- 选择主流池子
```

**4. 智能合约风险**
```
DEX合约漏洞
代币合约问题

控制：
- 使用知名DEX
- 检查合约审计
- 小额测试
```

**5. 跨链风险**
```
桥接延迟
资金卡在桥上
桥被黑

控制：
- 选择可靠桥
- 分散跨链
- 关注桥状态
```

### 风险评估框架

```
每次搬砖前评估：

□ 价差是否足够（>1%）
□ 池子流动性是否充足
□ Gas成本是否可控
□ 执行时间是否充裕
□ MEV风险是否可接受

总分 > 80分才执行
```

## 多链搬砖生态

### 以太坊主网

**特点**：
- 流动性最大
- 差价机会最多
- Gas成本高

**适合**：大资金（>$50K）
**策略**：闪电贷套利、大额套利

### BSC（币安智能链）

**特点**：
- Gas极低（$0.1-1）
- 速度快
- 套利竞争激烈

**适合**：小额高频
**DEX**：PancakeSwap, Biswap, Apeswap

### Arbitrum

**特点**：
- L2低Gas（$0.5-5）
- 以太坊安全性
- 流动性增长快

**适合**：中频套利
**DEX**：Uniswap V3, SushiSwap, Camelot

### Optimism

**特点**：
- L2低Gas
- OP奖励
- 生态系统成熟

**DEX**：Uniswap V3, Velodrome

### Polygon

**特点**：
- 超低Gas
- 快速确认
- 广泛采用

**DEX**：QuickSwap, Uniswap V3, Curve
