# 搬砖套利基础

## 什么是搬砖套利

搬砖套利（Arbitrage）是利用同一资产在不同市场的价格差异，低买高卖赚取无风险利润的交易策略。

### 核心原理

```
市场A价格: $100
市场B价格: $102

操作:
1. 在市场A买入: -$100
2. 在市场B卖出: +$102
利润: $2 (2%)
```

### 效率市场假说

理论上，套利机会会被迅速消除。但在加密货币市场：
- 信息传播有延迟
- 资金流动有摩擦
- 不同DEX独立定价
- 持续存在套利空间

## DEX搬砖类型

### 1. 简单跨DEX套利

**同一链，同一代币对**

```
Uniswap V3 ETH/USDC: 1 ETH = 3,500 USDC
SushiSwap ETH/USDC: 1 ETH = 3,520 USDC

套利操作:
1. Uniswap: 3,500 USDC → 1 ETH
2. SushiSwap: 1 ETH → 3,520 USDC
3. 利润: 20 USDC (0.57%)
```

**关键要素**：
- 同步监控多个DEX价格
- 快速执行（价格会变化）
- 考虑Gas成本
- 滑点估算

### 2. 三角套利

**三种代币循环套利**

```
池子价格:
- USDC/ETH = 1/3500
- ETH/WBTC = 1/0.065
- WBTC/USDC = 65,000

套利路径:
USDC → ETH → WBTC → USDC

计算:
10,000 USDC → 2.857 ETH → 0.1857 WBTC → 12,070 USDC
利润: 2,070 USDC (20.7%)
```

**注意**：实际中三角套利机会很少，且滑点会吃掉利润。

### 3. 跨链套利

**同一资产在不同链价格不同**

```
ETH主网: 1 ETH = $3,500
Arbitrum: 1 ETH = $3,480

套利操作:
1. Arbitrum: 3,480 USDC → 1 ETH
2. 跨链桥接到主网
3. 主网: 1 ETH → 3,500 USDC
4. 利润: 20 USDC (0.57%)
```

**挑战**：
- 桥接时间（5-30分钟）
- 桥接费用
- 价格可能在此期间变化

### 4. 闪电贷套利

**无本金套利**

```
流程:
1. 闪电贷借入 1,000,000 USDC
2. DEX A: USDC → ETH
3. DEX B: ETH → USDC
4. 还款 1,000,000 + 利息
5. 剩余为利润

特点:
- 零资金门槛
- 单笔完成
- 原子性（全部成功或回滚）
```

## 套利利润计算

### 毛利润计算

```
毛利润 = 卖出所得 - 买入成本

示例:
买入: 1 ETH @ $3,500 = $3,500
卖出: 1 ETH @ $3,520 = $3,520
毛利润 = $20
```

### 净利润计算

```
净利润 = 毛利润 - 所有成本

成本项:
1. Gas费
2. 协议手续费
3. 滑点损失
4. 桥接费（跨链）
5. 闪电贷利息

示例:
毛利润: $20
Gas费: $15
手续费: $10 (0.3% * 2)
滑点: $5
净利润: -$10 ❌
```

### 盈亏平衡分析

```
盈亏平衡价差 = 总成本 / 本金

示例（本金$10K）:
Gas: $20
手续费: $30 (0.3% * 2)
滑点: $10
总成本: $60

盈亏平衡价差 = $60 / $10,000 = 0.6%

结论: 价差 > 0.6% 才有利可图
```

### 不同链成本对比

| 链 | Gas费 | 手续费 | 最小可套利价差（$10K本金） |
|---|-------|--------|---------------------------|
| 以太坊主网 | $20-100 | 0.3-0.6% | 0.8-1.5% |
| Arbitrum | $0.5-5 | 0.3-0.6% | 0.4-0.6% |
| BSC | $0.1-1 | 0.2-0.3% | 0.3-0.4% |
| Polygon | $0.01-0.1 | 0.2-0.3% | 0.25-0.35% |

## 套利机会来源

### 1. 交易量不平衡

```
大额买单推高DEX A价格
DEX B价格滞后
产生套利窗口

持续时间: 几秒到几分钟
```

### 2. 新币上市

```
新币在不同DEX上线时间不同
早期价格发现不完善
大量套利机会

风险: 流动性不足，滑点大
```

### 3. 市场波动

```
剧烈波动时价格不同步
不同DEX反应速度不同
套利空间扩大

注意: Gas可能暴涨
```

### 4. 跨链延迟

```
资金跨链需要时间
链间价格暂时失衡
套利者平衡价格

持续时间: 桥接确认时间
```

## 套利执行策略

### 手动套利

**适用场景**：
- 低频大额
- 学习阶段
- 特殊机会

**步骤**：
1. 发现机会（监控工具）
2. 计算收益（利润计算器）
3. 执行买入（DEX 1）
4. 等待确认
5. 执行卖出（DEX 2）
6. 确认利润

### 半自动套利

**监控 + 手动执行**

```
机器人:
- 监控价格
- 计算利润
- 发送警报

人工:
- 审核机会
- 点击执行
- 风险控制
```

### 全自动套利

**机器人全自动**

```
监控 → 计算 → 执行 → 确认

完全自动化
7x24运行
需要完善的风控
```

## 套利风险

### 执行风险

```
买入后价格变化
卖出前流动性消失
交易失败
```

### Gas风险

```
Gas暴涨
交易卡住
成本超预期
```

### MEV风险

```
被三明治攻击
被抢先交易
利润被抢
```

### 合约风险

```
DEX合约漏洞
代币问题
资金损失
```

### 跨链风险

```
桥接延迟
桥被攻击
资金卡住
```

## 成功要素

### 1. 速度

- 低延迟节点
- 优化交易流程
- 预批准代币

### 2. 资本

- 足够本金
- 分散多链
- 应急资金

### 3. 技术

- 自动化脚本
- MEV保护
- 监控系统

### 4. 风控

- 严格止损
- 风险评估
- 应急预案
