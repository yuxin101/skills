# 爆款文案创作指南

## 核心原则

### 1. 一句话法则
每条文案必须是一句话，15-50 字，适合移动端快速阅读。

### 2. 情绪驱动
文案必须携带情绪：好奇、焦虑、惊喜、困惑、期待。

### 3. 互动导向
每条文案必须包含以下互动引导之一：
- ❓ 疑问结尾
- ⏰ 时效压迫
- 🙏 求教姿态
- 💬 邀请讨论

---

## 文案类型模板

### 对比争议型
制造选择困难，引发站队讨论。

**模板**：
- "A 还是 B？哪个更...？"
- "用了 A 再换 B，差距..."
- "A 和 B 的真实对比，没想到..."

**示例**：
- Claude 3.5 和 GPT-4o 写代码，真实体验差距有多大？求用过的大佬说说！
- Cursor 还是 Windsurf？AI 编程工具选哪个不踩坑？在线等！
- Kimi2.5 对比 Qwen3.5-plus，中文场景谁更强？

---

### 求教互动型
以请教姿态引发大佬分享欲。

**模板**：
- "求用过的大佬说说..."
- "有没有人试过...？"
- "...是什么体验？求分享"

**示例**：
- 新出的 ClaudeCode 真有那么神？用过的来聊聊真实感受！
- OpenClaw 的自动更新功能好用吗？求真实反馈！
- 有人用 MiniMax-M2.5 写过代码吗？效果如何？

---

### 体验分享型
强调"真实体验"增加可信度。

**模板**：
- "真实体验下来，..."
- "用了 X 天，发现..."
- "亲测...，结论是..."

**示例**：
- 真实体验下来，Qwen3.5-plus 写 Python 比 GPT-4 还顺！
- 用 Cursor 写了 7 天代码，这 3 个功能真香！
- 亲测 ArkClaw 自动部署，效率提升不止一倍！

---

### 时效热点型
绑定最新发布的模型/产品。

**模板**：
- "刚发布的 XX，..."
- "新出的...真有那么神？"
- "XX 更新了，这次..."

**示例**：
- 刚发布的 MiniMax-M2.7，号称超越 GPT-5？有人测了吗？
- 新出的 WorkBuddy 真能自动写周报？在线等测评！
- Claude 3.7 更新了，这次加了什么狠活？

---

## 高互动文案公式

```
[产品/功能] + [具体场景] + [情绪/疑问] + [互动引导]
```

**拆解示例**：
- "Cursor 的自动补全" + "写 React 时" + "真的准吗" + "求用过的大佬说说"
- = "Cursor 写 React 时的自动补全真的准吗？求用过的大佬说说！"

## 避坑指南

### ❌ 避免
- 过长文案（超过 50 字）
- 陈述句无互动
- 专业术语堆砌
- 多条信息混杂
- 虚假夸大宣传

### ✅ 推荐
- 口语化表达
- 具体场景描述
- 真实体验背书
- 单一核心信息
- 适度悬念设置

## 文案自检清单

- [ ] 是否为一句话（15-50 字）？
- [ ] 是否包含互动引导？
- [ ] 是否有具体产品/功能名称？
- [ ] 是否携带情绪或悬念？
- [ ] 是否口语化易读？
- [ ] 信息是否准确（已搜索核实）？

---

## 快速创作流程

1. **搜索热点** → 确定要讨论的产品/话题
2. **选择类型** → 对比/求教/体验/时效
3. **套用模板** → 填入具体产品名和场景
4. **添加互动** → 确保有疑问/求教/邀请
5. **核实信息** → 确认产品和功能真实存在
