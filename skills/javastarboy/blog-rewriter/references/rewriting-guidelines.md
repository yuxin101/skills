# 文章改写操作指南

## 改写流程详解

### 步骤 1: 内容获取

**输入是链接:**
```
1. 优先使用 web_fetch 工具提取 markdown 内容
2. 如页面复杂（需要登录/JS 渲染），使用 browser 工具
3. 提取：标题、正文、图片位置、广告话术
```

**输入是文本:**
```
直接进入分析处理流程
```

### 步骤 2: 原文分析

分析维度：
- 字数统计
- AI 痕迹检测（见 ai-detection-patterns.md）
- 广告/引流话术识别
- 图片数量与位置
- 关键事实/数据提取

### 步骤 3: 事实核实

对以下内容调用 `tavily-search` 核实：
- 具体数据（如"77 万字"、"2024 年 Q3"）
- 人名、公司名、产品名
- 时间敏感信息
- 存疑的统计数字

### 步骤 4: 深度改写

**降重技巧:**
1. 重组句式结构（主动↔被动，长句↔短句）
2. 替换同义词（使用近义词词典）
3. 调整段落顺序（不影响逻辑的前提下）
4. 改变叙述视角（第三人称↔第一人称）
5. 增加/删减修饰语

**去 AI 化:**
- 删除程式化连接词
- 增加个人化表达
- 加入情绪色彩
- 打破工整排比

**去广告:**
- 识别原文引流话术
- 删除或替换为用户指定话术

### 步骤 5: 封面生成

使用 `write` 工具创建 HTML 文件：
```
1. 选择合适模板（见 html-cover-templates.md）
2. 填入主标题 + 副标题
3. 保存到 workspace 临时目录
4. 告知用户文件路径供截图使用
```

### 步骤 6: 输出检查

使用质量检查清单逐项核对：
- [ ] 核心事实准确
- [ ] 无明显 AI 痕迹
- [ ] 广告话术已替换
- [ ] 图片占位标记完整
- [ ] 开头足够吸引人
- [ ] 有互动引导
- [ ] 5 个标题风格差异化
- [ ] 封面 4:3 比例

## 改写示例

### 原文（AI 痕迹重）
```
首先，我们需要了解人工智能的发展历程。其次，我们要分析当前的技术趋势。最后，我们可以展望未来的发展方向。综上所述，人工智能正在快速发展。
```

### 改写后（人类化）
```
人工智能这玩意儿，到底是怎么火起来的？

看看这几年的技术突破就知道了——从最初的规则系统，到现在的深度学习，AI 的发展速度简直让人咋舌。

那你可能会问：接下来会怎样？

我的判断是：真正的爆发才刚刚开始。
```

## 常见场景处理

### 技术文章
- 保持专业术语准确
- 代码块保留原样
- 增加实战案例/个人经验

### 情感文章
- 强化情绪表达
- 增加细节描写
- 用对话式语言

### 资讯文章
- 核实所有数据
- 注明信息来源
- 增加个人观点

### 干货文章
- 结构化呈现（列表/步骤）
- 增加可操作性
- 提供具体案例
