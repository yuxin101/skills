# OpenClaw双重清理系统 v1.3.0 变更日志

## 🚀 **版本重大更新**
**发布日期：** 2026-03-29  
**更新类型：** 重大功能重构  
**执行人：** 罗汉  
**要求来源：** 杜（主人）在2026-03-29 00:41明确要求

## 🎯 **核心变更内容**

### **1. 文件数量触发阈值调整**
- **变更前：** 会话文件数 > 30个触发清理
- **变更后：** **会话文件数 > 10个立即触发清理**（杜的要求）

### **2. 自动清理策略彻底重构**
- **变更前：** 三层保障架构
  - 第1层：智能心跳检测
  - 第2层：周度轻度清理（周日凌晨）
  - 第3层：月度深度清理（每月1日）
- **变更后：** 全新简化架构
  - **第1层：智能心跳检测**（低阈值：>10个文件）
  - **第2层：周一深度清理**（唯一自动清理）
  - **第3层：手动命令清理**（用户完全控制）

### **3. 删除的自动清理功能**
按照杜"不做每月、每日的自动清理"的要求：
- ❌ **删除：月度深度清理**（不再有每月1日凌晨3点的清理）
- ❌ **删除：周度轻度清理**（不再有每周日凌晨2点的清理）
- ✅ **保留：周一9:00深度清理**（唯一的自动清理）
- ✅ **保留：智能心跳触发**（>10个文件立即清理）

### **4. 新增网关补偿机制**
- **新增功能：** 如果周一9:00时网关未开，系统会自动记录等待任务
- **智能检测：** 网关启动后检查是否为周一，执行等待的清理任务
- **过期清理：** 自动清理过期的等待文件，保持系统整洁

### **5. 新增脚本文件**
```
scripts/
├── monday-deep-clean.ps1          # 周一深度清理脚本
├── gateway-start-hook.ps1         # 网关启动钩子脚本
└── clean-sessions-dual.ps1        # 原有的双重清理脚本
```

## 🔧 **技术实现细节**

### **周一深度清理逻辑**
```powershell
# 主逻辑流程
1. 检查是否是周一
2. 检查文件数量是否超过10个
3. 检查网关是否在运行
4. 如果网关运行 → 执行深度清理
5. 如果网关未运行 → 创建等待任务
```

### **网关启动钩子逻辑**
```powershell
# 钩子逻辑流程
1. 检查是否有等待的周一清理任务
2. 验证任务创建日期（仅当天有效）
3. 如果是周一且任务有效 → 执行补偿清理
4. 清理等待任务文件
```

### **智能心跳配置更新**
- HEARTBEAT.md文件已更新：
  - 文件数量触发：改为 >10个文件
  - 周度清理策略：改为周一深度清理
  - 网关补偿机制：已整合到心跳检查中
