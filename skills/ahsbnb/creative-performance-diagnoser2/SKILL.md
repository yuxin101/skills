# SKILL: creative-performance-diagnoser2

## 名称

投放素材效果数据诊断与迭代优化策略报告生成

## 描述

这是一个**半自动**的数据分析技能，用于诊断广告投放素材（创意）的效果。用户提供平台导出的数据报表（CSV 格式），通过简单的列名配置，即可生成一份结构化的诊断报告，包含整体表现概览、素材排名、四象限分类、关键词洞察以及可执行的优化建议。该技能**完全通用**，可适配淘宝万象台、巨量千川、腾讯广告、快手磁力引擎等任何平台的报表。

**核心模式**：半自动。数据获取由人工完成，数据分析由脚本自动执行，最终由主 AI（openclaw）结合业务背景提供更丰富的落地建议。

## 目标

**首要行动指令**: 无论分析维度如何，最终目标必须是清晰、直接地识别出“优质素材”（值得放大）和“劣质素材”（必须淘汰）。报告的核心是为决策提供依据。

自动化分析广告投放素材的效果数据，输出一份清晰、可执行的诊断报告，为后续的内容迭代和素材优化提供明确依据。

## 黄金标准：动态诊断SOP

根据用户`an`的反馈，本技能必须遵循“动态诊断”的原则，以确保报告的信息饱满度和决策价值。

**核心原则**:

1.  **动态调整结构**: 当输入数据缺少关键指标（如“花费”、“点击量”）时，绝不能机械地输出一个充满“0”的、单薄的标准化报告。必须主动切换分析框架。
2.  **创造衍生指标**: 在缺少ROI、CTR等标准指标时，必须主动创造新的、有意义的衍生指标来评估内容效率。例如：
    *   **RPV (Revenue Per View / 单次观看价值)** = 总成交金额 / 总观看次数。
3.  **分析与数据强关联**: 每一个核心数据表格（如“明星素材榜”）都必须被视为一个独立的论点，其后必须紧跟针对该论据的深度文字分析（说明“为什么”）和具体行动建议（指导“怎么办”）。
4.  **关键词与业务结果挂钩**: 关键词分析不能只停留在“词频”层面，必须深入分析“关键词”与“成交额”之间的关联，以识别出真正的“高转化词”，而非仅仅是“高频词”。
5.  **明确分类**: 报告必须在核心位置清晰地列出“优质素材清单”和“劣质素材清单”，所有后续分析都应围绕这个核心分类展开。

这个SOP是本技能的最高优先级指令，确保了交付物的质量与深度。

## 执行流程

### 1. 数据输入

- **核心输入**：平台导出的投放数据报表（CSV 格式）。
- **列映射**：用户必须指定报表中各关键指标对应的列名。支持两种方式：
  1. **命令行参数**：直接通过 `--xxx-col` 参数指定列名。
  2. **配置文件**：编写一个 JSON 文件，存放列名映射，通过 `--config` 引用。

### 2. 核心处理

- **数据解析**：自动识别 CSV 编码（UTF-8 with BOM 或 GBK），按用户提供的列映射读取数据。
- **数据清洗**：将指定列转换为数值类型，处理空值和异常值。
- **聚合计算**：按素材（创意）ID 和名称进行分组汇总，计算累计展现、点击、花费、成交金额、成交笔数等。
- **指标衍生**：
  - 点击率 (CTR) = 点击量 / 展现量
  - 点击转化率 (CVR) = 成交笔数 / 点击量
  - 投入产出比 (ROI) = 成交金额 / 花费
  - 平均点击花费 (CPC) = 花费 / 点击量
  - 购物车率 = 购物车数 / 点击量（如有购物车列）
  - 收藏成本 = 花费 / 收藏数（如有收藏列）
- **整体统计**：计算所有素材汇总后的核心指标。
- **分类评级**：基于整体平均值，将每个素材的 CTR 和 CVR 分为高、中、低三档，并划分至四个象限：
  - **明星素材**：高点击高转化
  - **引流型素材**：高点击低转化
  - **转化型素材**：低点击高转化
  - **问题素材**：低点击低转化
- **关键词洞察**：从素材名称中提取高频词，并分析包含特定关键词的素材平均表现（可选）。

### 3. 报告生成

- 将所有分析结果整合为 Markdown 格式的诊断报告，直接打印在终端（可重定向保存为 `.md` 文件）。
- 报告内容包含：
  - 报告标题（含日期）、平台名称、数据期间、分析素材数量
  - 整体表现概览表格
  - ROI 排名 TOP10 和 BOTTOM10
  - 四象限分类及各类别素材名单
  - 创意关键词洞察（高频词及关键词表现对比）
  - **初步优化建议**（基于数据特征的通用建议）

### 4. 主 AI 建议丰富

- 生成的报告将由 openclaw 主 AI 接收。
- 主 AI 会结合**店铺背景、行业知识、用户历史行为**等额外信息，在报告基础上提供更丰富、更落地的优化建议，例如：
  - 针对明星素材，建议具体的二次创作方向（如强化某种视觉风格、复用某类文案钩子）。
  - 针对引流型素材，指出可能的落地页问题，并建议测试不同的人群定向或促销话术。
  - 针对转化型素材，建议优化封面图、标题的 A/B 测试方案。
  - 结合关键词洞察，推荐新的内容主题或卖点组合。
  - 给出预算重分配策略，指导下一阶段的投放重点。
- 主 AI 的建议会以自然语言附加在报告末尾，或通过其他沟通渠道（如飞书、邮件）发送给用户。

## 使用方法

### 方式一：命令行参数直接指定列名（适合快速测试）

```bash
& "F:\python 3.10\python.exe" "C:\Users\EDY\.openclaw\skills\creative-performance-diagnoser2\scripts\analyzer_generic.py"\
    --file "C:\Users\EDY\.openclaw\skills\creative-performance-diagnoser2\input\内容报表_20260314_092445.csv" \
    --platform "淘宝万象台" \
    --id-col "主体ID" \
    --name-col "主体名称" \
    --impressions-col "展现量" \
    --clicks-col "点击量" \
    --spend-col "花费" \
    --revenue-col "总成交金额" \
    --orders-col "总成交笔数" \
    --cart-col "总购物车数" \
    --favorite-col "总收藏数" \
    --date-col "日期"
```

### 方式二：使用 JSON 配置文件（适合长期复用）

1. 创建配置文件，例如 `config/taobao.json`，内容如下：

```json
{
    "id": "主体ID",
    "name": "主体名称",
    "impressions": "展现量",
    "clicks": "点击量",
    "spend": "花费",
    "revenue": "总成交金额",
    "orders": "总成交笔数",
    "cart": "总购物车数",
    "favorite": "总收藏数",
    "date": "日期"
}
```

2. 运行脚本：

```bash
& "F:\python 3.10\python.exe" "C:\Users\EDY\.openclaw\skills\creative-performance-diagnoser2\scripts\analyzer_generic.py" --file "C:\Users\EDY\.openclaw\skills\creative-performance-diagnoser2\input\内容报表_20260314_092445.csv"  --platform "淘宝万象台"  --config "C:\Users\EDY\.openclaw\skills\creative-performance-diagnoser2\config\taobao.json"
```

## 注意事项

- 确保 CSV 文件的列名与配置完全一致（包括空格和标点）。
- 数值列中不应包含百分比符号（如 `3.45%`），如果有，请先在 Excel 中转换为数值格式。脚本不会自动处理此类格式。
- 如果数据中没有成交（成交金额为 0），ROI 将显示为 0，不影响分析。
- 关键词分析依赖素材名称列，如果名称列缺失，该部分会自动跳过。
