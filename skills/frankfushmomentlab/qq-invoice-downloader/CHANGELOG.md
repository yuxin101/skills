## v7.2 (2026-03-09) - 增强稳定版 ✨

### 核心优化

v7.2 在 v7.1 的基础上进一步增强稳定性和成功率。

4. **增强错误信息**

   - 更详细的失败原因说明
   - 调试日志记录完整异常

5. **实时进度显示**
   - C类处理显示当前进度 [1/N]
   - 实时显示成功/失败状态
   - 更好的用户体验

## v7.0 (2026-03-09) - 终极优化版 🚀

### 核心优化

v7.0 是目前最先进的版本，相比 v6.1.1 再提升 **30-50%** 性能。

### 五大优化

1. **HTTP连接池**
   - 使用 `requests.Session` + `HTTPAdapter`
   - 复用TCP连接，减少握手开销
   - 连接池大小：10连接，最大20

2. **浏览器池（3实例）**
   - 3个Playwright浏览器实例并行
   - C类发票处理速度提升 **3x**
   - 信号量控制并发访问

3. **自适应线程数**
   - A类（附件）：8线程（IO密集型）
   - B类（链接）：5线程
   - C类（浏览器）：3线程（受浏览器池限制）

4. **批量IO优化**
   - 减少磁盘操作次数
   - 批量写入文件

5. **智能预加载**
   - IMAP连接复用
   - 减少登录开销

### 性能对比

场景：100封发票（A:B:C = 5:3:2）

## v6.0.1 (2026-03-09) - 目录统一修复

### 问题
浏览器自动化下载的发票被保存到了 `v5` 文件夹，而不是主下载目录 `v1`。

### 修复
- **browser_processor.py**: 添加 `get_main_output_dir()` 函数，自动检测 `*_v1` 文件夹
- **process_remaining.py**: 同样使用自动目录检测机制

### 代码变更
```python
# 修复前（硬编码）
OUTPUT_DIR = r"Z:\OpenClaw\InvoiceOC\260106-260308_v5"

# 修复后（自动检测）
def get_main_output_dir():
    """获取主输出目录（v1 文件夹）"""
    if os.path.exists(BASE_DIR):
        for d in sorted(os.listdir(BASE_DIR), reverse=True):
            if '_v1' in d:
                return os.path.join(BASE_DIR, d)
    return os.path.join(BASE_DIR, "260102-260309_v1")

OUTPUT_DIR = get_main_output_dir()
```

### 影响
- ✅ 所有发票现在统一保存在同一个主目录中
- ✅ 浏览器下载和直接下载的发票不再分散
- ✅ 便于后续管理和统计
