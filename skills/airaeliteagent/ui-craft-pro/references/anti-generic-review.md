# Anti-Generic UI Review

Use this after implementation when the output looks 'fine' but still feels like obvious AI UI.

## Common smells

### 1. Equal-weight everything
Symptoms:
- all cards are the same size
- every badge is bright
- headline and supporting copy compete

Fix:
- choose one dominant headline
- create primary / secondary / tertiary emphasis
- reduce the number of loud accents

### 2. Gradient addiction
Symptoms:
- hero gradient + button gradient + card gradients + glow borders

Fix:
- keep one hero moment
- let the rest use neutral surfaces and token discipline

### 3. Inconsistent component family
Symptoms:
- buttons, cards, inputs, and modals feel like four different products

Fix:
- lock radius, borders, shadows, and interaction timing globally

### 4. Fake premium
Symptoms:
- dark background + blur + huge padding but no strong hierarchy

Fix:
- improve typography and layout first
- then add only one premium signature effect

### 5. Bento noise
Symptoms:
- feature tiles look modern at first glance, but nothing guides the eye

Fix:
- create one anchor tile
- simplify copy
- vary emphasis intentionally

### 6. Template copy
Symptoms:
- headings could belong to any AI product on the internet

Fix:
- write specific claims
- show proof near the first CTA
- reduce empty hype words

### 7. Screenshot-first design
Symptoms:
- design depends on fake app screenshots to feel complete

Fix:
- ensure layout and hierarchy still work with weak media

## Final test
Ask:
- If I removed gradients and illustrations, would hierarchy still hold?
- If I swapped the logo, would this still feel like a deliberate product?
- Is the page winning through clarity or just effects?

If the page only works because of effects, it is not done.
