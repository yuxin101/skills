# Product Archetypes

Use this file when the first generated design direction feels technically valid but emotionally wrong for the product.

The goal here is not to override all search output. The goal is to correct drift when broad product categories pull the UI toward a generic pattern.

## Privacy-first tools

Common product signals:
- privacy
- secure
- encrypted
- personal
- confidential
- trusted
- local-first

Prefer:
- restrained layouts
- clear hierarchy
- strong readability
- calm confidence rather than flashy momentum
- proof sections that reinforce trust
- clear CTA language without hype

Usually avoid:
- loud marketing energy
- playful social-app styling
- over-animated gradients
- visual tricks that make the product feel less trustworthy

Good visual directions:
- editorial minimal
- paper / e-ink
- premium dark trust surfaces
- quiet modern SaaS

## Journaling / reflection / writing tools

Common product signals:
- journaling
- notes
- reflection
- writing
- mindful
- long-form
- thinking
- clarity

Prefer:
- reading-first typography
- calmer pacing
- less UI chrome
- more whitespace
- fewer CTA variants
- a sense of focus and emotional safety

Usually avoid:
- app-store landing clichés unless the product is clearly consumer-download-first
- loud dashboard styling
- gaming/social visual energy
- too many badges, stickers, ratings, or shiny proof gimmicks

Good visual directions:
- paper / editorial
- calm serif + sans hierarchy
- soft premium minimal
- distraction-free writing surfaces

## Wellness / calm consumer software

Common product signals:
- calm
- wellness
- mindful
- peaceful
- sleep
- habit
- self-care

Prefer:
- soft pacing
- breathable spacing
- warm or natural neutrals
- low-friction CTA strategy
- gentle hierarchy

Usually avoid:
- harsh contrast without purpose
- dense enterprise structure
- excessive card noise

Good visual directions:
- soft premium minimal
- warm editorial
- paper-like calm interfaces

## Enterprise / admin / analytics / operator tools

Common product signals:
- dashboard
- operations
- analytics
- admin
- control
- compliance
- workflow

Prefer:
- higher information clarity
- predictable components
- strong visual systems
- data-readable typography
- compact but legible rhythm

Usually avoid:
- over-romantic editorial styling
- decorative layouts that hide data

Good visual directions:
- premium dashboard
- technical minimal
- dark trust surfaces
- controlled glass when used sparingly

## Correction rule

If the first generated result is mismatched, do not blindly follow it.
Instead:
1. identify the true archetype
2. run narrower domain searches
3. lock a corrected direction
4. then implement

Typical correction examples:
- privacy journaling app → do not accept loud app-store styling by default; prefer paper/editorial/calm trust
- secure fintech ops tool → do not accept playful consumer gradients by default; prefer trust, structure, premium restraint
- reflective writing product → optimize for reading and focus before conversion theatrics
