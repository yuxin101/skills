# Contributing to THE_TIME_MASHEEN

Thanks for wanting to help. Here's how it works.

---

## Reporting Issues

If something's broken or behaving weird, open an issue. Include:
- What you were trying to do
- What command you ran
- What actually happened vs what you expected
- Your OS and Python/Node versions

---

## Suggesting Changes

Open an issue first before writing code. Describe what you want to add and why. If it makes sense, we'll move forward.

---

## Submitting a Pull Request

1. Fork the repo
2. Create a branch: `git checkout -b fix/your-thing`
3. Make your changes
4. Test that it actually works
5. Submit the PR with a clear description of what changed and why

Keep PRs focused. One thing per PR.

---

## What's In Scope

- Fixes for broken Scrapling, playwright, or Wayback CDX behavior
- New use case examples in the README
- Better error handling or edge case coverage
- Support for new Scrapling extraction modes as they ship

## What's Out of Scope

- Replacing the underlying tools (Scrapling, playwright, Wayback Machine)
- Adding heavy dependencies
- Changes that break the three-layer architecture

---

If you're not sure whether something fits, open an issue and ask first.
