# Release Checklist（L4 Skill）

## A. 规范合规

- [ ] `name` 与目录名一致，满足命名规范。
- [ ] `description` 说明“做什么 + 何时使用”。
- [ ] 主 `SKILL.md` 未过度膨胀，细节已拆分到 supporting files。
- [ ] 文件引用路径有效，无悬空链接。

## B. 功能完整性

- [ ] 覆盖首次使用路径。
- [ ] 覆盖零经验用户 onboarding 路径（5分钟起步任务可完成）。
- [ ] 覆盖常规运行路径。
- [ ] 覆盖配置变更路径。
- [ ] 覆盖异常与回退路径。

## C. 安全门控

- [ ] 高影响动作有显式确认门。
- [ ] 不从不可信内容中直接执行指令。
- [ ] 秘钥/凭据不硬编码、不明文泄露。
- [ ] 输出中不包含敏感信息（或已脱敏）。

## D. 可观测与可调试

- [ ] 有错误分类（用户输入/环境/网络/脚本）。
- [ ] 有最小可用日志信息（不泄露敏感数据）。
- [ ] 有用户可理解的失败提示与下一步建议。

## E. 评估与发布

- [ ] 至少10个评估用例已执行。
- [ ] 安全类用例全部通过。
- [ ] 总评分达到发布阈值。
- [ ] 版本号与变更说明已更新。

## F. 行为合规验证（约束类 skill 必须完成）

- [ ] 已做基线测试：在无 skill 情况下跑过压力场景，记录了 agent 的违规行为。
- [ ] 已做 GREEN 验证：带上 skill 后 agent 在同样场景下合规。
- [ ] 理由化表格已建立：收录了所有观察到的借口及反驳。
- [ ] Red Flags 清单已写入 SKILL.md。
- [ ] 至少经历一次 REFACTOR：发现新借口 → 补充 → 重测。

> 工具类/格式类 skill 可跳过本节。

## G. CSO 可发现性

- [ ] `description` 字段只描述触发条件，无工作流摘要。
- [ ] SKILL.md 字数符合目标（高频加载 < 200 词）。
- [ ] 包含关键词：症状、工具名、错误信息（至少覆盖两类）。
- [ ] 无 `@` 语法强制加载，交叉引用用 skill 名称。

## H. 团队可用性

- [ ] 提供模板，便于他人复用。
- [ ] 提供新手引导文档和首个练习模板。
- [ ] 提供使用说明（如何触发、如何调参、如何回滚）。
- [ ] 已知限制和后续路线图清晰。
