# 零经验 Onboarding（0→1）

目标：让没有技能开发经验的人，在 5-15 分钟内完成第一个可用 skill。

## 第0步：先说人话（给用户看的解释）

- Skill 就是“可复用的任务说明书 + 小工具包”。
- 你不需要一次做到完美，只要先做出一个能稳定完成单一任务的 v0。

## 第1步：选一个最小任务（5分钟）

只选下面之一：
1. 生成固定格式的周报摘要
2. 代码改动后输出标准化 changelog
3. 根据输入问题返回结构化答复（含风险与下一步）

不要一开始做“多平台自动投递 + 定时任务 + 复杂状态机”。

## 第2步：填最少信息

你只需要给 3 个信息：
- 任务名称（skill 名）
- 触发场景（何时使用）
- 输出格式（你想看到什么）

其余默认值：
- 语言：跟随当前会话
- 安全策略：高风险动作先确认
- 评估阈值：85/100

## 第3步：用模板生成骨架

1) 用脚手架创建目录
- `node scripts/scaffold-skill.js <target-dir> <skill-name> "<description>"`

2) 打开以下文件按顺序填写
- `SKILL.md`
- `references/l4-standard.md`
- `assets/checklists/release-checklist.md`
- `assets/evals/eval-cases.md`

## 第4步：做第一次可运行验证

- 运行评分：`node scripts/score-skill.js <skill-dir>`
- 目标：先到 75 分以上（L3）
- 再补 onboarding/安全/评估项冲到 90+（L4）

## 第5步：首个复盘（必须）

交付后问 3 个问题：
1. 哪一步最卡？
2. 输出哪里不符合预期？
3. 下一版最该改什么？

把答案写进变更记录，再做 v0.2。

## 常见新手坑

- 一上来就追求全自动，导致不可控。
- 只写主 SKILL，不写评估和检查表。
- 没有高风险确认门，埋安全雷。
- 没有首个成功样例，用户不知道“做对了长什么样”。
