# 对标说明：本技能与官方示例模板的差异

> 结论：官方示例覆盖“如何写一个技能”，本技能覆盖“如何把技能做成可发布的生产资产”。

## 对标对象

- Agent Skills 规范（结构与字段）
- Claude Skills 文档（触发、frontmatter、supporting files）
- OpenAI Skills 文档（版本化与安全风险）

## 共同点（对齐官方）

- 遵循标准目录和 `SKILL.md` 前置元数据。
- 使用 supporting files 做渐进加载（progressive disclosure）。
- 强调描述字段要包含“做什么 + 何时用”。

## 增强点（超出常规模板）

1. **L4门禁化**
   - 提供发布检查清单与量化门槛，而非“写完即用”。
2. **安全内建**
   - 默认要求高影响动作显式确认；对注入与数据外泄有专门检查项。
3. **评估内建**
   - 内置评估用例集（正常/边界/攻击路径）。
4. **工程化自动化**
   - 提供评分脚本和脚手架脚本，支持团队批量复用。
5. **可维护性策略**
   - 明确版本与回滚、变更记录、后续迭代基线。

## 注意

- “超过官方”并不意味着替代官方规范；本技能以官方标准为底座，增强工程实践层。
- 任何高风险场景仍需结合你所在组织的合规策略进行二次审计。
