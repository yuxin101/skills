## 一、汇报类型

### 1. 任务领取汇报

**触发：** 虾领取新任务时

**模板：**
```
🦞 任务领取
- 任务：[角色] 任务描述
- 负责人：虾名字
- 预计完成：YYYY-MM-DD HH:MM
- 状态：已开始
```

**示例（终端虾）：**
```
🦞 任务领取
- 任务：[CPMO] 编写项目 v1.2.0 周报
- 负责人：终端虾
- 预计完成：2026-03-09 15:00
- 状态：已开始
```

**示例（Telegram 虾）：**
```
⚡ 任务领取
- 任务：[CGO] 写一篇小红书 AI 绘画教程
- 负责人：Telegram 虾
- 预计完成：2026-03-09 17:00
- 状态：开始执行
```

### 3. 每小时进展简报（COO 统筹）

**触发：** 每小时心跳（8:00-18:00）

**模板：**
```
📊 三只虾进展简报（HH:MM）

【任务状态】
- [CPMO] 终端虾：任务名称（状态 进度%）
- [COO] 飞书虾：任务名称（状态 进度%）
- [CGO] Telegram 虾：任务名称（状态 进度%）

【统计】
待处理：X | 进行中：X | 已完成：X | 阻塞：X

【备注】
- 需要关注的事项
- 需要 CEO 协助的事项（如有）
```

**示例：**
```
📊 三只虾进展简报（14:00）

【任务状态】
- [CPMO] 终端虾：编写项目周报（进行中 50%）
- [COO] 飞书虾：统筹今日汇报（已完成）
- [CGO] Telegram 虾：小红书文案（待处理）

【统计】
待处理：2 | 进行中：1 | 已完成：5 | 阻塞：0

【备注】
- Telegram 虾已上线，开始执行小红书文案
- 项目周报预计 15:00 完成
```

### 6. 任务阻塞汇报

**触发：** 任务无法继续时

**模板：**
```
🚨 任务阻塞
- 任务：[角色] 任务描述
- 负责人：虾名字
- 阻塞原因：具体原因
- 需要协助：需要谁/什么帮助
- 建议方案：解决方案建议
- 影响：如不解决的后果
```

**示例：**
```
🚨 任务阻塞
- 任务：[CPMO] 项目 v1.2.0 周报
- 负责人：终端虾
- 阻塞原因：产品组未提供最新数据
- 需要协助：需要 CEO 协调产品组
- 建议方案：1. 联系产品负责人 XXX 2. 先用现有数据
- 影响：周报可能延期到明天
```

## 四、注意事项

1. **简洁明了** - 汇报要简短，突出重点
2. **数据驱动** - 用数据说话（完成率、进度%）
3. **及时更新** - 任务状态变更立即汇报
4. **问题导向** - 遇到问题及时上报，带解决方案
5. **COO 统筹** - 飞书虾负责汇总和发送简报
