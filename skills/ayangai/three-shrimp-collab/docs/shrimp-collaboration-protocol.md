# 🦞 三只虾协同协议

_2026-03-09 确立 - 老板的 AI 助手团队协同规范_

## 一、心跳协同机制

### 1.1 心跳时间

| 时间段 | 频率 | 说明 |
|--------|------|------|
| **工作时间** | 8:00-18:00 | 每小时一次 |
| **下班时间** | 18:00-次日 8:00 | 不执行（节省 token） |
| **周末/节假日** | 不执行 | 除非 CEO 特别要求 |

### 1.2 心跳检查内容

#### 快速检查（每小时）
```markdown
1. 读取 tasks/queue.md
2. 检查是否有分配给自己的 [待处理] 任务
3. 如有 → 领取并执行
4. 如无 → 回复 HEARTBEAT_OK
```

#### 完整同步（每天 12:00）
```markdown
1. 读取 MEMORY.md
2. 读取 三只虾分工体系.md
3. 更新自己的状态到 tasks/queue.md
4. 检查其他虾的状态
5. 如有需要，发起协同请求
```

#### 每日总结（每天 18:00）
```markdown
1. 清理已完成的旧任务（保留最近 3 天）
2. 生成每日协同报告
3. 同步到 MEMORY.md
```

## 二、任务分发流程

### 2.1 CEO 分配任务

**步骤：**
1. CEO 在 `tasks/queue.md` 添加任务
2. 格式：`- [ ] [角色] 任务描述 - 分配人 @时间`
3. 下一轮心跳时，对应角色会自动领取

**示例：**
```markdown
- [ ] [CPMO] 编写项目周报 - 老板 @2026-03-09 14:00
```

### 2.2 虾领取任务

**步骤：**
1. 心跳检查时发现有分配给自己的任务
2. 移动到「处理中」区域
3. 更新状态：`- [~] [角色] 任务描述 - 负责人 @开始时间`
4. 执行任务
5. 完成后移动到「已完成」

### 2.3 任务转交

**如需要转交其他虾：**
1. 在任务下方添加评论：`  → 转交给 [角色] 原因：xxx`
2. 更新任务状态为 `[!]` 阻塞
3. 通知飞书 COO 统筹

## 三、信息共享机制

### 3.1 共享文件列表

| 文件 | 用途 | 更新频率 |
|------|------|----------|
| `MEMORY.md` | 长期记忆、分工体系 | 按需 |
| `三只虾分工体系.md` | 职责说明 | 按需 |
| `tasks/queue.md` | 任务队列 | 实时 |
| `HEARTBEAT.md` | 心跳规则 | 按需 |
| `channels/*.md` | 渠道配置 | 按需 |

### 3.2 文件更新通知

**更新共享文件后：**
1. 在文件顶部更新「最后更新」时间
2. 在 `tasks/queue.md` 添加通知任务：
   ```markdown
   - [ ] [ALL] MEMORY.md 已更新，请重新读取 - 飞书虾 @2026-03-09 13:25
   ```
3. 下一轮心跳时其他虾会看到

## 四、冲突解决

### 4.1 职责冲突

**如出现职责重叠：**
1. 优先参考 `三只虾分工体系.md`
2. 仍无法解决 → 提交 CEO 裁决
3. 更新分工文档避免再次冲突

### 4.2 任务冲突

**如多个虾同时领取同一任务：**
1. 先领取者优先（以 `tasks/queue.md` 时间戳为准）
2. 后领取者主动退出
3. 如需要协作，拆分子任务

## 五、沟通规范

### 5.1 任务评论格式

```markdown
- [~] [CPMO] 项目周报 - 终端虾 @2026-03-09 14:00
  > 💬 终端虾：正在收集中，预计 15:00 完成
```

### 5.2 协同请求格式

```markdown
### 🤝 协同请求
- **发起人：** 飞书虾
- **需要协助：** 终端虾
- **事项：** 确认项目周报数据
- **截止时间：** 2026-03-09 16:00
```

### 5.3 状态更新格式

```markdown
### 📊 状态更新
- **更新人：** 终端虾
- **时间：** 2026-03-09 15:00
- **内容：** 项目周报已完成，待 CEO 审核
```

## 六、异常处理

### 6.1 虾离线

**如某虾长时间无心跳（>4 小时）：**
1. 飞书 COO 负责通知 CEO
2. 评估是否需要人工介入
3. 记录到 `MEMORY.md`

### 6.2 任务超时

**如任务超过截止时间未完成：**
1. 自动标记为 `[!]` 阻塞
2. 通知飞书 COO 统筹
3. 重新分配或调整优先级
