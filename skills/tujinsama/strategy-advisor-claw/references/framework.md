# 战略分析框架参考

## 分析维度

每次分析时，从以下维度中选取与当前场景相关的维度进行深度思考。不需要每次都覆盖所有维度，根据问题的性质灵活选择。

### 1. 现状诊断
- 当前核心数据指标是什么？趋势如何？
- 关键瓶颈/卡点在哪里？
- 资源配置是否合理？（人力、资金、时间）
- 市场环境发生了什么变化？

### 2. 利益相关方分析
- 决策者是谁？他们关心什么？
- 执行者是谁？他们的能力和意愿如何？
- 外部合作方/竞争对手在做什么？
- 各方利益是否一致？冲突点在哪？

### 3. 机会与风险
- 短期机会（1-3个月）：容易被忽略的窗口期
- 中期机会（3-12个月）：需要布局的方向
- 关键风险：如果什么都不做会怎样？
- 如果执行建议方案，可能的负面后果？

### 4. 资源与约束
- 现有资源能支撑什么规模的行动？
- 硬约束（不可改变的）vs 软约束（可以想办法的）
- 最小可行投入是什么？

### 5. 优先级排序
- 影响力 vs 难度矩阵
- 时间敏感度（哪些事拖延会造成不可逆损失？）
- 依赖关系（哪些事必须先做？）

## 建议输出结构

每个建议应包含：
1. **具体行动** — 明确、可执行，不是笼统的方向
2. **预期收益** — 定性或定量的预期结果
3. **投入成本** — 所需资源和时间
4. **风险提示** — 可能出问题的地方
5. **优先级** — P0（立即）/ P1（本周）/ P2（本月）

## 思考质量自检

在给出最终建议前，自检以下问题：
- 这个建议是否基于具体信息，而非泛泛而谈？
- 如果我是对方，我能看懂并立即执行吗？
- 我是否遗漏了某个明显的风险？
- 有没有更简单、成本更低的替代方案？
