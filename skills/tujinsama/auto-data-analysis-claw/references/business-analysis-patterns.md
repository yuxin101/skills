# 业务分析模式

## 常见分析场景与思路

### 场景1：营收分析

**分析框架**：
1. 总量 → 趋势（同比/环比）
2. 结构 → 按产品/地区/渠道/客户分层
3. 质量 → 客单价、复购率、收入集中度
4. 预测 → 滚动预测、季节性调整

**关键问题**：
- 增长是靠量还是靠价？
- 哪个维度贡献了最大增量？
- 是否存在过度依赖单一来源？

### 场景2：成本分析

**分析框架**：
1. 成本结构 → 固定成本 vs 变动成本占比
2. 成本率趋势 → 成本/收入比变化
3. 差异分析 → 实际 vs 预算 vs 上期
4. 成本动因 → 驱动成本变化的关键因素

### 场景3：客户分析

**分析框架**：
1. RFM 分析 → 最近购买时间/频率/金额
2. 客户分层 → 高价值/成长型/潜力型/流失风险
3. 生命周期 → 获客成本(CAC)、生命周期价值(LTV)、LTV/CAC
4. 留存分析 → 按月的留存率、流失预警

### 场景4：运营效率分析

**分析框架**：
1. 人效 → 人均产出、人均收入
2. 资产效率 → 周转率、利用率
3. 流程效率 → 转化漏斗、处理时长
4. 质量效率 → 一次成功率、返工率

## 分析方法论

### 漏斗分析
适用于转化类场景（销售漏斗、用户旅程等）：
```
曝光 → 点击 → 注册 → 激活 → 付费 → 留存
```
关注每一步转化率和整体漏斗形状。

### 同期群分析（Cohort Analysis）
按时间窗口分组，追踪各组的行为随时间变化：
- 按获客月分群 → 追踪留存/复购
- 识别不同获客渠道的用户质量差异

### 帕累托分析（80/20 法则）
- 识别贡献 80% 收入的 20% 客户/产品
- 识别消耗 80% 成本的 20% 项目
- 用于资源优先级排序

### 因果推断思路
当发现关联性时：
1. 确认时间先后（因在果之前）
2. 排除混杂因素
3. 寻找自然实验或对照组
4. 量级合理性检查
