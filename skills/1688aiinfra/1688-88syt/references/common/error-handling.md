# 通用错误处理

所有命令在遇到以下 HTTP / 业务错误时，遵循统一处理策略。

## 错误码与 Agent 应对

| 错误码 | 含义 | Agent 应对 |
|--------|------|-----------|
| — | AK 未配置（命令入口预检查） | 输出 AK 引导话术（见 SKILL.md） |
| 400 | 参数不合法 | 检查用户输入（关键词、渠道、商品ID等）是否正确 |
| 401 | 鉴权无效（AK 错误或已过期） | 输出 AK 引导话术（见 SKILL.md），引导用户重新配置 |
| 429 | 请求被限流 | 建议用户稍后重试（通常等待 1-2 分钟） |
| 500 | 服务端异常 | 建议用户稍后重试，如持续出现建议联系客服 |

## 网络异常

CLI 已内置 3 次重试（指数退避），重试耗尽后返回 `success: false`。

Agent 应对：告知用户"网络异常，请检查网络连接后重试"。

## 识别方式

当 CLI 输出 `success: false` 时：

1. 输出 `markdown` 字段（用户可读的错误描述）
2. 检查 `markdown` 中的关键词（如 "AK 未配置"、"401"、"授权过期"、"限流"），按 SKILL.md 异常处理表追加对应引导

## 各能力特有异常

通用错误外的业务异常，见各能力文档的"业务异常处理"段。
