# 提前停止规则与质量门禁

## 一、提前停止规则

### 可提前停止的场景

#### 停在 Step 3（Clean Only）

**适用条件**（全部满足）：
1. goal_mode = "informational"
2. 不涉及时效敏感信息（canonical_time_frame 无严格要求）
3. 结论风险低（非决策依赖型任务）

**典型场景**：
- 知识整理："总结RAG技术的核心原理和主流框架"
- 历史回顾："2024年AI行业重大事件梳理"
- 概念解释："什么是MoE架构"

**停止后输出**：pipeline_steps 仅包含 step1-3，summary 中 stop_reason = "stopped_after_clean"

---

#### 停在 Step 4（Clean + Freshness）

**适用条件**（全部满足）：
1. goal_mode = "tactical" 或 "informational" + 时效需求
2. 未检测到高风险特征
3. 不涉及战略级决策

**典型场景**：
- 新闻编译："最近一周AI领域重要动态"
- 趋势追踪："2025年Q1新能源车销量趋势"
- 快速评估："某公司最新财报的核心变化"

**停止后输出**：pipeline_steps 包含 step1-4，summary 中 stop_reason = "stopped_after_freshness"

---

### 必须跑满5步的条件

**任一条件满足即必须跑满**：

| 条件编号 | 条件描述 | 示例 |
|---------|---------|------|
| R1 | goal_mode = "strategic" | "评估全球AI芯片竞争格局" |
| R2 | 涉及投资决策或资金风险评估 | "是否应该增持某股票" |
| R3 | 涉及政策影响预测 | "制裁将导致什么后果" |
| R4 | 涉及竞争格局判断 | "某公司市场份额变化" |
| R5 | Step 3 发现直接矛盾证据 | 正反方向证据强度相当 |
| R6 | Step 4 检测到关键证据过时 | 核心依赖的证据新鲜度=stale |
| R7 | 检测到单线叙事风险 | >80%证据指向同一方向且无反证 |
| R8 | 用户显式要求 | "请务必做反证分析" |

## 二、质量门禁

### 每步最低输出要求

#### Step 1: query-planner

| 指标 | 最低要求 | 不达标处理 |
|------|---------|-----------|
| planned_queries 数量 | ≥ 3 条 | 补充通用查询方向 |
| counter_queries 数量 | ≥ 2 条 | 补充反向查询 |
| 维度覆盖 | ≥ 3 个独立维度 | 合并相近维度并补充 |
| 查询 specificity | 每条至少含1个限定词/时间词 | 重写过于宽泛的查询 |

**强制要求**: goal_mode=strategic 时，planned_queries ≥ 5，counter_queries ≥ 3

---

#### Step 2: source-router

| 指标 | 最低要求 | 不达标处理 |
|------|---------|-----------|
| 每条查询的来源分配 | ≥ 1 个来源 | 使用默认搜索源降级 |
| 来源多样性 | ≥ 2 种不同来源类型 | 强制分配备用来源 |
| fallback_source 字段 | 100% 填充 | 标注"无降级方案" |

**来源类型分类**:
- search_engine: tavily, web_search, duckduckgo 等
- api: 专用API接口
- rss: RSS订阅源
- direct_fetch: web_fetch 直接抓取
- database: 本地知识库/缓存

---

#### Step 3: evidence-cleaner

| 指标 | 最低要求 | 不达标处理 |
|------|---------|-----------|
| 原始结果保留率 | ≥ 50% | 标注"高过滤率"，检查过滤是否过严 |
| 信源评级分布 | S+A级占比 ≥ 30% | 在summary中标注信源质量偏低 |
| 维度覆盖率 | ≥ 60% 维度有命中 | 在coverage_report中列出零命中维度 |
| 证据结构化 | 100% 包含 content + source | 无法提取的标记为raw_text |

---

#### Step 4: freshness-judge

| 指标 | 最低要求 | 不达标处理 |
|------|---------|-----------|
| 新鲜度标注率 | 100%（含unknown） | 无降级，强制标注 |
| fresh 证据占比 | ≥ 20%（时效性任务）或无要求（历史任务） | 在freshness_report中标注"证据偏旧" |
| staleness_warnings | 对所有 stale 证据必须生成 | 无降级，强制生成 |

---

#### Step 5: counter-evidence-hunter

| 指标 | 最低要求 | 不达标处理 |
|------|---------|-----------|
| mainline_synthesis | 必须输出（1-3句话） | 无降级，强制提取 |
| counter_evidence | ≥ 1条有效反证 或 明确声明"未发现有效反证" | **不可编造反证**，零发现时必须输出声明 |
| flip_conditions | 对每条 hard 级反证必须生成 | 无降级，强制生成 |
| confidence_assessment | 必须输出 | 无降级，强制输出 |

**零反证声明模板**:
```
"经过 [搜索范围描述] 的反证搜索，未发现与主线直接矛盾的有效证据。
这可能表明：(1) 主线判断较为可靠；(2) 搜索范围不够充分（盲区：[列出]）。
建议在最终分析中标注'反证搜索覆盖有限'。"
```

## 三、强制继续信号

### 信号检测规则

以下信号一旦在任意步骤中被检测到，**必须继续执行到下一步**，即使当前步骤已达提前停止条件：

| 信号 | 检测位置 | 触发动作 |
|------|---------|---------|
| 🔴 **直接矛盾**: 同一维度存在强度相当的正面和反面证据 | Step 3 | 继续到 Step 5 |
| 🔴 **高不确定性**: 关键维度只有单一来源且信源评级 ≤ B | Step 3 | 继续到 Step 5 |
| 🔴 **核心过时**: target_variable 直接依赖的证据新鲜度 = stale | Step 4 | 继续到 Step 5 |
| 🔴 **单线叙事**: >80% 证据指向同一方向，counter方向零命中 | Step 3-4 | 继续到 Step 5 |
| 🔴 **信息缺口**: 覆盖率 < 50% 的维度数 ≥ 2 | Step 3 | 继续到 Step 4 至少 |
| 🔴 **高风险关键词**: 任务描述中含"决策""投资""政策""制裁""预测" | 输入解析时 | 标记为 must_full_run |

### 信号传播机制

当某步检测到强制继续信号时：
1. 在当前步骤输出中添加 `force_continue: true` 字段
2. 在 summary 中添加 `critical_gaps` 条目
3. 下游步骤的输入自动继承该标记
4. 最终输出中 stop_reason 会被覆盖为 "full_run"

## 四、停止决策流程图

```
开始
  ↓
goal_mode == "strategic"? ──Yes──→ 跑满5步 → 输出
  │ No
  ↓
执行 Step 1-3
  ↓
检测到强制继续信号? ──Yes──→ 继续下一步
  │ No
  ↓
goal_mode == "informational" 且无时效需求? ──Yes──→ 停在Step 3 → 输出
  │ No
  ↓
执行 Step 4
  ↓
检测到强制继续信号? ──Yes──→ 执行Step 5 → 输出
  │ No
  ↓
goal_mode == "tactical"? ──Yes──→ 停在Step 4 → 输出
  │ No
  ↓
跑满5步 → 输出
```
