# 反证类型分类与评级标准

## 一、反证五大类型

### 1. 直接反驳 (Direct Contradiction)

**定义**：存在与主线命题直接矛盾的事实证据。

**特征**：
- 证据本身直接否定主线命题的核心主张
- 不需要额外假设或推理链条
- 来源可追溯、可验证

**示例**：
- 主线："台积电3nm产能已满载"
- 反证："台积电2025年Q4财报显示3nm产能利用率降至78%"
- 反证类型：直接反驳

---

### 2. 条件反转 (Conditional Reversal)

**定义**：主线索引成立依赖特定前提条件，而该条件正在或可能发生变化。

**特征**：
- 不直接否定主线，但指出主线的成立条件正在松动
- 通常涉及趋势变化、政策转向、技术突破等
- 具有预测性质

**示例**：
- 主线："中国芯片自给率将在2027年达到70%"
- 反证："美国正推动将EDA工具禁令从7nm扩展至14nm，若执行，将直接阻断中芯国际14nm以上产线的扩产计划"
- 反证类型：条件反转

---

### 3. 量级质疑 (Magnitude Challenge)

**定义**：主线命题方向正确，但量级或显著性被高估或低估。

**特征**：
- 不否定方向，质疑"有多大"或"有多快"
- 常见于市场规模、增长速度、影响力评估
- 依赖替代数据源或更精确的测量方法

**示例**：
- 主线："AI将导致全球30%的岗位被自动化替代"
- 反证："麦肯锡2024年更新报告将预期下调至15-20%，且强调'替代'与'增强'的边界模糊，多数场景为任务级替代而非岗位级替代"
- 反证类型：量级质疑

---

### 4. 因果替代 (Causal Alternative)

**定义**：主线归因存在替代解释，观察到的现象可能由其他原因导致。

**特征**：
- 承认现象存在，但挑战因果归因
- 提出同样能解释观测数据的替代因果链
- 常见于政策效果、经济指标、技术影响评估

**示例**：
- 主线："2024年全球半导体收入增长主要由AI需求驱动"
- 反证："拆分增长结构后发现，汽车芯片和工业半导体贡献了增长的40%，AI芯片贡献约为35%，增长动力实际上是多因素叠加"
- 反证类型：因果替代

---

### 5. 样本偏差 (Sample Bias)

**定义**：主线索引所基于的数据或案例存在系统性偏差。

**特征**：
- 挑战数据代表性
- 指出幸存者偏差、选择偏差、报告偏差等
- 常见于案例分析、调查报告、行业统计

**示例**：
- 主线："90%的企业已采用AI进行业务转型"
- 反证："该数据来自Gartner对全球500强企业的调查，中小企业采纳率实际低于30%，存在严重的样本偏差"
- 反证类型：样本偏差

---

## 二、什么算反证 vs 什么不算反证

### ✅ 算反证

| 判定标准 | 说明 |
|---------|------|
| 直接矛盾主线核心命题 | 主线说A是B，证据证明A不是B |
| 挑战主线成立的前提条件 | 主线假设X为真，证据表明X可能为假 |
| 质疑主线的量级/显著性 | 主线说影响巨大，证据表明影响有限 |
| 提出合理的替代解释 | 观察到的现象可以用完全不同的原因解释 |
| 揭示数据来源的系统性偏差 | 主线依赖的数据本身不可靠 |

### ❌ 不算反证

| 伪反证类型 | 说明 |
|-----------|------|
| 无关反对 | 对主线完全无关的负面信息（如：主线讨论技术趋势，反证讨论公司高管个人丑闻） |
| 循论证伪 | 用与主线相同的前提去反驳主线 |
| 证实偏差的产物 | 只是因为结论"听起来太好了"就怀疑，没有任何实质证据 |
| 过度泛化 | 用极端个案否定总体趋势 |
| 时间错位 | 用过时数据反驳最新结论（除非主线本身依赖过时数据） |
| 动机揣测 | "这个研究有利益冲突" 而非挑战研究内容本身 |

---

## 三、反证强度评级

### 硬反证 (Hard Counter-Evidence)

**评级标准**：
- 🟢 来源可验证、时间新鲜（在 canonical_time_frame 内或之后）
- 🟢 直接挑战主线核心命题（非边缘假设）
- 🟢 来自权威/一手来源（官方数据、同行评审论文、权威机构报告）
- 🟢 不需要额外假设即可成立

**输出标记**：`"strength": "hard"`

**处理规则**：必须进入 flip_conditions 提炼流程

---

### 软反证 (Soft Counter-Evidence)

**评级标准**：
- 🟡 来源可追溯但权威性有限
- 🟡 不直接否定核心命题，但质疑其前提、量级或适用范围
- 🟡 需要合理推理才能建立与主线的矛盾关系
- 🟡 可能存在替代解释（即反证本身也可以被反驳）

**输出标记**：`"strength": "soft"`

**处理规则**：作为条件性翻转条件纳入评估

---

### 噪声 (Noise)

**评级标准**：
- 🔴 与主线命题无直接逻辑关系
- 🔴 来源不可验证或已知不可靠
- 🔴 证实偏差产物（只因"感觉不对"而无实质依据）
- 🔴 过度泛化或时间错位

**输出标记**：`"strength": "noise"`

**处理规则**：记录但标注为噪声，不纳入 flip_conditions 提炼

---

## 四、强度评级快速判定流程图

```
证据是否直接挑战主线核心命题？
  ├─ 是 → 来源是否权威且可验证？
  │       ├─ 是 → 是否需要额外假设？
  │       │       ├─ 否 → 硬反证 ✅
  │       │       └─ 是 → 软反证 🟡
  │       └─ 否 → 软反证 🟡
  └─ 否 → 是否挑战前提条件或量级？
          ├─ 是 → 来源是否可追溯？
          │       ├─ 是 → 软反证 🟡
          │       └─ 否 → 噪声 🔴
          └─ 否 → 噪声 🔴
```

## 动态反证查询生成规则 (V2)

### 触发条件

当 counter-evidence-hunter 接收到 `unexpected_findings` 输入时，进入动态查询生成模式。

### unexpected_findings 分类与查询生成

#### A. assumption_crack（假设裂缝）

**定义**: 主线搜索中发现了与主线假设直接矛盾或严重质疑的证据。

**查询生成策略**:
- 围绕裂缝点深挖，寻找更多支持裂缝的证据
- 搜索"失败案例""问题""风险""挑战"等负面关键词
- 限定时间窗为裂缝发生后的时间段

**示例**:
```json
{
  "finding": "NVIDIA Blackwell延迟交付比例高达30%",
  "finding_type": "assumption_crack",
  "suggested_counter_query": "NVIDIA Blackwell 交付延迟 客户流失 竞争对手替代 2025 2026"
}
```

#### B. overconfident_signal（过度自信信号）

**定义**: 主线证据高度一致地支持同一方向，缺乏对立面（>80%同向）。

**查询生成策略**:
- 生成极端反面测试查询
- 搜索"崩溃""泡沫""逆转""失败"等强负面关键词
- 主动寻找历史类比（类似情境下的翻转案例）

**示例**:
```json
{
  "finding": "15条证据中14条支持NVIDIA主导地位",
  "finding_type": "overconfident_signal",
  "suggested_counter_query": "NVIDIA 市场份额下降 竞争对手超越 历史案例 市场逆转"
}
```

#### C. new_dimension（全新维度）

**定义**: 搜索中发现了原始查询未预设的重要维度或角度。

**查询生成策略**:
- 围绕新维度生成探索性查询
- 搜索该维度对主线判断的潜在影响
- 寻找该维度的反面证据

**示例**:
```json
{
  "finding": "发现AI芯片市场正在从训练向推理快速转移",
  "finding_type": "new_dimension",
  "suggested_counter_query": "AI推理芯片 市场规模 训练芯片衰退 推理替代训练 证据"
}
```

### 查询合并与去重

```
预设 counter_queries (来自 query-planner)
    +
dynamic_queries (来自 unexpected_findings)
    ↓
去重（基于语义相似度 > 80%）
    ↓
all_counter_queries (完整反证查询列表)
```
